## API 情報

このコネクタは以下のベンダー API を使用します：

* API 名: Yahoo! JAPAN API
* API バージョン: v18
* API エンドポイント: `https://ads-display.yahooapis.jp`
* ドキュメント: [Yahoo! JAPAN API](https://ads-developers.yahoo.co.jp/developercenter/en/)

## 構成の構成

**コネクタマーケットプレイス**にアクセスし、新しいコネクタを追加します。コネクタを追加する一般的な手順については、[コネクタについて](https://docs.tealium.com/ja/server-side/connectors/manage/)の記事を参照してください。

コネクタを追加した後、以下の構成を構成します：

* **クライアント ID**: (必須) アプリケーションのクライアント ID。
* **クライアント シークレット**: (必須) アプリケーションのクライアント シークレット。

コネクタの構成が完了したら、**完了**をクリックします。

## バッチ制限

このコネクタは、ベンダーへの大量データ転送をサポートするためにバッチリクエストを使用します。並列処理により、イベントがベンダーに順不同で到達する可能性があります。順序が重要な場合は、イベントにシーケンス値を追加してください。詳細については、[バッチ処理アクション](https://docs.tealium.com/batched-actions/)を参照してください。リクエストは、次のいずれかの閾値に達するか、プロファイルが公開されるまでキューに入れられます：

* 最大リクエスト数: 200
* 最古のリクエストからの最大時間: 360分
* リクエストの最大サイズ: 10 MB

## アクション

|アクション名| AudienceStream| EventStream|
|---| ---| ---|
| ユーザーリストを追加 | ✓ | ✗ |

**続行**をクリックして、コネクタアクションを構成します。アクションの名前を入力し、アクションタイプを選択します。

次のセクションでは、各アクションのパラメータとオプションの構成方法について説明します。

### ユーザーリストを追加

#### 標準パラメータ

|**パラメータ**| **説明**|
|---| ---|
|アカウント ID| (必須) アカウント ID。|
|オーディエンスリスト ID| (必須) オーディエンスリスト ID。|
|ベースアカウント ID | (必須) アカウント ID がMCCアカウントの場合のベースアカウント ID。 |

#### ユーザー ID

|**パラメータ**| **説明**|
|---| ---|
|Email (既に SHA256 ハッシュ化されている)| 空白をトリムし、小文字に変換し、SHA256でハッシュ化されたメールアドレスを提供してください。|
|Email (SHA256 ハッシュを適用)| プレーンテキストのメールアドレスを提供し、コネクタがこの値を空白トリム、小文字変換、SHA256ハッシュを使用してハッシュ化します。|
|IDFA| iOSデバイスの広告識別子。|
|AAID| Androidデバイスの広告識別子。|
|Unknown| 未知の値。|
