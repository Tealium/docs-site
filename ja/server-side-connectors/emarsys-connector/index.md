## 構成

コネクタマーケットプレイスにアクセスし、新しいコネクタを追加します。コネクタを追加する一般的な手順については、[コネクタについて](https://docs.tealium.com/about-connectors/)を参照してください。

コネクタを追加した後、以下の構成を構成します：

* **認証タイプ**：
  * **OAuth**：クライアント資格情報を使用して Emarsys v3 API で認証します。詳細については、[Emarsys: v3 API での認証](https://dev.emarsys.com/docs/emarsys-core-api-guides/b3c3a1eba8515-authentication-in-v3-api)を参照してください。
  * **WSSE**：ユーザー名とパスワードを使用して Emarsys v2 API で認証します。詳細については、[Emarsys: v2 WSSE 認証](https://dev.emarsys.com/docs/emarsys-core-api-guides/branches/main/b3c3a1eba8515-authentication)を参照してください。
* **クライアント ID**：  
OAuth 認証タイプに必要です。Emarsys v3 API で認証するためのクライアント ID。
* **クライアントシークレット**：  
OAuth 認証タイプに必要です。Emarsys v3 API で認証するためのクライアントシークレット。
* **ユーザー名**：  
WSSE 認証タイプに必要です。Emarsys APID へのアクセス権を持つユーザー名。Emarsys APID ユーザー名についての詳細は、[Emarsys: アカウントの構成](https://dev.emarsys.com/docs/core-api-reference/branches/main/a12e08c686f0f-2-set-up-your-account)を参照してください。
* **パスワード**：  
WSSE 認証タイプに必要です。提供されたユーザー名のパスワード。
* **環境**：  
API コールを行う環境。

#### バッチ制限

このコネクタは、バッチリクエストを使用してベンダーへの大量データ転送をサポートします。詳細については、[バッチアクション](https://docs.tealium.com/batched-actions/)を参照してください。リクエストは、次のいずれかの閾値に達するか、プロファイルが公開されるまでキューに入れられます：

* 最大リクエスト数：1000
* 最古のリクエストからの最大時間：4分45秒
* リクエストの最大サイズ：10 MB

### メールコンタクトの作成または更新（リアルタイム）

#### パラメータ

|**パラメータ**| **説明**|
|---| ---|
|存在しない場合は作成| 有効にすると、コンタクトが存在しない場合は自動的に作成されます。|

|コンタクトデータ|  <ul><li>(必須) コンタクトを識別するためのフィールドと対応するフィールドデータ。</li><li>コンタクトを識別するために少なくとも1つのフィールドが必要です。</li><li>コンタクトフィールドでサポートされている唯一の日付形式は `YYYY-MM-DD` です。</li><li>一部のコンタクトフィールドは受け入れ可能な値が限られています。詳細については、[Emarsys: コア API リファレンス](https://dev.emarsys.com/docs/emarsys-api/1a17c3d6fcebc-list-available-choices-of-a-single-choice-field)を参照してください。</li><li>ターゲットコンタクトフィールドのIDを知っている場合は、カスタム値を使用して手動で構成できます。</li></ul> |

#### オプション

|**パラメータ**| **説明**|
|---| ---|
