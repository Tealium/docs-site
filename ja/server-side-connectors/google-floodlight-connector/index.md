Google Floodlightコネクタは、Google Campaign Manager 360 APIを通じてオフラインコンバージョンをFloodlightコンバージョントラッキングおよびアトリビューションシステムに挿入します。


<blockquote>
Floodlightは、Googleマーケティングプラットフォーム全体で使用されるコンバージョントラッキングシステムです。すべてのGMP製品（Campaign Manager 360*、Search Ads 360、Display & Video 360）は、コンバージョンの測定にFloodlightを使用できます。
</blockquote>


## 要件

* Campaign Manager 360（CM360）アカウントを持っている必要があります。
* CM360アカウントはAPIアクセスが有効になっている必要があります。
* Floodlightトラッキングに使用される広告主アカウントへのアクセス権を持つCM360ユーザープロファイルを持っている必要があります。
* コンバージョンデータへのアクセスを許可するために、CM360アカウントで次のOAuthスコープを有効にします：`https://www.googleapis.com/auth/ddmconversions`。
* **api@tealium.com** ユーザーをアカウントに追加し、**オフラインコンバージョンの挿入**権限でCM360アカウントへのアクセスを許可します。
* **api@tealium.com** ユーザーをCM360アカウントに追加した後、コネクタの構成ページまたはアクションページで生成されたプロファイルIDを使用します。両方の場所で定義されている場合、アクションページで定義されたプロファイルIDが優先されます。

## API情報

このコネクタは以下のベンダーAPIを使用します：

* API名：Google Campaign Manager 360 API
* APIバージョン：v5
* APIエンドポイント：`https://www.googleapis.com/dfareporting/`
* ドキュメント：[Campaign Manager 360 API](https://developers.google.com/doubleclick-advertisers/rest/v5)

### バッチ制限

このコネクタは、ベンダーへの大量データ転送をサポートするためにバッチリクエストを使用します。並列処理により、イベントがベンダーに順不同で到達する可能性があります。順序が重要な場合は、イベントにシーケンス値を追加してください。詳細については、[Batched Actions](https://docs.tealium.com/batched-actions/)を参照してください。リクエストは、次のいずれかの閾値が満たされるか、プロファイルが公開されるまでキューに入れられます：

* 最大リクエスト数：1000
* 最古のリクエストからの最大時間：60分
* リクエストの最大サイズ：1 MB

## 遅延アクション

Googleのベストプラクティスに従い、オフラインコンバージョンのアップロードを容易にするために、アクションは30分遅延されます。Google Campaign Manager 360 Floodlightコネクタを使用した遅延アクションについての詳細は、[Google: オフラインコンバージョンのアップロードに関するベストプラクティス](https://support.google.com/campaignmanager/answer/2823388?hl=en)を参照してください。


<blockquote>
[trace](https://docs.tealium.com/about-trace/)をEventSreamまたはAudienceStreamで使用する場合、アクションは遅延されません。
</blockquote>


## 構成

コネクタマーケットプレースにアクセスし、新しいコネクタを追加します。コネクタを追加する方法については、[Connector Overview](https://docs.tealium.com/about-connectors/)記事を読んで一般的な指示を参照してください。

コネクタを追加した後、次の構成を構成します：

* **FloodlightプロファイルID**  
  * Campaign Manager 360から提供されるFloodlightプロファイルIDです。これは、`api@tealium.com` ユーザーをCampaign Manager 360アカウントに追加した後に与えられます。


<blockquote>
このコネクタを追加すると、ベンダーのデータプラットフォームポリシーを受け入れるように求められます。
</blockquote>


## アクション

| アクション名 | AudienceStream | EventStream |
| --- | :---: | :---: |
|Track Floodlight Conversion| ✓| ✓|

**次へ**をクリックするか、**アクション**タブに進みます。ここでコネクタアクションの構成を構成します。

このセクションでは、各アクションのパラメータとオプションの構成方法について説明します。

### Track Floodlight Conversion

#### 構成IDパラメータ

|パラメータ| 説明|
|---| ---|
|プロファイルID| CM360が提供するFloodlightプロファイルIDです。ここで構成された場合、構成で定義された値に上書きされます。 |
|Floodlight構成ID|  (必須) このコンバージョンのFloodlight構成IDです。 |
|FloodlightアクティビティID|  (必須) このコンバージョンのFloodlightアクティビティIDです。 |

#### ユーザー/デバイスID

* 複数の属性がマッピングされている場合、最初の空でない値が送信されます。
* すべてのコンバージョンは30分の遅延後に送信されます。

|パラメータ| 説明|
|---| ---|
|暗号化されたユーザーID| 1つ以上の暗号化されたユーザーIDが指定されている場合、暗号化エンティティタイプ、ID、およびソースセクションが必要です。 |
|モバイルデバイスID| 単一のデバイスID、単一のクリックID、単一のユーザーID、マッチID、または複数のユーザー候補IDを指定します。単一のユーザー候補IDパラメータに候補IDのリストを使用することが許可されています。 |
|GoogleクリックID| 単一のデバイスID、単一のクリックID、単一のユーザーID、マッチID、または複数のユーザー候補IDを指定します。単一のユーザー候補IDパラメータに候補IDのリストを使用することが許可されています。 |
|ディスプレイクリックID| 単一のデバイスID、単一のクリックID、単一のユーザーID、マッチID、または複数のユーザー候補IDを指定します。単一のユーザー候補IDパラメータに候補IDのリストを使用することが許可されています。 |
|マッチID| 単一のデバイスID、単一のクリックID、単一のユーザーID、マッチID、または複数のユーザー候補IDを指定します。単一のユーザー候補IDパラメータに候補IDのリストを使用することが許可されています。 |
|インプレッションID| 単一のデバイスID、単一のクリックID、単一のユーザーID、マッチID、または複数のユーザー候補IDを指定します。単一のユーザー候補IDパラメータに候補IDのリストを使用することが許可されています。 |

#### 追加パラメータ

|パラメータ| 説明|
|---| ---|
|序数|  (必須) コンバージョンの序数です。同じユーザーと日のコンバージョンの重複を排除するために使用されます。 |
|タイムスタンプ（マイクロ秒単位）|  コンバージョンのタイムスタンプで、Unixエポックマイクロ秒です。提供されていない場合は自動生成されます。 |
|広告トラッキングの制限|  `true`に構成すると、ターゲティングではなくレポートのみにコンバージョンを使用します。 |
|子供向けの処理|  子供向けのコンバージョンに対して`true`に構成します。 |
|非パーソナライズ広告|  コンバージョンが非パーソナライズ広告の場合、`true`に構成します。 |
|未成年者向けの処理|  リクエストが未成年者からの可能性がある場合（GDPRを参照）、`true`に構成します。 |


#### ユーザー識別子

| **パラメータ** | **説明** |
| --- | --- |
| メールアドレス（既にSHA256ハッシュ化済み） | すでに空白を削除し、小文字に変換し、SHA256でハッシュ化されたメールアドレスを提供してください。`gmail.com`および`googlemail.com`のメールアドレスでドメイン名の前にあるピリオド(`.`)を削除してからハッシュ化してください。 |
| メールアドレス（SHA256ハッシュを適用） | プレーンテキストのメールアドレスを提供し、コネクタが`gmail.com`および`googlemail.com`のメールアドレスでドメイン名の前にあるピリオド(`.`)を削除し、空白を削除し、小文字に変換し、この値をSHA256ハッシュでハッシュ化します。 |
| 電話番号（既にSHA256ハッシュ化済み） | すでに空白を削除し、SHA256でハッシュ化された電話番号を提供してください。 |
| 電話番号（SHA256ハッシュを適用） | プレーンテキストの電話番号を提供し、コネクタがすべての非数字記号を削除し、番号の前にプラス記号(`+`)を付け、空白を削除し、この値をSHA256ハッシュでハッシュ化します。 |
| 住所情報：名（既にSHA256ハッシュ化済み） | すでに空白を削除し、小文字に変換し、SHA256でハッシュ化された名前を提供してください。 |
| 住所情報：名（SHA256ハッシュを適用） | プレーンテキストの名前を提供し、コネクタが空白を削除し、小文字に変換し、この値をSHA256ハッシュでハッシュ化します。 |
| 住所情報：姓（既にSHA256ハッシュ化済み） | すでに空白を削除し、小文字に変換し、SHA256でハッシュ化された姓を提供してください。 |
| 住所情報：姓（SHA256ハッシュを適用） | プレーンテキストの姓を提供し、コネクタが空白を削除し、小文字に変換し、この値をSHA256ハッシュでハッシュ化します。 |
| 住所情報：番地（既にSHA256ハッシュ化済み） | 特殊文字を含まない番地を提供し、すでに空白を削除し、小文字に変換し、SHA256でハッシュ化されているものを提供してください。 |
| 住所情報：番地（SHA256ハッシュを適用） | プレーンテキストの番地を提供し、コネクタがすべての特殊文字を削除し、空白を削除し、小文字に変換し、この値をSHA256ハッシュでハッシュ化します。 |
| 住所情報：市区町村 | ユーザーの住所の市区町村。 |
| 住所情報：州 | ユーザーの住所の州コード。 |
| 住所情報：郵便番号 | ユーザーの住所の郵便番号。 |
| 住所情報：国コード | ユーザーの住所の2文字の国コード（ISO 3166-1 alpha-2形式）。 |

#### 暗号化

|パラメータ| 説明|
|---| ---|
|暗号化エンティティタイプ| 暗号化されたユーザーIDが指定されている場合に必要です。広告配信またはデータ転送の暗号化構成と一致する必要があります。 |
|暗号化エンティティID| 暗号化されたユーザーIDが指定されている場合に必要です。広告配信またはデータ転送の暗号化構成と一致する必要があります。 |
|暗号化ソース| 暗号化されたユーザーIDが指定されている場合に必要です。暗号化されたクッキーが広告配信から受け取られたかデータ転送から受け取られたかを説明します。 |

#### 数量と価値

|パラメータ| 説明|

#### カスタムデータ

必要に応じて`U`カスタム変数に属性をマッピングします。値が入力されているもののみが送信されます。
このセクションのマッピング先は、**カスタムデータ配列 - レガシー**セクションのマッピングを上書きします。

|パラメータ| 説明|
|---| ---|
|カスタム変数値| カスタム値のコレクション。各値は50文字を超えてはなりません。 |
|カスタム変数タイプ| `Uxxx`の形式の文字列のコレクション。ここで`xxx`は`1`から`100`までの数字です。 |

#### カートデータ

| **パラメータ** | **説明** |
| --- | --- |
| マーチャントID | アイテムがアップロードされるマーチャントセンターID。マーチャントセンターIDを提供することで、正確なオファーの詳細を特定する曖昧さが減ります。 |
| マーチャントフィードラベル | アイテムがアップロードされるフィードに関連するフィードラベル。 |
| マーチャントフィード言語 | アイテムがアップロードされるフィードに関連する言語。ISO 639-1言語コードを使用します。 |
| アイテムID | （必須）アイテムのショッピングID。マーチャントセンターの製品識別子と等しくなければなりません。 |
| 数量 | （必須）販売されたアイテムの数。 |
| 単価 | （必須）税、送料、および取引レベルの割引を除く単価。CM360 Floodlight構成の親広告主の通貨コードで解釈されます。 |
