MNTN Audienceコネクタを使用すると、TealiumからMNTNオーディエンスセグメントを作成および管理できます。このコネクタを使用して、バッチでファーストパーティのIDを同期し、接続されたテレビ（CTV）のターゲティング、リターゲティング、および抑制のためにオーディエンスメンバーを追加、更新、または削除できます。手動でのファイルアップロードやカスタムAPIコードは不要です。



## API情報

このコネクタは以下のベンダーAPIを使用します：

* API名：MNTN Audience API
* APIバージョン：v2026.0.1
* APIエンドポイント：`https://integrations.ex.mountain.com`

## 構成

コネクタマーケットプレイスにアクセスし、新しいコネクタを追加します。コネクタの追加方法については、[コネクタについて](https://docs.tealium.com/about-connectors/)を参照してください。

コネクタを追加した後、以下の構成を構成します：

* **MNTN Audience APIキー**：（必須）あなたのMNTN Audience APIキー。

## アクション

| アクション名 | AudienceStream | EventStream |
| ----------- | :------------: | :---------: |
| バッチでオーディエンスメンバーをアップサート | ✓ | ✓ |
| バッチでオーディエンスメンバーを削除 | ✓ | ✓ |

### バッチでオーディエンスメンバーをアップサート

#### バッチ制限

このアクションは、ベンダーへの高容量データ転送をサポートするためにバッチリクエストを使用します。並列処理により、イベントがベンダーに順不同で到達する可能性があります。順序が重要な場合は、イベントにシーケンス値を追加してください。詳細については、[バッチアクション](https://docs.tealium.com/batched-actions/)を参照してください。以下のいずれかの閾値に達するか、プロファイルが公開されるまでリクエストはキューに入れられます：

* 最大リクエスト数：10,000
* 最古のリクエストからの最大時間：60分
* リクエストの最大サイズ：2 MB

#### パラメータ

| パラメータ | 説明 |
| --- | --- |
| セグメントID | 既存のMNTNオーディエンスセグメントにIDを挿入または更新するためのID。 |
| ID ID | （必須）セグメント内のオーディエンスメンバーのユニークID（例：Tealium訪問IDまたはCRM ID）。 |

#### 識別子

| パラメータ | 説明 |
| --- | --- |
| Email (SHA256適用) | プレーンテキストのメールアドレス。コネクタは正規化（小文字化、トリム）を行い、SHA256ハッシュを適用します。 |
| Email (SHA256済み) | ハッシュ済みのメールアドレス。ハッシュする前に値を小文字化してトリムしてください。 |
| Phone (SHA256適用) | プレーンテキストの電話番号。コネクタは数字以外を削除し、国コードを先頭に付けてSHA256ハッシュを適用します。 |
| Phone (SHA256済み) | ハッシュ済みの電話番号。ハッシュする前に数字以外を削除し、10桁の米国番号の場合は`1`を先頭に付けてください。 |
| IPv4 | 生のIPv4アドレス。 |
| IPv4 (SHA256済み) | ハッシュ済みのIPv4アドレス。 |
| IPv6 | 生のIPv6アドレス。 |
| IPv6 (SHA256済み) | ハッシュ済みのIPv6アドレス。 |
| MAID | UUID形式の生のモバイル広告IDで、CTVまたはモバイルターゲティングに使用されます（例：`3f097372-f01e-4b64-984c-395ae5828ee6`）。 |


<blockquote>
IPv4 (SHA256済み)、IPv6、IPv6 (SHA256済み)、およびMAIDは、MNTNキャンペーンターゲティングで選択的にサポートされています。ターゲティングにこれらの識別子を使用する前に、MNTNアカウントチームに確認してください。
</blockquote>


### バッチでオーディエンスメンバーを削除

#### バッチ制限

このアクションは、ベンダーへの高容量データ転送をサポートするためにバッチリクエストを使用します。並列処理により、イベントがベンダーに順不同で到達する可能性があります。順序が重要な場合は、イベントにシーケンス値を追加してください。詳細については、[バッチアクション](https://docs.tealium.com/batched-actions/)を参照してください。以下のいずれかの閾値に達するか、プロファイルが公開されるまでリクエストはキューに入れられます：

* 最大リクエスト数：1,000
* 最古のリクエストからの最大時間：60分
* リクエストの最大サイズ：2 MB

#### パラメータ

| パラメータ | 説明 |
| --- | --- |
| オーディエンスセグメント | メンバーを削除するMNTNオーディエンスセグメントを選択するか、セグメントIDを手動で入力します。 |
| ID ID | （必須）セグメントから削除するオーディエンスメンバーのID ID（例：Tealium訪問IDまたはCRM ID）。 |

## ベストプラクティス

* 本番量にスケールする前に、小規模なテストセグメントから始めてください。
* リクエストペイロードを効率的に保ちます。MNTNは、バッチサイズを約2 MBにし、最大リクエストサイズの8 MBを下回ることを推奨します。
* 重複または矛盾するレコードを避けるために、一貫したID戦略を使用してください。
* MNTNに識別子を送信する前に、同意およびプライバシー要件を確認してください。

詳細については、[MNTNヘルプセンター：Tealiumを接続する](https://help.mountain.com/en/articles/14696352-connect-tealium)を参照してください。
