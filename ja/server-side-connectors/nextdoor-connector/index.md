## API情報

このコネクタは以下のベンダーAPIを使用します：

* API名：Nextdoor API
* APIバージョン：v2
* APIエンドポイント：`https://ads.nextdoor.com/v2/api`
* ドキュメント：[Nextdoor API](https://developer.nextdoor.com/reference/conversion-api)

## 構成

コネクタマーケットプレイスにアクセスし、新しいコネクタを追加します。詳細については、[コネクタについて]()を参照してください。

コネクタを追加した後、以下の構成を構成します：

* **APIトークン**：（必須）ログインしているユーザーとその関連アカウントに固有の認証トークンをNextdoor Ads Managerアカウントページで取得します。

## アクション

| アクション名 | AudienceStream | EventStream |
| --- | :---: | :---: |
| コンバージョン送信 | ✗ | ✓ |

アクション名を入力し、ドロップダウンメニューからアクションタイプを選択します。

次のセクションでは、各アクションのパラメータとオプションの構成方法について説明します。

### コンバージョン送信

#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| イベント名 | コンバージョン追跡イベントのタイプ。リストから値を選択するか、カスタム文字列値を入力します。利用可能な値には、`Add to Cart`、`Initiate Checkout`、`Search`、`Add to Wishlist`、`Subscribe`、`View Content`があります。|
| アクションソース | コンバージョンが発生するチャネル。ドロップダウンリストから値を選択します。 |

#### コンバージョンイベント

| **パラメータ** | **説明** |
| --- | --- |
| クライアントID | Nextdoor Ads Manager広告主ID。**データソースID**または**クライアントID**が必要です。 |
| データソースID | Nextdoor Ads ManagerデータソースID。**データソースID**または**クライアントID**が必要です。これにより、Conversions APIとピクセルを通じてイベントを送信する広告主のイベントの重複をNextdoorが排除できます。 |
| パートナーID | 単一のパートナーに関連付けられたNextdoor生成のID。この値はすべてのConversion APIコールで必要です。 |
| アクションソースURL | イベントが発生したブラウザのURL。URLは`http://`または`https://`で始まり、検証済みのドメインと一致する必要があります。この値はウェブサイトイベントに必要です。 |
| イベントID | 一意のイベントを識別できる文字列またはハッシュ化されたID。これは、CAPIとピクセル追跡からのイベントの重複排除に必要です。**イベントID**または**自動重複排除**が必要です。 |
| イベント時間 | [ISO 8601](https://en.wikipedia.org/wiki/ISO_8601)形式のイベントのタイムスタンプ。**イベント時間エポック**または**イベント時間**が必要です。 |
| イベント時間エポック | `1970-01-01`以降の秒数。ミリ秒は含まれません。**イベント時間エポック**または**イベント時間**が必要です。 |
| テストイベント | このイベントがテストイベントであるかどうかを示します。デフォルト値は`False`です。 |
| イベントタイムゾーン | コンバージョンイベントが発生した場所のタイムゾーン。 |
#### 顧客データ

統合には、以下のパラメータのうち少なくとも1つまたは組み合わせが必要です：**メールアドレス**、**電話番号**、または**クリックID**。

| **パラメータ** | **説明** |
| --- | --- |
| メールアドレス（既にSHA256ハッシュ化済み） | 既にSHA256ハッシュ化されたメールアドレスを提供してください。 |
| メールアドレス（SHA256ハッシュを適用） | 平文のメールアドレスを提供し、コネクタがこの値を空白トリム、小文字化し、SHA256ハッシュを使用してハッシュ化します。 |
| 外部ID | 広告主からのユニークID、例えばロイヤルティ会員ID、ユーザーID、外部クッキーIDなどです。特定のイベントに対して1つ以上の外部IDを送信できます。他のチャネルを通じて外部IDを送信する場合は、Conversions APIと同じ形式を使用してください。|
| 電話番号（既にSHA256ハッシュ化済み） | 既にSHA256ハッシュ化された電話番号を提供してください。 |
| 電話番号（SHA256ハッシュを適用） | 平文の電話番号を提供し、コネクタがこの値を空白トリムし、SHA256ハッシュを使用してハッシュ化します。 |
| 名（既にSHA256ハッシュ化済み） | 既にSHA256ハッシュ化された名前を提供してください。 |
| 名（SHA256ハッシュを適用） | 平文の名前を提供し、コネクタがこの値を空白トリム、小文字化し、SHA256ハッシュを使用してハッシュ化します。 |
| 姓（既にSHA256ハッシュ化済み） | 既にSHA256ハッシュ化された姓を提供してください。 |
| 姓（SHA256ハッシュを適用） | 平文の姓を提供し、コネクタがこの値を空白トリム、小文字化し、SHA256ハッシュを使用してハッシュ化します。 |
| 生年月日（既にSHA256ハッシュ化済み） | `YYYYMMDD`形式で既にSHA256ハッシュ化された生年月日を提供してください。 |
| 生年月日（SHA256ハッシュを適用） | 平文の生年月日を提供し、コネクタがこの値を空白トリムし、`YYYYMMDD`形式でSHA256ハッシュを使用してハッシュ化します。 |
| 住所（既にSHA256ハッシュ化済み） | 既にSHA256ハッシュ化された住所を提供してください。 |
| 住所（SHA256ハッシュを適用） | 平文の住所を提供し、コネクタがこの値を空白トリム、小文字化し、SHA256ハッシュを使用してハッシュ化します。 |
| 市区町村（既にSHA256ハッシュ化済み） | 既にSHA256ハッシュ化された市区町村を提供してください。 |
| 市区町村（SHA256ハッシュを適用） | 平文の市区町村を提供し、コネクタがこの値を空白トリム、小文字化し、SHA256ハッシュを使用してハッシュ化します。 |
| 州（既にSHA256ハッシュ化済み） | 2文字の[ANSI省略コード](https://en.wikipedia.org/wiki/Federal_Information_Processing_Standard_state_code)で、既に小文字化された州の省略形を提供してください。 |
| 州（SHA256ハッシュを適用） | 州の省略形を提供し、コネクタがこの値を空白トリム、小文字化し、2文字の[ANSI省略コード](https://en.wikipedia.org/wiki/Federal_Information_Processing_Standard_state_code)でSHA256ハッシュを使用してハッシュ化します。 |
| 郵便番号（既にSHA256ハッシュ化済み） | 既にSHA256ハッシュ化された郵便番号を提供してください。 |
| 郵便番号（SHA256ハッシュを適用） | 平文の郵便番号を提供し、コネクタがこの値を空白トリム、小文字化し、SHA256ハッシュを使用してハッシュ化します。 |
| 国（既にSHA256ハッシュ化済み） | [ISO 3166-1 alpha-2](https://en.wikipedia.org/wiki/ISO_3166-1_alpha-2)形式で、既に空白トリムされ、SHA256ハッシュ化された2文字の国コードを提供してください。 |
| 国（SHA256ハッシュを適用） | 平文の国コードを提供し、コネクタがこの値を空白トリム、小文字化し、[ISO 3166-1 alpha-2](https://en.wikipedia.org/wiki/ISO_3166-1_alpha-2)形式でSHA256ハッシュを使用してハッシュ化します。 |
| 性別（既にSHA256ハッシュ化済み） | 既にSHA256ハッシュ化された性別を提供してください。小文字の頭文字を使用します：女性の場合はf、男性の場合はm、ノンバイナリーの場合はnです。言及したくない場合は空欄にしてください。 |
| 性別（SHA256ハッシュを適用） | 平文の性別を提供し、コネクタがこの値を空白トリム、小文字化し、SHA256ハッシュを使用してハッシュ化します。小文字の頭文字を使用します：女性の場合はf、男性の場合はm、ノンバイナリーの場合はnです。言及したくない場合は空欄にしてください。 |
| Pixel ID | Nextdoor Pixel IDです。これにより、NextdoorはConversions APIとピクセルの両方を通じてイベントを送信する広告主のイベントを重複排除できます。 |
| クリックID | クリックスルーURLに付随する`ndclid`パラメータと同じです。 |
| クライアントIPアドレス | &lt;ul&gt;&lt;li&gt;イベントに対応するブラウザのIPアドレスは、有効なIPV4またはIPV6アドレスでなければなりません。IPV6が有効なユーザーにはIPV4よりもIPV6が望ましいです。&lt;/li&gt;&lt;li&gt;常に正確なイベントレポートを保証するために、実際のIPアドレスを提供してください。&lt;/li&gt;&lt;li&gt;この値をハッシュ化しないでください&lt;/li&gt;&lt;li&gt;すべてのイベントに対して**クライアントIPアドレス**と**ユーザーエージェント**のパラメータを送信することで、イベントマッチングとキャンペーンの広告配信が向上する可能性があります。&lt;/li&gt;&lt;li&gt;この情報はブラウザを通じて送信されるイベントに自動的に追加されますが、サーバーを通じて送信されるイベントの場合は手動で構成する必要があります。&lt;/li&gt;&lt;/ul&gt;|
| クライアントユーザーエージェント | イベントに対応する未ハッシュ化のブラウザユーザーエージェントです。このパラメータは、[Conversions API](https://developers.facebook.com/docs/marketing-api/conversions-api)を使用して共有されるウェブサイトイベントに必要です。すべてのイベントに対して**クライアントIPアドレス**と**ユーザーエージェント**のパラメータを送信することで、イベントマッチングとキャンペーンの広告配信が向上する可能性があります。この情報はブラウザを通じて送信されるイベントに自動的に追加されますが、サーバーを通じて送信されるイベントの場合は手動で構成する必要があります。 |
| データソースID | Nextdoor広告マネージャーデータソースIDです。**データソースID**または**クライアントID**が必要です。これにより、NextdoorはConversions APIとピクセルの両方を通じてイベントを送信する広告主のイベントを重複排除できます。 |
| 配信最適化 | &lt;ul&gt;&lt;li&gt;`True`: データを最適化に使用できることを示します。&lt;/li&gt;&lt;li&gt;`False`: データは帰属のみに使用されることを示します。&lt;/li&gt;&lt;/ul&gt; |

#### カスタムおよびアプリイベント

| **パラメータ** | **説明** |
| --- | --- |
| 注文値 | 購入イベントに必要です。[ISO 4217](https://en.wikipedia.org/wiki/ISO_4217)形式の通貨コードに続いて、イベントに関連する合計数値値（小数点以下2桁まで）を指定します。 |
| 注文ID | トランザクションの注文IDです。オフラインイベントにはこの値が必要です。 |
| アプリID | 広告主のユニークIDに対応します。App StoreおよびGoogle Play StoreからモバイルアプリIDを使用してください。 |
| プラットフォーム | ユーザーのプラットフォームです。値は`iOS`または`Android`になります。 |
| アプリバージョン | モバイルアプリのバージョンです。 |
| 配信カテゴリ | &lt;ul&gt;&lt;li&gt;`In Store`: 顧客が購入した商品を受け取るために店内に入る必要があります。&lt;/li&gt;&lt;li&gt;`Curbside`: 顧客が店に車で行き、車内で待機して注文を受け取ります。&lt;/li&gt;&lt;li&gt;`Home Delivery`: 購入した商品が顧客の自宅に配送されます。&lt;/li&gt;&lt;/ul&gt; |

#### 製品コンテキスト

| **パラメータ** | **説明** |
| --- | --- |
| ID | (必須) 製品ID。 |
| コンテンツ名 | 製品名。 |
| 数量 | 製品数量。 |
| アイテム価格 | 製品のアイテム価格。 |
| アプリ追跡有効 | ATTのユーザーオプトアウト構成です。アプリイベントの追跡を行い、Conversions APIを通じてNextdoorに送信するかどうかを指定してください。&lt;ul&gt;&lt;li&gt;`False`: アプリイベントの追跡を無効にします。Nextdoorにアプリイベントは送信されません。&lt;/li&gt;&lt;li&gt;`True`: アプリイベントの追跡を有効にします。アプリイベントはNextdoorに送信され、帰属および測定のために使用されます。&lt;/li&gt;&lt;/ul&gt;  |

| **パラメータ** | **説明** |

| 自動重複排除 | Tealium iQタグIDを提供すると、コネクタはTealium iQが送信する**イベントID**値を自動的に検索します。 |
