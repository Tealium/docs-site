## 構成

コネクタマーケットプレイスにアクセスし、新しいコネクタを追加します。コネクタを追加する一般的な手順については、[コネクタについて]()を参照してください。

コネクタを追加した後、以下の構成を構成します：

* **Agency Secret Key**: The Trade Deskから提供される秘密鍵。
* **Merchant ID**: （オプション）The Trade Deskのマーチャントアカウントに関連付けられたID。The Trade DeskがこのIDを提供します。

## アクション

| **アクション名** | **AudienceStream**| **EventStream**|
|---| ---| ---|
| オフラインコンバージョンを送信 | ✓ | ✓ |

アクションの名前を入力し、ドロップダウンメニューからアクションタイプを選択します。

次のセクションでは、各アクションのパラメータとオプションの構成方法について説明します。

### オフラインコンバージョンを送信

#### バッチ制限

このコネクタは、ベンダーへの大量データ転送をサポートするためにバッチリクエストを使用します。詳細については、[バッチアクション]()を参照してください。リクエストは、次のいずれかのしきい値に達するか、プロファイルが公開されるまでキューに入れられます：

* 最大リクエスト数：1000
* 最古のリクエストからの最大時間：10分
* リクエストの最大サイズ：1 MB

#### データプロバイダーID

|**パラメータ**| **説明**|
|---| ---|
| データプロバイダーID | TTDシステム内のデータプロバイダーのID。The Trade Deskのテクニカルアカウントマネージャーがこの値を提供できます。 |

#### データID

* Trade Desk ID、デバイス広告ID、Unified ID、またはIdentityLinkをマッピングします。
* 複数のIDがマッピングされている場合、最初の空でないIDが次の優先順位で選択されます：Trade Desk ID、デバイス広告ID、Unified ID、IdentityLink。
* デバイス広告IDは、iOS IDFAまたはAndroid Advertising ID (AAID)で表されます。
* ハッシュされていないIDFAおよびAAIDのIDのみが、ダッシュを含む36文字のGUIDとして受け入れられます。

|**パラメータ**| **説明**|
|---| ---|
| The Trade Desk ID (TDID) (`TDID`) | このユーザーのThe Trade Desk 36文字のGUID（ダッシュを含む）。 |
| デバイス広告ID (`DAID`) | このユーザーの生のデバイスIDで、36文字のGUID形式（ダッシュを含む）で送信します。iOS IDFAまたはAndroidのAAIDを使用します。 |
| Unified ID (`UID2`) | 生のUID2値、別名UID2。この値は大文字と小文字を区別します。生のUID2はUID2 APIを使用して生成および管理されます。 |
| UID2トークン（暗号化された広告トークン） | 暗号化されたUID2広告トークン。このトークンは大文字と小文字を区別します。 |
| IdentityLink (`IDL`) | 49文字または70文字のRampID（以前はIdentityLinkとして知られていました）。これはLiveRampからThe Trade Desk用にマッピングされたRampIDでなければなりません。 |

#### アイテムデータ

|**パラメータ**| **説明**|
|---| ---|
| トラッキングタグID | （必須）トラッキングタグAPIを使用して作成されたトラッキングタグID。例：`abc1234` |
| コンバージョンタイムスタンプ | （必須）UTCでのコンバージョン日時。オフラインコンバージョンが発生した時間。例：`2019-02-07T13:25:00Z` |
| インプレッションID | ダッシュを含む36文字の生の（ハッシュされていない）GUID。 |
| 金額 |  金額。例：`0.02` |
| 通貨 |  金額の通貨。例：`USD` |
| 国 |  国。例：`United States`。 |
| 地域 |  地域。例：`New York`。 |
| メトロエリア |  メトロエリア。例：`501` |
| 市 |  市。例：`Brooklyn`。|
| 郵便番号 | 郵便番号。例：`&#39;11223&#39;`。 |
| イベント名 | パートナープラットフォームによって定義されるイベントのタイプ。詳細については、[The Trade Desk: イベントマッピング](https://partner.thetradedesk.com/v3/portal/data/doc/post-providerapi-offlineconversion#event-mapping)を参照してください。  |
| 注文ID | トランザクションまたはコンバージョンイベントの一意の識別子で、最大64文字です。 |

#### 追加データ

|**パラメータ**| **説明**|
|---| ---|
| `TD1 - TD10` |  カテゴリーやその他の組織用のオプション報告フィールドで、最大64文字です。|

#### ラインアイテム

マーチャントの製品カタログラインアイテム情報を含むプロパティ。これらのマッピングは配列属性を受け入れます。

|**パラメータ**| **説明**|
|---| ---|
| アイテムコード | アイテム識別子（例：SKU）。 |
| アイテム名 | アイテムコードに関連付けられた名前で、最大150文字です。 |
| アイテム数量 | アイテムコードの注文内のアイテム数。 |
| アイテム価格 | アイテムの単価。 |
| アイテムカテゴリ | アイテムカテゴリID。 |
