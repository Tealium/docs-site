Adformセグメントコネクタ構成ガイド

この記事では、Adformセグメントコネクタの構成方法について説明します。

API情報

このコネクタは、以下のベンダーAPIを使用します：

  • API名：Adform API
  • APIバージョン：v1
  • APIエンドポイント：https://api.adform.com/v1/dmp
  • ドキュメンテーション：Adform API

コネクタアクション

アクション名 AudienceStream EventStream
ユーザーをオーディエンスセグメントに追加 ✓ ✓
ユーザーをオーディエンスセグメントから削除 ✓ ✓

構成の構成

コネクタマーケットプレイスに移動し、新しいコネクタを追加します。コネクタの追加方法については、コネクタ概要の記事を参照してください。

コネクタを追加したら、以下の構成を構成します：

  • クライアントID
    必須。クライアントIDを提供してください。
  • クライアントシークレット
    必須。クライアントシークレットを提供してください。

コネクタの構成が完了したら、完了をクリックします。

オーディエンスセグメント

セグメントは作成および削除が可能で、ユーザーをオーディエンスセグメントに追加したり、オーディエンスセグメントからユーザーを削除したりすることができます。AudienceStreamはAPIエンドポイントを利用して、Tealium AudienceStreamインスタンス内のセグメントを取得、作成、削除します。

デフォルトでは、動的にオーディエンスを作成する機能は利用できません。この機能を使用するには、Adformの管理者にこの機能を利用可能にするよう依頼してください。

動的なオーディエンス生成を利用可能にしたり、すべての動的に生成されたオーディエンスのデフォルト構成を調整したりすることは、アカウント管理者のみが可能です。管理者は価格を構成し、生存期間（TTL）を日数で指定し、データタイプを指定することができます。

セグメントを作成するには、以下のパラメーターについてセグメントを作成をクリックします：

パラメーター

パラメーター 説明
オーディエンス名 (必須) Adformに表示されるオーディエンスの名前。
AdformパートナーID (必須) あなたのアカウントのAdformパートナーID。
カテゴリーID (必須) オーディエンスが所属するべきカテゴリーID。
オーナーID (必須) オーディエンスのオーナーIDの値。
価格 (必須) オーディエンスごとに構成する価格。
データタイプ データ共有計画に応じてデータタイプを構成します：
  • 1st Party：データはあなただけが使用します。
  • 2nd Party：データをパートナーまたはパートナーと共有する予定です。
  • 3rd Party：データをブランドデータプロバイダーとして全員と共有する予定です。
生存期間（TTL） (必須) オーディエンスセグメントのデータの寿命。
頻度 (必須) 識別子が資格を得る回数。この数に達すると、定義された基準がオーディエンスリストに追加されます。

セグメントを削除するには、オーディエンスを削除をクリックし、削除するオーディエンスセグメントを選択します。

また、セグメントのステータスは、オーディエンスセグメント名の隣の括弧内に表示されます。

例：カート放棄者（アクティブ）

アクション構成 - パラメーターとオプション

続行をクリックしてコネクタアクションを構成します。アクションの名前を入力し、ドロップダウンメニューからアクションタイプを選択します。

Adformセグメントコネクタは、UIDパラメーターの送信をサポートしていません。UIDパラメーターが必要な場合、Adformは彼らのTealiumタグの使用を提案します。

アクション構成 - パラメーターとオプション

続行をクリックしてコネクタアクションを構成します。アクションの名前を入力し、ドロップダウンメニューからアクションタイプを選択します。

以下のセクションでは、各アクションのパラメーターとオプションの構成方法について説明します。

アクション - ユーザーをオーディエンスセグメントに追加

パラメーター

パラメーター 説明
オーディエンスセグメント (sg) オーディエンスセグメントを選択するか、オーナーIDを入力します。
AdformパートナーID (必須) あなたのアカウントのAdformパートナーID。
GAID (gaid) Google広告訪問ID属性
IDFA (idfa) Apple広告主向け識別子
ファーストパーティデータソース 含まれるIDのソースまたは技術プロバイダーを示すテキスト。これはトップレベルドメインであるべきです。例えば、netid.de
ファーストパーティエージェントタイプ ユーザー識別子がどのタイプのユーザーエージェントから収集されたかを識別する整数。可能なエージェントタイプの値：
  • 1 - デスクトップデバイスのブラウザから収集。
  • 2 - モバイルデバイスのアプリ（インアプリ）から収集（例：IDFAまたはAAID）。
  • 3 - 複数のデバイスとアプリケーションで同じユーザーベースのID。
  • 500+ - ベンダー固有のコンテキストとID、ベンダー固有のコード。
ファーストパーティID ソースプロバイダーからのファーストパーティIDの識別子。これは長い文字列またはGUIDである可能性があります。

パラメーター

パラメーター 説明
オーディエンスセグメント (sg) オーディエンスセグメントを選択するか、オーナーIDを入力します。
AdformパートナーID (必須) あなたのアカウントのAdformパートナーID。

ファーストパーティID

Adformは、現在利用可能なすべてのファーストパーティIDソリューションを受け入れます。ファーストパーティIDは、呼び出しで提供されるオーディエンスに割り当てられます。

以下のファーストパーティIDがサポートされています：

プロバイダー名 ソース名
NetID netid.de
Britepool britepool.com
Liveramp (IdentityLink) liveramp.com
PubCommonID id5-sync.com
Lotame (Panorama ID) crwdcntrl.net
Unified ID 2.0 uidapi.com
Prebid Shared ID sharedid.org
あなたの独自のID your-top-level-domain.com この値は、訪問のスティッチングに使用されるTealium訪問IDをAdformに送信するためのマッピングとなる可能性があります。
