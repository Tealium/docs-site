## API情報

このコネクタは以下のベンダーAPIを使用します：

* API名：Viant API
* APIバージョン：v1
* APIエンドポイント：`https://vdp-connect-api.viantinc.com`
* ドキュメント：[Viant API](https://docs.api.viantinc.com/reference/how-to-get-started)

## 構成

コネクタマーケットプレイスにアクセスし、新しいコネクタを追加します。コネクタの追加方法については、[コネクタについて]()を参照してください。

コネクタを追加した後、以下の構成を構成します：

* **クライアントID**：必須。Viant VDP API認証用のクライアントID。
* **クライアントシークレット**：必須。Viant VDP API認証用のクライアントシークレット。
* **アカウントID**：必須。ViantアカウントID。
* **広告主ID**：必須。Viant広告主ID。

## アクション

| アクション名 | AudienceStream | EventStream |
| --- | :---: | :---: |
| コンバージョンイベント送信 | ✓ | ✓ |
| ヘルスケアコンバージョンイベント送信 | ✓ | ✓ |

アクションの名前を入力し、ドロップダウンメニューからアクションタイプを選択します。

次のセクションでは、各アクションのパラメータとオプションの構成方法について説明します。

### コンバージョンイベント送信

#### バッチ制限

このアクションは、ベンダーへの大量データ転送をサポートするためにバッチリクエストを使用します。並列処理により、イベントがベンダーに順不同で到達する可能性があります。順序が重要な場合は、イベントにシーケンス値を追加してください。詳細については、[バッチアクション]()を参照してください。次のいずれかの閾値に達するか、プロファイルが公開されるまでリクエストはキューに入れられます：

* 最大リクエスト数：1,000
* 最古のリクエストからの最大時間：10分
* リクエストの最大サイズ：10 MB

#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| コンバージョンイベントタイプ | **PageView** - ページビューイベント&lt;br&gt;**LandingPage** - ランディングページイベント&lt;br&gt;**ItemView** - アイテムビューイベント&lt;br&gt;**AddToCart** - カートに追加イベント&lt;br&gt;**InitiateCheckout** - チェックアウト開始イベント&lt;br&gt;**AddPaymentInfo** - 支払情報追加イベント&lt;br&gt;**Purchase** - 購入イベント&lt;br&gt;**Lead** - リードイベント |

#### 売上データ

| **パラメータ** | **説明** |
| --- | --- |
| トランザクションID | トランザクションの一意の識別子。自動マッピングが適用され、属性をオーバーライドするためにマップします。 |
| コンバージョンタイムスタンプ | コンバージョンのISO 8601タイムスタンプ。例：`2025-07-28T15:32:00Z`。自動マッピングが適用され、属性をオーバーライドするためにマップします。 |
| ストアID | ストアまたはロケーションの一意の識別子。 |
| 金額 | 総コンバージョンまたは取引金額。自動マッピングが適用され、属性をオーバーライドするためにマップします。 |
| 通貨 | 通貨コード。例：`USD`, `EUR`。自動マッピングが適用され、属性をオーバーライドするためにマップします。 |
| 販売チャンネル | 販売チャンネル。例：`Online`, `In-store`。 |

#### 識別子

| **パラメータ** | **説明** |
| --- | --- |
| Email Apply SHA256 | 顧客のプレーンテキストのメールアドレス。コネクタは正規化を行い、SHA256ハッシュを適用します。 |
| Email Already SHA256 | SHA256ハッシュ化された顧客のメールアドレス。 |
| Address Apply SHA256 | 顧客のプレーンテキストの住所。コネクタは正規化を行い、SHA256ハッシュを適用します。 |
| Address Already SHA256 | SHA256ハッシュ化された顧客の住所。 |
| IPアドレス | 顧客の`IP`アドレス。 |
| モバイルID | 顧客のモバイル`ID`。 |
| Cookie | イベントクッキー。 |

#### 購入アイテム

| **パラメータ** | **説明** |
| --- | --- |
| アイテムID | (アイテムが存在する場合は必須) 商品SKUの一意の識別子。 |
| 商品名 | 商品の人間が読める名前。 |
| 商品カテゴリ | 商品のカテゴリ。 |
| 価格 | (アイテムが存在する場合は必須) 単位あたりの価格。 |
| 数量 | (アイテムが存在する場合は必須) 購入数量。 |
| カスタムデータ | 追加のコンテキスト用の最大10個のカスタムキー値メタデータペア。例：`キャンペーンID`, `ユーザーセグメント`。 |

#### コンバージョン場所

| **パラメータ** | **説明** |
| --- | --- |
| コンバージョン国 | コンバージョン場所の国。 |
| コンバージョン州 | コンバージョン場所の州。 |
| コンバージョン市 | コンバージョン場所の市。 |
| 自動マッピング無効化 | コネクタの自動マッピングを無効にします。 |

### ヘルスケアコンバージョンイベント送信

#### バッチ制限

このアクションは、ベンダーへの大量データ転送をサポートするためにバッチリクエストを使用します。並列処理により、イベントがベンダーに順不同で到達する可能性があります。順序が重要な場合は、イベントにシーケンス値を追加してください。詳細については、[バッチアクション]()を参照してください。次のいずれかの閾値に達するか、プロファイルが公開されるまでリクエストはキューに入れられます：

* 最大リクエスト数：1,000
* 最古のリクエストからの最大時間：10分
* リクエストの最大サイズ：10 MB

#### パラメータ

| パラメータ | 説明 |
| --- | --- |
| ヘルスケアイベント名 | (必須) Viantに送信する匿名化されたイベントコード。例：`Page View`, `Event 1`, `Event 2`。HIPAA要件に準拠するために、説明的なイベント名ではなく短い不透明なコードを使用します。 |

#### 売上データ

| パラメータ | 説明 |
| --- | --- |
| トランザクションID | (必須) トランザクションの一意の識別子。Impression IDとClick IDをグループ化するために使用されます。 |
| コンバージョンタイムスタンプ | コンバージョンのISO 8601タイムスタンプ。例：`2025-07-28T15:32:00Z`。提供されていない場合は現在の時間が使用されます。 |
| ストアID | ストアまたはロケーションの一意の識別子。 |
| 販売チャンネル | 販売チャンネル。例：`Online`, `In-store`。 |

#### ヘルスケア識別子

| パラメータ | 説明 |
| --- | --- |
| ViantインプレッションID | クリエイティブマクロ`${ADELPHIC_IMPRESSIONID}`からのViant匿名インプレッショントラッキングID。ビュー後のコンバージョン帰属に使用されます。 |
| ViantクリックID | クリエイティブマクロ`${ADELPHIC_CLICKID}`からのViant匿名クリックトラッキングID。クリック後のコンバージョン帰属に使用されます。 |
| カスタム識別子 | (オプション) 顧客定義の匿名キー。例：内部エンカウンターまたはファネルID。 |
| カスタムデータ | (オプション) 追加のコンテキスト用のキーバリューメタデータペア。`healthcare_capi`フラグが自動的に含まれます。 |

#### コンバージョン場所

| パラメータ | 説明 |
| --- | --- |
| コンバージョン国 | コンバージョン場所の国。 |
| コンバージョン州 | コンバージョン場所の州。 |
| コンバージョン市 | コンバージョン場所の市。 |
