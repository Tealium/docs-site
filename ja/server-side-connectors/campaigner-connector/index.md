## 構成

コネクタマーケットプレイスにアクセスし、新しいコネクタを追加します。コネクタを追加する一般的な手順については、[コネクタについて](https://docs.tealium.com/about-connectors/)を参照してください。

コネクタを追加した後、以下の構成を構成します：
* **API キー**  
  * Campaigner の API キー。購読者を追加、更新、または削除するために必要です。
* **インテグレーションキー**  
  * Campaigner のインテグレーションキー。Upsert contact アクションに必要です。

## アクション

| アクション名 | AudienceStream | EventStream |
| --- | :---: | :---: |
| Upsert contact | ✓ | ✗ |
| 購読者を追加 | ✓ | ✗ |
| 購読者を更新 | ✓ | ✗ |
| 購読者を削除 | ✓ | ✗ |

アクションの名前を入力し、ドロップダウンメニューからアクションタイプを選択します。

次のセクションでは、各アクションのパラメータとオプションの構成方法について説明します。

### Upsert contact

#### 連絡先

| **パラメータ** | **説明** |
| --- | --- |
| 再購読 | `true`に構成すると、連絡先をメール、SMSチャネル、または両方に再購読させます。 |
| Contact Id | Campaigner によって割り当てられた連絡先 ID。提供されている場合、この連絡先の情報が更新されます。省略された場合、主キーフィールドが更新または挿入を決定します。 |

#### カスタムフィールド

以下のフィールドに対して文字列の配列をマッピングします。文字列がマッピングされた場合、サイズ1の配列として扱われます。

| **パラメータ** | **説明** |
| --- | --- |
| フィールドID | カスタムフィールドの一意の識別子。 |
| 値 | 連絡先のカスタムフィールドの値。 |
| リストID | この購読者に関連付けるリストID。チェックボックスを選択してリストを割り当てます。 |

#### 固定フィールド

| **パラメータ** | **説明** |
| --- | --- |
| メール | 連絡先のメールアドレス。 |
| 名 | 連絡先の名前。 |
| 姓 | 連絡先の姓。 |
| 電話 | 連絡先の電話番号。 |
| 携帯電話 | 連絡先の携帯電話番号。 |
| ファックス | 連絡先のファックス番号。 |
| メールメッセージ形式 | 連絡先の希望メールメッセージ形式。 |
| メール状態 | 連絡先のメール状態。 |
| SMS状態 | 連絡先のSMS状態。 |
| テスト連絡先か | この連絡先がテスト連絡先としてマークされているかどうか。 |
| オーナーメール | 連絡先のオーナーのメールアドレス。 |
| オーナー名 | 連絡先のオーナーの名前。 |
| オーナー姓 | 連絡先のオーナーの姓。 |

#### トリガーワークフロー

| **パラメータ** | **説明** |
| --- | --- |
| すべてのワークフローをトリガー | この連絡先が作成または更新された後にCampaignerが適用可能なすべてのワークフローをトリガーする場合は `true`に構成します。デフォルトは `false`です。 |
| 特定のワークフローをトリガー | この連絡先の作成または更新のためにトリガーするワークフローIDの配列。`triggerAllWorkflows`が `true`に構成されていて`triggerSpecificWorkflows`に値が指定されている場合、エラーが発生します。 |

### 購読者を追加

#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| メールアドレス | 購読者のメールアドレス。 |

#### カスタムフィールド

以下のフィールドに対して文字列の配列をマッピングします。文字列がマッピングされた場合、サイズ1の配列として扱われます。

| **パラメータ** | **説明** |
| --- | --- |
| フィールド名 | データベースの列名。 |
| 値 | データベースの列の値。 |
| ソースID | この購読者に関連付けるソースのID。提供されていない場合、最初のソースが使用されます。 |
| 公開物 | 購読者に関連付ける公開物ID。公開物が指定されていない場合、購読者は非アクティブとなりキャンペーンを受信しません。購読者が以前に公開物から削除された場合、Force publicationsが `true`に構成されていない限り、再追加されません。 |
| リストID | この購読者に関連付けるリストID。 |

#### 注文データ

| **パラメータ** | **説明** |
| --- | --- |
| SKU | 商品のSKU。 |
| 数量 | 購入したアイテムの数。デフォルトは `1`です。 |
| 重量 | 注文アイテムの総重量。デフォルトは `0`です。 |
| 状態 | 注文アイテムの状態。 |
| 注文番号 | 注文番号。 |
| 購入日 | 注文が行われた日付を `YYYY-MM-DD` 形式（UTC）で。提供されていない場合、現在の日付がデフォルトです。 |
| 商品名 | 商品の名前。 |
| 単価 | 単位あたりの価格。デフォルトは `0`です。 |

#### 強制

| **パラメータ** | **説明** |
| --- | --- |
| 強制 | 以前に削除されたりバウンスした購読者を追加する場合は `true`に構成します。`false`または提供されていない場合、購読者は追加されず、`HTTP 400 Bad Request`エラーが返されます。オプトアウトした購読者を追加しないように注意してこのオプションを使用してください。 |
| 公開物を強制 | 以前に削除を要求した公開物にこの購読者を強制的に戻す場合は `true`に構成します。 |
