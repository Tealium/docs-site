## API情報

このコネクタは以下のベンダーAPIを使用します：

* API名：TikTok HTTP API
* APIバージョン：v1.3
* APIエンドポイント：`https://business-api.tiktok.com/open_api/v1.3`
* ドキュメント：[TikTok オフラインイベントAPI](https://ads.tiktok.com/marketing_api/docs?id=1758427539356673)

## 構成の構成

コネクタマーケットプレイスにアクセスして新しいコネクタを追加します。コネクタを追加する一般的な手順については、[コネクタについて]()を参照してください。

コネクタを追加した後、以下の構成を構成します：

* **アクセストークン**：（必須）TikTokで構成されたアクセストークン。
* **広告主ID**：（必須）TikTok広告マネージャーからの広告アカウントID。詳細については、[TikTok: 広告アカウントIDの検索方法](https://ads.tiktok.com/help/article/find-your-ad-account-id?lang=en)を参照してください。

## アクション

| アクション名 | AudienceStream | EventStream |
| --- | :---: | :---: |
| オフラインイベントの送信（リアルタイム） | ✗ | ✓ |
| オフラインイベントの送信（バッチ処理） | ✗ | ✓ |

**続行**をクリックしてコネクタアクションを構成します。アクションの名前を入力し、**アクションタイプ**を選択します。

次のセクションでは、各アクションのパラメータとオプションの構成方法について説明します。

### オフラインイベントの送信（リアルタイム）

メールアドレスまたは電話番号のパラメータの提供を推奨します。

**コンテンツ名**、**コンテンツタイプ**、**コンテンツカテゴリ**、**コンテンツID**、**数量**、**価格**に対してデータタイプ属性を提供して、複数のアイテムを追加します。配列データタイプ属性は同じ長さでなければなりません。

単一値属性を使用することができ、それぞれのアイテムに適用されます。

**Tealium iQ Tag ID**を構成すると、コネクタはTealium iQタグ管理から送信された**イベントID**値を使用します。

#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| イベント名 | （必須）変換イベント名。 |
| カスタムイベント名 | 変換イベント名。 |
| イベントセットID | イベントセットID。 |
| イベントID | 各イベントの一意の値。このIDを使用してTikTokとデータを照合できます。 |
| タイムスタンプ | ISO 8601形式のイベントのタイムスタンプ。例：`2019-06-12T19:11:01:152Z`。タイムスタンプが提供されていない場合は、現在のタイムスタンプが使用されます。 |
| メール（既にSHA256ハッシュ化されている） | すでにSHA256ハッシュ化されているクリアテキストのメールアドレス（`@`記号を含む必要があります）。 |
| メール（SHA256ハッシュを適用） | プレーンテキストのメールアドレス。コネクタはこの値を空白トリム、小文字化し、SHA256ハッシュを使用してハッシュ化します。 |
| 電話番号（既にSHA256ハッシュ化されている） | SHA256ハッシュ化された電話番号を提供します。 |
| 電話番号（SHA256ハッシュを適用） | プレーンテキストの電話番号。コネクタはこの値を空白トリムしてハッシュ化します。 |
| 注文ID | トランザクションの注文ID。 |
| ショップID | トランザクションのショップID。 |
| 通貨 | 通貨コード、ISO 4217。 |
| 価値 | 推奨。全コンテンツの収益。 |
| イベントチャンネル | 推奨。イベントチャンネル。`email`、`website`、`phone_call`、`in_store`、`crm`、`other`をサポートします。 |
| コンテンツ名 | 製品またはコンテンツ名。複数のコンテンツを追加するために配列データタイプ属性を使用します。 |
| コンテンツタイプ | 製品またはコンテンツタイプ。 |
| コンテンツカテゴリ | 製品またはコンテンツカテゴリ。 |
| コンテンツID | 製品またはコンテンツ識別子。 |
| 数量 | アイテム数。 |
| 価格 | 製品またはコンテンツの価格。 |
| Tealium iQ Tag ID | Tealium iQからのイベントIDを使用します。 |

### オフラインイベントの送信（バッチ処理）

メールアドレスまたは電話番号のパラメータの提供を推奨します。

**コンテンツ名**、**コンテンツタイプ**、**コンテンツカテゴリ**、**コンテンツID**、**数量**、**価格**に対してデータタイプ属性を提供して、複数のアイテムを追加します。配列データタイプ属性は同じ長さでなければなりません。

単一値属性を使用することができ、それぞれのアイテムに適用されます。

**Tealium iQ Tag ID**を構成すると、コネクタはTealium iQタグ管理から送信された**イベントID**値を使用します。

#### バッチ制限

このアクションは、ベンダーへの大量データ転送をサポートするためにバッチリクエストを使用します。詳細については、[バッチアクション]()を参照してください。リクエストは、次のいずれかの閾値が達成されるか、プロファイルが公開されるまでキューに入れられます：

* 最大リクエスト数：100
* 最古のリクエストからの最大時間：10分
* リクエストの最大サイズ：10MB
