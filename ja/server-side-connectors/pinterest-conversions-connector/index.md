## 要件

* コンバージョンイベントを送信する権限を持つ Pinterest 広告アカウント。
* `user_data` で以下の有効なユーザーIDのいずれかを送信する能力:
    * Email（既に SHA256 ハッシュ化済み）(`em`)。
    * Appleの広告識別子（既に SHA256 ハッシュ化済み）または Google 広告 ID（既に SHA256 ハッシュ化済み）(`hashed_maids`)。
    * クライアント IP アドレスとクライアント ユーザーエージェント (`client_ip_address` と `client_user_agent`)。
    * アプリ ID（該当する場合）。  Pinterest は少なくとも1つの有効なユーザーIDが必要です。PII制限がある組織は、この統合を構成する前に承認を得る必要があります。
* オフラインイベントの場合:
    * コネクタ構成で **イベントアクションソース** を `Offline` に構成する必要があります。
    * オフラインイベントには、必要なすべてのイベントパラメータを含める必要があります。詳細については、[Pinterest: サーバーイベントパラメータの形式](https://developers.pinterest.com/docs/track-conversions/track-conversions-in-the-api/#format-server-event-parameters)を参照してください。

## API 情報

このコネクタは以下のベンダー API を使用します:

* API 名: Pinterest API
* API バージョン: v5
* API エンドポイント: `https://api.pinterest.com/`
* ベンダー文書: [Pinterest: コンバージョンガイド](https://developers.pinterest.com/docs/conversions/conversion-management/)

## バッチ制限

このコネクタは、ベンダーへの高容量データ転送をサポートするためにバッチリクエストを使用します。詳細については、[バッチアクション]()を参照してください。リクエストは、次のいずれかの閾値が満たされるか、プロファイルが公開されるまでキューに入れられます:

* 最大リクエスト数: 1000
* 最古のリクエストからの最大時間: 10分
* リクエストの最大サイズ: 2 MB

## 動作方法

Pinterest API for Conversions は、Pinterest タグを使用せずに広告主が Pinterest にコンバージョンイベントを送信することを可能にします。コンバージョンを Pinterest キャンペーンにマッピングすることで、キャンペーンの効果とコンバージョンを駆動するアクションのより正確なビューが提供されます。このアプローチは、ターゲティングの最適化、透明性の向上、およびレポートと分析の改善を促進します。

Pinterest API for Conversions は、ウェブ、インアプリ、またはオフラインのコンバージョンを受け取ることができます。イベント発生から1時間以内に受信されたイベントは、ウェブまたはアプリイベントとして報告されます。そのウィンドウの外で受信されたイベントは、オフラインイベントとして分類されます。

Pinterest Conversions コネクタと Pinterest タグを併用することで、可視性を最大化し、コンバージョンを追跡することをお勧めします。Pinterest Conversions コネクタは単独のソリューションとして利用可能ですが、両方を併用することで最良の結果が得られます。

追跡可能なイベントには以下が含まれます:

* **カートに追加**: アイテムがショッピングカートに追加されます。
* **チェックアウト**: トランザクションが完了します。
* **カスタム**: カスタムイベントを追跡します。このイベント名を使用して、コンバージョンレポートに含めたい特別なイベントを追跡します。
* **リード**: 製品またはサービスへの関心。
* **ページ訪問**: 製品ページや記事ページなどの主要ページの閲覧。
* **検索**: あなたのウェブサイトでの検索。
* **サインアップ**: あなたの製品やサービスへのサインアップ。
* **カテゴリ表示**: カテゴリページの閲覧。
* **ビデオ視聴**: ビデオ視聴。
* **ユーザー定義イベント**: オーディエンスターゲティングのために定義した任意の追加イベント。ユニークなイベントはコンバージョンレポートで利用できません。

## 重複排除

冗長な実装が存在する場合（例えば、Pinterest API for Conversions と [Pinterest タグ]()）、重複排除が必要です。重複排除を使用することで、コネクタまたはタグソリューションが独立して使用される場合よりも多くのコンバージョンが帰属されることが期待されます。

[Pinterest タグ]() と Pinterest API for Conversions は、重複排除のためにイベント ID を使用します。複数のソースを使用してコンバージョンデータを追跡する場合、この値を含めることを確認してください。イベント ID を見つけるには、次の手順を完了してください:

1. [Pinterest ビジネスアカウント](http://pinterest.com/business/create/)にログインします。
1. **広告 &gt; レポート**に移動します。
1. テーブルの上部右上にあるメニューアイコンをクリックし、**パフォーマンス**を選択します。

サイトに追加したイベントコードに応じて、選択した期間中に発生した回数とともにテーブル全体にリストされます。

次の命名規則を使用してイベント属性を探します:

```none
pinterest_event_id_{PINTEREST_EVENT}_{TAG_UID}
```

例: `pinterest_event_id_Checkout_32`

各タイプのイベント ID に対して別のアクションを使用します。イベント固有のイベント ID 属性をイベント ID にマッピングし、一致するイベント名をイベント名にマッピングします。例えば:

| **Tealium 属性/値** | Pinterest パラメータ |
|---|---|
| `pinterest_event_id_Checkout_32` | イベント ID |
| `Checkout` (ドロップダウン選択) | イベント名 |

詳細については、[Pinterest タグ構成ガイド]()を参照してください。

### 重複排除の確認

Pinterest は重複排除されたコンバージョン指標を表示しません。代わりに、次の項目を検証することで重複排除を確認します:

* `event_id` がウェブと API のペイロードの両方に存在します。
* 両バージョンをトリガーした後、レポートでコンバージョンが1回だけ表示されます。

Pinterest 広告マネージャーで `event_id` を表示するには、次の手順を使用します:

1. **広告マネージャー &gt; コンバージョン &gt; イベント**に移動します。
1. 広告アカウントを選択します。
1. イベント（例えば、チェックアウト）を選択します。
1. **イベント診断**を開きます。
1. イベントペイロードを展開するためにクリックします。
1. 次のことを確認します:
    * `event_source` = `web` または `api`
    * `event_id` が一致します
    * **ユーザーデータステータス** = `accepted`

## 検証

コネクタアクションを検証するには、次のようにテストイベントコードを使用します:

1. **送信コンバージョンイベント**アクションで**テストモード**を選択します。
1. テストイベントを送信します。
1. Pinterestで次の手順を使用します:
    * **コンバージョン &gt; イベント &gt; テストイベント**に移動します。
    * 受領とペイロードの整合性を確認します。

## 構成

コネクタマーケットプレースに移動して新しいコネクタを追加します。コネクタを追加する一般的な手順については、[コネクタについて]()の記事を参照してください。

このコネクタを追加すると、ベンダーのデータプラットフォームポリシーを受け入れるように求められます。

コネクタを追加した後、次の構成を構成します:

* **EULA 同意**: (必須) Pinterest のデータプラットフォームポリシーを受け入れるためのチェックボックスを選択します。
* **アクセストークン**: (必須) Pinterest 広告マネージャーを通じて生成されたアクセストークン。詳細については、[Pinterest: コンバージョンガイド](https://developers.pinterest.com/docs/conversions/conversion-management/#Authenticating%20for%20the%20Conversion%20Tracking%20endpoint)を参照してください。
* **広告アカウントID**: (必須) 広告アカウントの一意の識別子。Pinterest アカウントにログインし、広告アカウントを所有するアカウントに移動して **広告 &gt; 概要**にアクセスします。**広告アカウントID**は URL パスに表示されます。例: `ads.pinterest.com/advertiser/{ad_account_id}/...` .

コネクタの構成が完了したら、**完了**をクリックします。

### オフラインコンバージョン

オフラインイベントを通じて、ウェブサイトやアプリの外で発生する広告の影響を測定できます。これには、物理的な小売店での購入、相互作用、およびコンバージョンが含まれます。オフラインイベントとコンバージョンを処理するために、必要なパラメータを含めるように Pinterest Conversions コネクタを構成します。
#### 構成手順

Pinterest Conversionsコネクターを構成してオフラインイベントを送信するには：

1. **アクション**タブで、新しい**送信コンバージョンイベント**アクションを追加します。
1. **イベントアクションソース**パラメータを`Offline`にマッピングします。
1. 必要なパラメータをマッピングし、イベントタイプに必要に応じてオプションのパラメータを含めます。
    * Pinterestは、オフラインイベントの必須イベントIDパラメータにトランザクションID、注文ID、またはTealium Randomをマッピングすることを推奨しています。
1. Pinterest広告マネージャーで確認します。

#### ユーザーデータ

データマッピングは以下の要件を満たす必要があります：

* 配列型属性をマッピングして複数の値を追加します。
* 単一値属性をマッピングして単一の値を追加します。**クライアントIPアドレス**および**クライアントユーザーエージェント**フィールドには単一値属性のみをマッピングしてください。

以下のパラメータが必要です：

| **パラメータ** | **説明** |
| --- | --- |
| Email（既にSHA256ハッシュ化済み） | 空白トリム、小文字化、SHA256ハッシュ化が既に行われたメールアドレスを提供してください。 |
| Email（SHA256ハッシュを適用） | プレーンテキストのメールアドレスを提供し、コネクタが空白トリム、小文字化、SHA256ハッシュを使用してこの値をハッシュ化します。 |
| Appleの広告主識別子（既にSHA256ハッシュ化済み） | 空白トリムおよびSHA256ハッシュ化が既に行われたAppleの広告主識別子を提供してください。 |
| Appleの広告主識別子（SHA256ハッシュを適用） | プレーンテキストのAppleの広告主識別子を提供し、コネクタが空白トリムおよびハッシュ化を行います。 |
| Google広告ID（既にSHA256ハッシュ化済み） | 空白トリムおよびSHA256ハッシュ化が既に行われたGoogle広告IDを提供してください。 |
| Google広告ID（SHA256ハッシュを適用） | プレーンテキストのGoogle広告IDを提供し、コネクタが空白トリムおよびハッシュ化を行います。 |
| クライアントIPアドレス | ユーザーのIPアドレスで、IPv4またはIPv6形式のいずれかです。 |
| クライアントユーザーエージェント | ユーザーのウェブブラウザのユーザーエージェント文字列です。 |
| 電話番号（既にSHA256ハッシュ化済み） | 数字のみを含む電話番号で、空白トリムおよびSHA256ハッシュ化が既に行われています。ハッシュ化する前に先頭のゼロを削除してください。 |
| 電話番号（SHA256ハッシュを適用） | プレーンテキストの電話番号を提供し、コネクタがすべての非数字文字および先頭のゼロを削除し、空白トリムおよびSHA256ハッシュを使用してこの値をハッシュ化します。 |
| 性別（既にSHA256ハッシュ化済み） | 空白トリム、小文字化、SHA256ハッシュ化が既に行われた性別を提供してください。 |
| 性別（SHA256ハッシュを適用） | プレーンテキストの性別を提供し、コネクタが空白トリム、小文字化、ハッシュ化を行います。 |
| 生年月日（既にSHA256ハッシュ化済み） | 空白トリム、小文字化、SHA256ハッシュ化が既に行われた生年月日を提供してください。 |
| 生年月日（SHA256ハッシュを適用） | プレーンテキストの生年月日を提供し、コネクタが空白トリム、小文字化、ハッシュ化を行います。 |
| 名（既にSHA256ハッシュ化済み） | 空白トリム、小文字化、SHA256ハッシュ化が既に行われた名を提供してください。 |
| 名（SHA256ハッシュを適用） | プレーンテキストの名を提供し、コネクタが空白トリム、小文字化、ハッシュ化を行います。 |
| 姓（既にSHA256ハッシュ化済み） | 空白トリム、小文字化、SHA256ハッシュ化が既に行われた姓を提供してください。 |
| 姓（SHA256ハッシュを適用） | プレーンテキストの姓を提供し、コネクタが空白トリム、小文字化、ハッシュ化を行います。 |
| 市区町村（既にSHA256ハッシュ化済み） | 空白トリム、小文字化、SHA256ハッシュ化が既に行われた市区町村を提供してください。ハッシュ化する前にスペースと句読点を削除してください。 |
| 市区町村（SHA256ハッシュを適用） | プレーンテキストの市区町村を提供し、コネクタがスペースと句読点を削除し、空白トリム、小文字化、ハッシュ化を行います。 |
| 州（既にSHA256ハッシュ化済み） | 空白トリム、小文字化、SHA256ハッシュ化が既に行われた州を提供してください。 |
| 州（SHA256ハッシュを適用） | プレーンテキストの州を提供し、コネクタが空白トリム、小文字化、ハッシュ化を行います。 |
| 郵便番号（既にSHA256ハッシュ化済み） | 数字のみを含む郵便番号で、空白トリムおよびSHA256ハッシュ化が既に行われています。 |
| 郵便番号（SHA256ハッシュを適用） | プレーンテキストの郵便番号を提供し、コネクタがすべての非数字文字を削除し、空白トリムおよびハッシュ化を行います。 |
| 国コード（既にSHA256ハッシュ化済み） | 空白トリム、小文字化、SHA256ハッシュ化が既に行われた国コードを提供してください。 |
| 国コード（SHA256ハッシュを適用） | プレーンテキストの国コードを提供し、コネクタが空白トリム、小文字化、ハッシュ化を行います。 |
| 外部ID（既にSHA256ハッシュ化済み） | 空白トリムおよびSHA256ハッシュ化が既に行われた1つ以上の外部IDを提供してください。複数の外部IDがある場合は、配列としてフォーマットしてください。詳細については、[Pinterest: 外部IDとは何ですか？](https://developers.pinterest.com/docs/api-features/conversion-best-practices/#what-is-an-external-id)を参照してください。 |
| 外部ID（SHA256ハッシュを適用） | 1つ以上のプレーンテキストの外部IDを提供し、コネクタが空白トリムおよびハッシュ化を行います。複数の外部IDを提供する場合は、配列としてフォーマットしてください。 |
| クリックID  | ドメイン上の`_epik`クッキーまたはURL内の`&amp;epik=`クエリパラメータに格納されている一意の識別子です。これにより、ROAS/CPAなどのレポートパフォーマンスが向上します。 |

| 値 | イベントの総価値。コネクタは文字列値を受け取り、それを[倍精度浮動小数点数](https://en.wikipedia.org/wiki/Double-precision_floating-point_format)に解析します。複数のアイテムがあるチェックアウトイベントの場合、すべてのアイテムの合計価格を使用します。 |
| コンテンツID | 製品IDのリスト。 |
| アイテム数 | イベントの製品の総数。たとえば、チェックアウトイベントで購入されたアイテムの総数です。 |
| 注文ID | 注文ID。 |
| 検索文字列 | ユーザーのコンバージョンイベントに関連する検索文字列。 |
| オプトアウトタイプ | 個人情報の共有をオプトアウトするさまざまなプライバシー権法に関するフラグ。値はコンマで区切られるべきです。配列型属性はマッピング可能であり、コンマ区切りの文字列に変換されます。 |

#### コンテンツデータ

* コンバージョンに複数のアイテムを追加するために配列型属性を提供します。配列型属性は同じ長さでなければなりません。
* 単一値属性をパラメータにマッピングする場合、その値はコンバージョンの配列内のすべてのアイテムに適用されます。

| **パラメータ** | **説明** |
| --- | --- |
| アイテム価格 | コンバージョン内の個々のアイテムの価格。コネクタは文字列値を受け取り、それを[倍精度浮動小数点数](https://en.wikipedia.org/wiki/Double-precision_floating-point_format)に解析します。複数のアイテムがあるチェックアウトイベントの場合、各アイテムの価格を提供します。 |
| 数量 | 製品の数量。 |
| ID | 製品のID。**AddToCart**および**Checkout**イベントに推奨されます。 |
| アイテム名 | 製品の名前。 |
| アイテムカテゴリ | 製品のカテゴリ。 |
| アイテムブランド | 製品のブランド。 |
