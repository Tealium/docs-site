## API情報

このコネクタは以下のベンダーAPIを使用します：

* API名：Microsoft API
* APIバージョン：v13
* APIエンドポイント：`https://campaign.api.bingads.microsoft.com/Api/Advertiser/CampaignManagement/v13/CampaignManagementService.svc`
* ドキュメント：[キャンペーン管理 API](https://learn.microsoft.com/en-us/advertising/campaign-management-service/campaign-management-service-reference?view=bingads-13)

## コネクタのアクション

| アクション名 | AudienceStream | EventStream |
| --- | :---: | :---: |
| コンバージョンイベントの送信 | ✗ | ✓ |

## 構成の構成

コネクタマーケットプレイスにアクセスして新しいコネクタを追加します。コネクタの追加方法については、[コネクタについて]()を参照してください。

コネクタを追加した後、以下の構成を構成します：

* **顧客ID**  
（必須）Microsoft Ads UIにログインした後、URLの`CID=`の後に顧客IDが見つかります。
* **アカウントID**  
（必須）Microsoft Ads UIにログインした後、URLの`AID=`の後にアカウントIDが見つかります。

**接続の確立**をクリックしてMicrosoftに接続します。画面の指示に従って接続プロセスを完了します。

### オフラインコンバージョン目標の作成

**オフラインコンバージョン目標の作成**をクリックして、コンバージョンの目標を構成します。以下の情報を入力します：

1. コンバージョン目標のためのユニークな**名前**を入力します。名前の最大長は100文字です。
1. **目標カテゴリー**を選択して、コンバージョン目標をセグメント化します。
1. **スコープ**を選択して、目標がすべてのアカウントに適用されるか、指定されたアカウントのみに適用されるかを決定します。スコープを構成した後、スコープを変更するには、新しいコンバージョン目標を作成して古いものを一時停止する必要があります。
1. **ID**の下に、コンバージョン目標のためのMicrosoft Advertising識別子を入力します。
1. **コンバージョンウィンドウ（分）**の下に、クリック後にコンバージョンを追跡したい時間（分）を入力します。デフォルト値は`43200`です。サポートされる最小値は`1`分、最大値は`129600`分（90日）です。コンバージョンウィンドウを短くすると、アカウントが記録するコンバージョン数が減少します。
1. **カウントタイプ**を選択して、コンバージョンウィンドウ内でのコンバージョンの記録方法を決定します。デフォルトは`All`です：
    * **All**：広告クリック後に発生する異なるオフラインコンバージョン時間のすべてのコンバージョンがカウントされます。これは販売に一般的な選択です。
    * **Unique**：広告クリック後に発生する最初のコンバージョンのみがカウントされます。これはリードに一般的な選択です。
1. **入札から除外**の下で、この目標を入札に含めるか除外するかを選択します。これにより、このコンバージョン目標に関連するデータがパフォーマンスレポートの列のサブセットに表示されるかどうかが決まります。
1. **収益値**の下に、各コンバージョンがビジネスにとってどれだけの価値があるかを決定する値を入力します。
1. **収益コード**の下に、レポートで使用する通貨タイプを入力します。
1. （オプション）**ステータス**の下で、`Active`、`Paused`、または`Deleted`を選択して、コンバージョン目標の可能なユーザー決定ステータス値を定義します。
1. **作成して閉じる**をクリックします。

## アクション

アクションの名前を入力し、ドロップダウンメニューからアクションタイプを選択します。

次のセクションでは、各アクションのパラメータとオプションの構成方法について説明します。

### コンバージョンイベントの送信

#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| コンバージョン名 | (必須) コンバージョン目標名。この名前は既存のコンバージョン目標名と一致する必要があります。そうでない場合、オフラインコンバージョン目標データは適用されません。|
| Microsoft Click ID | (必須) Microsoft Advertisingによって構成されたランディングページURLのユニークID。  |
| 外部帰属モデル | 帰属モデル名を指定します。このフィールドは、外部帰属を使用するコンバージョンアクションのみに構成できます。 |
| 外部帰属クレジット | 各クリックに帰属するコンバージョンの割合を表します。値は0より大きく1以下でなければなりません。このフィールドは、外部帰属を使用するコンバージョンアクションのみに構成できます。 |
| コンバージョン値 | オフラインコンバージョン値。 |
| コンバージョン時間 | (必須) ユーザーがアクションを完了した後、Bing Adsがコンバージョンデータを記録する時間。 |
| 通貨コード | オフラインコンバージョンの通貨コード。オフラインコンバージョンの通貨コードを指定しない場合、目標の&lt;a href=&#34;https://learn.microsoft.com/en-us/advertising/campaign-management-service/conversiongoalrevenue?view=bingads-13&#34; target=&#34;_blank&#34;&gt;Conversion Goal Revenue&lt;/a&gt;の通貨コード要素が使用されます。 |
| メールアドレス（既にSHA256ハッシュ化済み） | すでに空白をトリミングし、小文字に変換し、SHA256でハッシュ化されたメールアドレスを提供します。 |
| メールアドレス（SHA256ハッシュを適用） | プレーンテキストのメールアドレスを提供し、コネクタが空白をトリミングし、小文字に変換し、SHA256を使用してこの値をハッシュ化します。 |
| 電話番号（既にSHA256ハッシュ化済み） | E.164標準に従って、すでに空白をトリミングし、小文字に変換し、SHA256でハッシュ化された電話番号を提供します。 |
| 電話番号（SHA256ハッシュを適用） | プレーンテキストの電話番号を提供し、コネクタがすべての非数字記号を削除し、プラス記号（`&#43;`）を番号の前に付け、空白をトリミングし、小文字に変換し、SHA256を使用してこの値をハッシュ化します。 |
