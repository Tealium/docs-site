Google Analytics 4 測定プロトコルコネクタ構成ガイド

この記事では、Google Analytics 4 測定プロトコルコネクタの構成方法について説明します。

コネクタのアクション

アクション名 AudienceStream EventStream
Send PageView Event (gtag.js) 廃止予定 ✓ ✓
Send Event (gtag.js) ✓ ✓
Send Firebase Event ✓ ✓

Send PageView Eventアクションは現在廃止予定です。page_viewイベントを送信するには、Send Eventアクションを使用してください。

Google Analytics 4 測定プロトコルは、イベントの検証用に次のエンドポイントを提供します：https://google-analytics.com/debug/mp/collect?。TealiumのGoogle Analytics 4 測定プロトコルコネクタは、Trace中にこのエンドポイントを使用し、イベントデータの検証に関する洞察を提供します。Google Analytics 4 測定プロトコルの検証メッセージは、Trace内のAPI呼び出しのHTTPレスポンスコンテンツで表示できます。

構成の構成

コネクタマーケットプレイスに移動し、新しいコネクタを追加します。コネクタの追加方法については、About Connectorsを参照してください。

コネクタを追加したら、次の構成を構成します：

  • API Secret (必須)
    Google Analytics UIで生成されたAPI Secret。新しいシークレットを作成するには、次の場所に移動します：Admin > Data Streams > ストリームを選択 > Measurement Protocol > Create。
  • Measurement ID (オプション)
    ストリームに関連付けられた測定ID。Google Analytics UIの次の場所で見つけることができます：Admin > Data Streams > ストリームを選択 > Measurement ID。
  • Firebase App ID (オプション)
    Firebase App ID。Firebaseアプリの識別子。Firebaseコンソールの次の場所で見つけることができます：Project Settings > General > Your Apps > App ID。

アクション構成 - パラメータとオプション

アクションに名前を入力し、ドロップダウンメニューからアクションタイプを選択します。

次のセクションでは、各アクションのパラメータとオプションの構成方法について説明します。

クライアントIDのマッピング

クライアントIDは、ウェブクライアントのユーザーインスタンスを一意に識別するための必須パラメータです。

Google Analyticsは、クライアントIDを_gaという名前のクッキーに保存します。このクッキーの形式はGA1.2.12349876.1500644855で、クライアントIDは12349876.150064485です。clientIdの値はGoogle Analyticsトラッカーから直接取得する必要があります。

詳細については、以下を参照してください：

  • Google tag API reference: get.
  • Google Analytics Measurement Protocol > Parameter Reference > Client ID.

同意

コネクタはデフォルトでは同意信号を送信しません。同意を指定しない場合、Google Analyticsはクライアントインスタンスの対応するオンラインインタラクションから同意構成を使用します。ad_user_dataとad_personalizationの同意を構成するとき、同意状態は許可または拒否のいずれかになります。
