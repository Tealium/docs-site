## 必要条件

LINEコネクタには以下が必要です：

* LINE: チャネルアクセストークン。詳細については、[メッセージングAPI概要](https://developers.line.biz/en/docs/messaging-api/overview/)を参照してください。

## API情報

このコネクタは以下のベンダーAPIを使用します：

* API名：LINE Messaging API
* APIバージョン：v2
* APIエンドポイント：`https://api.line.me/v2`
* ドキュメント：[LINE: Audience API](https://developers.line.biz/en/docs/messaging-api/)

## バッチ制限

このアクションは、ベンダーへの大量データ転送をサポートするためにバッチリクエストを使用します。詳細については、[バッチアクション](https://docs.tealium.com/batched-actions/)を参照してください。リクエストは、次のいずれかの閾値に達するか、プロファイルが公開されるまでキューに入れられます：

* 最大リクエスト数：10,000
* 最古のリクエストからの最大時間：30分
* リクエストの最大サイズ：1 MB

## 構成の構成

コネクタマーケットプレイスにアクセスして、新しいLINEコネクタを追加します。コネクタの追加方法については、[コネクタ概要](https://docs.tealium.com/about-connectors/)の記事を参照してください。

ベンダーを構成するには、次の手順に従います：

1. **構成**タブで、コネクタインスタンスのタイトルを入力します。
1. LINEアカウント用に構成したチャネルアクセストークンを入力します。
1. 実装に関する追加のノートを提供します。

### アクション - メッセージ送信 - テキスト

#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| コンタクトID | 必須。このパラメータは、LINEビジネスセンターの**チャネルコンソール**で構成されたウェブフックから受け取ったIDです。ユーザーがアカウントを追加したりメッセージを送信したりすると、HTTPS POSTリクエストがウェブフックURLに送信されます。|
| メッセージデータ | 必須。ユーザーに送信するメッセージの内容。このフィールドは2,000文字に制限され、ユニコード絵文字をサポートし、Tealiumテンプレートエンジンをサポートします。詳細については、[about-connector-templates](https://docs.tealium.com/about-connector-templates/)を参照してください。|
| テンプレート変数  | これらのフィールドは、メッセージデータで使用される変数名に属性をマッピングします。ネストされたテンプレート変数はドット表記で名前を付けます。例えば、`items.name`。詳細については、[connector-template-variables](https://docs.tealium.com/connector-template-variables/)を参照してください。 |

### メッセージ送信 - イメージマップ


#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| Contact ID | 必須。このパラメータは、**Channel Console** に構成されたウェブフックから受け取ったIDです。アカウントの追加やメッセージの送信などのユーザーのアクションが行われると、ウェブフックURLにHTTPS POSTリクエストが送信されます。 |
| Base URL | 必須。ベース画像のURL。URLはHTTPSプロトコルを使用し、1,000文字を超えてはいけません。 |
| Alternative Text | 必須。ベース画像の代替テキストで、最大400文字です。この代替テキストは通知やチャットリストに表示されます。 |
| Base Size - Width | 必須。ベース画像の幅（ピクセル）。この値は `1040` に構成します。 |
| Base Size - Height | 必須。ベース画像の高さ（ピクセル）。この値は画像の幅が1040ピクセルのときの高さに構成します。 |

#### アクションデータ

すべてのアクションデータパラメータは配列です。

| **パラメータ** | **説明** |
| --- | --- |
| Type | アクションのタイプ。可能な値は `uri` と `message` です。 |
| Area - X | イメージマップエリアの左端からのアクションエリアの水平位置。最小値は `0` です。 |
| Area - Y | イメージマップエリアの上端からのアクションエリアの垂直位置。最小値は `0` です。 |
| Area - Width | アクションエリアの幅。 |
| Area - Height | アクションエリアの高さ。 |
| Label | アクションのラベル。 |
| Link URI | `uri` タイプのアクションの場合、アクションのリンクのURI。 |
| Text | メッセージタイプのアクションの場合、アクションのテキスト。 |
| Clipboard Text | クリップボードにコピーされるテキスト。 |

#### ビデオデータ

イメージマップにビデオを構成する場合、以下のパラメータが必要です：

| **パラメータ** | **説明** |
| --- | --- |
| Original Content URL | ビデオファイルのURL。URLは2,000文字を超えてはいけませんし、HTTPSプロトコルを使用する必要があります。ビデオは.mp4形式で、200MBを超えてはいけません。 |
| Preview Image URL | プレビュー画像のURL。URLは2,000文字を超えてはいけませんし、HTTPSプロトコルを使用する必要があります。画像はJPEGまたはPNG形式を使用し、1MBを超えてはいけません。 |
| Area - X | イメージマップエリアの左端からのビデオエリアの水平位置。最小値は `0` です。 |
| Area - Y | イメージマップエリアの上端からのビデオエリアの垂直位置。最小値は `0` です。 |
| Area - Width | ビデオエリアの幅。 |
| Area - Height | ビデオエリアの高さ。 |

ビデオがイメージマップ上で再生され、ビデオ終了後にラベルが表示される場合、以下のパラメータが必要です：

| **パラメータ** | **説明** |
| --- | --- |
| External Link - Link URI | ウェブページのURL。URLは1,000文字を超えてはいけませんし、`http`, `https`, `line`, `tel` のプロトコルを使用できます。 |
| External Link - Label | ビデオが終了した後に表示されるラベル。ラベルは30文字を超えてはいけません。 |

### プッシュメッセージ - テンプレートメッセージ（ボタンテンプレート）

#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| Contact ID | 必須。このパラメータは、**Channel Console** に構成されたウェブフックから受け取ったIDです。アカウントの追加やメッセージの送信などのユーザーのアクションが行われると、ウェブフックURLにHTTPS POSTリクエストが送信されます。 |
| Alternative Text | 必須。メッセージの代替テキストで、テキストは400文字を超えてはいけません。このテキストは通知やチャットリストに表示されます。 |
| Template - Text | 必須。メッセージテキスト。画像やタイトルがないテキストは400文字を超えてはいけませんし、画像やタイトルがある場合のテキストは60文字を超えてはいけません。 |
| Template - Thumbnail Image URL | 必須。画像のサムネイルのURL。URLは2,000文字を超えてはいけませんし、HTTPSプロトコルを使用する必要があります。 |
| Template - Image Aspect Ratio | 必須。画像のアスペクト比。許可される値は `rectangle` または `square` です。デフォルト値は `rectangle` です。 |
| Template - Image Size | 必須。画像のサイズ。許可される値は `cover` または `contain` です。デフォルト値は `cover` です。 |
| Template - Image Background Color | 必須。画像の背景色（16進数形式）。デフォルト値は `#FFFFFF`（白）です。 |
| Template - Title | 必須。メッセージのタイトル。タイトルは40文字を超えてはいけません。 |
