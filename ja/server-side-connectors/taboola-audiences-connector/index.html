Taboola Audiences コネクタ構成ガイド

この記事では、Taboola Audiencesコネクタの構成方法について説明します。

API情報

このコネクタは以下のベンダーAPIを使用します：

  • API名：Taboola API
  • APIバージョン：v1.0
  • APIエンドポイント：https://backstage.taboola.com/backstage/api/1.0
  • ドキュメンテーション：Taboola API

アクション

アクション名 AudienceStream EventStream
ユーザーをオーディエンスに追加 ✓ ✓
ユーザーをオーディエンスに追加（バッチ処理） ✓ ✓
ユーザーをオーディエンスから削除 ✓ ✓
ユーザーをオーディエンスから削除（バッチ処理） ✓ ✓

構成

コネクタマーケットプレイスに移動し、新しいコネクタを追加します。コネクタの追加方法の一般的な指示については、About Connectorsを参照してください。

このコネクタを追加するときは、ベンダーのデータプラットフォームポリシーを受け入れる必要があります。

コネクタを追加した後、以下の構成を行います：

  • アカウントID：割り当てられたTaboolaアカウントID。

アクションパラメータ

以下のセクションでは、各アクションでサポートされているパラメータをリストします。

ユーザーをオーディエンスに追加

パラメータ

パラメータ 説明
オーディエンスID ユーザーが追加されるオーディエンスのID。オーディエンスIDを確認するには、オーディエンスIDを確認をクリックします。新しいオーディエンスを作成するには、以下の手順を使用します：
  1. オーディエンスを作成をクリックします。
  2. オーディエンス名フィールドに一意のオーディエンス名を入力します。
  3. ユーザーがオーディエンスに残るべき時間をTTL in Hoursフィールドに入力します。デフォルト値は8670（1年）です。
  4. 新規および既存のキャンペーンからオーディエンスを除外するには、Exclude from CampaignsフィールドにTrueを入力します。デフォルト値はFalseです。
  5. Doneをクリックします。
ユーザーID SHA256ハッシュ化されたメールアドレスまたはMAIDデバイスIDとしてのユーザーID
タイプ ユーザーIDのタイプ（DEVICE_IDまたはEMAIL_IDのいずれか）

ユーザーをオーディエンスに追加（バッチ処理）

バッチ制限

このアクションは、ベンダーへの大量のデータ転送をサポートするためにバッチリクエストを使用します。詳細については、Batched Actionsを参照してください。リクエストは、以下のいずれかの閾値が達成されるか、プロファイルが公開されるまでキューに入れられます：

  • リクエストの最大数：10000
  • 最も古いリクエストからの最大時間：10分
  • リクエストの最大サイズ：10MB

パラメータ

パラメータ 説明
オーディエンスID ユーザーが追加されるオーディエンスのID。オーディエンスIDを確認するには、オーディエンスIDを確認をクリックします。新しいオーディエンスを作成するには、以下の手順を使用します：
  1. オーディエンスを作成をクリックします。
  2. オーディエンス名フィールドに一意のオーディエンス名を入力します。
  3. ユーザーがオーディエンスに残るべき時間をTTL in Hoursフィールドに入力します。デフォルト値は8670（1年）です。
  4. 新規および既存のキャンペーンからオーディエンスを除外するには、Exclude from CampaignsフィールドにTrueを入力します。デフォルト値はFalseです。
  5. Doneをクリックします。
ユーザーID SHA256ハッシュ化されたメールアドレスまたはMAIDデバイスIDとしてのユーザーID
タイプ ユーザーIDのタイプ（DEVICE_IDまたはEMAIL_IDのいずれか）

ユーザーをオーディエンスから削除

パラメータ

パラメータ 説明
オーディエンスID ユーザーが追加されるオーディエンスのID。オーディエンスIDを確認するには、オーディエンスIDを確認をクリックします。新しいオーディエンスを作成するには、以下の手順を使用します：
  1. オーディエンスを作成をクリックします。
  2. オーディエンス名フィールドに一意のオーディエンス名を入力します。
  3. ユーザーがオーディエンスに残るべき時間をTTL in Hoursフィールドに入力します。デフォルト値は8670（1年）です。
  4. 新規および既存のキャンペーンからオーディエンスを除外するには、Exclude from CampaignsフィールドにTrueを入力します。デフォルト値はFalseです。
  5. Doneをクリックします。
ユーザーID SHA256ハッシュ化されたメールアドレスまたはMAIDデバイスIDとしてのユーザーID
タイプ ユーザーIDのタイプ（DEVICE_IDまたはEMAIL_IDのいずれか）

ユーザーをオーディエンスから削除（バッチ処理）

バッチ制限

このアクションは、ベンダーへの大量のデータ転送をサポートするためにバッチリクエストを使用します。詳細については、Batched Actionsを参照してください。リクエストは、以下のいずれかの閾値が達成されるか、プロファイルが公開されるまでキューに入れられます：

  • リクエストの最大数：10000
  • 最も古いリクエストからの最大時間：10分
  • リクエストの最大サイズ：10MB
