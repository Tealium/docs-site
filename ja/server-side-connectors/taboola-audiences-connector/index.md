## API情報

このコネクタは以下のベンダーAPIを使用します：

* API名：Taboola API
* APIバージョン：v1.0
* APIエンドポイント：`https://backstage.taboola.com/backstage/api/1.0`
* ドキュメンテーション：[Taboola API](https://developers.taboola.com/backstage-api/reference/add-or-remove-users)

## アクション

| アクション名 | AudienceStream | EventStream |
| --- | :---: | :---: |
| ユーザーをオーディエンスに追加 | ✓ | ✓ |
| ユーザーをオーディエンスに追加（バッチ処理） | ✓ | ✓ |
| ユーザーをオーディエンスから削除 | ✓ | ✓ |
| ユーザーをオーディエンスから削除（バッチ処理） | ✓ | ✓ |

## 構成

コネクタマーケットプレイスに移動し、新しいコネクタを追加します。コネクタの追加方法の一般的な指示については、[About Connectors]()を参照してください。

このコネクタを追加するときは、ベンダーのデータプラットフォームポリシーを受け入れる必要があります。

コネクタを追加した後、以下の構成を行います：

* **アカウントID**：割り当てられたTaboolaアカウントID。

## アクションパラメータ

以下のセクションでは、各アクションでサポートされているパラメータをリストします。

### ユーザーをオーディエンスに追加

#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| オーディエンスID | ユーザーが追加されるオーディエンスのID。**オーディエンスID**を確認するには、**オーディエンスIDを確認**をクリックします。新しいオーディエンスを作成するには、以下の手順を使用します：&lt;ol&gt;&lt;li&gt;**オーディエンスを作成**をクリックします。&lt;/li&gt;&lt;li&gt;**オーディエンス名**フィールドに一意のオーディエンス名を入力します。&lt;/li&gt;&lt;li&gt;ユーザーがオーディエンスに残るべき時間を**TTL in Hours**フィールドに入力します。デフォルト値は`8670`（1年）です。&lt;/li&gt;&lt;li&gt;新規および既存のキャンペーンからオーディエンスを除外するには、**Exclude from Campaigns**フィールドに`True`を入力します。デフォルト値は`False`です。&lt;/li&gt;&lt;li&gt;**Done**をクリックします。&lt;/li&gt;&lt;/ol&gt;  |
| ユーザーID | SHA256ハッシュ化されたメールアドレスまたはMAIDデバイスIDとしてのユーザーID  |
| タイプ | ユーザーIDのタイプ（`DEVICE_ID`または`EMAIL_ID`のいずれか） |


### ユーザーをオーディエンスに追加（バッチ処理）

#### バッチ制限

このアクションは、ベンダーへの大量のデータ転送をサポートするためにバッチリクエストを使用します。詳細については、[Batched Actions]()を参照してください。リクエストは、以下のいずれかの閾値が達成されるか、プロファイルが公開されるまでキューに入れられます：

* リクエストの最大数：10000
* 最も古いリクエストからの最大時間：10分
* リクエストの最大サイズ：10MB

| ユーザーID | SHA256ハッシュ化されたメールアドレスまたはMAIDデバイスIDとしてのユーザーID  |
| タイプ | ユーザーIDのタイプ（`DEVICE_ID`または`EMAIL_ID`のいずれか） |
