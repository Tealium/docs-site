## API情報

このコネクタは以下のベンダーAPIを使用します：

* API名：Bloomreach API
* APIバージョン：v2.0
* APIエンドポイント：`https://api-engagement.bloomreach.com`
* ドキュメント：[Bloomreach API](https://documentation.bloomreach.com/engagement/reference/about)

## 構成

コネクタマーケットプレイスにアクセスし、新しいコネクタを追加します。コネクタを追加する一般的な手順については、[コネクタについて](https://docs.tealium.com/about-connectors/)を参照してください。

コネクタを追加した後、以下の構成を構成します：
* **APIキー**  
APIキー。
* **APIシークレット**  
APIシークレット。
* **プロジェクトトークン**  
あなたのプロジェクトトークン。これは、Engagementウェブアプリの**プロジェクト構成 > アクセス管理 > API**で見つけることができます。
* **APIドメイン**  
あなたのBloomreach APIドメイン。デフォルト値は `https://api-engagement.bloomreach.com` です。

## バッチ制限

このアクションは、ベンダーへの大量データ転送をサポートするためにバッチリクエストを使用します。詳細については、バッチアクションを参照してください。リクエストは、次のいずれかの閾値に達するか、プロファイルが公開されるまでキューに入れられます：

* 最大リクエスト数：50
* 最古のリクエストからの最大時間：5分
* リクエストの最大サイズ：30 MB

| --- | --- |

#### メール内容

| **パラメータ** | **説明** |
| --- | --- |
| テンプレートID | 送信したいメールテンプレートのID。これは、選択された事前定義されたテンプレートのURLで見つけることができます。 |
| 件名 | メールの件名を定義します。このフィールドではJinjaは評価されません。 |
| 送信者アドレス | 送信者のメールアドレス。 |
| 送信者名 | 通常、送信者の名前です。 |
| 添付ファイルのファイル名 | 添付ファイルの名前。 |
| 添付ファイルの内容 | ファイルのBase64エンコードされた内容。 |
| 添付ファイルのコンテンツタイプ | 添付ファイルの形式。 |
| Html | `<html>` と `</html>` タグの間のメールの内容を定義します。CSSコンテンツを含めることができます。Jinjaはこのフィールドでは評価されません。 |
| メール内容パラメータ | テンプレートで使用されるメールのパーソナライゼーションに必要なすべてのデータを提供します。形式：`param_name`:"value"。例：`first_name`:"Marian" |
| キャンペーン名 | メールキャンペーンの名前。 |

#### 受信者

| **パラメータ** | **説明** |
| --- | --- |
| メール | 顧客のメールを指定します。メールが提供されていない場合、顧客IDに基づいてBloomreach Engagementの顧客プロファイルから取得されます。 |
| 言語 | テンプレートの言語バージョン。テンプレートに複数の言語バージョンが作成されている場合、このメールに使用するバージョンを選択できます。 |

#### 受信者顧客ID

| **パラメータ** | **説明** |
| --- | --- |
| 転送アイデンティティ | リンク転送機能を有効にするかどうか。可能な値：<ul><li>`Enabled` - デフォルト値、すべてのクリックで識別する`iitt`パラメータを追加します。</li><li>`Disabled` - リンクに何も追加せず、アイデンティティ転送はありません。</li><li>`First Click` - 最初のクリックのみを識別する`xnpe_tifc`を追加します。</li></ul> |

#### メールテンプレート構成

| **パラメータ** | **説明** |
| --- | --- |
| ユーザーアイデンティティの転送 | リンク転送機能を有効にするかどうか。可能な値：<ul><li>`enabled` (デフォルト値)</li><li>`disabled`</li><li>`iitt` - リンクにアイデンティティを転送しない。</li></ul> この構成は、リクエストの本文に直接指定される`transferidentity`パラメータと同じです。どちらか一方のみを指定する必要があります。構成の一部として転送アイデンティティを指定し、単独で指定するとエラーが発生します。 |
| 同意カテゴリ | Engagementプラットフォームでの同意を尊重するために、顧客の同意カテゴリを指定します。 |
| トラッキング同意カテゴリ | Engagementプラットフォームでのトラッキング同意を尊重するために、顧客のトラッキング同意カテゴリを指定します。 |
| メールテンプレートカスタムイベントプロパティ | カスタム属性。 |
| メールテンプレートカスタムヘッダー | メールプロバイダに送信するリクエストヘッダ。 |
| メールテンプレートURLパラメータ | セッション開始イベントでのトラッキングに使用されるURLパラメータ。 |
