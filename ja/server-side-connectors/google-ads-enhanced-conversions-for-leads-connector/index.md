## コネクタのアクション

| アクション名 | AudienceStream | EventStream |
| --- | :---: | :---: |
|コンバージョン送信| ✓| ✓|

## API 情報

このコネクタは以下のベンダー API を使用します：

* API 名: Google Ads API
* API バージョン: v18
* API エンドポイント: `https://googleads.googleapis.com/`
* ドキュメント: [Google Ads API: リード向け拡張コンバージョンのアップロード](https://developers.google.com/google-ads/api/docs/conversions/upload-identifiers)

## バッチ制限

このコネクタは、ベンダーへの大量データ転送をサポートするためにバッチリクエストを使用します。詳細については、[バッチアクション]()を参照してください。リクエストは、次のいずれかの閾値に達するか、プロファイルが公開されるまでキューに入れられます：

* 最大リクエスト数: 2,000
* 最古のリクエストからの最大時間: 10分
* リクエストの最大サイズ: 2 MB

### 同意

コネクタはデフォルトで `adUserData` の値として `GRANTED` を送信します。デフォルトを上書きするには、関連する EventStream 属性を `Ad User Data Consent` にマッピングし、値を `DENIED` に構成します。

## 構成の構成

コネクタマーケットプレイスに移動し、新しいコネクタを追加します。コネクタを追加する一般的な手順については、[コネクタについて](/ja/server-side/connectors/manage/)の記事を参照してください。

コネクタを追加した後、以下の構成を構成します：

* **顧客ID**  
必須：広告アカウントの顧客IDを提供してください。
* **マネージャー顧客ID**  
オプション。クライアントを代表してアカウントにアクセスする場合、マネージャー顧客IDを構成する必要があります。顧客IDは、Google Adsにログインし、**構成 &gt; アクセスとセキュリティ &gt; マネージャー**に移動することで見つけることができます。

コネクタの構成が完了したら、**完了**をクリックします。

## アクション構成 — パラメータとオプション

コネクタアクションの構成を続行するには、**続行**をクリックします。アクションの名前を入力し、ドロップダウンメニューからアクションタイプを選択します。

次のセクションでは、各アクションのパラメータとオプションの構成方法について説明します。

### アクション — コンバージョン送信

#### パラメータ

|**パラメータ**| **説明**|
|---| ---|
|コンバージョンアクション| **UPLOAD_CLICKS**タイプで拡張コンバージョンが有効になっている既存のコンバージョンアクションを選択します。詳細については、[Google Ads 拡張コンバージョントラッキング](https://support.google.com/google-ads/answer/9888656)を参照してください。|

#### ユーザーデータ

|**パラメータ**| **説明**|
|---| ---|
|メールアドレス（既にSHA256ハッシュ化済み）| すでに空白をトリミングし、小文字に変換し、SHA256ハッシュ化されたメールアドレスを提供してください。ハッシュ化する前に、`gmail.com`および`googlemail.com`のメールアドレスのドメイン名の前にあるすべてのピリオド（`.`）を削除してください。|
|メールアドレス（SHA256ハッシュを適用）| プレーンテキストのメールアドレスを提供し、コネクタが`gmail.com`および`googlemail.com`のメールアドレスのドメイン名の前にあるすべてのピリオドを削除し、空白をトリミングし、小文字に変換し、この値をSHA256ハッシュでハッシュ化します。|
|電話番号（既にSHA256ハッシュ化済み）| [E164標準](https://en.wikipedia.org/wiki/E.164)に従って、すでに空白をトリミングし、SHA256ハッシュ化された電話番号を提供してください。|
|電話番号（SHA256ハッシュを適用）| プレーンテキストの電話番号を提供し、コネクタがすべての非数字記号を削除し、プラス記号（`&#43;`）で番号をプレフィックスし、空白をトリミングし、この値をSHA256ハッシュでハッシュ化します。|
|gclid| コンバージョンに関連付けられたGoogleクリックID（gclid）。注意：`gclid`と`user_identifiers`（メール/電話）をコンバージョンに提供する場合、Google Adsは`user_identifiers`を無視します。|
|住所情報：名（既にSHA256ハッシュ化済み）| すでに空白をトリミングし、小文字に変換し、SHA256ハッシュ化された名を提供してください。|
|住所情報：名（SHA256ハッシュを適用）| プレーンテキストの名を提供し、コネクタが小文字に変換し、空白をトリミングし、この値をSHA256ハッシュでハッシュ化します。|
|住所情報：姓（既にSHA256ハッシュ化済み）| すでに空白をトリミングし、小文字に変換し、SHA256ハッシュ化された姓を提供してください。|
|住所情報：姓（SHA256ハッシュを適用）| プレーンテキストの姓を提供し、コネクタが小文字に変換し、空白をトリミングし、この値をSHA256ハッシュでハッシュ化します。|
|住所情報：住所（既にSHA256ハッシュ化済み）| すでに空白をトリミングし、小文字に変換し、SHA256ハッシュ化された住所を提供してください。|
|住所情報：住所（SHA256ハッシュを適用）| プレーンテキストの住所を提供し、コネクタが小文字に変換し、空白をトリミングし、この値をSHA256ハッシュでハッシュ化します。|
|住所情報：市区町村| プレーンテキストの市区町村を提供してください。|
|住所情報：州| プレーンテキストの州を提供してください。|
|住所情報：郵便番号| プレーンテキストの郵便番号を提供してください。|
|住所情報：国| プレーンテキストの国を提供してください。|

#### コンバージョンデータ

|**パラメータ**| **説明**|
|---| ---|
|コンバージョン値| コンバージョンの金銭的価値。|
|コンバージョン通貨| コンバージョンの通貨コード。|
|コンバージョン時間| 元のコンバージョンが発生した日時。タイムゾーンを指定する必要があります。形式は &lt;code&gt;yyyy-mm-dd hh:mm:ss&#43;&amp;#124;-hh:mm&lt;/code&gt; です。たとえば、`2022-01-01 12:32:45-08:00`または`2022-01-01 12:32:45&#43;08:00`です。値がマッピングされていない場合、値は現在の時間に構成されます。|
|注文ID| コンバージョンに関連付けられた注文ID。注文IDは、コンバージョンアクションごとに1つのコンバージョンにのみ使用できます。|
|カスタム変数| このコンバージョンに関連付けられたカスタム変数。値をカスタム変数IDにマッピングします。コネクタは次の形式で `conversion_custom_variable` フィールドを送信します：`customers/{customer_id}/conversionCustomVariables/{custom_variable_id}`。|

#### カートデータ

| **パラメータ** | **説明** |
| --- | --- |
|マーチャントID| アイテムがアップロードされるマーチャントセンターID。|
|フィード国コード| アイテムがアップロードされるフィードの国コード。|
|フィード言語コード| アイテムがアップロードされるフィードの言語コード。|
|ローカルトランザクションコスト| カート全体のすべてのトランザクションレベルの割引（送料無料やクーポン割引など）の合計。|
|商品ID| アイテムのショッピングID。マーチャントセンターの商品識別子と一致する必要があります。複数のアイテムを追加するために配列タイプの属性を提供します。配列タイプの属性は同じ長さでなければなりません。単一値の属性を使用することもでき、各アイテムに適用されます。|
|数量| 販売されたアイテムの数。複数のアイテムを追加するために配列タイプの属性を提供します。配列タイプの属性は同じ長さでなければなりません。単一値の属性を使用することもでき、各アイテムに適用されます。|
|単価| 税金、送料、およびすべてのトランザクションレベルの割引を除いた単価。複数のアイテムを追加するために配列タイプの属性を提供します。配列タイプの属性は同じ長さでなければなりません。単一値の属性を使用することもでき、各アイテムに適用されます。|
|顧客IDオーバーライド| （オプション）コネクタ構成の顧客IDを上書きするための顧客IDを提供します。このオプションを使用する場合、コンバージョンアクションIDのオーバーライドも使用する必要があります。 |
|マネージャー顧客IDオーバーライド| （オプション）顧客IDオーバーライドを使用してクライアントの顧客にアクセスする場合、マネージャー顧客IDを提供します。 |
|コンバージョンアクションオーバーライド| （オプション）**必須**セクションのコンバージョンアクションを上書きするためのコンバージョンアクションIDを提供します。このパラメータを使用すると、EventStream変数を使用してコンバージョンアクションIDを入力できます。コンバージョンアクションIDを見つけるには、Google Adsダッシュボードに移動し、コンバージョンをクリックします。コンバージョンアクションIDは、ウェブサイトURLの `ctId` クエリパラメータの値です。たとえば、URLが `https://ads.google.com/aw/conversions/detail?ctId=987654321` の場合、コンバージョンアクションIDは `987654321` です。 |
|広告ユーザーデータの同意| オプション。広告ユーザーデータの同意。デフォルト値は `GRANTED` です。有効な値は `GRANTED` または `DENIED` です。 |
