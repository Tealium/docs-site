## 前提条件

*   Tealium EventStreamまたはAudienceStream
*   Pega Customer Decision Hubへのアクセス、以下が必要です：
    * クライアントID、クライアントシークレット、アクセストークンURL
    * [Tealium Customer Data Connector](https://docs.pega.com/bundle/components/page/customer-decision-hub/components/tealium-integration-component.html) コンポーネントの実装

## 動作原理

顧客の好みや次の最適な行動を正確に予測するには、その顧客と最近の全チャネルにわたる相互作用をよく理解することが重要です。Tealiumは、正確で豊富な顧客データを簡単に接続して、決定や推奨に役立てることで、PegaのCustomer Decision Hubをエンリッチメントします。

Pegasystemsコネクタは、以下の二つの方法で貴重な顧客データを接続します：

1.  TealiumとPegaは、顧客の360度ビューに基づいて統合された顧客情報を共有し、標準化された顧客プロファイルにすべてのクロスチャネルシグナルを集約します。
1.  Tealiumは、ウェブ、アプリ、コールセンター、その他のチャネルを通じて顧客の相互作用をリアルタイムでPegaにストリーミングし、意思決定のためのその場のコンテキストを提供します。

Pegasystemsコネクタは、EventStreamおよびAudienceStreamアクションをサポートし、EventStreamを通じてページビュー/相互作用を追跡することができます。顧客識別子またはクッキーIDを使用して、異なるページタイプを構成し、相互作用履歴をPegaに送信するようにマッピングできます。

さらに、顧客識別子を使用して、オーディエンスセグメントとステータスをPegaに送信することで、Pega Customer Decision Hub内で次の最適なアクションをトリガーします。

## 構成の構成

コネクタマーケットプレイスに移動し、新しいコネクタを追加します。コネクタを追加する一般的な手順については、[コネクタについて]()の記事を参照してください。

コネクタを追加した後、次の構成を構成します：

* **クライアントID**：（必須）クライアントに割り当てられた一意のIDです。
* **クライアントシークレット**：（必須）クライアントがPegaプラットフォームの認証モジュールで認証するためにシステムが割り当てるパスワードです。
* **アクセストークンURL**：（必須）認証後にクライアントのアクセストークンをレスポンスとして受け取るために使用するリクエストURLです。
* **認証方法**：（必須）Pegaプラットフォームにアクセスするために使用される認証方法です。
  * **POST**：クライアント認証情報をPOSTリクエストの本文に送信します。
  * **BASIC**：クライアント認証情報（クライアントIDおよびクライアントシークレット）を認証ヘッダーの一部として外部アプリケーションに送信します。

クライアントによって使用されるRESTサービス認証には、**クライアントID**および**クライアントシークレット**の認証情報が必要です。

コネクタの構成が完了したら、**完了**をクリックします。

## アクション構成 — パラメータとオプション

コネクタアクションの構成を続けるために**続行**をクリックします。アクションの名前を入力し、ドロップダウンメニューからアクションタイプを選択します。

次のセクションでは、各アクションのパラメータとオプションの構成方法について説明します。

### アクション — セグメントメンバーシップデータの更新（リアルタイム）

このアクションを使用して、Pegaの次の最適なアクション決定をトリガーするために、オーディエンスデータをPegaに送信します。リアルタイムで意思決定を行うために、その訪問のプロファイルに関連する追加パラメータとともにオーディエンスデータを送信します。

#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| サービスエンドポイントURL | Pegaシステムで定義されたサービスエンドポイントURL。&lt;br&gt;**サービスエンドポイントURL**は以下の構造を使用しますが、カスタマイズ可能であり、Pegaの担当者から提供される場合があります：`https://&lt;host&gt;:&lt;port&gt;/prweb/PRRestService/&lt;servicepackage&gt;/&lt;serviceclass&gt;/&lt;servicemethod&gt;` |
| 顧客ID | (必須) Pega顧客プロファイルの主キーです。 |
| 名前 | (必須) Pegaプラットフォームに送信されるオーディエンス名です。Pegaに渡すために、オーディエンス名または類似の値を文字列として構成することをお勧めします。 |
| ステータス | (必須) オーディエンスに参加または離脱するためのオーディエンストリガーアクションです。選択したアクションに基づいて、セグメントステータスフィールドに**参加**または**離脱**の値をマッピングすることをお勧めします。 |
| 名前空間 | (必須) Pegaに提供されるオーディエンスデータを補完するために使用されるサブカテゴリパラメータです。 |
| 最終資格取得時間 | (必須) ユーザーがオーディエンスに参加したタイムスタンプです。日付属性は[ISO 8601](https://en.wikipedia.org/wiki/ISO_8601)形式に変換されます。マッピングされていない場合、このパラメータは現在のタイムスタンプをISO 8601形式で初期化します。例：`2022-08-30T01:13:16.081Z`。 |

### アクション — セグメントメンバーシップデータの更新（バッチ処理）

#### バッチ制限

このアクションは、ベンダーへの大量データ転送をサポートするためにバッチリクエストを使用します。詳細については、[バッチ処理アクション]()を参照してください。リクエストは、次のいずれかの閾値が満たされるか、プロファイルが公開されるまでキューに入れられます：

* 最大リクエスト数：10000
* 最古のリクエストからの最大時間：60分
* リクエストの最大サイズ：100 MB

このアクションを使用して、Pegaの次の最適なアクション決定をトリガーするために、オーディエンスデータをPegaに送信します。リアルタイムで意思決定を行うために、その訪問のプロファイルに関連する追加パラメータとともにオーディエンスデータを送信します。

TealiumはPegaへのバッチ処理アクションをサポートしていますが、バッチ処理の必要性についてはPegaアカウントチームと相談することをお勧めします。

#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| サービスエンドポイントURL | Pegaシステムで定義されたサービスエンドポイントURL。&lt;br&gt;**サービスエンドポイントURL**は以下の構造を使用しますが、カスタマイズ可能であり、Pegaの担当者から提供される場合があります：`https://&lt;host&gt;:&lt;port&gt;/prweb/PRRestService/&lt;servicepackage&gt;/&lt;serviceclass&gt;/&lt;servicemethod&gt;` |
| 顧客ID | (必須) Pega顧客プロファイルの主キーです。 |
| 名前 | (必須) Pegaプラットフォームに送信されるオーディエンス名です。Pegaに渡すために、オーディエンス名または類似の値を文字列として構成することをお勧めします。 |
| ステータス | (必須) オーディエンスに参加または離脱するためのオーディエンストリガーアクションです。選択したアクションに基づいて、セグメントステータスフィールドに**参加**または**離脱**の値をマッピングすることをお勧めします。 |
| 名前空間 | (必須) Pegaに提供されるオーディエンスデータを補完するために使用されるサブカテゴリパラメータです。 |
| 最終資格取得時間 | (必須) ユーザーがオーディエンスに参加したタイムスタンプです。日付属性は[ISO 8601](https://en.wikipedia.org/wiki/ISO_8601)形式に変換されます。マッピングされていない場合、このパラメータは現在のタイムスタンプをISO 8601形式で初期化します。例：`2022-08-30T01:13:16.081Z`。 |

### アクション — カスタムイベントデータの送信（リアルタイム）

このEventStreamアクションを使用して、主にページビューのリアルタイムイベントデータをPegaに送信し、Pegasystemsエンドポイント内で集約します。
#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| サービスエンドポイントURL | あなたのPegaシステムで定義されたサービスエンドポイントURLの値です。&lt;br&gt;**サービスエンドポイントURL**は以下の構造を使用しますが、カスタマイズ可能であり、Pegaの担当者から提供される場合があります: `https://&lt;host&gt;:&lt;port&gt;/prweb/PRRestService/&lt;servicepackage&gt;/&lt;serviceclass&gt;/&lt;servicemethod&gt;` |
| ページタイプ | （必須）訪問されたページのタイプです。 |
| デバイスタイプ | （必須）イベント生成に使用されたデバイスです。 |
| イベントタイプ | （必須）イベントの名前です。`tealium_event`の値を使用することを推奨します。 |
| 顧客ID | Pega顧客プロファイルの主キーです。**Cookie ID**が提供されていない場合に必要です。 |
| Cookie ID | **顧客ID**が提供されていない場合に必要です。両方が提供されている場合は、**顧客ID**のみが使用されます。 |
| イベントタイムスタンプ | イベントが生成された時間です。日付属性はISO [8601](https://en.wikipedia.org/wiki/ISO_8601)形式に変換されます。マッピングされていない場合、現在のタイムスタンプをISO 8601形式で初期化します。例: `2022-08-30T01:13:16.081Z`。 |
| ページ位置 | **PageView**イベントが提供される場合、ページURLが必要です。 |
| ページ閲覧者クティブタイム | 訪問がページに滞在した時間（秒）。 |

### アクション — カスタムイベントデータの送信（バッチ処理）

#### バッチ制限

このアクションは、ベンダーへの大量データ転送をサポートするためにバッチリクエストを使用します。詳細については、[バッチアクション]()を参照してください。リクエストは、次のいずれかの閾値が達成されるか、プロファイルが公開されるまでキューに入れられます：

* 最大リクエスト数：10000
* 最古のリクエストからの最大時間：60分
* リクエストの最大サイズ：100 MB

このEventStreamアクションを使用して、ページビューを主にPegaにバッチでイベントデータを送信し、Pegasystemsエンドポイント内で集約します。

TealiumはPegaへのバッチアクションをサポートできますが、バッチ処理の必要性と構成についてPegaアカウントチームと相談することをお勧めします。

### アクション — IDのマージ

このアクションを使用して、顧客の行動情報をマージし、累積的な行動データに基づいてより正確な推奨事項や次善のアクション決定を行います。

#### Tealiumでの推奨構成

以下のPegasystemsコネクタ構成を推奨します：

1. 次の属性を作成します：
    * **Pega顧客ID**：（EventStream文字列）データレイヤーから継承されます。
    * **Pega顧客ID群**：（AudienceStream文字列の配列）Tealiumによって縫い合わされた**Pega顧客ID**です。
    * **Pega顧客ID数**：（AudienceStream数）**Pega顧客ID群**の文字列セットの長さです。
1. **Pega match check**という名前のオーディエンスを作成し、条件は`Pega Customer IDs count &gt; 1`です。
1. 既存のPegaコネクタに**顧客行動データのマージ**アクションを追加します：
    * **トリガー**：`訪問の終わりにオーディエンスにいます`。
    * **チェックを外す**：`このオーディエンスの訪問開始時にはメンバーではありませんでした`。

#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| プライマリID / セカンダリID | （必須）プライマリIDとセカンダリIDを含む文字列の配列です。最初の値はプライマリIDで、その後の値はセカンダリIDです。例：`[&#34;ID12345&#34;,&#34;ID12346&#34;,&#34;ID12347&#34;]`。 |
| アプリケーション名 | 既知の顧客と未知の見込み客のデータが保存されているアプリケーションのIDです。この値が渡されない場合、サービスURLのアプリケーションエイリアスの値からアプリケーションIDが特定されます。 |
| コンテキスト名 | コンテキスト名を選択するか、カスタムコンテキスト名を入力します：**顧客**、**提供者**、または**加入者**。 |
