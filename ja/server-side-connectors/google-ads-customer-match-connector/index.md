## Google Data Manager API アクションへの移行

以下のアクションは古いGoogle Ads APIを使用しています：

* リマーケティングリストへのユーザー追加
* リマーケティングリストからのユーザー削除

以下の代替アクションを使用してください。これらはGoogle Data Manager APIに基づいています：

* リストへのユーザー追加（Data Manager API）
* リストからのユーザー削除（Data Manager API）

詳細については、[Google Data Manager API 移行ガイド]()を参照してください。

## 構成

コネクタマーケットプレイスにアクセスし、新しいコネクタを追加します。コネクタの追加方法については、[コネクタについて]()の記事を参照してください。

このコネクタを追加する際には、ベンダーのデータプラットフォームポリシーを受け入れるよう求められます。

コネクタを追加した後、以下の構成を構成します：

* **カスタマーID**：（必須）管理する広告アカウントのカスタマーIDを提供してください。
* **マネージャーカスタマーID**：（オプション）クライアントのカスタマーにアクセスする場合、マネージャーカスタマーIDを構成し、広告アカウントへの完全なアクセス権を持っている必要があります。

### ユーザーリストの作成

ユーザーリストを作成するには、**ユーザーリストの作成**をクリックし、次の情報を入力します：

| **パラメータ** | **説明** |
| --- | --- |
| リスト名 | （必須）ユーザーリストの名前。 |
| リストタイプ | （必須）リストのタイプ。これにより、このリストで使用されるユーザー識別情報のタイプが決まります：&lt;ul&gt;&lt;li&gt;`CONTACT_INFO`：顧客情報（メールアドレス、電話番号、物理的住所など）からユーザーを照合します。&lt;/li&gt;&lt;li&gt;`CRM_ID`：広告主が生成し割り当てたユーザーIDからユーザーを照合します。&lt;/li&gt;&lt;li&gt;`MOBILE_ADVERTISING_ID`：モバイルデバイスID（広告ID/IDFA）からユーザーを照合します。&lt;/li&gt;&lt;/ul&gt; |
| アプリID | モバイル広告リストタイプに必要です。データをGoogle Ads APIに送信するモバイルアプリケーションを一意に識別する文字列です。 |
| リストメンバーシップの寿命 | （オプション）ユーザーがリストに追加されてからリストに残る日数。0から540の間でなければなりません。デフォルトの寿命は540日です。 |
| リストの説明 | （オプション）リストの説明。 |

### ユーザー識別子

各アクションにはユーザー識別子が必要です。ベンダーはこれらの値が正規化され、SHA-256でハッシュ化されていることを要求します。マップされたユーザー識別子の値は、次の要件を満たすことが期待されます：

* 小文字
* テキストの先頭と末尾からの空白のトリミング
* SHA-256でのハッシュ化

すでに正規化されハッシュ化された属性をマッピングすることも、コネクタに正規化とハッシュ化を行わせることもできます。シナリオに適したマッピングを選択してください。

必要なユーザー識別子は、選択されたユーザーリストタイプに基づいています。ユーザーリストタイプは次のいずれかになります：

* Google Ads API：
  * `CONTACT_INFO`
  * `CRM_ID`
  * `MOBILE_ADVERTISING_ID`
* Google Data Manager API：
  * `CONTACT_ID`
  * `MOBILE_ID`
  * `USER_ID`

次のユーザー識別子フィールドがサポートされています：

| ユーザー識別子フィールド | 説明 |
|---| ---|
| `CONTACT_INFO` (Google Ads API)|  ハッシュ化されたメール、ハッシュ化された電話番号、または住所情報を提供します。このユーザーデータタイプには、以下にリストされているすべての住所情報フィールドが必要です。&lt;ul&gt;&lt;li&gt;**住所情報：国コード**：ユーザーの住所の2文字の国コード、ISO 3166-1 alpha-2形式。&lt;/li&gt;&lt;li&gt;**住所情報：名（すでにSHA256ハッシュ化されている）**：空白をトリミングし、小文字にし、SHA256でハッシュ化された名を提供します。&lt;/li&gt;&lt;li&gt;**住所情報：名（SHA256ハッシュを適用）**：プレーンテキストの名を提供し、コネクタがこの値をSHA256ハッシュでハッシュ化します。&lt;/li&gt;&lt;li&gt;**住所情報：姓（すでにSHA256ハッシュ化されている）**：空白をトリミングし、小文字にし、SHA256でハッシュ化された姓を提供します。&lt;/li&gt;&lt;li&gt;**住所情報：姓（SHA256ハッシュを適用）**：プレーンテキストの姓を提供し、コネクタがこの値をSHA256ハッシュでハッシュ化します。&lt;/li&gt;&lt;li&gt;**住所情報：郵便番号**：ユーザーの住所の郵便番号を提供します。&lt;/li&gt;&lt;li&gt;**メールアドレス（すでにSHA256ハッシュ化されている）**：空白をトリミングし、小文字にし、SHA256でハッシュ化されたメールアドレスまたはメールアドレスの配列を提供します。&lt;/li&gt;&lt;li&gt;**メールアドレス（SHA256ハッシュを適用）**：プレーンテキストのメールアドレスまたはメールアドレスの配列を提供し、コネクタがこれらの値をSHA256ハッシュでハッシュ化します。&lt;/li&gt;&lt;li&gt;**電話番号（すでにSHA256ハッシュ化されている）**：空白をトリミングし、SHA256でハッシュ化された電話番号または電話番号の配列を提供します。&lt;/li&gt;&lt;li&gt;**電話番号（SHA256ハッシュを適用）**：プレーンテキストの電話番号または電話番号の配列を提供し、コネクタがこれらの値をSHA256ハッシュでハッシュ化します。&lt;/li&gt;&lt;/ul&gt; |
|`CRM_ID` (Google Ads API)|  &lt;ul&gt;&lt;li&gt;**ユーザーID**：（必須）広告主が割り当てたユーザーIDで、カスタマーマッチのアップロードに使用します。&lt;/li&gt;&lt;/ul&gt; |
|`MOBILE_ADVERTISING_ID` (Google Ads API)|  &lt;ul&gt;&lt;li&gt;**モバイルID**：（必須）モバイルデバイスID（広告ID/IDFA）。&lt;/li&gt;&lt;/ul&gt; |
|`CONTACT_ID` (Data Manager API)| **連絡先情報**：メールアドレス、電話番号、住所。 |
|`MOBILE_ID` (Data Manager API)|  **モバイルID**：モバイルデバイスID。 |
|`USER_ID` (Data Manager API)|  **ユーザーID**：第一者識別子。 |

### リストへのユーザー追加（Data Manager API）

#### API情報

このコネクタは次のベンダーAPIを使用します：

* API名：Google Data Manager API
* APIバージョン：v1
* APIエンドポイント：`https://datamanager.googleapis.com`
* ドキュメント：[Google Data Manager API](https://developers.google.com/data-manager/api/reference/rest)

#### バッチ制限

このアクションは、ベンダーへの大量データ転送をサポートするためにバッチリクエストを使用します。並行処理により、イベントがベンダーに順不同で到達する可能性があります。順序が重要な場合は、イベントにシーケンス値を追加してください。詳細については、[バッチアクション]()を参照してください。次のいずれかの閾値が達成されるか、プロファイルが公開されるまでリクエストはキューに入れられます：

* 最大リクエスト数：10,000
* 最古のリクエストからの最大時間：60分
* リクエストの最大サイズ：40 MB

#### パラメータ

| パラメータ | 説明 |
| --- | --- |
| ユーザーリスト | Data Manager APIからユーザーリストを選択します。詳細については、[Data Manager API: カスタマーマッチの概要](https://developers.google.com/data-manager/api/devguides/audiences/google-ads/customer-match)を参照してください。 |

#### ユーザー識別子

| パラメータ | 説明 |
| --- | --- |
| カスタマーIDの上書き | （オプション）コネクタ構成のカスタマーIDを上書きするカスタマーIDを提供します。このオプションを使用する場合は、ユーザーリストIDの上書きも使用する必要があります。 |
| ユーザーリストIDの上書き | （オプション）必要セクションのユーザーリストを上書きするユーザーリストIDを提供します。このフィールドを使用すると、イベント属性を使用してユーザーリストIDを入力できます。カスタマーIDの上書きを使用する場合は必須です。値はリストIDでなければなりません。例：`9310271231`。 |

### リマーケティングリストへのユーザー追加（非推奨）

#### バッチ制限

このコネクタアクションは、ベンダーへの大量データ転送をサポートするためにバッチリクエストを使用することができます。詳細については、[バッチアクション]()を参照してください。次のいずれかの閾値が達成されるか、プロファイルが公開されるまでリクエストはキューに入れられます：

* 最大リクエスト数：33,000
* 最古のリクエストからの最大時間：60分
* リクエストの最大サイズ：40 MB

#### API情報

このコネクターは以下のベンダーAPIを使用しています：

* API名：Google Ads API
* APIバージョン：v18
* APIエンドポイント：`https://googleads.googleapis.com/`
* ドキュメント：[Google Ads API](https://developers.google.com/google-ads/api/docs/start)

#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| リマーケティングリスト | リマーケティングユーザーリストを選択します。詳細については、[Google Ads: リマーケティングとオーディエンスターゲティング](https://developers.google.com/google-ads/api/docs/remarketing/overview)を参照してください。利用可能なオプションには、ファーストパーティCRMデータタイプのユーザーリストのみが含まれます。ユーザーリストを作成するには、[ユーザーリストの作成](#create-user-list)を参照してください。|
| カスタマーIDオーバーライド | （オプション）コネクタ構成のカスタマーIDをオーバーライドするためのカスタマーIDを提供します。このオプションを使用する場合は、**ユーザーリストIDオーバーライド**も入力する必要があります。 |
| マネージャーカスタマーIDオーバーライド | （オプション）**カスタマーIDオーバーライド**を使用してクライアントカスタマーにアクセスする場合は、マネージャーカスタマーIDを提供します。 |
| ユーザーリストIDオーバーライド | （オプション）ユーザーリストのリソース名（リストIDとタイプ）をオーバーライドするために提供します。このフィールドでは、ユーザーリストIDを入力するためにEventStream変数を使用できます。**カスタマーIDオーバーライド**を使用する場合は必須です。値は次の形式である必要があります `ID:TYPE`、例えば `123456:CONTACT_INFO`。 |

#### 同意

**リマーケティングリストへのユーザー追加**アクションを使用する際、コネクタは`adUserData`および`adPersonalization`パラメータに対して`GRANTED`の値を送信します。同意していない訪問がリストに追加されないようにするためには、訪問のプロファイル同意状態をチェックするルールを持つオーディエンスを作成し、リストから同意していない訪問を削除するために**リマーケティングリストからの削除**アクションを使用します。
