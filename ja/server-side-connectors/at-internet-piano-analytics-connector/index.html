ATインターネットピアノアナリティクスコネクタ構成ガイド

この記事では、ATインターネットピアノアナリティクスコネクタの構成方法について説明します。

バッチ制限

このコネクタは、ベンダーへの大量のデータ転送をサポートするためにバッチリクエストを使用します。詳細については、バッチアクションを参照してください。リクエストは、以下のいずれかの閾値が達成されるか、プロファイルが公開されるまでキューに入れられます：

  • リクエストの最大数：100
  • 最古のリクエストからの最大時間：5分
  • リクエストの最大サイズ：1 MB

コネクタアクション

アクション名 AudienceStream EventStream
イベント送信 ✓ ✓

構成の構成

コネクタマーケットプレイスに移動し、新しいコネクタを追加します。コネクタの追加方法の一般的な指示については、コネクタについての記事を参照してください。

コネクタを追加した後、以下の構成を構成します：

  • コレクションドメイン
    必須：あなたの組織のコレクションエンドポイント。
  • コレクションパス
    必須：コレクションパス。
  • サイトID
    必須：データソースのサイトID。

コネクタの構成が完了したら、完了をクリックします。

アクション構成 - パラメータとオプション

続行をクリックしてコネクタアクションを構成します。アクションの名前を入力し、ドロップダウンメニューからアクションタイプを選択します。

次のセクションでは、各アクションのパラメータとオプションの構成方法について説明します。ATインターネットピアノアナリティクスコネクタパラメータについての詳細は、コレクションAPIのドキュメンテーションを参照してください。

アクション - イベント送信

パラメータ

パラメータ 説明
イベントタイプ すべての標準イベントがサポートされています。
カスタム、非標準イベントを送信するには、カスタムオプションを選択し、イベントプロパティセクションでカスタムイベントタイプの値を提供します。
日付形式 イベントプロパティを記入する際の日付形式を指定します。
エポックUTC（またはCDH日付属性）からのミリ秒のみがフォーマット可能で、それ以外の場合はアクションが失敗します。
次の日付形式をマッピングします：YYYYMMDD、YYYY-MM-DD、YYYY-MM-DDTHH:mm:ssZ、UNIX。
IPアドレス クライアントのIPアドレス。
リファラー データを送信するページのアドレス。
サイトID データが所属するサイトのID。このパラメータは、値が空でない場合、構成タブで定義された値を上書きします。
ユーザーエージェント クライアントのユーザーエージェント。このフィールドがマッピングされていない場合、Tealium EventStream API Hubのユーザーエージェント属性とTealium AudienceStream CDPの訪問の最後のユーザーエージェント属性に自動マッピングされます。
訪問ID 訪問の一意の識別子。このフィールドがマッピングされていない場合、Tealium訪問IDに自動マッピングされます。
前のURL 前のページのURL。このフィールドがマッピングされていない場合、EventStreamの現在のURL属性とAudienceStreamのエントリURL属性に自動マッピングされます。
デバイスタイムスタンプUTC 秒単位のデバイスタイムスタンプUTC。日付属性はUNIXタイムスタンプに変換されます。
イベントコレクションプラットフォーム 使用されるコレクションプラットフォーム。
イベントコレクションバージョン コレクションプラットフォームのバージョン。
デバイス製造者 デバイスの製造者。
デバイスモデル デバイスのモデル。
デバイス表示高さ ビューポートの高さ。
デバイス表示幅 ビューポートの幅。
デバイス画面高さ デバイス画面の高さ。
デバイス画面幅 デバイス画面の幅。
