## API情報

このコネクタは以下のベンダーAPIを使用します：

* API名：The Trade Desk CRM API
* APIバージョン：v3
* APIエンドポイント：`https://api.thetradedesk.com/v3/crmdata/`
* ドキュメント：[The Trade Desk CRM API](https://partner.thetradedesk.com/v3/portal/data/doc/DataIntegrateCRMData)

## バッチ制限

このコネクタは、ベンダーへの大量データ転送をサポートするためにバッチリクエストを使用します。リクエストは、以下のいずれかの閾値に達するまでキューに入れられます：

* 最大リクエスト数：10,000
* 最古のリクエストからの最大時間：720分
* リクエストの最大サイズ：100 MB

## 構成

コネクタマーケットプレイスにアクセスして新しいコネクタを追加します。コネクタを追加する一般的な手順については、[コネクタについて](https://docs.tealium.com/about-connectors/)を参照してください。

コネクタを追加した後、以下の構成を構成します：
* **広告主ID**
  * （必須）あなたのTrade Desk広告主ID。
* **APIトークン**
  * （必須）長期間有効なAPIトークン。
  * トークンを生成するには、The Trade Deskパートナーポータルの**ユーザープロファイル**メニューの**APIトークンの管理**セクションにアクセスしてください。
  * 詳細については、[The Trade Desk：APIトークン認証](https://partner.thetradedesk.com/v3/portal/api/doc/Authentication)を参照してください。

## アクション

| アクション名 | AudienceStream | EventStream |
| --- | :---: | :---: |
| ターゲティングデータのアップロード | ✓ | ✓ |
| コンバージョンデータのアップロード | ✓ | ✓ |

アクション名を入力し、ドロップダウンメニューからアクションタイプを選択します。

次のセクションでは、各アクションのパラメータとオプションの構成方法について説明します。

### ターゲティングデータのアップロード

#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| ターゲティングセグメント | リストからTrade Deskターゲティングセグメントを選択します。**ターゲティングセグメントの作成**をクリックして、ターゲティングセグメントを作成します。 |

#### ユーザー識別子

オプションのパラメータ：

| **パラメータ** | **説明** |
| --- | --- |
| メールアドレス | プレーンテキストのメールアドレスを提供します。 |
| メールアドレス（既にSHA256ハッシュ化済み） | 空白をトリムし、小文字に変換し、SHA256でハッシュ化されたメールアドレスを提供します。コネクタは、The Trade Deskに送信する前に生のバイトをBase64でエンコードします。 |
| 電話番号 | プレーンテキストの電話番号を提供します。EUセグメントには電話番号をアップロードできないことに注意してください。 |
| 電話番号（既にSHA256ハッシュ化済み） | 空白をトリムし、SHA256でハッシュ化された電話番号を提供します。コネクタは、The Trade Deskに送信する前に生のバイトをBase64でエンコードします。EUセグメントには電話番号をアップロードできないことに注意してください。 |

### コンバージョンデータのアップロード

#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| コンバージョンセグメント | コンバージョンセグメントIDを入力します。The Trade Desk UIではトラッキングタグIDとして表示されます。<ul><li>**コンバージョンセグメントの作成**をクリックして新しいコンバージョンセグメントを作成します。</li><li>作成後、セグメントIDが**コンバージョンセグメント**フィールドに表示されます。</li><li>アップロードする前に、コンバージョンセグメントをThe Trade Desk UIのキャンペーンにアタッチする必要があります。このステップが完了するまで、アップロードされたデータをキャンペーンに帰属させることはできません。</li></ul> |

#### ユーザー識別子

オプションのパラメータ：

| **パラメータ** | **説明** |
| --- | --- |
| メールアドレス | プレーンテキストのメールアドレスを提供します。 |
| メールアドレス（既にSHA256ハッシュ化済み） | 空白をトリムし、小文字に変換し、SHA256でハッシュ化されたメールアドレスを提供します。コネクタは、The Trade Deskに送信する前に生のバイトをBase64でエンコードします。 |
| 電話番号 | プレーンテキストの電話番号を提供します。EUセグメントには電話番号をアップロードできないことに注意してください。 |
| 電話番号（既にSHA256ハッシュ化済み） | 空白をトリムし、SHA256でハッシュ化された電話番号を提供します。コネクタは、The Trade Deskに送信する前に生のバイトをBase64でエンコードします。EUセグメントには電話番号をアップロードできないことに注意してください。 |

#### コンバージョンデータ

オプションのパラメータ：

| **パラメータ** | **説明** |
| --- | --- |
| タイムスタンプ | コンバージョンのタイムスタンプ。指定されていない場合、コネクタは現在のタイムスタンプを送信します。 |
| 値 | コンバージョンの値を小数点以下2桁で。 |
| 通貨 | 価値の金額の通貨。例：`USD`。 |
| 国 | コンバージョンが発生した国の正式名称。 |
| 地域 | **国**フィールドがアメリカ合衆国に構成されている場合は必須。コンバージョンが発生した地域の正式名称またはコード。例：`NY`または`New York`。 |
| メトロ | コンバージョンが発生した都市圏の数値地域ターゲティングコード。 |
| 市 | コンバージョンが発生した市。 |
| 注文ID | トランザクションまたはコンバージョンイベントの一意の識別子。最大長は64文字です。 |
| プライバシータイプ | ユーザーの地理的位置に基づいて適用されるプライバシー規制を指定します（`GDPR`、`GPP`）。 |
| 適用可能 | プライバシータイププロパティに指定された値が適用可能かどうかを示します。 |
| 同意文字列 | プライバシー規制が有効な場合のユーザーの同意（`TCF`文字列、`GPP`文字列）。 |
