Amazon Ads オーディエンス管理は、Amazon Marketing Cloud (AMC) 内でのオーディエンスのアップロード、管理、使用を効率化します。Amazon Ads オーディエンスコネクタを使用して、オーディエンス管理サービスを通じてオーディエンスを作成およびアップロードします。これらのオーディエンスは、Amazon DSP、AMCユーザーインターフェースワークフロー、またはAMCルールベースのオーディエンスなどのAPIアクションを通じてフォローアップアクションの対象となります。

 このコネクタは、AMCまたはDSPのオーディエンスに訪問を追加することができます。AMCオーディエンスの場合、[**構成**]()プロセス中にAMC接続を作成する必要があります。DSPオーディエンスの場合、[DSPオーディエンスを作成する]()ときに広告主IDを選択する必要があります。

## API情報

このコネクタは以下のベンダーAPIを使用します：

* API名：Amazon Ads API
* APIバージョン：v1.0
* APIエンドポイント：`https://advertising-api.amazon.com`

## バッチ制限

このコネクタは、ベンダーへの大量データ転送をサポートするためにバッチリクエストを使用します。詳細については、[バッチアクション]()を参照してください。リクエストは、次のいずれかの閾値に達するか、プロファイルが公開されるまでキューに入れられます：

* 最大リクエスト数：6,000
* 最古のリクエストからの最大時間：60分
* リクエストの最大サイズ：1 MB

## 構成

コネクタマーケットプレイスにアクセスして新しいコネクタを追加します。コネクタを追加する一般的な手順については、[コネクタについて]()を参照してください。

コネクタを追加した後、次の構成を構成します：

* **Amazon Ads アカウントリージョン**  
  * (必須) Amazon Ads アカウントのリージョンを選択します。

### Amazon認証

コネクタをAmazonで認証するには：

1. **Amazon Ads アカウントリージョン**リストからリージョンを選択します。
1. **接続を確立**をクリックします。
1. Amazon **サインイン**ダイアログで、Amazonの認証情報を入力します。

### AMCオーディエンス接続

AMCオーディエンスには、オーディエンスを構成するための接続オブジェクトが必要です。Amazonは、AMCインスタンスとDSPオーディエンスのマッピングを格納するオブジェクトを保持しています。

AmazonでAMC接続を確立するには：

1. **AMC接続を作成**をクリックします。
1. **AMC接続を作成**ダイアログで、オーディエンスアップロードに使用するAMCインスタンスを**AMCインスタンス**リストから選択します。
1. (オプション) **DSP広告主ID**リストからDSP広告主IDを選択または入力して、オーディエンスをDSPに同期します。
1. **完了**をクリックし、その後**作成**をクリックします。

### 国コード

国コードは、ISO 3166形式の2文字の文字列で、データが収集された国を示します。**国コード**はオプションのパラメータですが、データのアップロード時に含めることを強くお勧めします。レコードに追加された国コードは、オーディエンスに追加された国コードを上書きします。

詳細については、[Amazon Ads: 国コード](https://advertising.amazon.com/API/docs/en-us/guides/amazon-marketing-cloud/audiences/audience-management-service#country-code)を参照してください。

## アクション

| アクション名 | AudienceStream | EventStream |
| --- | :---: | :---: |
| AMCオーディエンスへの訪問追加 | ✓ | ✓ |
| DSPオーディエンスへの訪問追加 | ✓ | ✓ |
| オーディエンスからの訪問削除 | ✓ | ✓ |

アクションの名前を入力し、リストからアクションタイプを選択します。

次のセクションでは、各アクションのパラメータとオプションの構成方法について説明します。

### AMCオーディエンスへの訪問追加

#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| オーディエンスID | 訪問を追加するオーディエンスID。&lt;ol&gt;&lt;li&gt;新しいオーディエンスを作成するには、**オーディエンスを作成**をクリックします。&lt;/li&gt;&lt;li&gt;**オーディエンスを作成**画面で、リストから**AMC接続**を選択します。&lt;/li&gt;&lt;li&gt;**オーディエンス名**を入力します。&lt;/li&gt;&lt;li&gt;**説明**フィールドにオーディエンスの説明を入力します。&lt;/li&gt;&lt;li&gt;**国コード**を入力します。これは、ISO 3166形式の2文字の文字列で、オーディエンス内のユーザーデータが収集された場所を表します。構成されていない場合、デフォルト値**UNKNOWN**が適用されます。&lt;/li&gt;&lt;li&gt;**完了**をクリックします。&lt;/li&gt;&lt;li&gt;**オーディエンスIDを確認**をクリックして、オーディエンスIDを確認します。&lt;/li&gt;&lt;/ol&gt;このコネクタで作成されたオーディエンスは、Amazonに送信される前にオーディエンス名にプレフィックス`Tealium:`を追加します。 |
| AMC接続 | (必須) AMC接続を選択します。&lt;ul&gt;&lt;li&gt;必要に応じて、コネクタ構成画面で新しい接続を作成します。&lt;/li&gt;&lt;li&gt;接続IDは、オーディエンスが作成されたときに使用されたものと一致している必要があります。&lt;/li&gt;&lt;/ul&gt; |
| 外部ユーザーID | Tealium訪問ID属性など、各訪問に固有であると知られている非AmazonユーザーIDを選択します。 |
| 外部ID (SHA256ハッシュを適用) | 外部ユーザーIDにSHA256ハッシュを適用する場合は、このチェックボックスを選択します。 |
| 国コード | ISO 3166形式の2文字の文字列を提供して、データが収集された国を示します。レコードに追加された国コードは、オーディエンスに追加された国コードを上書きします。 |

#### ユーザー識別子

| **パラメータ** | **説明** |
| --- | --- |
| Email Address (already SHA256 hashed) | すでに空白トリム、小文字化、SHA256ハッシュ処理が施されたメールアドレスを提供します。 |
| Email Address (apply SHA256 hash) | プレーンテキストのメールアドレスを提供し、コネクタがこの値を空白トリム、小文字化し、SHA256ハッシュを使用してハッシュ処理します。 |
| Phone Number (already SHA256 hashed) | すでに空白トリムおよびSHA256ハッシュ処理が施された電話番号を提供します。 |
| Phone Number (apply SHA256 hash) | プレーンテキストの電話番号を提供し、コネクタがこの値を空白トリムし、SHA256ハッシュを使用してハッシュ処理します。 |
| First Name (already SHA256 hashed) | すでに空白トリム、小文字化、SHA256ハッシュ処理が施された名を提供します。 |
| First Name (apply SHA256 hash) | プレーンテキストの名を提供し、コネクタがこの値を空白トリム、小文字化し、SHA256ハッシュを使用してハッシュ処理します。 |
| Last Name (already SHA256 hashed) | すでに空白トリム、小文字化、SHA256ハッシュ処理が施された姓を提供します。 |
| Last Name (apply SHA256 hash) | プレーンテキストの姓を提供し、コネクタがこの値を空白トリム、小文字化し、SHA256ハッシュを使用してハッシュ処理します。 |
| City (already SHA256 hashed) | すでに空白トリム、小文字化、SHA256ハッシュ処理が施された市を提供します。 |
| City (apply SHA256 hash) | プレーンテキストの市を提供し、コネクタがこの値を空白トリム、小文字化し、SHA256ハッシュを使用してハッシュ処理します。 |
| State (already SHA256 hashed) | すでに空白トリム、小文字化、SHA256ハッシュ処理が施された州を提供します。 |
| State (apply SHA256 hash) | プレーンテキストの州を提供し、コネクタがこの値を空白トリム、小文字化し、SHA256ハッシュを使用してハッシュ処理します。 |
| Postal Code (already SHA256 hashed) | すでに空白トリム、小文字化、SHA256ハッシュ処理が施された郵便番号を提供します。 |
| Postal Code (apply SHA256 hash) | プレーンテキストの郵便番号を提供し、コネクタがこの値を空白トリム、小文字化し、SHA256ハッシュを使用してハッシュ処理します。 |

#### 同意

**AMCオーディエンスへの訪問追加**アクションを使用する場合、コネクタは`amznAdStorage`および`amznUserData`パラメータに対して`GRANTED`の値を送信します。同意を与えていない訪問をリストに追加しないようにするために、訪問プロファイルの同意状態をチェックするルールを持つオーディエンスを作成します。その後、**オーディエンスからの訪問削除**アクションを使用して、同意を与えていない訪問をリストから削除します。

### DSPオーディエンスへの訪問追加


#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| Audience ID | 訪問を追加するAudience ID。&lt;ol&gt;&lt;li&gt;新しいオーディエンスを作成するには、**Create Audience**をクリックします。&lt;/li&gt;&lt;li&gt;**Create Audience**画面で、リストから**DSP Advertiser**を選択または入力します。&lt;/li&gt;&lt;li&gt;**Audience Name**を入力します。&lt;/li&gt;&lt;li&gt;**Description**フィールドにオーディエンスの説明を入力します。&lt;/li&gt;&lt;li&gt;**Country Code**を入力します。これは、オーディエンスのユーザーデータが収集された場所を表すISO 3166形式の2文字の文字列です。構成されていない場合はデフォルト値**UNKNOWN**が適用されます。&lt;/li&gt;&lt;li&gt;**Done**をクリックします。&lt;/li&gt;&lt;li&gt;オーディエンスIDを確認するには、**Verify Audience ID**をクリックします。&lt;/li&gt;&lt;/ol&gt; このコネクタで作成されたオーディエンスは、Amazonに送信される前にオーディエンス名に`Tealium:`の接頭辞が付けられます。 |
| External User ID | Amazon以外のユーザーIDを選択します。例えば、Tealiumの訪問ID属性や、訪問ごとに一意であると分かっている他の内部顧客IDなどです。 |
| External ID (apply SHA256 hash) | このチェックボックスを選択して、External User IDにSHA256ハッシュを適用します。 |

### オーディエンスから訪問を削除

#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| Audience ID | 既存のTealium作成のAmazonオーディエンスを使用する場合は、そのIDを入力し、**Verify Audience ID**をクリックしてオーディエンスが正しいことを確認します。**Add to Audience**アクションの構成で作成されたオーディエンスIDをコピーすることができます。 |
| AMC Connection | (必須) AMC接続を選択します。&lt;ul&gt;&lt;li&gt;必要に応じて、コネクタ構成画面で新しい接続を作成します。&lt;/li&gt;&lt;li&gt;接続IDは、オーディエンスが作成されたときに使用されたものと一致する必要があります。&lt;/li&gt;&lt;/ul&gt; |
| External User ID | Amazon以外のユーザーIDを選択します。例えば、Tealiumの訪問ID属性や、訪問ごとに一意であると分かっている他の内部顧客IDなどです。 |
