## コネクタアクション

| アクション名 | AudienceStream | EventStream |
| --- | :---: | :---: |
|キューまたはトピックへのイベントデータ送信| ✗| ✓|
|キューまたはトピックへの訪問データ送信| ✓| ✗|
|キューまたはトピックへのカスタマイズデータ送信| ✓| ✓|

## 必要条件

* Microsoft Azure アカウント
* Service Bus 共有アクセスポリシー認証情報
* データを送信するキューまたはトピック


## 構成

コネクタマーケットプレイスに移動し、新しいコネクタを追加します。コネクタを追加する一般的な手順については、[コネクタについて]()の記事を参照してください。

ベンダーを構成するには、次の手順に従います：

1. 「構成」タブでコネクタインスタンスのタイトルを入力します。

1. Service Bus 名前空間、共有アクセスポリシー名、キーを入力します。詳細については、[名前空間と共有アクセスポリシー](https://docs.microsoft.com/en-us/azure/service-bus-messaging/service-bus-create-namespace-portal)を参照してください。

1. **接続テスト**をクリックして、提供された認証情報でAPI接続性を確認します。

共有アクセスポリシーは、`Send`、`Listen`、`Manage`の3種類の権限を主張できます。提供されたポリシーは少なくとも`Send`権限を主張する必要があります。

## アクション構成 - パラメータとオプション

**次へ**をクリックするか、**アクション**タブに進みます。ここでアクションのトリガーを構成します。

このセクションでは、各アクションのパラメータとオプションの構成方法について説明します。

### アクション - キューまたはトピックへのイベントデータ送信

#### パラメータ

| パラメータ | 説明 |
|---|---|
| キューまたはトピック | (必須): このリストからキューまたはトピックオプションを選択します。これらのオプションは、構成されたポリシーに**Manage**権限が付与されている場合にのみ利用可能です。権限が付与されていない場合は、カスタム値としてキュー/トピックを手動で入力する必要があります。 |
| ユーザープロパティ | **To**ドロップダウンでメッセージユーザープロパティに属性をマッピングします（カスタム値として入力）。 |
| ブローカープロパティ | **To**ドロップダウンで選択したメッセージブローカープロパティに属性をマッピングします。以下のオプションを参照してください。 |
| 属性名の表示 | このボックスをチェックすると、メッセージペイロード内の各イベントデータ属性の名前が表示されます。 |

すべてのアクションはメッセージユーザーおよびブローカープロパティをサポートしています。これらはキーと値のペアを使用してメッセージに関する便利な情報を提供しますが、HTTPヘッダーとして構成され、メッセージ本体内には存在しません。詳細については、[メッセージユーザーおよびブローカープロパティ](https://docs.microsoft.com/en-us/rest/api/servicebus/message-headers-and-properties)を参照してください。

#### オプション - ブローカープロパティ

|**オプション**| **説明**|
|---| ---|
|Content Type| メッセージ消費者用にHTTPヘッダー `Content-Type` を構成します|
|Correlation ID| 相関関係の一意の識別子|
|Session ID| 特定の消費者にメッセージをパーティション分割してルーティングするために使用できます|
|Message ID| デフォルトの自動生成メッセージIDを上書きします|
|Label| アプリケーションラベル|
|Reply To| メッセージ消費者に対してレスポンスを送信する場所を指定するために使用できます|
|Time to Live in Seconds| X秒数後にメッセージを期限切れにします（キュー/トピックレベルを上書き）|
|To| 送信先アドレス|
|Scheduled Enqueued Time| メッセージがエンキューされる予定の日時|
|Reply to Session ID| Reply Toと似ていますが、レスポンスのターゲットとしてセッションを指定します|
|Partition Key| パーティション分割されたキューまたはトピックの一意の識別子|

### アクション - キューまたはトピックへの訪問データ送信

#### パラメータ

| パラメータ | 説明 |
|---|---|
| キューまたはトピック | 必須。このリストからキューまたはトピックオプションを選択します。これらのオプションは、構成されたポリシーに**Manage**権限が付与されている場合にのみ利用可能です。権限が付与されていない場合は、カスタム値としてキュー/トピックを手動で入力する必要があります。 |
| ユーザープロパティ | **To**ドロップダウンでメッセージユーザープロパティに属性をマッピングします（カスタム値として入力）。 |
| ブローカープロパティ | **To**ドロップダウンで選択したメッセージブローカープロパティに属性をマッピングします。利用可能な[ブローカープロパティ](#options---broker-properties)オプションを参照してください。 |
| 現在の訪問データを訪問データに含める | ペイロードに現在の訪問データを追加します。これには、現在の訪問イベントデータが除外されていない場合、イベント訪問データが含まれます。 |
| 現在の訪問イベントデータを除外 | 現在の訪問データからイベントデータを除外します。 |
| 属性名の表示 | このボックスをチェックすると、メッセージペイロード内の各訪問データ属性の名前が表示されます。 |

### アクション - キューまたはトピックへのカスタマイズデータ送信（高度）

#### パラメータ

| パラメータ | 説明 |
|---|---|
| キューまたはトピック | (必須): このリストからキューまたはトピックオプションを選択します。これらのオプションは、構成されたポリシーに**Manage**権限が付与されている場合にのみ利用可能です。権限が付与されていない場合は、カスタム値としてキュー/トピックを手動で入力する必要があります。 |
| ユーザープロパティ | **To**ドロップダウンでメッセージユーザープロパティに属性をマッピングします（カスタム値として入力）。 |
| ブローカープロパティ | **To**ドロップダウンで選択したメッセージブローカープロパティに属性をマッピングします。利用可能な[ブローカープロパティ](#options---broker-properties)オプションを参照してください。 |
| メッセージデータ | (必須): カスタムメッセージ本体を構築します。属性を単一レベルのJSON形式の名前にマッピングするか、テンプレート名（二重波括弧で囲まれている）を参照し、**カスタムメッセージ定義**オプションを選択します。 |
| テンプレート変数 | 属性をテンプレート変数名にマッピングします。テンプレート変数は、すべてのテンプレートの置換およびレンダリングに利用可能です。詳細については、を参照してください。 |
| テンプレート | 一つ以上のテンプレートを提供します。通常、メッセージデータを構築するために単一のテンプレートが使用されます。一般的な構文と拡張機能については、[テンプレートガイド]()を参照してください。詳細については、を参照してください。 |

必要に応じて、テンプレートはユーザーおよびブローカープロパティにも参照され、注入されることがあります。
