## API 情報

このコネクタは以下のベンダー API を使用します：

* API 名: Google Ads API
* API バージョン: v18
* API エンドポイント: `https://googleads.googleapis.com/`
* ドキュメント: [Google Ads API](https://developers.google.com/google-ads/api/docs/start)

## コネクタのアクション

| アクション名 | AudienceStream | EventStream |
| --- | :---: | :---: |
| トランザクションのアップロード | ✓ | ✓ |
| 複数識別子でのトランザクションのアップロード | ✓ | ✓ |

## 構成の構成

コネクタマーケットプレイスに移動して、新しいコネクタを追加します。コネクタの追加方法については、[コネクタについて]()を参照してください。

このコネクタを追加する際には、ベンダーのデータプラットフォームポリシーを受け入れるよう求められます。

コネクタを追加した後、以下の構成を構成します：

* **顧客 ID**  
(必須) Google Ads アカウントの顧客 ID。
* **マネージャー顧客 ID**  
(オプション) クライアントを代表してアカウントにアクセスする場合、マネージャー顧客 ID を構成する必要があります。顧客 ID は、Google Ads にログインして **構成 &gt; アクセスとセキュリティ &gt; マネージャー** に移動することで見つけることができます。

## アクション構成 - パラメータとオプション

アクションの名前を入力し、ドロップダウンメニューからアクションタイプを選択します。

次のセクションでは、各アクションのパラメータとオプションの構成方法について説明します。

### アクション - トランザクションのアップロード

#### バッチ制限

このアクションは、ベンダーへの大量データ転送をサポートするためにバッチリクエストを使用します。詳細については、[バッチアクション]()を参照してください。リクエストは、次のいずれかの閾値に達するか、プロファイルが公開されるまでキューに入れられます：

* 最大リクエスト数: 100000
* 最古のリクエストからの最大時間: 60分
* リクエストの最大サイズ: 40 MB

#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| コンバージョンアクション | `customers/{customer_id}/conversionActions/{conversion_action_id}` 形式の有効な Google Ads コンバージョンアクションのリソース名。コンバージョンアクション ID は、Google Ads URL の `ctId` クエリパラメータです。コンバージョンアクションは `STORE_SALES` タイプでなければなりません。詳細については、[Google Ads API: 店舗売上のコンバージョンのインポート](https://developers.google.com/google-ads/api/docs/conversions/upload-store-sales-transactions)を参照してください。 |

##### ユーザー識別子

| **パラメータ** | **説明** |
| --- | --- |
| 住所情報: 国コード | ユーザーの住所の国コードを ISO 3166-1 alpha-2 形式の2文字で指定します。 |
| 住所情報: 名 (既に SHA256 ハッシュ化済み) | ユーザーの名を既に空白トリム、小文字化、SHA256 ハッシュ化したものを提供します。 |
| 住所情報: 名 (SHA256 ハッシュを適用) | ユーザーの名をプレーンテキストで提供し、コネクタがこの値を空白トリム、小文字化し、SHA256 ハッシュを適用します。 |
| 住所情報: 姓 (既に SHA256 ハッシュ化済み) | ユーザーの姓を既に空白トリム、小文字化、SHA256 ハッシュ化したものを提供します。 |
| 住所情報: 姓 (SHA256 ハッシュを適用) | ユーザーの姓をプレーンテキストで提供し、コネクタがこの値を空白トリム、小文字化し、SHA256 ハッシュを適用します。 |
| 住所情報: 郵便番号 | ユーザーの住所の郵便番号。 |
| メールアドレス (既に SHA256 ハッシュ化済み) | ユーザーのメールアドレスを既に空白トリム、小文字化、SHA256 ハッシュ化したものを提供します。 |
| メールアドレス (SHA256 ハッシュを適用) | ユーザーのメールアドレスをプレーンテキストで提供し、コネクタがこの値を空白トリム、小文字化し、SHA256 ハッシュを適用します。 |
| 電話番号 (既に SHA256 ハッシュ化済み) | ユーザーの電話番号を既に空白トリム、SHA256 ハッシュ化したものを提供します。 |
| 電話番号 (SHA256 ハッシュを適用) | ユーザーの電話番号をプレーンテキストで提供し、コネクタがこの値を空白トリムし、SHA256 ハッシュを適用します。 |
| サードパーティユーザー ID | サードパーティに割り当てられたユーザー ID。 |

##### トランザクションデータ

| **パラメータ** | **説明** |
| --- | --- |
| トランザクション日時 | (必須) トランザクションが発生したタイムスタンプを `YYYY-MM-DD HH:MM:SS[&#43;/-HH:MM]` 形式で指定します。ここで `[&#43;/-HH:MM]` は UTC からのオプションのタイムゾーンオフセットです。 |
| トランザクション金額 | (必須) トランザクション金額をマイクロ単位で指定します。ここで、100万が1通貨単位に相当します。 |
| 通貨コード | (必須) トランザクションの通貨コードを ISO 4217 の3文字形式で指定します。 |
| トランザクションアップロード比率 | (必須) 顧客に関連付けられる全売上に対してアップロードされる売上の比率。この値は0から1の間でなければならず（0を除く）、例えば顧客に関連付け可能な売上の半分をアップロードする場合、この値は0.5になります。 |
| トランザクション注文 ID | トランザクションの注文 ID。 |
| カスタムキー | トランザクションに適用可能な店舗売上のカスタム変数キーの名前。後でカスタムセグメンテーションのレポートで使用するために定義されたキーです。 |
| カスタム値 | 各トランザクションのカスタム変数の値。 |
| 店舗コード | Google に事前にアップロードされたユニークな店舗 ID。 |
| 商品 ID | 商品のユニークな識別子。Merchant Center の商品 ID または GTIN (グローバルトレードアイテムナンバー) のいずれかです。 |
| 数量 | 販売された商品の数。構成されていない場合、デフォルトは1です。 |
| 国コード | フィードにアップロードされた商品に関連する国の Common Locale Data Repository (CLDR) 領域コード。 |
| 言語コード | フィードにアップロードされた商品に関連する言語の ISO 639-1 コード |
| マーチャント ID | Merchant Center アカウントの ID。 |

##### 同意

コネクタはデフォルトで `adUserData` と `adPersonalization` の同意に `GRANTED` の値を送信します。デフォルトを上書きするには、関連する EventStream 属性を `Ad User Data Consent` および/または `Ad Personalization Consent` にマップし、値を `DENIED` に構成します。

| **パラメータ** | **説明** |
| --- | --- |
| Ad User Data Consent | 広告ユーザーデータの同意。デフォルト値は `GRANTED` です。有効な値は `GRANTED` と `DENIED` です。 |
| Ad Personalization Consent | 広告パーソナライゼーションの同意。デフォルト値は `GRANTED` です。有効な値は `GRANTED` と `DENIED` です。 |

#### パラメータ
