## アクション

| アクション名 | AudienceStream | EventStream |
| --- | :---: | :---: |
| コンバージョンポストバックを送信 | ✗ | ✓ |
| イベントポストバックを送信 | ✗ | ✓ |

## 設定

コネクタマーケットプレイスに移動し、新しいコネクタを追加します。コネクタの追加方法の一般的な指示については、[コネクタについて](https://docs.tealium.com/about-connectors/)を参照してください。

コネクタを追加した後、次の設定を行います：

* **ネットワークID**：TUNEクライアント識別子。

## アクション設定 - パラメータとオプション

次のセクションでは、各アクションでサポートされているパラメータをリストします。

### アクション - コンバージョンポストバックを送信

#### コンバージョンパラメータ

| **パラメータ** | **説明** |
| --- | --- |
| トランザクションID | (必須) TUNEトランザクションID |
| adv sub | 広告主サブID 1から5。このパラメータに最大5つの値の配列、または単一の文字列または数値属性を割り当てます。詳細については、[TUNE: Advertiser Sub IDs](https://support.tune.com/hc/en-us/articles/1500008231142-Advertiser-Sub-Ids)を参照してください。|
| adv unique | 広告主ユニークサブID 1から5。このパラメータに最大5つの値の配列、または単一の文字列または数値属性を割り当てます。 |
| aff ID | オファーをプロモーションするパートナーのID |
| プロモーションコード | プロモーションコード |
| 金額 | コンバージョン時の生成販売額 |
| コンバージョンユニークID | 有効期限のないユニークID |
| ゴールID | オファーのゴールのID |
| ゴールRef | オファーのゴールの参照ID。このパラメータは、ゴールIDが提供されていない場合にのみ、アトリビューションに影響します。 |
| オファーID | TUNEのオファーID |
| ペイアウト | パブリッシャーへの指定オファーのペイアウト |
| 収益 | 広告主からの指定オファーの収益 |
| セキュリティトークン | コンバージョン認証のための広告主のユニークなトークン文字列 |
| ステータス | コンバージョンステータス。許可される値は `approved`、`rejected`、または `pending`です。 |

#### モバイルパラメータ

| **パラメータ** | **説明** |
| --- | --- |
| Google aid | Google Playストアのアプリに対するインストールをクリックにアトリビュートするために使用されるGoogle Android広告識別子 |
| Google aid sha1 | Google Android広告IDのSHA1ハッシュ |
| iOS ifa | Apple iOS広告識別子（iOS 6以降） |
| iOS ifa sha1 | iOSデバイスIFA IDのSHA1ハッシュ |
| iOS ifv | Apple iOSベンダー識別子（iOS 6以降） |
| UNID | 未知のモバイルデバイス識別子のためのキャッチオール |
| ユーザーID | アプリ開発者によって生成されたアプリケーション固有のユーザーID |
| Windows aid | Windowsデバイス広告識別子 |
| Window aid sha1 | Windows広告IDのSHA1ハッシュ |
