## API情報

このコネクタは以下のベンダーAPIを使用します：

* API名：Awin API
* APIエンドポイント：`https://www.awin1.com`
* ドキュメント：[Awin: Awin API](https://developer.awin.com/apidocs/conversion-api)
* 認証タイプ：OAuth2.0
* タグドキュメント：[Awinタグ]()

## 動作原理

タグとコネクタを使用したハイブリッド構成を推奨します。この構成により、データの耐久性が最大化され、サーバーサイドデータとのイベントマッチングが改善され、リアルタイムのクライアントサイドデータを保持しながらデータフローの連続性が維持されます。コネクタは、トラフィックのソースを識別するために`awc`クエリ文字列パラメータを含む紹介リンクからのコンバージョンを帰属するためにAwinタグを使用します。Tealiumの拡張機能を使用するか、`awc`パラメータを保持してサーバーサイドコネクタから送信することで、購入が正しいコミッショングループにクレジットされることを保証します。

ブラウザの更新により、Awinはピクセルのみの実装をサポートしなくなったため、このコネクタでは[Awin S2S API](https://developer.awin.com/docs/direct-s2s)実装をフォールバックとして使用できます。

## 構成

コネクタマーケットプレイスにアクセスして新しいコネクタを追加します。コネクタの追加方法についての一般的な説明については、[コネクタについて]()を参照してください。

コネクタを追加した後、以下の構成を構成します：

* **トークン**  
必須。このトークンは個人のユーザーアカウントにリンクされています。トークンの作成についての詳細は、[AWIN: AW認証と承認](https://developer.awin.com/apidocs/api-authentication)を参照してください。
* **マーチャント（広告主ID）**  
適用されるAwin広告主ID。サポートについては、Awinアカウントマネージャーに連絡してください。

コネクタの構成が完了したら、**完了**をクリックします。

### 重複排除

複数のマーケティングチャネルやネットワークを使用している場合、同じコンバージョンに対して複数のトラフィックソースにコミッションが支払われないように重複排除を行う必要があります。詳細については、以下のチャネルパラメータと[Awin: 重複排除](https://advertiser-success.awin.com/s/article/What-is-De-Duplication)を参照してください。

### チャネルパラメータ

Awinは、すべてのクライアントに**チャネルパラメータ**の使用を強く推奨しています。このパラメータを使用することで、トラフィックのソースに基づいて各コンバージョンをどのように処理するかをシステムに指示します。

そのパラメータに勝利チャネルコードを入力することで、Tealiumは以下の4つの方法でリクエストを処理できます：

* 処理しない。
* 任意のAwinクッキーが存在する場合に処理する。
* Awinクリッククッキーが存在する場合のみ処理する。
* Awinポストビュークッキーが存在する場合のみ処理する。

### コンバージョン送信

サーバーからサーバーへのトラッキングは、トラッキングの必須要素です。要件は、コンバージョンデータがTealium Customer Data HubからAwinのサーバーへ直接HTTPSリクエストを介して送信されることです。

#### パラメータ

| **パラメータ**| **説明**|
|:---|:--- |
| 合計金額 | 必須。適用される割引後の注文の小計金額ですが、税金、配送料、または関連するサービス料などの追加料金は含まれません。値はfloatデータタイプでなければならず、千の区切りは許可されず、小数点は標準のドット(`.`)文字でなければなりません。 |
| 注文参照 | 必須。ユニークな予約または注文ID。|
| マーチャント（広告主ID） | 広告プログラムID。この値は構成画面からの値を上書きします。|
| コミッショングループコード | 推奨。取引を帰属させたいコミッショングループのコードまたはコード。このパラメータは単一グループの文字列または複数グループの配列であることができます。受け入れられる文字は大文字の英数字、アンダースコア、およびハイフンです。このパラメータが入力されていない場合、Awinはデフォルトのコミッショングループを使用します。 |
| コミッショングループ金額 | 各コミッショングループに割り当てたいコミッション金額。このパラメータは単一グループの文字列または複数グループの配列であることができます。配列値はコミッショングループ配列と同じ順序で表示されなければなりません。マッピングされていない場合、デフォルト値はマッピングされた合計金額です。 |
| チャネル | 最後のクリックリファラーのチャネル名。常に**aw**をAwin用に使用してください。これがデフォルト値です。この値は20文字以内の英数字でなければなりません。このパラメータを含めることを強くお勧めします。これにより、Awinシステムはトラフィックのソースに基づいて各コンバージョンをどのように処理するかを知ることができます。例としては、`aw`（Awin用に常に使用する必要があります）、`display`、または`ppc`があります。詳細については、[Awin: チャネルパラメータ](https://developer.awin.com/docs/channel-parameter)を参照してください。 |
| AWCパラメータ | この値は、クリックのソースを識別するためにAwinによってランディングページURLに追加されます。**AWCパラメータ**または**バウチャーコード**のいずれかの値を提供する必要があります。`awc`値は、クリックのソースを識別するためにAwinによってランディングページURLに追加されるパラメータです。エンドユーザーがサイトに到着したときに、サーバーサイドのスクリプト言語を使用して`awc`値を記録するようにサイトを構成する必要があります。この値は、コンバージョン時に`cks`パラメータを入力するために使用されます。この機能はすべてのページで呼び出す必要があります。awc値をクッキーでキャプチャしている場合、以下のガイドラインに従ってください：&lt;ul&gt;&lt;li&gt;クッキーはHTTPレスポンスヘッダーで構成され、クライアントサイドのスクリプトではなく構成される必要があります。たとえば、PHPで構成されたクッキーは問題ありません。&lt;/li&gt;&lt;li&gt;クッキーはHTTP Onlyフラグでクッキーを構成する必要があります。&lt;/li&gt;&lt;li&gt;クッキーはセキュアフラグで構成される必要があります。&lt;/li&gt;&lt;/ul&gt; |
| バウチャーコード | トランザクションのURLエンコードされたバウチャーコード。**AWCパラメータ**または**バウチャーコード**のいずれかの値を提供する必要があります。|
| 通貨コード| &lt;ul&gt;&lt;li&gt;推奨。トランザクションで使用された通貨のISO通貨コード、たとえば`GBP`または`USD`です。&lt;/li&gt;&lt;li&gt;正確なトラッキングを維持するために、このパラメータは非常に推奨されます。&lt;/li&gt;&lt;/ul&gt; |
| カスタムパラメータ | 単一または複数のカスタムトラッキングパラメータの入力データ。 |
| テストモード| 受信トラッキングリクエストはテスト目的で処理されません。  |

### 商品レベルのコンバージョン

商品レベルのトラッキングにより、広告主は個々の製品のパフォーマンスを測定できるより詳細なレポートを作成できます。
#### パラメータ

| パラメータ | 説明 |
|:---|:---|
| 合計金額 | 必須。適用される割引後の注文の小計金額ですが、税金、配送料、その他の関連サービス料金は含まれません。値は浮動小数点型で、千の区切りは許可されず、小数点は標準のドット(`.`)文字でなければなりません。 |
| 注文参照 | 必須。ユニークな予約または注文IDです。|
| マーチャント（広告主ID） | 広告プログラムIDです。この値は構成画面からの値を上書きします。 |
| コミッショングループコード | 推奨。取引に帰属させたいコミッショングループのコードまたはコードです。このパラメータは単一グループの場合は文字列、複数グループの場合は配列であることができます。許可される文字は大文字の英数字、アンダースコア、ハイフンです。このパラメータが入力されていない場合、Awinはデフォルトのコミッショングループを使用します。コミッショングループコードまたはコードはpartsパラメータにマッピングされ、製品レベルでもマッピングされます。 |
| コミッショングループ金額 | 各コミッショングループに割り当てたいコミッション金額です。このパラメータは単一グループの場合は文字列、複数グループの場合は配列であることができます。配列値はコミッショングループ配列と同じ順序で表示されなければなりません。マッピングされていない場合、デフォルト値はマッピングされた合計金額です。 |
| チャネル | 最後のクリックリファラーのチャネル名です。Awinの場合は常にデフォルト値`aw`を使用してください。この値は20文字以下の英数字でなければなりません。このパラメータを含めることを強くお勧めします。これにより、Awinシステムはトラフィックの出所に基づいて各変換をどのように処理するかを知ることができます。例としては、`aw`（Awin用に常に使用されるべきです）、`display`、または`ppc`があります。 |
| AWCパラメータ | Awinによってクリックの出所を識別するためにランディングページURLに追加される値です。**AWCパラメータ**または**バウチャーコード**のいずれかの値を提供する必要があります。awc値をクッキーでキャプチャする場合は、以下のガイドラインに従ってください：&lt;ul&gt;&lt;li&gt;クッキーはHTTPレスポンスヘッダーで構成され、クライアントサイドのスクリプトではなく、例えばPHPで構成されたクッキーは問題ありません。クッキーはHTTP Onlyフラグで構成する必要があります。&lt;/li&gt;&lt;li&gt;クッキーはセキュアフラグで構成されるべきです。&lt;/li&gt;&lt;/ul&gt; |
| バウチャーコード| トランザクションのURLエンコードされたバウチャーコードです。**AWCパラメータ**または**バウチャーコード**のいずれかの値を提供する必要があります。|
| 通貨コード | 推奨。トランザクションで使用された通貨のISO通貨コードです。例えば、`GBP`や`USD`です。|
| 商品名| 必須。トランザクション内の商品名の配列です。|
| カスタムパラメータ | 単一または複数のカスタムトラッキングパラメータの入力データです。 |
| テストモード| トラッキングリクエストはテスト目的で処理されません。 |
