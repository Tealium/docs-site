## 使用制限

Google Sheets コネクタはテスト目的のみであり、低容量の使用を想定しています。

このコネクタには、アカウント内のすべてのアクションに対して、1日あたりのリクエストの総数が100回という制限があります。この制限を超えると、スプレッドシートへの更新が失敗する可能性が高くなります。この制限は変更できません。

## 必要条件

* Google アカウント
* Google スプレッドシート - スプレッドシートはあなたの Google ドライブに存在し、最初の行に列名が必要です。

## サポートされるアクション

|アクション名| AudienceStream| EventStream|
|---| ---| ---|
|行の追加または更新| ✓| ✓|

## 構成

コネクタマーケットプレイスにアクセスし、新しいコネクタを追加します。コネクタの追加方法については、[コネクタ概要]() の記事を参照してください。

## アクション

**次へ** をクリックするか、**アクション** タブに進みます。ここでコネクタアクションを構成します。

このセクションでは、各アクションのパラメータとオプションの構成方法について説明します。

#### バッチ制限

このコネクタは、ベンダーへの大量データ転送をサポートするためにバッチリクエストを使用します。並列処理により、イベントがベンダーに順不同で到達する可能性があります。順序が重要な場合は、イベントにシーケンス値を追加してください。詳細については、[バッチアクション]() を参照してください。リクエストは、次のいずれかの閾値に達するか、プロファイルが公開されるまでキューに入れられます：

* 最大レコード数: 10,000
* 最古のリクエストからの最大時間: 60分
* リクエストの最大サイズ: 2 MB

### 行の追加または更新

#### パラメータ

|**パラメータ**| **説明**|
|---| ---|
|行操作|  &lt;ul&gt;&lt;li&gt;ドロップダウンリストから行操作のタイプを選択します:  &lt;ul&gt;&lt;li&gt;**追加のみ** - ルックアップなしで新しい行を追加します。&lt;/li&gt;&lt;li&gt;**更新のみ** - 既存の行を検索して更新します。&lt;/li&gt;&lt;li&gt;**追加または更新** - 既存の行を検索し、見つかった場合は更新します。それ以外の場合は新しい行を追加します。&lt;/li&gt;&lt;/ul&gt; &lt;/li&gt;&lt;/ul&gt; |
|スプレッドシート ID|  &lt;ul&gt;&lt;li&gt;既存のスプレッドシート ID を入力します。&lt;/li&gt;&lt;li&gt;これは、書き込みを行いたいスプレッドシートの ID です。&lt;/li&gt;&lt;/ul&gt; |
|ワークシート名|  &lt;ul&gt;&lt;li&gt;既存のワークシート名を選択します。&lt;/li&gt;&lt;li&gt;スプレッドシートには複数のワークシートが含まれている場合があります。&lt;/li&gt;&lt;li&gt;ターゲットとするワークシートのタイトルを入力します。&lt;/li&gt;&lt;li&gt;ワークシートには、最初のヘッダー行に列名が含まれている必要があります&lt;/li&gt;&lt;/ul&gt; |
|行データ|  &lt;ul&gt;&lt;li&gt;列名はワークシートの最初のヘッダー行に既に存在している必要があります。&lt;/li&gt;&lt;li&gt;空の行値はスキップされ、含まれません。&lt;/li&gt;&lt;li&gt;属性とスプレッドシートの列名を指定する必要があります。&lt;/li&gt;&lt;li&gt;行値を列名にマッピングするには: &lt;ul&gt;&lt;li&gt;**マップ** ドロップダウンリストから属性を選択し、**To** フィールドに列名（カスタム値）を入力します。アクションが発火すると、指定された列で属性値を検索します。&lt;/li&gt;&lt;li&gt;値が存在する場合、一致する行のセルは行データで指定された通りに更新されます。&lt;/li&gt;&lt;li&gt;値が存在しない場合、行 ID 値と行データで指定された属性を持つ新しい行が作成されます。&lt;/li&gt;&lt;/ul&gt; &lt;/li&gt;&lt;/ul&gt; |
|ルックアップ (オプション)|  &lt;ul&gt;&lt;li&gt;**更新のみ** および **追加または更新** の行操作に必要です。&lt;/li&gt;&lt;li&gt;既存の行を検索するためにヘッダーに値をマッピングします。&lt;/li&gt;&lt;li&gt;Google Sheets API は、値による行の検索をネイティブでサポートしていません。したがって、ルックアップを明示的に有効にする必要があります。これは、画面の左上にある **ルックアップを有効にする** ボタンを使用して行います。&lt;/li&gt;&lt;li&gt;ルックアップは、各個別のスプレッドシート ID を使用してスプレッドシートごとに一度有効にされます。&lt;/li&gt;&lt;/ul&gt; |

## FAQ

### ルックアップを有効にするために私のスプレッドシート ID をどのように見つけますか？

次の URL で、スプレッドシート ID は 12345 です: `https://docs.google.com/spreadsheets/d/**12345**/edit#gid=0`.

### ルックアップが有効になっている Google Sheets を表示するトラッキングは利用できません

ルックアップが有効になっている Google Sheets を表示するトラッキングは利用できません。一度ルックアップが有効になると、別の時間にアカウントにログインした場合、特定のシートに対してルックアップが有効になっているかを確認する方法はありません。

### アクション動的ドロップダウンリストはリアルタイムで表示されません

アクション動的ドロップダウンリストの更新はリアルタイムではなく、リアルタイムで表示されません。たとえば、Google Sheets に新しい列を追加している場合、シートの変更は自動的に継承されず、ドロップダウンリストに反映されません。

最新のデータを表示するには、コネクタページの **更新** アイコンをクリックするか、スプレッドシートの選択を解除して再度選択します。これらの方法のいずれかを使用すると、ドロップダウンリストが更新されて最新の更新が反映されます。
