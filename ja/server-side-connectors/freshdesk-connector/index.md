## 構成の構成

コネクタマーケットプレイスにアクセスし、新しいコネクタを追加します。コネクタの追加方法については、[コネクタ概要]()の記事を参照してください。

コネクタを追加した後、以下の構成を構成します：

* **アカウントドメイン**: アカウントのドメイン名を提供します。例えば、あなたのFreshdeskアカウントAPIエンドポイントが `https://exampledomain.freshdesk.com/api/v2/` の場合、`exampledomain` を入力します。
([Freshdesk API](https://developers.freshdesk.com/api/#getting-started))
* **ユーザー名**: 認証には、Freshdeskにログインする際に使用するユーザー名とパスワードを使用します。
* **パスワード**: 認証には、Freshdeskにログインする際に使用するユーザー名とパスワードを使用します。

## アクション構成 - パラメータとオプション

**次へ**をクリックするか、**アクション**タブに進みます。ここでコネクタアクションを構成します。

このセクションでは、各アクションのパラメータとオプションの構成方法について説明します。

### アクション - コンタクトの作成

#### パラメータ

| **パラメータ**      | **説明**                                                                                                                                                                                |
|:-------------------|:-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 名前               | コンタクトの名前。このエンドポイントの詳細については、[Freshdeskのドキュメント](https://developers.freshdesk.com/api/#create_contact)を参照してください。                                          |
| メール              | コンタクトの主要なメールアドレス。このコンタクトに追加のメールを関連付けるには、`other_emails` 属性を使用します。                                                                      |
| 電話番号              | コンタクトの電話番号。                                                                                                                                                               |
| 携帯番号             | コンタクトの携帯番号。                                                                                                                                                                  |
| Twitter ID         | コンタクトのXハンドル。                                                                                                                                                                 |
| ユニークな外部ID | コンタクトの外部ID。                                                                                                                                                                    |
| その他のメール       | コンタクトに関連付けられた追加のメール。文字列または文字列のセット属性。                                                                                                             |
| 会社ID         | このコンタクトが属する主要な会社のID。                                                                                                                                       |
| すべてのチケットを見る   | `True` または `false`。コンタクトが所属する会社に関連するすべてのチケットを見ることができるようにする場合は `true` に構成します。                                                             |
| 住所            | コンタクトの住所。                                                                                                                                                                        |
| カスタムフィールド      | カスタムフィールドの名前と値を含むキーバリューペア。カスタム日付フィールドの入力として受け入れられるのは `YYYY-MM-DD` 形式の日付のみです。                                         |
| 説明        | コンタクトの簡単な説明。                                                                                                                                                            |
| 職種          | コンタクトの職種。                                                                                                                                                                      |
| 言語           | コンタクトの言語。デフォルトの言語は `en` です。この構成は、複数言語機能が有効になっている場合（Gardenプラン以上）にのみ構成できます。                                                   |
| タグ               | このコンタクトに関連付けられたタグ。`set of strings` または `array of strings` 属性タイプでなければなりません。ここに1つのアイテムのみが必要な場合、それをこれらの属性タイプのいずれかにフォーマットする必要があります。 |
| タイムゾーン          | コンタクトのタイムゾーン。デフォルト値はドメインのタイムゾーンです。この構成は、複数タイムゾーン機能が有効になっている場合（Gardenプラン以上）にのみ構成できます。                             |

### アクション - コンタクトの完全削除

#### パラメータ

| **パラメータ** | **説明**                                                                                                                                                   |
|:--------------|:------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Contact ID    | 完全に削除したいコンタクトのIDです。このアクションは、コンタクトがすでにソフト削除されているか、`force` パラメータが `true` に構成されている場合にのみ実行できます。 |
| Force         | `True` または `false`。ソフト削除されていないコンタクトを強制的に完全削除する場合は `true` に構成します。                                                              |
