## 構成

コネクタマーケットプレイスに移動し、新しいコネクタを追加します。コネクタを追加する一般的な手順については、[コネクタについて]()を参照してください。

コネクタを追加した後、以下の構成を構成します：

* **ホスト**  
 必須。環境に関連付けられたホストを選択します。
セッションAIサンドボックス環境を指す場合は `https://csb.zineone.com` を使用します。本番環境を指す場合は `https://cloud.zineone.com` を使用します。
* **APIキー**  
APIキーを生成するには、**Session AI &gt; Integrations &gt; Data Channel** でチャネルを作成します。

## アクション

次のセクションでは、各アクションでサポートされるパラメータをリストします。

### CCPA同意の更新

#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| オプトインアクション | 可能な値：&lt;ul&gt;&lt;li&gt;`true` - オプトインリクエストを選択します。&lt;/li&gt;&lt;li&gt;`false` - オプトアウトリクエストを選択します。&lt;/li&gt;&lt;/ul&gt; |

#### ユーザー識別子

| **パラメータ** | **説明** |
| --- | --- |
| `id` | 顧客プロファイルの一意のIDです。 |
| `email` | 顧客の一意のメールIDです。 |
| `deviceId` | 顧客の一意のデバイスIDです。 |

### イベントの送信

#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| データチャネル | イベントを送信するチャネル。 |
| イベント名 | イベント名を選択するか、カスタム名を入力します。 |

#### 共通属性

| **パラメータ** | **説明** |
| --- | --- |
| `session_id `| (必須) セッションまたは訪問を一意に識別するIDです。 |
| `visitor_id `| (必須) サインインしていない場合のユーザーを識別するIDです。 |
| `customer_id` | サインインしている場合のユーザーを識別するIDです。 |
| `user_agent` | 利用可能な場合は、ブラウザのユーザーエージェント文字列です。 |
| `device_channel` | (必須) デバイスのタイプです。例：デスクトップ、タブレット、モバイル、アプリなど。 |
| `referrer_url` | (必須) 前のページのURLを参照するURLです。例えば、ユーザーがサードパーティのウェブサイトからあなたのサイトを初めて訪れた場合、これはサードパーティのURLになります。ユーザーがあなたのサイトのセッションの途中にいる場合、前のページのURLが現在のページのリファラーURLになります。 |
| `campaign_info` | キャンペーン名または構造です。例えば、`utm_campaign` の値です。 |
| `os` | デバイスのオペレーティングシステムです。 |
| `os_version` | オペレーティングシステムのバージョンです。 |
| `platform_os` | オペレーティングシステムのプラットフォームです。 |
| `language` | デバイスの言語です。 |
| `tz` | デバイスのタイムゾーンです。 |
| `devicetype` | デバイスのタイプです。 |
| `browser` | デバイスで使用されるブラウザです。 |
| `browserversion` | デバイスで使用されるブラウザのバージョンです。 |
| `resolution` | デバイスのディスプレイの解像度です。 |
