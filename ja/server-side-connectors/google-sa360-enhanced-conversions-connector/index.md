Google SA360 エンハンスドコンバージョンコネクタは、コンバージョン測定の精度を向上させ、より強力な入札を可能にします。既存のクライアントサイドのコンバージョンタグを補完し、プライバシーを保護した方法でウェブサイトからGoogleへハッシュ化された第一者データを送信します。

コンバージョンはクライアントサイドの[Floodlight (`gtag.js`) タグ]()を通じて記録され、Googleによってインデックスされます。Google SA360 エンハンスドコンバージョンコネクタは、サーバーからサーバーへのAPIコールを使用して、個人を特定できる情報（PII）を使用してこれらのコンバージョンをエンリッチメントするために使用できます。

## 要件

このコネクタには以下が必要です：

* コネクタ構成で使用するプロファイルIDを取得するために、`api@tealium.com` ユーザーをGoogleキャンペーンマネージャー360アカウントに追加してください。このユーザーはオフラインコンバージョンの更新権限を持っている必要があります。
* このコネクタがPIIデータをタグペイロードに更新するためには、[Floodlight `gtag.js` タグ]()が必要です。
* Floodlight `gtag.js` タグを最新バージョンに更新してください。
* `tealium_random` のような一意の値を含む属性をタグの `match_id` 変数にマッピングしてください。Googleがコンバージョンをエンリッチメントするために同じ値をコネクタで使用してください。
* 非購入コンバージョンの場合、Floodlight `gtag.js` タグで `ordinal` に一意の値を含む属性をマッピングしてください。例えば、`tealium_random`。

### 権限

* 任意のソース（バルクシート、Search Ads 360 API、またはCampaign Manager 360 APIなど）からコンバージョンをアップロードするには、コンバージョンをアップロードするサブマネージャーアカウントへの編集アクセスが必要です。詳細については、[Google: ユーザー権限についての役割の割り当て](https://support.google.com/searchads/answer/1409946)を参照してください。
* 共有Floodlight構成では、アップロードする各子サブマネージャーアカウントへの編集アクセスが必要です。親サブマネージャーやアップロードしていない子サブマネージャーへのアクセスは必要ありません。詳細については、[Google: Floodlight命令の編集](https://support.google.com/sa360/answer/13400844)を参照してください。
* コンバージョンを追跡するためにクリックIDを使用する場合、そのクリックIDを生成したサブマネージャーへの編集アクセスが必要です。編集できないサブマネージャーからのクリックIDを使用する場合、アップロードは失敗します。詳細については、[Google: ダウンロードしたバルクシートの編集](https://support.google.com/sa360/answer/13607450)を参照してください。

## API情報

このコネクタは次のベンダーAPIを使用します：

* API名：Google Campaign Manager 360 API
* APIバージョン：v5
* APIエンドポイント：`https://dfareporting.googleapis.com/dfareporting`
* ドキュメント：[Google: Campaign Manager 360 API - エンハンスドコンバージョン](https://developers.google.com/doubleclick-advertisers/guides/conversions_ec)

## バッチ制限

このコネクタは、ベンダーへの大量データ転送をサポートするためにバッチリクエストを使用します。並行処理により、イベントがベンダーに順不同で到達する可能性があります。順序が重要な場合は、イベントにシーケンス値を追加してください。詳細については、[バッチ処理アクション]()を参照してください。次のいずれかの閾値が満たされるか、プロファイルが公開されるまでリクエストはキューに入れられます：

* 最大リクエスト数：1000
* 最古のリクエストからの最大時間：60分
* リクエストの最大サイズ：1 MB

さらに、コネクタはクライアントサイドのコンバージョンがGoogleにインデックスされるのを待ってから2時間の遅延を持ってバッチコンバージョンを送信します。

## トレース

Tealiumの[Trace]()機能は、リアルタイムデバッグを可能にするために構成されたバッチ処理と遅延をバイパスします。クライアントサイドのコンバージョンのGoogleインデックスは即時ではないため、このコネクタを使用してTraceを使用すると`404 Not Found`エラーが発生します。

## PIIマッピングの制限

Floodlight (`gtag.js`) タグをこのコネクタで使用する際は、以下のPIIデータの制限を守ってください：

* コネクタが機能するためにはPIIデータを送信する必要があります。コネクタがGoogleに接続する際にPIIが含まれていない場合、コネクタはPIIが欠落しているエラーを生成し、Googleにデータを送信しません。PIIデータには少なくとも以下のいずれかが含まれている必要があります：
    * メールアドレス
    * 電話番号
    * すべての住所情報（名、姓、住所、市町村、州、郵便番号、国コード）
* PIIデータをFloodlight (`gtag.js`) タグまたはGoogle SA360 エンハンスドコンバージョンコネクタのいずれか一方を通じて送信してくださいが、両方ではないでください。両方にPIIをマッピングすると、コネクタで`conversion already enhanced`エラーが発生します。
* コネクタエラーを避けるために、PIIが必要なイベントフィードやイベント仕様に条件を追加することをお勧めします。

## 構成

コネクタマーケットプレースにアクセスして新しいコネクタを追加してください。コネクタを追加する一般的な手順については、[コネクタについて]()の記事を参照してください。

 このコネクタを追加する際には、ベンダーのデータプラットフォームポリシーを承諾する必要があります。 

コネクタを追加した後、次の構成を構成してください：

* **FloodlightプロファイルID**：Campaign Manager 360に`api@tealium.com`ユーザーを追加した後に提供されるFloodlightプロファイルIDです。

コネクタの構成が完了したら、**完了**をクリックしてください。

| オーディナル | (必須) 変換の序数です。同じユーザーと日に対して同じ値（例：`1`）を各イベントに渡すことで、変換の重複を排除するために使用できます。重複排除を無効にするには、購入イベントの場合は`order_id`、非購入イベントの場合は`tealium_random`などの一意の値を使用してください。このコネクターとFloodlightの`gtag.js`タグの両方で同じ属性値を使用してください。タグでトランザクションIDが使用されている場合は、その値を含む属性をマッピングしてください。 |

| 数量 | (必須) 変換の数量です。配列をマッピングする場合、コネクターは配列内の値を合計します。このコネクターとFloodlightの`gtag.js`タグの両方で同じ属性値を使用してください。値を更新する場合は除きます。 |
| 値 | (必須) 変換の値です。このコネクターとFloodlightの`gtag.js`タグの両方で同じ属性値を使用してください。値を更新する場合は除きます。 |
| プロファイルID | コネクターの構成タブで定義された値を上書きします。 |

### エラーメッセージ

このコネクターの潜在的なエラーメッセージは以下の通りです：

|エラーメッセージ| 説明|
|---| ---|
| `NOT_FOUND` | &#34;MatchId `{matchId}`が見つかりません。&#34; Googleはクライアント側のタグピングから一致する変換を見つけることができませんでした。これはコネクターの問題ではなく、iQタグ管理のタグ構成の調査が必要かもしれません。これはまた、[Google Consent Mode tag]()の使用によるもので、タグに対する同意が拒否されたが、コネクターを通じて変換エンリッチメントが送信された結果でもあります。 |
| `NOT_FOUND` | &#34;指定された変換は存在しません。&#34; 変換はクライアント側のタグから受信されましたが、Googleによってエンリッチメントのためにまだインデックスされていません。Tealiumはこれらの呼び出しを最大3回まで再試行します。|
