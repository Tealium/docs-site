## API情報

このコネクタは以下のベンダーAPIを使用します：

* API名：Singular API
* APIバージョン：v1
* APIエンドポイント：`https://s2s.singular.net/api/v1`
* ドキュメント：[Singular API](https://support.singular.net/hc/en-us/articles/31394799175963-Server-to-Server-SESSION-Endpoint-API-Reference#Session_API_Endpoint)

## 構成

コネクタマーケットプレイスにアクセスし、新しいコネクタを追加します。コネクタの追加方法については、[コネクタについて](https://docs.tealium.com/about-connectors/)を参照してください。

コネクタを追加した後、以下の構成を行います：

* **SDK APIキー**
    * （必須）SDKキーはAPIリクエストの`a`パラメータとして渡されます。
    * Singular UIのメインメニューにある[開発者ツール](https://app.singular.net/?#/react/sdk_integration)からSDKキーを取得します。
    * レポーティングAPIキーを使用するとデータが拒否されるため、使用しないでください。

## アクション

| アクション名 | AudienceStream | EventStream |
| --- | :---: | :---: |
| セッションデータ送信 | ✗ | ✓ |
| イベントデータ送信 | ✗ | ✓ |

アクション名を入力し、**アクションタイプ**を選択します。

次のセクションでは、各アクションのパラメータとオプションの構成方法について説明します。

### セッションデータ送信

#### パラメータ

| パラメータ | 説明 |
| --- | --- |
| プラットフォーム | アプリのプラットフォーム。利用可能な値は：`Android`, `iOS`, `PC`, `Xbox`, `Playstation`, `Nintendo`, `MetaQuest`, `CTV`です。 |

#### セッションデータパラメータ

すべてのパラメータはURLデコードされます。

| パラメータ | 説明 |
| --- | --- |
| IPv4アドレス | （必須）デバイスの公開IPv4アドレス。 |
| OSバージョン | （必須）セッション時のデバイスのOSバージョン。 |
| アプリ識別子 | （必須）アプリ識別子。 |
| アプリケーションバージョン | （必須）アプリケーションのバージョン。 |
| インストール | （必須）このセッションがインストールまたは再インストール後の最初のセッションであるかどうかを示すパラメータ。 |
| クライアント定義識別子 | 他のすべての識別子が利用不可能な特殊なケースでのクライアント定義の識別子。 |
| タイムスタンプ | セッションのUNIX時間（秒）。 |
| タイムスタンプ（ミリ秒） | 13桁のUNIX時間（ミリ秒）。 |
| グローバルプロパティ | 最大5つのキーと値のペアを含むJSONオブジェクト。 |
| IP使用 | このパラメータは、HTTPリクエストからIPv4アドレスパラメータではなくIPアドレスを抽出するようSingularに指示します。 |
| 国 | セッション時のユーザーのISO 3166-1 alpha-2二文字国コード。 |
| ユーザーエージェント | デバイスのユーザーエージェント。 |
| データ共有オプション | 情報共有に関するエンドユーザーの同意。 |
| 内部ユーザーID | 内部ユーザーID。 |

#### プラットフォーム固有のデータパラメータ

ベンダーでプラットフォーム固有のパラメータにイベント属性をマッピングします。
