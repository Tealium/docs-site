AWS Kinesis Streams (Tealiumが提供する認証情報) コネクタ構成ガイド

この記事では、AWS Kinesis Streams (Tealiumが提供する認証情報) コネクタの構成方法について説明します。

コネクタのアクション

アクション名 AudienceStream EventStream
Kinesis Streamにイベントデータを送信 ✗ ✓
Kinesis Streamにビジターデータを送信 ✓ ✗
Kinesis Streamにカスタマイズデータを送信 (高度) ✓ ✓
Kinesis Streamにバッチ処理されたイベントデータを送信 ✗ ✓
Kinesis Streamにバッチ処理されたビジターデータを送信 ✓ ✗
Kinesis Streamにバッチ処理されたカスタマイズデータを送信 (高度) ✓ ✓

AWSアカウントにTealiumロールを追加

コネクタを構成する前に、TealiumアカウントIDのロールを作成し、信頼されたエンティティとして構成する必要があります：

  1. Identity and Access Management (IAM) コンソールで、Details タブのKinesisストリームのAmazon Resource Name (ARN)を探します。
  2. Policies > Create Policy に移動します。これが最初に作成するポリシーの場合は、Get Started と Create Policy をクリックします。
  3. Kinesis StreamのARNを入力します。
  4. {} JSON をクリックします。
  5. 次のポリシーを入力し、arn:aws:kinesis:us-east-1:123456789:stream/example をステップ1のARNに置き換えます。
{
  "Version": "2023-06-01",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "kinesis:PutRecord",
        "kinesis:PutRecords"
                ],
      "Resource": "arn:aws:kinesis:us-east-1:123456789:stream/example"
    }
  ]
}
  1. Next Step をクリックします。
  2. Policy Name の下に、TealiumKinesis を入力します。
  3. Create Policy をクリックします。
  4. Roles > Create Role に移動します。
  5. Trusted entity type の下で、AWS Account を選択します。
  6. Another AWS account を選択します。
  7. Account ID の下に、757913464184 を入力します。
  8. Next: Permissions をクリックします。
  9. TealiumKinesis ポリシーを選択します。
  10. Next: Review をクリックします。
  11. 名前には、TealiumUser を入力します。
  12. Create Role をクリックします。

Kinesisは、次のようなロールのARNを生成します：

arn:aws:kinesis:us-east-1:123456789:role/example

詳細については、Controlling Access to Amazon Kinesis Data Streams Resources Using IAM および Create an IAM Policy and User を参照してください。

構成の構成

コネクタマーケットプレイスに移動し、新しいコネクタを追加します。コネクタの追加方法の一般的な指示については、About Connectorsを参照してください。

コネクタを追加した後、次の構成を構成します：

  • Region
    必須：リージョンを選択します。
  • Assume Role: ARN
    オプション：引き受けるロールのAmazon Resource Name (ARN)を提供します。例：arn:aws:iam::222222222222:role/myrole 参照：Switching to an IAM Role。
  • Assume Role: Session Name
    必須：引き受けるロールセッションの識別子を提供します。長さ制約：最小長さ2、最大長さ64。
  • Assume Role: External ID
    オプション：外部識別子を提供します。参照：How to Use an External ID。

コネクタの構成が完了したら、Test Connection をクリックして、コネクタがAmazonに接続できることを確認します。

アクション

アクションの名前を入力し、ドロップダウンメニューからアクションタイプを選択します。

次のセクションでは、各アクションのパラメータとオプションの構成方法について説明します。

  • 最古のリクエストからの最大時間：10分
