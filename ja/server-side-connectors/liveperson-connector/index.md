この記事では、LivePersonコネクタの構成方法について説明します。

## API情報

このコネクタは以下のベンダーAPIを使用します：

* API名：LivePerson Proactive Messaging API
* APIバージョン：v1.0
* APIエンドポイント：`https://proactive-messaging.z1.fs.liveperson.com`
* ドキュメンテーション：[LivePerson API](https://developers.liveperson.com/proactive-messaging-api.html)


## 構成

コネクタマーケットプレイスに移動し、新しいコネクタを追加します。コネクタの追加方法の一般的な指示については、[About Connectors](https://docs.tealium.com/about-connectors/)を参照してください。


コネクタを追加した後、以下の構成を行います：
* **アカウントID**：LivePersonサイトID
* **クライアントID**：クライアントIDを提供します
* **クライアントシークレット**：クライアントシークレットを提供します
* **APIドメイン**：認証用のAPIドメイン（ホスト）

## アクション

| アクション名 | AudienceStream | EventStream |
| --- | :---: | :---: |
| アウトバウンドメッセージを送信 | ✓ | ✓ |
| バッチ処理でアウトバウンドメッセージを送信 | ✓ | ✓ |

アクションの名前を入力し、ドロップダウンメニューからアクションタイプを選択します。

次のセクションでは、各アクションのパラメータとオプションの構成方法について説明します。

### アウトバウンドメッセージを送信

#### メッセージパラメータ
必要なパラメータ：

| **パラメータ** | **説明** |
| --- | --- |
| キャンペーン名 | キャンペーンの名前。 |
| スキル | キャンペーンのスキル`ID`。 |
| テンプレートID | キャンペーンのテンプレート`ID`またはハンドオフ`ID`。 |
| 同意 | このキャンペーンを有効にすることで、企業は消費者にメッセージを送信するすべての適用可能な法律と規制を遵守する責任があることを認め、同意します。 |
| アウトバウンド番号 | 消費者にメッセージを送信するための番号。 |

#### 消費者コンテンツ


| **パラメータ** | **説明** |

| 消費者コンテンツApple | 消費者のApple`ID`。 |
| 消費者コンテンツin-app | 消費者のin-app識別子。 |

### バッチ処理でアウトバウンドメッセージを送信

#### バッチ制限

このアクションは、ベンダーへの大量のデータ転送をサポートするために、バッチリクエストを使用します。リクエストは、以下のいずれかの閾値が満たされるまでキューに入れられます：

* リクエストの最大数：100
* 最古のリクエストからの最大時間：60分
* リクエストの最大サイズ：1MB
