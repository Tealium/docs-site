## API情報

このコネクタは以下のベンダーAPIを使用します：

* API名：X変換API
* APIバージョン：v12
* APIエンドポイント：`https://ads-api.x.com/`
* レート制限：アカウントあたり15分間に60,000イベント
* ドキュメント：[X：Web変換API](https://developer.x.com/en/docs/x-ads-api/measurement/web-conversions/api-reference/conversions)

## 前提条件

* [X広告API](https://developer.x.com/en/docs/x-ads-api/getting-started)アクセス
* [X開発者](http://developer.x.com)アカウント
* 広告アカウントにイベントを送信する権限を持つXユーザーアカウント

既に活用しているX広告APIアプリケーションがある場合、そのアプリケーションと既存のアクセストークンを変換APIに使用できます。詳細については、[X：変換API構成](https://developer.x.com/en/docs/x-ads-api/measurement/web-conversions/conversion-api)を参照してください。

## 仕組み

X変換APIを使用して、別のサーバーサイドアプリケーションを作成し、Xでの最適化、測定、およびキャンペーン全体の帰属に使用する変換イベントをXに渡します。このAPIを使用すると、Xウェブサイトピクセルを必要とせずに、広告の実行がXに与える影響を測定するために変換イベントを送信できます。

現在、広告の影響を測定するための2つの主要なユースケースがあります：

* XクリックIDを使用したウェブサイトトラフィックキャンペーンのクリックスルー変換
* すべてのXキャンペーンのビュースルーおよびクリックスルー変換にメールを使用

広告キャンペーンに応じて、次のウェブサイトアクションの1つ以上を変換トラッキングで測定できます：

* **サイト訪問**：ユーザーが広告主のサイトのランディングページを訪問
* **購入**：ユーザーが広告主のサイトで商品またはサービスの購入を完了
* **ダウンロード**：ユーザーが広告主のサイトからファイル（ホワイトペーパーやソフトウェアパッケージなど）をダウンロード
* **サインアップ**：ユーザーが広告主のサービス、ニュースレター、またはメール通信にサインアップ
* **カスタム**：上記のカテゴリに該当しないカスタムアクション

### X変換APIの使用

X変換APIを使用するには、X広告マネージャーで新しい変換イベントを作成するか、既に作成されているXピクセルで使用されているイベントを使用する必要があります。XピクセルとX変換APIリクエストの間で変換を重複排除するには、ピクセル用に作成した既存のイベントを使用します。

#### 広告マネージャーで既存の変換イベントを使用する

Xピクセルで使用されている既存のイベントを使用するには、そのイベントのイベントIDを使用します。同じイベントにXピクセルとX変換APIの両方を使用する場合は、XピクセルのコードスニペットとX変換APIリクエストの両方で重複排除キー（`conversion_id`として）を使用することを確認してください。重複排除についての詳細は、Xドキュメントの[変換API構成](https://developer.x.com/en/docs/x-ads-api/measurement/web-conversions/conversion-api)のテストイベントと重複排除セクションを参照してください。

#### 変換イベントの識別子

X変換APIで変換イベントを渡すためには、少なくとも1つのXクリックID（`twclid`）またはメールアドレスがイベント識別子として構成されている必要があります。

X変換APIに変換イベントを正常に構成するためには、次の構成を推奨します：

* URLクエリパラメータに`twclid`値が存在する場合は常に解析してTealium EventStream API Hubに渡すこと。
* 関連するフォームフィールドまたは変換イベント情報とともにデータを保存すること。

#### 重複排除

ページビューイベントには、Xが定義する**サイト訪問**および**ランディングページビュー**イベントを含む30分間の重複排除ウィンドウがあります。他のイベントタイプ、たとえば**購入**や**リード**は重複排除されません。

`conversion_id`イベントパラメータを使用してイベントの重複を管理します。このパラメータはピクセルまたはX変換APIリクエストに含めることができます。

## 構成

X変換はあなたのXアカウントでのホワイトリストが必要です。ホワイトリストのサポートについては、あなたのX担当者に連絡してください。

コネクタマーケットプレースにアクセスして新しいコネクタを追加します。コネクタを追加する一般的な手順については、[コネクタについて](https://docs.tealium.com/about-connectors/)を参照してください。

コネクタを追加した後、次の構成を構成します：

* **イベントID**  
特定のイベントのbase36 IDです。この広告アカウント内に事前に構成されたイベントに一致します。これはイベントマネージャーの対応するイベントで**ID**と呼ばれます。
この値は次の方法で作成できます：
  * シングルイベントウェブサイトタグ構成内で。[https://ads.x.com/](https://ads.x.com/)にログインし、**ツール**の下にある**イベントマネージャー**を選択します。
  * APIを使用します。詳細については、[X：ウェブイベントタグ](https://developer.x.com/en/docs/x-ads-api/measurement/api-reference/web-event-tags)を参照してください。
  * あなたのXアカウントマネージャーに連絡してください。

* **ピクセルID**  
広告アカウントのユニバーサルウェブサイトタグ（UWT）IDです。これは広告アカウントのUWT IDのbase36エンコード値を表します。
  ピクセルIDを取得するには：
  1. [https://ads.x.com/](https://ads.x.com)にログインします。
  1. インターフェースの左上隅で、**ツール**の下にある**イベントマネージャー**をクリックします。
  1. まだ左サイドバーにXピクセルイベントソースがない場合、ページの右上隅で**イベントソースを追加**を選択します。
  1. インターフェースはこのイベントのピクセルIDとイベントIDを表示します。

## コネクタアクション

| アクション名 | AudienceStream | EventStream |
| --- | :---: | :---: |
|変換イベントを送信| ✓| ✓|

**続行**をクリックしてコネクタアクションを構成します。アクションの名前を入力し、ドロップダウンメニューからアクションタイプを選択します。

次のセクションでは、各アクションのパラメータとオプションの構成方法について説明します。

### 変換イベントを送信

#### バッチ制限

このコネクタは、ベンダーへの高容量データ転送をサポートするためにバッチリクエストを使用します。詳細については、[バッチアクション](https://docs.tealium.com/batched-actions/)を参照してください。リクエストは、次のいずれかの閾値が満たされるか、プロファイルが公開されるまでキューに入れられます：

* 最大リクエスト数：500
* 最古のリクエストからの最大時間：10分
* リクエストの最大サイズ：1 MB

#### ユーザーデータ

少なくとも次のいずれかのパラメータを提供する必要があります：

| **パラメータ** | **タイプ** |**説明** |
| --- | --- | -- |
| XクリックID | 文字列 | クリックスルーURLから解析されたXクリックID。XクリックIDまたはメールのいずれかを指定する必要があります。両方を使用すると一致率が向上します。 |
| メールアドレス（既にSHA256ハッシュ化されている） | 文字列 | すでに空白トリム、小文字化、SHA256ハッシュ化されているメールアドレスを提供します。 |
| メールアドレス（SHA256ハッシュを適用） | 文字列 | プレーンテキストのメールアドレスを提供し、コネクタがこの値を空白トリム、小文字化し、SHA256ハッシュを使用してハッシュ化します。 |
| 電話番号（既にSHA256ハッシュ化されている） | 文字列 | すでに空白トリム、小文字化、SHA256ハッシュ化されており、E164形式の電話番号を提供します。 |
| 電話番号（SHA256ハッシュを適用） | 文字列 | プレーンテキストの電話番号を提供し、コネクタがこの値を空白トリム、小文字化し、SHA256ハッシュを使用してハッシュ化します。 |
| `ip_address` | 文字列 | この値はドットで区切られた10進数表記で、4つの数字がピリオドで区切られています。 |
| `user_agent` | 文字列 | この識別子を使用すると、サーバーはリクエストを行うユーザーエージェントのアプリケーション、オペレーティングシステム、ベンダー、および/またはバージョンを識別できます。 |


#### 追加パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| イベントIDオーバーライド | 広告マネージャーおよび広告APIでは**ウェブサイトタグID**と呼ばれ、X広告プラットフォームの**イベントマネージャー**で見つけることができます。このパラメータは必須フィールドであり、構成セクションで使用される**イベントソースID**を上書きします。 |
| ピクセルIDオーバーライド | 広告アカウントのユニバーサルウェブサイトタグ（UWT）ID。このパラメータは、広告アカウントのUWT IDのbase36エンコードされた値を表します。例：`o5h34`。この構成は、構成セクションで使用される**ピクセルID**を上書きします。 |
| コンバージョン時間 | （必須）コンバージョンイベントの時間。イベントの日時を[ISO 8601形式](https://en.wikipedia.org/wiki/ISO_8601)で指定します。提供されていない場合、値はイベントの時間です。例：`2020-11-01T08:00:00.000Z`。 |
| コンバージョン値 | 購入されるアイテムの価格値で、`price_currency`通貨で表されます。 |
| アイテム数 | 購入されるアイテムの数。ゼロより大きい正の数でなければなりません。**アイテム数**が提供されていない場合、**コンテンツ**配列の要素の総数が使用されます。 |
| 通貨 | 結果をフィルタリングするための通貨の種類で、[ISO 4217形式](https://en.wikipedia.org/wiki/ISO_4217)を使用して識別されます。これは、`USD`のような3文字の文字列です。 |
| コンバージョンID | ピクセルとコンバージョンAPIのコンバージョン間の重複排除のため。同じイベントタグ内のWebピクセルとコンバージョンAPIのコンバージョン間で重複排除に使用できるコンバージョンイベントの識別子。詳細については、[X: コンバージョンAPIセットアップ](https://developer.x.com/en/docs/x-ads-api/measurement/web-conversions/conversion-api)を参照してください。 |
| 説明 | コンバージョンに関する追加情報の説明。 |
| 検索文字列 | 広告主のウェブサイトで検索されたテキスト。 |

#### コンテンツ属性

オプションのパラメータ:

| **パラメータ** | **説明** |
| --- | --- |
| コンテンツID | コンテンツを表すSKUまたはGTIN（グローバルトレードアイテムナンバー）識別子。 |
| コンテンツグループID | 製品バリアントのグループに関連付けられたID。 |
| コンテンツ名 | 製品またはサービスの名前。 |
| コンテンツ価格 | 製品またはサービスの価格。 |
| コンテンツタイプ | 購入された製品のカテゴリ。 |
| アイテム数 | 購入された製品の数。 |
