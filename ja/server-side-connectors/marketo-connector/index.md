## 構成

コネクタマーケットプレイスにアクセスし、新しいコネクタを追加します。コネクタを追加する一般的な手順については、[コネクタについて]()を参照してください。

コネクタを追加した後、以下の構成を構成します：

* **REST APIクライアントID**：（必須）**Marketo &gt; 管理 &gt; LaunchPoint**で見つかるクライアントIDを提供してください。APIユーザーは**読み取り-書き込みリード**、**キャンペーンの読み取り/実行**、および**リストの読み取り**の権限を持っている必要があります。詳細については、[Marketo: ReST APIで使用するカスタムサービスを作成する](http://docs.marketo.com/display/public/DOCS/Create&#43;a&#43;Custom&#43;Service&#43;for&#43;Use&#43;with&#43;ReST&#43;API)を参照してください。
* **REST APIクライアントシークレット**：（必須）**Marketo &gt; 管理 &gt; LaunchPoint**で見つかるクライアントシークレットを提供してください。
* **REST APIエンドポイントURL**：（必須）**Marketo &gt; 管理 &gt; Webサービス**で見つかるREST APIエンドポイント。例：`https://123-ABC-123.mktorest.com/rest`。
* **REST APIアイデンティティURL**：（必須）**Marketo &gt; 管理 &gt; Webサービス**で見つかるREST APIアイデンティティ。例：`https://123-ABC-123.mktorest.com/identity`。

次のセクションでは、各アクションのパラメータとオプションの構成方法について説明します。

### リストにリードを追加

#### パラメータ

|**パラメータ**| **説明**|
|---| ---|
| ターゲット静的リスト | （必須）リストを選択します。プラットフォームは最大300のリストをロードします。リストが表示されない場合は、リストIDを入力してください。 |
| ターゲットリードID | ユーザーの**リードID**がわかっている場合に使用します。**リードID検索**は空白のままにします。 |
| リードID検索 | ユーザーの**リードID**が不明な場合に使用します。**ターゲットリードID**を空白のままにし、このフィールドを使用してさまざまなフィルターでリードIDを検索します。可能な値：`id`, `cookies`, `email`, `twitterId`, `facebookId`, `linkedInId`, `sfdcAccountId`, `sfdcContactId`, `sfdcLeadId`, `sfdcLeadOwnerId`, `sfdcOpptyId`。 |
| リードを追加 | リードが存在しない場合は追加します。 |

### リストにリードを追加（バッチ処理）

#### バッチ制限

このアクションは、バッチ処理されたリクエストを使用してベンダーへの大量データ転送をサポートします。詳細については、[バッチ処理されたアクション]()を参照してください。リクエストは、次のいずれかのしきい値が満たされるか、プロファイルが公開されるまでキューに入れられます：

* 最大リクエスト数：300
* 最古のリクエストからの最大時間：30分
* リクエストの最大サイズ：1 MB

### トランザクションメールを送信

#### パラメータ

|**パラメータ**| **説明**|
|---| ---|
| ターゲットメールキャンペーン | トランザクションメール用に作成されたターゲットスマートキャンペーンを選択します。詳細については、[Marketo: トランザクションメール](https://experienceleague.adobe.com/en/docs/marketo-developer/marketo/rest/assets/transactional-email)を参照してください。 |
| ターゲットリードID | （必須）ターゲットリードIDを提供してください。 |
| 構成するトークンデータ | （オプション）トークン値をトークン名にマッピングします。Marketo UIからの完全なトークン形式を使用してください。例：`{ { my.message } }`。 |

### リードを作成または更新

#### パラメータ

|**パラメータ**| **説明**|
|---| ---|
| 更新戦略 | （必須）更新戦略を選択します。 |
| リードルックアップフィルター | &lt;ul&gt;&lt;li&gt;（オプション）既存のリードを検索するためのフィルターフィールドを選択します。&lt;/li&gt;&lt;li&gt;選択されたルックアップフィルターフィールドには、**リードデータ**セクションで値が構成されている必要があります。&lt;/li&gt;&lt;/ul&gt; |

### リードを一括インポート（オプションでリストに追加）

#### バッチ制限

このアクションは、バッチ処理されたリクエストを使用してベンダーへの大量データ転送をサポートします。リクエストは、次のいずれかのしきい値が満たされるまでキューに入れられます：

* 最大リクエスト数：25,000
* 最古のリクエストからの最大時間：30分
* リクエストの最大サイズ：10 MB
#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| ターゲット静的リスト | (必須) リストを選択してください。プラットフォームは最大300のリストを読み込みます。リストが表示されない場合は、リストIDを入力してください。 |
| リード検索フィールド | (必須) リード検索に使用するフィールド。&lt;ul&gt;&lt;li&gt;選択された検索フィールドには、**リードデータ**セクションで値が構成されている必要があります。&lt;/li&gt;&lt;li&gt;可能な値: `id`, `cookies`, `email`, `twitterId`, `facebookId`, `linkedInId`, `sfdcAccountId`, `sfdcContactId`, `sfdcLeadId`, `sfdcLeadOwnerId`, `sfdcOpptyId`。&lt;/li&gt;&lt;/ul&gt; |
| リードデータ | (必須) リードのプロパティ名に値をマッピングします。検索フィールドもマッピングする必要があります。 |
