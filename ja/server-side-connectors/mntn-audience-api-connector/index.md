MNTN Audience API コネクタを使用すると、TealiumからMNTNオーディエンスセグメントを作成および管理できます。このコネクタを使用して、バッチでファーストパーティのIDを同期し、接続されたテレビ（CTV）のターゲティング、リターゲティング、および抑制のためにオーディエンスメンバーを追加、更新、または削除できます。手動でのファイルアップロードやカスタムAPIコードは不要です。

## API情報

このコネクタは以下のベンダーAPIを使用します：

* API名：MNTN Audience API
* APIバージョン：v2026.0.1
* APIエンドポイント：`https://integrations.ex.mountain.com`

## 構成

コネクタマーケットプレイスにアクセスし、新しいコネクタを追加します。コネクタの追加方法については、[コネクタについて]()を参照してください。

コネクタを追加した後、以下の構成を構成します：

* **MNTN Audience APIキー**：（必須）あなたのMNTN Audience APIキー。

## アクション

| アクション名 | AudienceStream | EventStream |
| ----------- | :------------: | :---------: |
| バッチでオーディエンスメンバーをアップサート | ✓ | ✓ |
| バッチでオーディエンスメンバーを削除 | ✓ | ✓ |

### バッチでオーディエンスメンバーをアップサート

#### バッチ制限

このアクションは、ベンダーへの高容量データ転送をサポートするためにバッチリクエストを使用します。並列処理により、イベントがベンダーに順不同で到達する可能性があります。順序が重要な場合は、イベントにシーケンス値を追加してください。詳細については、[バッチアクション]()を参照してください。以下のいずれかの閾値に達するか、プロファイルが公開されるまでリクエストはキューに入れられます：

* 最大リクエスト数：10,000
* 最古のリクエストからの最大時間：60分
* リクエストの最大サイズ：2 MB

#### パラメータ

| パラメータ | 説明 |
| --- | --- |
| セグメントID | 既存のMNTNオーディエンスセグメントにIDを挿入または更新するためのID。 |
| ID ID | （必須）セグメント内のオーディエンスメンバーのユニークID（例：Tealium Visitor IDまたはCRM ID）。 |
| ソース | IDイベントのソースシステムラベル。 |
| ソース時間 (RFC 3339) | IDイベントが発生したタイムスタンプ、RFC 3339形式（例：`2026-05-01T12:00:00Z`）。マッピングされていない場合はイベントタイムスタンプがデフォルトです。 |

#### 識別子

| パラメータ | 説明 |
| --- | --- |
| メール（SHA256適用） | プレーンテキストのメールアドレス。コネクタは正規化（小文字化、トリム）を行い、SHA256ハッシュを適用します。 |
| メール（既にSHA256） | 事前にハッシュ化されたメールアドレス。値を小文字化してトリムした後にハッシュ化を確認してください。 |
| 電話（SHA256適用） | プレーンテキストの電話番号。コネクタは数字以外を削除し、国コードを先頭に付けてSHA256ハッシュを適用します。 |
| 電話（既にSHA256） | 事前にハッシュ化された電話番号。数字以外を削除し、10桁のUS番号の場合は`1`を先頭に付けてください。 |
| IPv4 | 生のIPv4アドレス。 |
| IPv4（既にSHA256） | 事前にハッシュ化されたIPv4アドレス。 |
| IPv6 | 生のIPv6アドレス。 |
| IPv6（既にSHA256） | 事前にハッシュ化されたIPv6アドレス。 |
| MAID | UUID形式の生のモバイル広告IDで、CTVまたはモバイルターゲティングに使用されます（例：`3f097372-f01e-4b64-984c-395ae5828ee6`）。 |


IPv4（既にSHA256）、IPv6、IPv6（既にSHA256）、およびMAIDは、MNTNキャンペーンターゲティングで選択的にサポートされています。ターゲティングにこれらの識別子を頼る前に、MNTNアカウントチームに連絡してください。


### バッチでオーディエンスメンバーを削除

#### バッチ制限

このアクションは、ベンダーへの高容量データ転送をサポートするためにバッチリクエストを使用します。並列処理により、イベントがベンダーに順不同で到達する可能性があります。順序が重要な場合は、イベントにシーケンス値を追加してください。詳細については、[バッチアクション]()を参照してください。以下のいずれかの閾値に達するか、プロファイルが公開されるまでリクエストはキューに入れられます：

* 最大リクエスト数：1,000
* 最古のリクエストからの最大時間：60分
* リクエストの最大サイズ：2 MB

#### パラメータ

| パラメータ | 説明 |
| --- | --- |
| オーディエンスセグメント | メンバーを削除するMNTNオーディエンスセグメントを選択するか、セグメントIDを手動で入力します。 |
| ID ID | （必須）セグメントから削除するオーディエンスメンバーのID ID（例：Tealium Visitor IDまたはCRM ID）。 |

## ベストプラクティス

* 本番量にスケールする前に、小規模なテストセグメントから始めてください。
* リクエストペイロードを効率的に保ちます。MNTNは、バッチを約2 MBにターゲティングし、最大リクエストサイズの8 MBを下回ることを推奨します。
* 重複または矛盾するレコードを避けるために、一貫したID戦略を使用してください。
* MNTNに識別子を送信する前に、同意およびプライバシー要件を確認してください。
