## API情報

このコネクタは以下のベンダーAPIを使用します：

* API名：Amazon Ads Events API
* APIバージョン：v1
* APIエンドポイント：`https://advertising-api.amazon.com`
* ドキュメント：[Amazon Ads Events API](https://advertising.amazon.com/API/docs/en-us/guides/events/events)
* タグドキュメント：[Amazon Advertising Tag](https://docs.tealium.com/amazon-advertising-tag/)

## コネクタのヒント

タグとコネクタを使用したハイブリッド構成を推奨します。この構成により、データの耐久性が最大化され、サーバーサイドのシグナルとのイベントマッチングが改善され、リアルタイムのクライアントサイドシグナルを保持しながらデータフローの連続性が維持されます。

## 構成

コネクタマーケットプレイスにアクセスして新しいコネクタを追加します。コネクタを追加する一般的な手順については、[About Connectors](https://docs.tealium.com/about-connectors/)を参照してください。

コネクタを追加した後、以下の構成を構成します：

* **APIベースURL**：（必須）APIベースURLを選択します。
* **Amazon広告主ID**：（必須）Amazon広告アカウントIDを入力します。

## Webイベントの重複排除

Amazon Adsサーバータグからのイベントを重複排除するために、次の命名規則でイベントIDをイベント属性として送信するようにこのコネクタを構成します：

```nohl
amazon_event_id_{EVENT_NAME}_{TAG_UID}
```

タグのUIDは、Tealium iQの**タグ**テーブルまたはタグ詳細画面で見つけることができます：

![](https://docs.tealium.com/images/server-side-connectors/amazon-ad-server-tag-uid.png)

例えば、UIDが`170`のタグからのチェックアウトイベントは、次の属性値を送信します：

```json
{
  "amazon_event_id_checkout_170": "028b2ade7478..."
}
```

同じタグからのページビューイベントは、次の属性と値を送信します：

```json
{
  "amazon_event_id_pageView_170": "028b2ade7478..."
}
```

各タイプのイベントに対して別々のアクションを使用します。イベント固有のイベントID属性を`Event ID`にマッピングし、対応するイベント名を`Event Name`にマッピングします。例えば：

| Tealium属性  | Amazonパラメータ  |
|:-------------------------|:-------------------|
| `amazon_event_id_checkout_170` (属性)   | Event ID           |
| `"Checkout"` (カスタムテキスト) | Event Name         |

### 自動重複排除

**自動重複排除**の値が入力されている場合、コネクタは対応するイベントID属性を探します。このセクションが入力されていてTealium iQを使用している場合、`Event ID`を追加する必要はありません。ただし、Tealium iQを使用せずにEventStreamを使用している場合は、以下のアプローチが依然として必要になる場合があります。

コネクタは次の操作順序に従います：

* **自動重複排除**が入力され、値がデータレイヤーで見つかった場合（例：`amazon_event_id_checkout_170`）、この値は`Event ID`にマッピングされたものより優先されます。
* **自動重複排除**が入力されているが、値が見つからない場合、入力されている場合はマッピングされた`Event ID`が使用されます。

### コンバージョン送信

#### バッチ制限

このアクションはバッチリクエストを使用して、ベンダーへの大量データ転送をサポートします。並行処理により、イベントがベンダーに順不同で到達する可能性があります。イベントの順序が重要な場合は、イベントにシーケンス値を追加します。詳細については、[Batched Actions](https://docs.tealium.com/batched-actions/)を参照してください。リクエストは、次のいずれかの閾値に達するか、プロファイルが公開されるまでキューに入れられます：

* 最大リクエスト数：500
* 最古のリクエストからの最大時間：10分
* リクエストの最大サイズ：1 MB

#### パラメータ

| パラメータ | 説明 |
| --- | --- |
| コンバージョンタイプ | インポートされるイベントのコンバージョンタイプを選択します。可能な値は`ADD_TO_SHOPPING_CART`, `APPLICATION`, `CHECKOUT`, `CONTACT`, `LEAD`, `OFF_AMAZON_PURCHASES`, `MOBILE_APP_FIRST_START`, `PAGE_VIEW`, `SEARCH`, `SIGN_UP`, `SUBSCRIBE`, `OTHER`です。 |
| コンバージョンソース | インポートされるイベントのコンバージョンソースを選択します。可能な値は`ANDROID`, `FIRE_TV`, `IOS`, `OFFLINE`, `WEBSITE`, `MEASUREMENT_PARTNER`です。 |

#### コンバージョンデータ

| パラメータ | 説明 |
| --- | --- |
| 名前 | (必須) インポートされるイベントの名前。例えば、ウェブサイト購入、ニュースレター登録など。 |
| 国コード | (必須) イベントを実行したユーザーが位置していた国を示すISO 3166-1 alpha-2国コード。 |
| データセット名 | このイベントを追加するイベントデータセット名。1〜100文字の文字列で、文字で始まり、その後は文字、数字、アンダースコア、ハイフンのみを含むことができます。 |
| イベント時間 | コンバージョンのタイムスタンプをISO形式（`YYYY-MM-DDThh:mm:ssTZD`）で。マッピングされていない場合、コネクタはイベント時間を使用します。 |
| 値 | イベントの値。 |
| 通貨コード | ISO-4217形式の通貨。`OFF_AMAZON_PURCHASES`のコンバージョンタイプにのみ適用されます。 |

| 重複排除ID | タグとコネクタの両方から送信されるコンバージョンイベントを重複排除するために使用される一意の識別子。重複したコンバージョンレポートを防ぐために、両方で同じ値を構成します。 |

#### API情報

#### 変換定義の作成

変換定義を作成するには、**Create Conversion Definition** をクリックし、次の情報を提供してください：

* **Advertising Profile**：（必須）この変換の広告プロファイルを選択します。
* **Conversion Definition Name**：（必須）変換定義の名前です。
* **Source Type**：（必須）変換のソースタイプを選択します。利用可能なタイプは `ADD_TO_SHOPPING_CART`, `APPLICATION`, `CHECKOUT`, `CONTACT`, `LEAD`, `OFF_AMAZON_PURCHASES`, `PAGE_VIEW`, `SEARCH`, `SIGN_UP`, `SUBSCRIBE`, `OTHER` です。
* **Conversion Type**：（必須）変換のタイプを選択します。利用可能なタイプは `ANDROID`, `FIRE_TV`, `IOS`, `OFFLINE`, `WEBSITE` です。
* **Counting Method**：（必須）変換定義のために、24時間のウィンドウ内でユーザーによって摂取された最初の変換イベントのみ、またはすべての変換イベントを使用するかを決定する方法を選択します。利用可能な方法は `EVERY` と `FIRST` です。
* **Conversion Value**：（オプション）変換の値です。デフォルト値は `1` で、**Send Conversion** アクションでマッピングを通じて上書きすることができます。

変換定義を保存するには、**Done** をクリックします。

広告データマネージャーコンソールの変換ページまたはTerms API（`POST /adm/terms`）を使用して、利用規約に同意してください。

#### バッチ制限

このアクションは、ベンダーへの大量データ転送をサポートするためにバッチリクエストを使用します。並行処理により、イベントがベンダーに順不同で到達する可能性があります。順序が重要な場合は、イベントにシーケンス値を追加してください。詳細については、[Batched Actions](https://docs.tealium.com/batched-actions/)を参照してください。次のいずれかの閾値が満たされるか、プロファイルが公開されるまでリクエストはキューに入れられます：

* 最大リクエスト数：100
* 最古のリクエストからの最大時間：10分
* リクエストの最大サイズ：1 MB
