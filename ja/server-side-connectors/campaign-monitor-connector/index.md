## コネクタアクション

| アクション名 | AudienceStream | EventStream |
| --- | :---: | :---: |
| スマートメール送信 | ✓ | ✗ |
| 加入者の追加または更新 | ✓ | ✗ |
| 加入者の削除 | ✓ | ✗ |
| 加入者の退会 | ✓ | ✗ |
| 加入者の更新 | ✓ | ✗ |

## 構成の構成

コネクタマーケットプレイスに移動して新しいコネクタを追加します。コネクタの追加方法についての一般的な説明は、[コネクタについて](https://docs.tealium.com/about-connectors/)を参照してください。

コネクタを追加した後、以下の構成を構成します：

* **APIキー**  
 <br>Campaign Monitor アカウントとのやり取りにはAPIキーが必要です。詳細については、<a href="http://help.campaignmonitor.com/topic.aspx?t=206" target="_blank">APIキーはどこで見つかりますか？</a>を参照してください。
* **クライアント名**  
 <br>クライアントアカウントを管理するための名前が必要です。

## アクション

アクションの名前を入力し、ドロップダウンメニューからアクションタイプを選択します。

次のセクションでは、各アクションのパラメータとオプションの構成方法について説明します。

### スマートメール送信

#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| スマートメール | 送信するスマートトランザクションメールを選択します。 |

### 加入者の追加または更新

#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| 加入者リスト | 加入者を追加するリストを選択 |
| 加入者のメールアドレス | (必須)  |
| 加入者の名前 | (必須)  |
| 加入者の再加入 |  |
| 加入者の自動応答の再開 |  |

### 加入者の削除

#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| 加入者リスト | 加入者を削除するリストを選択します。 |
| 加入者のメール | 選択したリストから削除するメールを提供します（参照：<a href="http://help.campaignmonitor.com/topic.aspx?t=140#remove-options" target="_blank">削除対退会</a>）。削除されたメールは抑制リストに追加されません<br>推奨：誰かが削除を求めた場合は退会アクションを使用し、何らかの理由で削除を選択した場合は削除アクションを使用します。 |

### 加入者の退会

#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| 加入者リスト | 加入者を退会させるリストを選択します。 |
| 加入者のメール | 選択したリストから退会させ、抑制リストに追加するメールを提供します。このメールは、退会構成が変更されたリストを除き、クライアントの他のリストからも退会されます。 |

### 加入者の更新

#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| 加入者リスト | 更新する加入者のリストを選択 |
| 加入者のメールアドレス | (必須)  |
| 加入者の名前 | (必須)  |
| 加入者の携帯番号 |  |
| 加入者のカスタムフィールド | クリアを選択した場合、値フィールドに値を入力するか、クリアを選択して属性名を入力して値をクリアします。カスタムフィールドの値をクリアするには、trueの値とカスタムフィールド名/値とともにClearパラメータを渡します。特定の複数値選択多数オプションを削除するには、値フィールドにオプション名とClear: trueパラメータを渡します。複数値選択多数フィールドのすべての値をクリアするには、値なしでClear: trueパラメータを渡します。また、日付タイプのカスタムフィールドは、「0000-00-00」の値を渡すことで依然としてクリア可能です。利用可能なフィールドは選択された加入者リストによって決まります（参照：<a href="http://help.campaignmonitor.com/topic.aspx?t=154" target="_blank">カスタムフィールドを作成して使用するにはどうすればよいですか？</a>）<br>日付属性は自動的にYYYY-MM-DD形式に変換されます |
| トラッキングの同意 | 加入者がメールの開封とクリックを追跡することを許可しているかどうか。この値は、同じクライアント内の同じメールアドレスのすべての加入者に適用されます。メールアドレスにunchangedの値を渡し、そのアドレスが現在クライアントに存在しない場合、またはConsentToTrackの既存の値がない場合、加入者は同意したとみなされます。 |
| SMS送信の同意 | 加入者がSMSを受信することに同意したかどうかを示すオプションのパラメータです。SMS送信の同意は、携帯番号が提供された場合にのみ記録されます。このパラメータを省略すると、既存の同意構成は変更されません。 |
| 再加入 | 加入者が非アクティブ状態にあるか、以前に退会していたり抑制リストに追加されていた場合、Resubscribeの入力値をtrueとして指定すると、リストに再追加されます。したがって、この方法は慎重に使用し、適切な場合にのみ使用する必要があります。Resubscribeをfalseとして指定した場合、加入者はアクティブリストに再追加されません。 |
| 定期購読ベースの自動応答の再開 | RestartSubscriptionBasedAutorespondersの入力値をtrueとして指定すると、任意のシーケンスが再開されます。RestartSubscriptionBasedAutorespondersは再加入する加入者にのみ影響し、省略された場合はfalseになります。 |
