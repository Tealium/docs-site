Facebookコンバージョンコネクタ構成ガイド

この記事では、アカウント内でFacebook Conversionsコネクタを構成する方法について説明します。

前提条件

WebイベントのFacebook Conversionsコネクタを使用している場合、クライアント側のFacebook PixelタグがWebブラウザで動作している必要があります。詳細については、Facebookヘルプセンターを参照してください。

Facebook広告アカウント構成でTealiumパートナー統合を有効にするには、次の手順を使用します：

  1. Facebook広告マネージャーアカウントにログインし、メニューからイベントマネージャーを選択します。
  2. メニューでデータソースを選択し、ピクセルを選択して構成タブをクリックします。
  3. サーバーサイドAPIセクションまでスクロールします。
  4. パートナー統合セクションで、パートナーを選択をクリックします。
  5. Tealium接続を承認を選択し、続行をクリックしてTealiumを有効にします。

パートナー統合を切り替えるオプションが表示されない場合は、Tealiumアカウントマネージャーにサポートを依頼してください。

Facebook Pixelタグは自動的にユーザーデータをハッシュ化します。このピクセル機能は変更できません。

API情報

このコネクタは次のベンダーAPIを使用します：

  • API名：Facebook Graph API - Facebook Pixel
  • APIバージョン：v20.0
  • APIエンドポイント：https://graph.facebook.com

アクション

アクション名 AudienceStream EventStream
イベントを送信 ✓ ✓
高度な測定イベントを送信 ✓ ✓
リードイベントを送信 ✓ ✓

構成

コネクタマーケットプレイスに移動し、新しいコネクタを追加します。コネクタの追加方法の一般的な手順については、コネクタについてを参照してください。

コネクタを追加した後、次の構成を行います：

  • データセットID
    • Facebook Pixel IDです。
    • Facebook Pixelが正しく動作するように、Facebook Pixelのパートナー統合を有効にするための前提条件で概説された手順を遵守していることを確認してください。
    • 詳細については、Meta Pixelの構成とインストール方法を参照してください。

接続テスト

  • ピクセルが許可リストに追加され、Tealiumで有効になっている場合、TealiumTestConnectorイベントがピクセルで追跡されます。接続テストをクリックして、TealiumConnectorTestという名前のテストイベントを送信します。
  • Facebookイベントマネージャーピクセル概要を確認して、イベントが受信されたことを確認します。
  • テストが失敗した場合は、構成を再確認してください。

Webイベントの重複排除

Facebook Pixelタグからのイベントを重複排除するようにこのコネクタを構成するには、次の命名規則でイベント属性として送信されるイベントIDを使用します：

fb_event_id_{FACEBOOK_EVENT}_{TAG_UID}

例えば、タグ#32からの購入イベントは次の属性と値を送信します：

{
  "fb_event_id_Purchase_32": "028b2ade7478..."
}

同じタグからのページビューイベントは次の属性と値を送信します：

{
  "fb_event_id_PageView_32": "084b1cda7461..."
}

各タイプのイベントに対して別々のアクションを使用します。イベント固有のイベントID属性をEvent IDにマッピングし、一致するイベント名をEvent Nameにマッピングします。例えば：

Tealium属性 Facebookパラメータ
fb_event_id_Purchase_32 (属性) Event ID
"Purchase" (カスタムテキスト) Event Name

イベントID属性をfb_event_id_{FACEBOOK_EVENT}（タグUIDなし）の命名形式でのみ受け取っている場合は、タグを更新することをお勧めします。詳細については、Facebook Pixelタグ構成ガイドを参照してください。

自動重複排除

自動重複排除の値が入力されている場合、コネクタは対応するイベントID属性を探します。このセクションが入力されており、Tealium iQを使用している場合、Event IDを追加する必要はもうありません。ただし、Tealium iQを使用せずにEventStreamを使用する場合は、以下のアプローチがまだ必要な場合があります。

コネクタは次の操作順序を守ります：

  • 自動重複排除が入力され、その値がデータレイヤーで見つかる場合（例：fb_event_id_ViewContent_30）、この値はEvent IDにマッピングされたものよりも優先されます。
  • 自動重複排除が入力されているが、タグIDバージョンを見つけることができず、レガシーバージョン（IDが付いていない）を見つけることができる場合、その値が使用されます（例：fb_event_id_ViewContent）。
  • 自動重複排除が入力されているが、その値が見つからず、Event IDがマッピングされている場合、そのEvent IDが使用されます。

オフラインコンバージョン

新しい統合のために、Facebookオフラインコンバージョンコネクタの代わりにFacebook Conversionsコネクタを使用することをお勧めします。

Facebook Conversionsコネクタを通じてオフラインおよびストアイベントを処理するには、action_sourceパラメータをphysical_storeに構成します。

アクション

次へをクリックするか、アクションタブをクリックします。アクションタブでコネクタアクションを構成します。

このセクションでは、各アクションのパラメータとオプションの構成方法について説明します。

イベントを送信

パラメータ

ユーザーデータパラメータについては、以下の表に一部のパラメータの重要度を記載しています。重要度は、パラメータがイベントマッチ品質に与える影響を説明します。例えば、Email (em)は最高の重要度を持ち、Phone Number (ph)は高の重要度を持ちます。

パラメータ 説明
イベントソースURL イベントが発生したブラウザのURL。アクションソースがウェブサイトに構成されている場合、このパラメータは必須です。
  • EventStream - アクションソースがウェブサイトに構成され、このフィールドがマッピングされていない場合、EventStreamの属性Current URLに自動的にマッピングされます。
  • AudienceStream - アクションソースがウェブサイトに構成され、このフィールドがマッピングされていない場合、AudienceStreamのLast Event URL属性に自動的にマッピングされます。
テストイベントコード テストIDをtest_event_codeパラメータで送信して、テストイベントウィンドウでイベントアクティビティを開始します。test_event_codeで送信されたイベントはドロップされず、代わりにイベントマネージャーに流れ込み、ターゲティングと広告測定の目的で使用されます。
データ処理オプション (必須) この値をLDUにマッピングして、FacebookがイベントをLimited Data Use制限で処理するようにします。それ以外の場合は、この値をNoneにマッピングして、イベントがLimited Data Use制限で処理されないように指定します。詳細については、Facebookデータ処理オプションを参照してください。
イベント名 (必須) Facebookピクセルの標準またはカスタムイベント名。
デバッグレベル イベントのトレースレベルを指定します。
  • 1はエラー
  • 2は情報
  • 3はデバッグ
データのハッシュ化を確認するためにTraceを使用する場合、Traceの上部に表示されるデータ、例えばuser_data - EMAILは、コネクタに入ってきたデータであり、Facebookに送信されたデータではありません。Facebookに送信されたデータはTraceのリクエストボディセクションに表示されます。
アクションソース (必須) このフィールドは、コンバージョンが発生した場所を指定します。有効なアクションソースは次のとおりです：
  • email
  • website
  • app
  • phone_call
  • chat
  • physical_store
  • system_generated
  • other
データセットIDオーバーライド この文字列値は、初期のコネクタ構成で使用される値をオーバーライドするために使用できます。提供されていない場合、コネクタ構成からのデータセットIDが使用されます。
データ処理オプション国 ユーザーイベントに関連付けられた国コードをマッピングします。このパラメータは、データ処理オプションがLDUにマッピングされている場合に必要です。受け入れられるコード値:
  • 1はアメリカ合衆国を表します
  • 0はFacebookによる国のジオロケーションを要求します
ジオロケーションには、クライアントIPアドレス属性のマッピングも必要です。
イベントID Facebookがサーバーとブラウザから送信される同じイベントを重複排除するために使用するID。この値がブラウザとサーバから送信されるイベントのペアに対する一意のIDであることを確認してください。
イベント時間 実際のイベントが発生したときのUnixタイムスタンプ（秒）。コネクタはマッピングされた日付属性を秒に変換し、属性が欠けている場合はイベント時間を自動的に補完します。
データ処理オプション州 このパラメータは、データ処理オプションがLDUにマッピングされている場合に必要です。ユーザーイベントに関連付けられた州コードをマッピングします。受け入れられるコード値:
  • 1000はカリフォルニアを表します
  • 0はFacebookによる州のジオロケーションを要求します
ジオロケーションには、クライアントIPアドレス属性のマッピングも必要です。
オプトアウト このイベントを広告配信の最適化に使用しないことを示すブール値。この値をtrueに構成すると、イベントは属性付けのためだけに使用されます。
郵便番号 (zip) 郵便番号。例: 94035。このパラメータは文字列または文字列の配列を受け入れます。例: ["<first value>", "<second value>"]。 重要度: 中
FBログインID 人がアプリのインスタンスに初めてログインしたときにFacebookによって発行されるID。このパラメータは、App-Scoped IDとも呼ばれます。この値をハッシュ化しないでください。
パートナーID パートナーのID。このパラメータは、LiverampやTradeDeskなどのサードパーティIDプロバイダーによって、追加のユーザー識別子として使用することができます。この値をハッシュ化しないでください。
パートナー名 パートナーの名前。このパラメータは、LiverampやTradeDeskなどのサードパーティIDプロバイダーによって、追加のユーザー識別子として使用することができます。この値をハッシュ化しないでください。
クライアントユーザーエージェント (ua) このパラメータは、アクションソースがウェブサイトに構成されている場合に必要です。イベントに対応するブラウザのユーザーエージェント。この値をハッシュ化しないでください。
  • EventStream - アクションソースがウェブサイトに構成されていて、このフィールドがマッピングされていない場合、EventStreamのユーザーエージェント属性に自動的にマッピングされます。
  • AudienceStream - アクションソースがウェブサイトに構成されていて、このフィールドがマッピングされていない場合、AudienceStreamの訪問の最後のユーザーエージェントに自動的にマッピングされます。
重要度: 高
クライアントIPアドレス (ip) イベントに対応するブラウザのIPアドレス。TealiumはクライアントのIPアドレスを取得することができますが、デフォルトでは利用できません。この機能を有効にするには、カスタマーサクセスマネージャーに連絡してください。詳細については、サーバーサイドTealium製品でのIPアドレス収集の有効化を参照してください。この値をハッシュ化しないでください。 重要度: 高
電話 (ph) 電話番号。国コード、地域コード、番号のみを含めます。例: 16505551212。このパラメータは文字列または文字列の配列を受け入れます。例: ["<first value>", "<second value>"]。 重要度: 高
州 (st) 米国の州の略称。例: ca。このパラメータは文字列または文字列の配列を受け入れます。例: ["<first value>", "<second value>"]。 重要度: 中
メール (em) メールアドレス。例: user@example.com。このパラメータは文字列または文字列の配列を受け入れます。例: ["<first value>", "<second value>"]。 重要度: 最高
姓 (ln) 姓を小文字で。例: smith。このパラメータは文字列または文字列の配列を受け入れます。例: ["<first value>", "<second value>"]。 重要度: 中
名 (fn) 名を小文字で。例: joe。このパラメータは文字列または文字列の配列を受け入れます。例: ["<first value>", "<second value>"]。 重要度: 中
リードID Facebook リード広告によって生成されたリードに関連付けられたID。この値をハッシュ化しないでください。
市区町村 (ct) スペースや句読点なしの小文字の市区町村。例: menlopark。このパラメータは文字列または文字列の配列を受け入れます。例: ["<first value>", "<second value>"]。 重要度: 中
国 (country) 2文字の国コードを小文字で。例: us。このパラメータは文字列または文字列の配列を受け入れます。例: ["<first value>", "<second value>"]。 重要度: 中
外部ID 広告主からの任意の一意のID、例えば、ロイヤルティメンバーシップID、ユーザーID、外部クッキーIDなど。複数のIDを追加するために配列型の属性をマッピングします。このマッピングは、自動マッピングが無効になっていない限り、Tealium Visitor IDで自動的に補完されます。
ブラウザID (fpb) Facebookの_fbpクッキーに保存されているブラウザIDの値。このパラメータは自動マッピングされますが、自動マッピングを無効にすることもできます。この値をハッシュ化しないでください。 重要度: 高

Send Lead Event

Batch Limits

このアクションは、ベンダーへの大量のデータ転送をサポートするためにバッチリクエストを使用します。詳細については、Batched Actionsを参照してください。リクエストは、以下のいずれかの閾値が満たされるか、プロファイルが公開されるまでキューに入れられます：

  • リクエストの最大数：1000
  • 最古のリクエストからの最大時間：10分
  • リクエストの最大サイズ：1 MB

Verify Events

イベントを送信した後、Facebookがそれらを受信したことを概要とブレークダウンビューで確認します：

  1. Business Manager > Pixelsに移動します。
  2. POSTリクエストのPIXEL_IDに対応するピクセルをクリックします。

Overview Tab

Overviewタブで、AllからServerにビューを変更して、正常に送信したイベントの数を確認します。

Breakdownタブでは、正常に送信されたイベントの内訳が表示されます。ドロップされたイベントはServer Events Received列に表示されません。

Conversions APIについての詳細情報は、FacebookのUsing the Conversions APIを参照してください。
