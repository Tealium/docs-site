Facebook Conversions Connector セットアップガイド

この記事では、アカウント内で Facebook Conversions コネクタを設定する方法について説明します。

前提条件

Web イベントのために Facebook Conversions コネクタを使用する場合、クライアントサイドの Facebook Pixel タグ が Web ブラウザで実行されている必要があります。詳細については、Facebook ヘルプセンターを参照してください。

Facebook Ads アカウントの設定で Tealium パートナー統合を有効にするには、次の手順を使用します。

  1. Facebook Ads Manager アカウントにログインし、メニューから イベントマネージャー を選択します。
  2. メニューで データソース を選択し、ピクセルを選択して 設定 タブをクリックします。
  3. サーバーサイド API セクションまでスクロールします。
  4. パートナー統合 セクションで パートナーを選択 をクリックします。
  5. Tealium 接続を承認 を選択し、Tealium を有効にするために 続行 をクリックします。

パートナー統合を切り替えるオプションが表示されない場合は、Tealium アカウントマネージャーに連絡してサポートを依頼してください。

Facebook Pixel タグはユーザーデータを自動的にハッシュ化します。このピクセルの機能は変更できません。

API 情報

このコネクタは次のベンダー API を使用します。

  • API 名: Facebook Graph API - Facebook Pixel
  • API バージョン: v18.0
  • API エンドポイント: https://graph.facebook.com

コネクタのアクション

アクション名 AudienceStream EventStream
イベントを送信する ✓ ✓
リードイベントを送信する ✓ ✓

設定の構成

コネクタマーケットプレイスに移動し、新しいコネクタを追加します。コネクタの追加方法の一般的な手順については、コネクタについてを参照してください。

コネクタを追加した後、次の設定を構成します。

  • データセット ID

    • Facebook Pixel が機能するためには、Facebook ピクセルのパートナー統合を有効にするために Facebook Server-Side Events コネクタセットアップガイド で説明されている手順に従っていることを確認してください。
    • 詳細については、Facebook Pixel セットアップガイドを参照してください。

  • 接続のテスト

    • ピクセルが Tealium にホワイトリスト登録され、Tealium で有効になっている場合、TealiumTestConnector イベントがピクセルでトラッキングされます。テスト接続 をクリックして、TealiumConnectorTest という名前のテストイベントを送信します。
    • Facebook イベントマネージャーのピクセルの概要を確認して、イベントが受信されたことを確認します。
    • テストが失敗した場合は、設定を再確認してください。

アクションの設定 - パラメータとオプション

次へ をクリックするか、アクション タブをクリックします。アクションタブでコネクタのアクションのパラメータとオプションを構成します。

このセクションでは、各アクションのパラメータとオプションの設定方法について説明します。

Facebook Pixel および Facebook Conversions API の重複排除

このコネクタアクションを設定して、Facebook Pixel タグからのイベント ID を iQ タグ管理アカウントで受け取るようにするには、次の命名規則を使用してイベント属性を構成します。

fb_event_id_<Facebook Event>_<Tag UID>

例: fb_event_id_Purchase_32 または fb_event_id_PageView

以前は fb_event_id_<Facebook Event> として生成されていた ID も引き続き生成されますが、これらはレガシーと見なされます。新しい ID は、単一のページで複数の Facebook タグが発火する場合に使用でき、関連するタグにより強力な関連付けが可能です。レガシーバージョンを使用している場合は、コネクタを新しい ID に変更する必要があります。

各種イベント ID ごとに別個のアクションを使用します。イベント固有のイベント ID 属性をイベント ID にマッピングし、マッチングするイベント名を Event Name にマッピングします。例:

Tealium 属性/値 Facebook パラメータ
fb_event_id_Purchase Event ID
“Purchase” (カスタムテキスト) Event Name

Facebook Pixel タグセットアップガイドを参照してください。

自動重複排除

自動重複排除 の値が入力されている場合、コネクタは UDO オブジェクト内の対応する変数を検索します。このセクションが入力されており、Tealium iQ を使用している場合、Event ID を追加する必要はありません。ただし、TealiumiQ を使用せずに EventStream を使用している場合は、次のアプローチが引き続き必要になる場合があります。

コネクタは次の操作順序を使用します。

  • 自動重複排除 が入力されており、データレイヤーでその値が見つかった場合（たとえば、fb_event_id_ViewContent_30）、この値は Event ID にマッピングされたものより優先されます。
  • 自動重複排除 が入力されており、タグ ID バージョンが見つからないが、レガシーバージョン（ID なし）が見つかった場合、その値が使用されます（たとえば、fb_event_id_ViewContent）。
  • 自動重複排除 が入力されており、値が見つからない場合、マッピングされた Event ID が入力されている場合は、それが使用されます。

オフラインコンバージョン

新しい統合には、Facebook オフラインコンバージョンコネクタ の代わりに Facebook Conversions コネクタを使用することをお勧めします。

Facebook Conversions コネクタを介してオフラインおよびストアイベントを処理するには、action_source パラメータを physical_store に設定します。

アクション - イベントを送信する

パラメータ

ユーザーデータ パラメータについて、次の表は、いくつかのパラメータの重要度レベルを示しています。重要度レベルは、イベントの一致品質に対するパラメータの影響度を説明しています。たとえば、Email (em) の重要度レベルは 最高 であり、Phone Number (ph) の重要度レベルは 高 です。

パラメータ 説明
Event Source URL イベントが発生したブラウザの URL。このパラメータは、Action Source がウェブサイトに設定されている場合に必要です。
  • EventStream - Action Source がウェブサイトに設定されており、このフィールドがマッピングされていない場合、EventStream の Current URL 属性に自動的にマッピングされます。
  • AudienceStream - Action Source がウェブサイトに設定されており、このフィールドがマッピングされていない場合、AudienceStream の Last Event URL 属性に自動的にマッピングされます。
Test Event Code テストイベントウィンドウでイベントアクティビティを表示するために、test_event_code パラメータでテスト ID を送信します。test_event_code で送信されるイベントは破棄されず、代わりにイベントマネージャーにフローし、ターゲティングおよび広告測定の目的で使用されます。
Data Processing Options (必須) この値を LDU にマッピングすると、Facebook は制限付きデータ使用でイベントを処理します。それ以外の場合は、この値を None にマッピングして、イベントを制限付きデータ使用で処理しないように指定します。詳細については、Facebook Data Processing Options を参照してください。
Event Name (必須) Facebook ピクセルの標準イベントまたはカスタムイベントの名前。
Debugging Level イベントのトレースレベルを指定します。
  • 1 はエラー
  • 2 は情報
  • 3 はデバッグ
データのハッシュ化を検証するために Trace を使用する場合、Trace の上部に表示される user_data - EMAIL などのデータは、コネクタに入力されたデータであり、Facebook に送信されたデータではありません。Facebook に送信されるデータは、Trace の Request Body セクション に表示されます。
Action Source (必須) コンバージョンが発生した場所を指定するフィールドです。有効なアクションソースは次のとおりです:
  • email
  • website
  • app
  • phone_call
  • chat
  • physical_store
  • system_generated
  • other
Dataset ID Override この文字列値を使用して、初期のコネクタ構成で使用される値を上書きできます。指定しない場合、コネクタ構成のデータセット ID が使用されます。
Data Processing Options Country ユーザーイベントに関連する国コードをマッピングします。Data Processing Options を LDU にマッピングする場合、このパラメータが必要です。受け入れられるコード値:
  • アメリカ合衆国の場合は 1
  • Facebook によるジオロケーションをリクエストする場合は 0
ジオロケーションには、Client IP Address 属性のマッピングも必要です。
Event ID Facebook がサーバーとブラウザの両方から送信された同じイベントを重複排除するために使用される ID。この値がブラウザとサーバーから送信されたイベントのペアに対して一意の ID であることを確認してください。
Event Time 実際のイベントが発生した Unix タイムスタンプ（秒単位）。コネクタはマッピングされた日付属性を秒単位に変換し、属性が存在しない場合はイベント時刻を自動的に設定します。
Data Processing Options State Data Processing Options を LDU にマッピングする場合、このパラメータが必要です。ユーザーイベントに関連する州コードをマッピングします。受け入れられるコード値:
  • カリフォルニア州の場合は 1000
  • Facebook によるジオロケーションをリクエストする場合は 0
ジオロケーションには、Client IP Address 属性のマッピングも必要です。
Opt Out 広告配信の最適化にこのイベントを使用しないことを示すブール値。この値を true に設定すると、イベントはアトリビューションにのみ使用されます。
Zip (zip) 郵便番号。例: 94035。 重要度: 中
FB Login ID アプリのインスタンスに最初にログインしたときに Facebook が発行する ID。このパラメータは App-Scoped ID とも呼ばれます。この値をハッシュ化しないでください。
Partner ID パートナーの ID。このパラメータは、Liveramp や TradeDesk などのサードパーティ ID プロバイダが追加のユーザー識別子として使用できます。この値をハッシュ化しないでください。
Partner Name パートナーの名前。このパラメータは、Liveramp や TradeDesk などのサードパーティ ID プロバイダが追加のユーザー識別子として使用できます。この値をハッシュ化しないでください。
Client User Agent (ua) このパラメータは、Action Source がウェブサイトに設定されている場合に必要です。イベントに対応するブラウザのユーザーエージェント。この値をハッシュ化しないでください。
  • EventStream - Action Source がウェブサイトに設定されており、このフィールドがマッピングされていない場合、EventStream の User Agent 属性に自動的にマッピングされます。
  • AudienceStream - Action Source がウェブサイトに設定されており、このフィールドがマッピングされていない場合、AudienceStream の訪問者の最後のユーザーエージェントに自動的にマッピングされます。
重要度: 高
Client IP Address (ip) イベントに対応するブラウザの IP アドレス。Tealium はクライアント IP アドレスをキャプチャできますが、デフォルトでは使用できません。この機能を有効にするには、カスタマーサクセスマネージャーに連絡してください。詳細については、サーバーサイド Tealium 製品での IP アドレスの収集の有効化を参照してください。この値をハッシュ化しないでください。 重要度: 高
Phone (ph) 電話番号。国コード、市外局番、番号のみを含めます。例: 16505551212。 重要度: 高
State (st) 米国の州の略称。例: ca。 重要度: 中
Email (em) メールアドレス。例: user@example.com。 重要度: 最高
Last Name (ln) 英小文字の姓。例: smith。 重要度: 中
First Name (fn) 英小文字の名。例: joe。 重要度: 中
Lead ID Facebook の リード広告 で生成されたリードに関連する ID。この値をハッシュ化しないでください。
City (ct) 小文字でスペースや句読点を含まない都市名。例: menlopark。 重要度: 中
Country (country) 小文字の 2 文字の国コード。例: us。 重要度: 中
External ID 広告主からの一意の ID。例えば、ロイヤリティメンバーシップ ID、ユーザー ID、または外部クッキー ID など。この値をハッシュ化しないでください。
Browser ID (fpb) Facebook の _fbp クッキーに格納されているブラウザ ID の値。このパラメータは自動マッピングされますが、自動マッピングを無効にしない限り、自動マッピングされません。この値をハッシュ化しないでください。 重要度: 高
Subscription ID このトランザクションのユーザーのサブスクリプション ID。これは個々の製品の注文 ID に類似しています。例: anid1234。この値をハッシュ化しないでください。
Gender 小文字の単一文字の性別値。オプション:
  • 女性の場合は f
  • 男性の場合は m
Date of Birth 年、月、日で指定された誕生日。例: 19971226（1997 年 12 月 26 日）。
Click ID (fbc) Facebook の _fbc クッキーに格納されている Facebook クリック ID の値。このパラメータは自動マッピングされますが、自動マッピングが無効になっていない限り、自動マッピングされません。この値をハッシュ化しないでください。重要度: 高
User Data is already hashed Facebook Conversions コネクタでは、Facebook Pixel タグが必要であり、ユーザーデータが自動的にハッシュ化されます。このピクセルの機能は変更できません。データを再度ハッシュ化しないようにするには、このオプションを選択します。
Currency 適用される場合、指定された値に関連する通貨。通貨は有効な ISO 4217 の 3 桁の通貨コードである必要があります。例: usd。
Content Name イベントに関連するページまたは製品の名前。例: lettuce。
Number of Items InitiateCheckout イベントで使用します。ユーザーがチェックアウト中に購入しようとするアイテムの数。例: 4。
Search String ユーザーが行った検索クエリ。検索イベントでのみ使用します。例: lettuce
Order ID このトランザクションの注文 ID。例: order1234。
Content Type この値は product または product_group のいずれかである必要があります。
  • Content IDs または Content Product に送信するキーが Content IDs にある場合は product に設定します。Content Product は製品を表します。
  • Content IDs に送信するキーがある場合は product_group に設定します。Content Product は製品グループを表します。
Predicted Lifetime Value コンバージョンイベントの予測されるライフタイムバリュー。例: 432.12。
Content IDs イベントに関連するコンテンツ ID（たとえば、AddToCart イベントのアイテムの製品 SKU）です。例: ['ABC123', 'XYZ789']。非配列のイベント属性が提供された場合、それは単一アイテムの配列に変換されます。Content Type が product の場合、このマッピングされた値は非配列のイベント属性または単一要素の配列である必要があります。
Value このイベントに関連する数値の値。これは通貨値または他のメトリックの値である場合があります。例: 142.54。
Content Category イベントに関連するコンテンツのカテゴリ。例: grocery。
Quantities 製品の数量。
Prices 製品の個別の価格。
IDs 製品の ID。
JSON Template Variables テンプレートにデータ入力としてテンプレート変数を提供します。詳細については、テンプレート変数ガイドを参照してください。ネストされたテンプレート変数は、ドット表記法で名前付けされます。例: items.name。ネストされたテンプレート変数は、通常、データレイヤーリスト属性から構築されます。
JSON Templates サポートされるフィールドに名前で参照される有効な JSON テンプレートを提供します。テンプレートは、ダブルカーリーブラケットで名前を挿入してフィールドに挿入されます。例: {{SomeTemplateName}}。詳細については、Webhook - Send Custom Request - Trimou Templating Engine ガイドを参照してください。
Tealium iQ Tag ID Facebook Pixel タグの ID。
Disable Automapping パラメータの自動マッピングを無効にします。
Automatic Deduplication Tealium iQ Tag ID を指定すると、コネクタは自動的に Tealium iQ が送信する Event ID 値を検索します。
Advertiser Tracking Enabled 必須です。このフィールドを使用して、iOS 14.5+ デバイスでの ATT 許可を指定します。無効にする場合は 0、有効にする場合は 1 に設定します。
Application Tracking Enabled 必須です。このパラメータはアプリレベルでの広告トラッキングを有効にします。SDK はアプリ開発者がアプリにオプトアウト設定を入れることができるようにする必要があります。このフィールドを使用して、ユーザーの選択を指定します。無効にする場合は 0、有効にする場合は 1 に設定します。
App Version 必須です。Android の場合は a2、iOS の場合は i2 にする必要があります。
OS Version 必須です。OS のバージョン。例: 13.4.1。
App Package Name アプリのパッケージ名。例: com.facebook.sdk.samples.hellofacebook。
Short Version 短いバージョン。例: 1.0。
Long Version 長いバージョン。例: 1.0 long。
Device Model Name デバイスのモデル名。例: iPhone5,1。
Locale ロケール。例: En_US。
Timezone Abbreviation タイムゾーンの略称。例: PDT。
Carrier キャリア。例: AT&T。
Screen Width 画面の幅。例: 320。
Screen Height 画面の高さ。例: 568。
Screen Density 画面の密度。例: 2。
CPU Cores CPU のコア数。例: 2。
Storage Size (GB) 外部ストレージのサイズ (GB)。例: 13。
Free Storage (GB) 外部ストレージの空き容量 (GB)。例: 8。
Device Timezone デバイスのタイムゾーン。例: USA/New York。
Campaign IDs Facebook からリンクをクリックしたユーザーがアウトバウンド URL（たとえば、ad_destination_url）またはディープリンク（App Aggregated Event Manager の場合）に追加される暗号化された文字列と非ユーザーメタデータ。
Install Referrer サードパーティのインストールリファラー。現在は Android のみで利用可能です。詳細については、Campaign Measurement を参照してください。
Installer Package Android SDK の内部で使用されます。
URL Schemes (Array) iOS および Android SDK の内部で使用されます。
Windows Attribution ID Windows 10 で使用されるアトリビューショントークン。
Anon ID インストール ID。このフィールドは、一意のアプリケーションインストールインスタンスを表します。
MADID モバイル広告主 ID。通常は Android デバイスの広告 ID または Apple デバイスの Advertising Identifier (IDFA) です。
Disable Automapping

コネクタはデフォルトで次のパラメータを自動マッピングします。

  • Browser ID (fbp)
  • User Agent (fbc)

自動マッピングを無効にするには、Disable Automapping を true に設定します。自動マッピングされたパラメータはベンダーに送信されません。

アクション - リードイベントを送信する

バッチ制限

このアクションは、高容量のデータ転送をサポートするためにバッチリクエストを使用します。詳細については、バッチアクションを参照してください。リクエストは、次のいずれかのしきい値が満たされるか、プロファイルが公開されるまでキューに入れられます。

  • 最大リクエスト数: 1000
  • 最も古いリクエストからの経過時間の最大値: 10 分
  • リクエストの最大サイズ: 1 MB

パラメータ

パラメータ 説明
Lead ID (必須) ダウンロードしたリードから取得した Facebook が生成した 15 または 16 桁の leadgen_id。この番号は、リードジェネレーションウェブフックの leadgen_id フィールドから取得され、user_data パラメータに含まれます。
Event Name (必須) CRM で使用するリードステージをキャプチャします。event_name パラメータは、CRM でのセールスファネルを通過するリードを示します。更新されるたびにすべてのステージを送信し、生のリードも含めて送信してください。
Event Time CRM によってリードステージの更新イベントが更新された Unix タイムスタンプ（秒単位）。タイムスタンプはリード生成時間より後である必要があります。このフィールドがマッピングされていない場合、現在のタイムスタンプで初期化されます。
Test Event Code サーバーイベントが Facebook に正しく受信されているかどうかを確認するために、イベントマネージャー の テストイベント 機能を使用します。ツールを見つけるには、イベントマネージャー > データソース > ピクセル > テストイベント に移動します。
Lead Event Source イベントが発生している CRM の名前。

イベントの検証

イベントを送信した後、Facebook がそれらを受信したかどうかを概要と詳細ビューで確認します。

  1. ビジネスマネージャー に移動し、ピクセル をクリックします。
  2. POST リクエストでの PIXEL_ID に対応するピクセルをクリックします。

概要タブ

概要 タブで、表示を すべて から サーバー に変更すると、正常に送信されたイベントの数が表示されます。

詳細タブ

詳細 タブでは、正常に送信されたイベントの詳細が表示されます。ドロップされたイベントは Server Events Received 列に表示されません。

イベントは、サーバーから送信してから 20 分以内に確認できます。

検証

検証には、Facebook の Payload Helper を使用して、データパラメータフィールドとともにペイロードを構造化する方法を確認します。詳細については、Facebook のトラブルシューティングのための サポート ページを参照してください。

Conversions API の詳細については、Facebook の Using the Conversions API を参照してください。
