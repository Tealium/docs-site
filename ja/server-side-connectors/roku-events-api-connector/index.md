## API情報

このコネクタは以下のベンダーAPIを使用します：

* API名：Roku API
* APIバージョン：v1
* APIエンドポイント：`https://events.ads.rokuapi.net`
* ドキュメント：[Roku API](https://help.ads.roku.com/en/articles/8880744-events-api)

## バッチ制限
このコネクタは、ベンダーへの大量データ転送をサポートするためにバッチリクエストを使用します。詳細については、バッチアクションを参照してください。リクエストは、次のいずれかの閾値に達するか、プロファイルが公開されるまでキューに入れられます：

* 最大リクエスト数：1,000
* 最古のリクエストからの最大時間：10分
* リクエストの最大サイズ：1 MB

## 構成

コネクタマーケットプレイスにアクセスして新しいコネクタを追加します。コネクタを追加する一般的な手順については、[コネクタについて](https://docs.tealium.com/about-connectors/)を参照してください。

コネクタを追加した後、以下の構成を構成します：
* **APIキー**  
  * APIキーを生成するには、Roku広告マネージャーのイベントタブ内の構成ページにアクセスしてください。
* **イベントグループID**  
  * イベントグループは、ウェブサイトやアプリ、チャンネルなどのオンラインプロパティを表すために使用されます。イベントグループIDの作成についての詳細は、[Roku: イベントグループ](https://help.ads.roku.com/en/articles/9451133-event-groups)を参照してください。

## アクション

| アクション名 | AudienceStream | EventStream |
| --- | :---: | :---: |
| イベント送信 | ✓ | ✓ |

アクション名を入力し、ドロップダウンメニューからアクションタイプを選択します。

次のセクションでは、各アクションのパラメータとオプションの構成方法について説明します。

### イベント送信

#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| イベント名 | イベントのタイプ。以下はRokuがサポートするイベントです：<ul><li>`AD_VIEW` - 広告表示。</li><li>`ADD_PAYMENT_INFO` - チェックアウトフローで支払い情報が追加されます。</li><li>`ADD_TO_CART` - 製品がショッピングカートに追加されます。</li><li>`ADD_TO_WISHLIST` - 製品がウィッシュリストに追加されます。</li><li>`APP_INSTALL` - モバイルアプリがインストールされます。</li><li>`COMPLETE_REGISTRATION` - 登録フォームが完了します。</li><li>`CONTACT` - お客様が電話、`SMS`、メール、チャット、その他の連絡方法を通じてビジネスに連絡を取ります。</li><li>`CUSTOMIZE_PRODUCT` - お客様が製品をカスタマイズします。</li><li>`DONATE` - あなたの組織や原因への寄付が行われます。</li><li>`DOWNLOAD` - ドキュメント、情報、またはサービスのダウンロード。</li><li>`FIND_LOCATION` - お客様がウェブサイトやアプリを通じて物理的な店舗の場所を検索し、訪問する意図を持っています。</li><li>`INITIATE_CHECKOUT` - お客様が完了前にチェックアウトフローに入ります。</li><li>`LEAD` - お客様がサインアップフォームを完了します。</li><li>`PAGE_VIEW` - ピクセルトラッキングページ訪問のデフォルト数。</li><li>`PURCHASE` - 購入またはチェックアウトフローが完了します。</li><li>`SCHEDULE` - お客様があなたのロケーションのいずれかを訪問するための予約をします。</li><li>`SEARCH` - 検索クエリが行われます。</li><li>`SIGN_UP` - サインアップイベント。</li><li>`START_TRIAL` - 無料トライアルが開始されます。</li><li>`SUBMIT_APPLICATION` - お客様が提供された製品、サービス、またはプログラムに応募します。</li><li>`SUBSCRIBE` - お客様が提供された製品やサービスの有料サブスクリプションを開始します。</li><li>`SUBSCRIPTION_CANCELLATION` - お客様が製品やサービスの有料サブスクリプションをキャンセルします。</li><li>`SUBSCRIPTION_RENEWAL` - お客様が製品やサービスの有料サブスクリプションを更新します。</li><li>`UNLOCK_ACHIEVEMENT` - お客様が成果や報酬を解除します。</li><li>`VIEW_CONTENT` - お客様が興味のあるページを訪れます。例えば、製品ページやランディングページです。このイベントはページ上のアクションについて具体的ではありません。このイベントは、特定のページや映画やテレビ番組などのコンテンツのアプリ内`VIEW`をキャプチャするためにも使用できます。</li></ul> |

#### イベントパラメータ

| **パラメータ** | **説明** |
| --- | --- |
| イベント時間 | イベントが発生したときのUNIXタイムスタンプ（エポック秒）。 |
| イベントソース | イベントのソース。ソースには`website`、`phone_call`、`physical_store`などが含まれます。ソースの完全なリストについては、[Roku: Events API](https://help.ads.roku.com/en/articles/8880744-events-api)を参照してください。 |
| イベントID | Rokuがサーバーとブラウザーの両方から送信された同じイベントを重複排除するために使用するIDです。IDがブラウザーとサーバーから送信されたイベントのペアに対して一意であることを確認してください。 |
| イベントソースURL | イベントが発生したページのURL。 |
| リファラーURL | 対象URLに着陸する直前にユーザーがいたページのURL。 |
| イベントタイプ | マッピングされていない場合、コネクタはデフォルトで`conversion`を送信します。 |
| オプトアウト | `true`の値は、限定データ使用フラグをトリガーします。詳細については、[Roku: 限定データ使用](https://help.ads.roku.com/en/articles/8766138-limited-data-use)を参照してください。 |

#### ユーザーデータ

| **パラメータ** | **説明** |
| --- | --- |
| Email（既にSHA256ハッシュ化されている） | すでに空白トリム、小文字化、SHA256ハッシュ化されたメールアドレスを提供してください。 |
| Email（SHA256ハッシュを適用） | プレーンテキストのメールアドレスを提供し、コネクタがこの値を空白トリム、小文字化し、SHA256ハッシュを使用してハッシュ化します。 |
| 電話番号（既にSHA256ハッシュ化されている） | 数字のみを含む電話番号を提供してください。ハッシュ化する前に先頭のゼロを削除し、空白トリムおよびSHA256ハッシュ化してください。 |
| 電話番号（SHA256ハッシュを適用） | プレーンテキストの電話番号を提供し、コネクタがすべての非数字文字と先頭のゼロを削除し、空白トリムしてSHA256ハッシュを使用してハッシュ化します。 |
| 名（既にSHA256ハッシュ化されている） | すでに空白トリム、小文字化、SHA256ハッシュ化された名を提供してください。 |
| 名（SHA256ハッシュを適用） | プレーンテキストの名を提供し、コネクタがこの値を空白トリム、小文字化し、SHA256ハッシュを使用してハッシュ化します。 |
| 姓（既にSHA256ハッシュ化されている） | すでに空白トリム、小文字化、SHA256ハッシュ化された姓を提供してください。 |
| 姓（SHA256ハッシュを適用） | プレーンテキストの姓を提供し、コネクタがこの値を空白トリム、小文字化し、SHA256ハッシュを使用してハッシュ化します。 |
| 生年月日（既にSHA256ハッシュ化されている） | yyyy-mm-dd形式の生年月日をすでに空白トリム、小文字化、SHA256ハッシュ化された状態で提供してください。 |
| 生年月日（SHA256ハッシュを適用） | yyyy-mm-dd形式のプレーンテキストの生年月日を提供し、コネクタがこの値を空白トリム、小文字化し、SHA256ハッシュを使用してハッシュ化します。 |

#### カスタムデータ

| **パラメータ** | **説明** |
| --- | --- |
| コンテンツカテゴリ | リクエストのページまたは製品のカテゴリ。 |
| コンテンツID | イベントに関連する製品ID。 |
| コンテンツ名 | ページまたは製品の名前。 |
| コンテンツタイプ | リクエストのコンテンツIDのタイプに応じて、`product`または`product_group`。 |
| 製品ID | 製品IDの配列。 |
| 製品数量 | 製品数量の配列。 |
| 製品価格 | 製品価格の配列。 |
| 通貨コード | 3文字の通貨コード。サポートされている通貨のリストについては、[Roku: Events API](https://help.ads.roku.com/en/articles/8880744-events-api)を参照してください。 |
| アイテム数 | チェックアウトが開始されたときのアイテム数。 |
| 予測LTV | 広告主によって定義された購読者の予測生涯価値で、正確な値として表現されます。 |
| 検索文字列 | 検索イベントのためにユーザーが入力した文字列。 |
| ステータス | 注文またはサービスのステータス。 |
| 値 | 購入されているすべての製品の合計値。 |
| 注文ID | トランザクションの注文ID。 |
| 購入タイプ | 受け入れられる購入タイプの値は`first_time_purchase`と`repeat_purchase`です。 |
| 配送カテゴリ | 購入イベントの配送タイプ。受け入れられる値は`in_store`、`curbside`、`home_delivery`です。 |


#### 自動重複排除

| **パラメータ** | **説明** |
| --- | --- |
| Tealium iQ タグ ID | Tealium iQ タグ ID が提供されると、コネクタは自動的に Tealium iQ から送信されたイベント ID 値を検索し、この値を Roku に送信します。 |
