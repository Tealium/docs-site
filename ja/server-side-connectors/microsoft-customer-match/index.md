## 必要条件

* Microsoft Ads アカウント
* Microsoft Ads テナント内でスーパーアドミンまたは書き込みアクセス権を持つ Microsoft ユーザー。
    * 認証ユーザーは、個人の Microsoft アカウント（MSA）である必要があります。
    * 仕事用のメールアドレスを使用することもできますが、SSOを介してアクセスするのではなく、個人アカウントとして登録する必要があります。組織が Microsoft ログインに SSO または MFA を使用している場合でも、個人アカウントとして登録することにより、仕事用のメールを使用することができます。登録時には、**個人アカウントを使用する** を選択してください。
* お客様のアカウント地域でカスタマーマッチが有効で承認されています。詳細については、[Microsoft: カスタマーマッチ - 独自のデータを使用して顧客を見つける](https://help.ads.microsoft.com/#apex/ads/en/56921/1)を参照してください。

### Microsoft サービス規約の受諾

Tealium に認証するために使用する Microsoft ユーザーがビジネスアカウントに新しい場合、まず以下の手順を実行する必要があります：

1. そのユーザーで Microsoft Ads にサインインします。
1. Microsoft カスタマーマッチの利用規約を受け入れるために、一度手動でオーディエンスを作成します。

これらの条件が受け入れられるまで、APIは Tealium からのオーディエンス作成リクエストを拒否します。

## API 情報

このコネクタは以下のベンダー API を使用します：

* API 名: Bulk API
* API バージョン: v13
* API エンドポイント: `https://bulk.api.bingads.microsoft.com/Api/Advertiser/CampaignManagement/V13/BulkService.svc`
* ドキュメント: [Microsoft: Bulk API](https://learn.microsoft.com/en-us/advertising/bulk-service/bulk-service-reference?view=bingads-13)

## バッチ制限

このコネクタは、ベンダーへの大量データ転送をサポートするためにバッチリクエストを使用します。詳細については、[Batched Actions]()を参照してください。リクエストは、次のいずれかのしきい値に達するか、プロファイルが公開されるまでキューに入れられます：

* 最大リクエスト数: 100000
* 最古のリクエストからの最大時間: 60分
* リクエストの最大サイズ: 100 MB

## 構成

コネクタマーケットプレイスにアクセスして、新しいコネクタを追加します。コネクタの追加方法についての一般的な説明については、[About Connectors]()を参照してください。

コネクタを追加した後、次の構成を構成します：

* **アカウント ID**: (必須) リクエスト内のエンティティを所有または関連付けている広告アカウントの識別子です。広告主が Microsoft Ads UI にログインするときの URL（例: `AID=`）で見つけることができます。
* **顧客 ID**: (必須) ユーザーがアクセスまたは操作しているマネージャーアカウント（顧客）の識別子です。広告主が Microsoft Ads UI にログインするときの URL（例: `CID=`）で見つけることができます。

**アカウント ID** と **顧客 ID** を構成した後、**接続を確立する** をクリックして Microsoft のサーバーに接続します。

### カスタマーマッチリストの作成

このオプションを使用して、広告対象とする加入者を追加するカスタマーマッチリストを作成します。リスト作成プロセスが完了したかどうかを判断するために、結果の **リクエスト ID** 値を使用することができます。

**作成** ボタンをクリックした後、確認メッセージを受け取るまで待ちます。顧客リストが正常にアップロードされた場合、インターフェースはアップロードの **リクエスト ID** を表示します。**リクエスト ID** をコピーして、アップロード状況を確認するために使用することができます。

| フィールド |  説明 |
| --- |--- |
| 名前 |(必須) 顧客リストの名前。このフィールドには最大128文字を含めることができます。 |
| 説明 | (任意) 顧客リストの説明。この顧客リストでターゲットにしているオーディエンスを覚えておくために説明を使用します。このフィールドには最大1,024文字を含めることができます。 |
| 親 ID | (任意) 顧客リストを含む顧客の Microsoft Advertising 識別子。 |
| クライアント ID | (任意) バルクアップロードファイルのレコードを結果ファイルのレコードと関連付けるために使用されます。このフィールドは最大100文字の有効な文字列であることができます。 |
| メンバーシップ期間 | (任意) 顧客リストを作成するとき、Microsoft Advertising が顧客リストの定義に一致するアクションを検索してリストに人々を追加するために遡る期間を指定することができます。&lt;ul&gt;&lt;li&gt;最小期間は `1` 日で、許可される最大期間は `180` 日です。&lt;/li&gt;&lt;li&gt;メンバーシップ期間が `-1` の場合、有効期限はありません。&lt;/li&gt;&lt;li&gt;メンバーシップ期間が null の場合、デフォルトの期間（30日）が使用されます。&lt;/li&gt;&lt;/ul&gt; |

### アップロード状況の確認

このオプションを使用して、リスト作成とアップロード状況を確認します。これにより、顧客が適切なリストに登録されていることを確認できます。

アップロード状況が `Completed` または `CompletedWithErrors` の場合、結果ファイルの URL が表示されます。この URL を使用して結果ファイルをダウンロードします。URLは、アップロード状況を表示する Check the Status 操作から5分後に期限切れになります。この期間内にダウンロードを開始しない場合、新しい URL を取得するために再度状況を確認する必要があります。

| フィールド | 説明 |
| --- |--- |
| リクエスト ID | アップロードリクエストの識別子。 |

## 使用例

#### Microsoft プロパティ全体での高価値リマーケティング

Bing、MSN、Outlook、Edge で高価値ユーザーセグメントをリアルタイムで同期し、精度を向上させてアクティブ化します。ファーストパーティ識別子による行動リマーケティングは、マッチ率を向上させ、ブラウザベースのトラッキングへの依存を減らし、ROASを増加させます。

例：

* 見た商品をカートに入れたが購入に至らなかったユーザーを再度関与させる。
* 以前にブランドと交流したユーザーにプロモーションをエンリッチメントする。
* CRMオーディエンスのリーチを Microsoft のネイティブ広告環境に拡大する。

#### 予算効率のための抑制リスト

見込み客キャンペーンや広範囲マッチキャンペーンから変換済みユーザーを除外します。広告主は無駄な予算を削減し、変換の可能性が高い新規ユーザーに支出を向けることで、入札効率を研ぎ澄ませます。

例：

* 最近の購入者を獲得キャンペーンから除外する。
* カスタマーサービスや返金リクエストを行った顧客を抑制する。
* 離脱したり退会したユーザーを除外する。

#### ライフサイクルと CRM からの広告アクティベーション

ロイヤリティ階層、サブスクリプションステータス、顧客ライフサイクルステージを直接 CDP から Microsoft Ads に同期します。これにより、生涯価値と顧客ステージに基づいてユーザーを異なる方法でターゲットにする強力な入札戦略が解放されます。

例：

* VIP専用のアップセルキャンペーンのためのロイヤリティ階層。
* サブスクリプション更新のリマインダー。
* 既知の店舗顧客をデジタルリマーケティングキャンペーンにプッシュする。

#### オムニチャネルのパーソナライゼーション

ハッシュ化されたファーストパーティ識別子を同期することで、オフラインとオンラインの相互作用を橋渡しします。

## アクション

| アクション名 | AudienceStream | EventStream |
| --- | :---: | :---: |
| ユーザーをオーディエンスに追加 | ✓ | ✗ |
| ユーザーをオーディエンスから削除 | ✓ | ✗ |

アクションの名前を入力し、**アクションタイプ**を選択します。

次のセクションでは、各アクションのパラメータとオプションの構成方法について説明します。

### ユーザーをオーディエンスに追加

#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| カスタマーマッチリスト ID | カスタマーマッチリスト ID を入力します。&lt;br&gt;カスタマーマッチリスト ID を見つけるには: [Microsoft Advertising UI](https://ui.ads.microsoft.com/) にログインし、**ツール** を選択してから **オーディエンス** タブをクリックします。&lt;br&gt;注意: 顧客リストデータをアップロードする前に、Microsoft Advertising UI で利用規約を受け入れる必要があります。&lt;br&gt; |
| Email Address (already SHA256 hashed) | すでに SHA256 ハッシュ化されたメールアドレスを提供します。 |
| Email Address (apply SHA256 hash) | プレーンテキストのメールアドレスを提供し、コネクタはこの値を空白トリム、小文字化し、SHA256 ハッシュを使用してハッシュ化します。 |

### ユーザーをオーディエンスから削除


#### パラメータ

| **パラメータ** | **説明** |
| --- | --- |
| 顧客マッチリストID | 顧客マッチリストIDを入力してください。&lt;br&gt;顧客マッチリストIDを見つけるには、[Microsoft Advertising UI](https://ui.ads.microsoft.com/)にログインし、**ツール**を選択してから**オーディエンス**タブをクリックしてください。&lt;br&gt;注意: 顧客リストデータをアップロードする前に、Microsoft Advertising UIで利用規約を受け入れる必要があります。&lt;br&gt; |
| メールアドレス（既にSHA256ハッシュ化済み） | 既にSHA256ハッシュ化されたメールアドレスを提供してください。 |
| メールアドレス（SHA256ハッシュを適用） | 平文のメールアドレスを提供し、コネクタがこの値を空白トリム、小文字化し、SHA256ハッシュを使用してハッシュ化します。 |
