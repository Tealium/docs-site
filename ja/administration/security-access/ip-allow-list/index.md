## 仕組み

Tealiumプラットフォームは、以下の操作中に外部システムと通信します：

* コネクタアクションでデータを送信する。
* [関数]()で外部サービスへのリクエストを行う。
* [ファイルインポートデータソース]()の処理（AWS S3を除く）。

外部システムが承認されたIPアドレスのリストにのみアクセスを制限している場合、以下のすべてのIPアドレスを許可リストに追加する必要があります：

OAuthコール、サーバーサイドインターフェースからのテスト接続、Snowflake Streamingコネクタ、SFTP接続、およびファイルインポートデータソースのテスト接続（AWS S3を除く）では、`us-west-1`地域のIPアドレスからデータが送信されます。これらの機能を有効にするためには、`us-west-1`地域のIPアドレスを許可リストに含めてください。&lt;br&gt;&lt;br&gt;さらに、**サーバーサイド構成** &gt; **地域**の下で見つかる特定のサーバーサイドプロファイルに関連する地域のIPアドレスを含めてください。&lt;br&gt;&lt;br&gt;OAuthコールとテスト接続中にはログイン情報のみが送信され、顧客データは送信されません。

## プライベートクラウド環境

プライベートクラウド環境を使用している場合は、サポートチームに連絡してアカウントに固有のIPアドレスを取得し、それを許可リストに追加してください。

`PC-HMT`環境を使用している場合は、52.36.206.227も許可リストに追加してください。

## AWS S3

Tealium [ファイルインポート]()は、TealiumのS3バケットまたはお客様のS3バケットからファイルを取得できます。お客様の会社のS3バケット（TealiumのS3バケットではない）からファイルを取得する場合、AWSバケットポリシーの許可リストには以下のVPCe ID（VPCエンドポイントID）と上記のIPアドレスが含まれている必要があります。

AWSバケットポリシーについての詳細は、AWSドキュメントの以下の記事を参照してください：

* [Amazon S3コンソールを使用してバケットポリシーを追加する](https://docs.aws.amazon.com/AmazonS3/latest/userguide/add-bucket-policy.html)
* [バケットポリシーでVPCエンドポイントからのアクセスを制御する](https://docs.aws.amazon.com/AmazonS3/latest/userguide/example-bucket-policies-vpc-endpoint.html)
