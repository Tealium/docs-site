## API情報

このコネクタは以下のベンダーAPIを使用します：

* API名：Spotify API
* APIバージョン：v1.0
* APIエンドポイント：`https://api-partner.spotify.com/ads/v1`

| コンバージョンイベントの送信|  ✓ | ✓| 

## 仕組み

Spotify Conversionsコネクタは、コンバージョンイベントのトラッキングをキャプチャし、このデータをSpotifyに送信し、ウェブキャンペーンの広告費を追跡し、測定するのに役立ちます。Conversion APIは、ピクセルを必要とせずにコンバージョンデータを共有できる直接的なサーバー間の統合です。

ウェブキャンペーンのコンバージョントラッキングを有効にするときに、以下の情報を含める必要があるかもしれません：

* **ページビュー**：ユーザーが広告主のサイトのランディングページを訪れます。
* **登録完了**：ユーザーが広告主のサービスの登録を完了します。
* **カートに追加**：ユーザーが広告主のサイトの製品またはサービスをカートに追加します。
* **チェックアウト**：ユーザーが広告主のサイトの製品またはサービスのチェックアウトを開始します。
* **購入**：ユーザーが広告主のサイトの製品またはサービスの購入を完了します。
* **検索**：ユーザーが広告主のサイトから検索を行います。
* **リード**：ユーザーが広告主のサイトからリードステータスに入ります。
* **購読**：ユーザーが広告主のサービス、ニュースレター、またはメールコミュニケーションにサインアップします。

Spotifyは、初期のアルファ版では**購入**イベントのみを使用することを推奨しています。

Spotify Conversion APIは、マッチング識別子としてメールアドレスのみをサポートしています。TealiumはSHA256ハッシングを適用する機能を提供しています。また、APIで使用するためにすでにハッシュ化された値をマッピングすることもできます。

## 構成の構成

**コネクタマーケットプレイス**に移動し、新しいコネクタを追加します。コネクタの追加方法については、[コネクタ概要](https://docs.tealium.com/about-connectors/)の記事を参照してください。

コネクタを追加したら、以下の構成を構成します：

* **コンバージョントラッカーID**：必須。
  * コンバージョントラッカーIDは、Spotify Conversions APIへのすべてのイベントリクエストに必要です。これは、イベントを特定の広告主のキャンペーンに帰属させるための一意のIDです。
  * **トラッカーIDの作成**をクリックしてIDを作成します。
    * **広告主名**：必須。広告主の名前。
    * **説明**：必須。トラッカーIDに関連するイベントの種類の説明。

パートナーがConversions APIを使用し始める前に、Spotifyに連絡してConversion Tracker IDを広告主/広告アカウントにリンクする必要があります。

コネクタの構成が完了したら、**完了**をクリックします。

## アクション構成 — パラメータとオプション

**続行**をクリックしてコネクタアクションを構成します。アクションの名前を入力し、ドロップダウンメニューからアクションタイプを選択します。

次のセクションでは、各アクションのパラメータとオプションの構成方法について説明します。

### アクション — コンバージョンイベントの送信


<blockquote>
コネクタのアルファリリース中、Spotifyは**購入**イベントのみを使用することを推奨しています。
</blockquote>


#### パラメータ

| パラメータ| 説明|
|---------|--------|
| イベント名 | コンバージョンイベントを説明します。 |
| イベントID | (必須) イベントID。 |
| イベント時間 | RFC 3339形式のタイムスタンプ。実際のイベントが発生したときを示します。提供されるタイムスタンプは過去の時間を表すべきです。例えば、`2022-12-25T15:30:50Z`。タイムスタンプが提供されていない場合、コネクタは現在のタイムスタンプを使用します。 |
| イベントソースURL | イベントが発生したブラウザのURL。URLは`http://`または`https://`で始まり、検証済みのドメインと一致するべきです。 |
| 金額 | 金銭的な価値。**購入**イベントに必須。 |
| 通貨 | ISO - 4217に準拠した有効な通貨コード。**購入**イベントに必須。|
| コンテンツ名 | 製品名またはページ名。 |
| コンテンツカテゴリ | イベントに関連するコンテンツのカテゴリ。コンテンツカテゴリは[Googleの製品分類](https://www.google.com/basepages/producttype/taxonomy.en-US.txt)に対応している必要があります。 |
|アクションソース | イベントの一部。以下のいずれかでなければなりません：<ul><li>`EMAIL` - コンバージョンがメールで行われました。</li><li>`WEBSITE` - コンバージョンがウェブサイトで行われました。</li><li>`MOBILE` - コンバージョンがモバイルアプリで行われました。</li><li>`PHONE_CALL` - コンバージョンが電話で行われました。</li><li>`CHAT` - コンバージョンがメッセージングアプリ、SMS、またはオンラインメッセージング機能を介して行われました。</li><li>`IN_PERSON` - コンバージョンがあなたの実店舗で直接行われました。</li><li>`PROGRAMMATIC` - コンバージョンがプログラム的に行われました。例えば、定期的な取引が月に一度発生するサブスクリプションサービス。</li></ul> |
| メールアドレス（すでにSHA256でハッシュ化されている） | すでにSHA256でハッシュ化されているメールアドレス（`@`を含む）を提供します。 |
| メールアドレス（SHA256ハッシュを適用） |  プレーンテキストのメールアドレスを提供し、コネクタはこの値を空白トリム、小文字化し、SHA256ハッシュを使用してハッシュ化します。ユーザーが複数のメールアドレスを持っている場合は、メールアドレスを配列としてリストします。 |
| IPアドレス |  イベントを送信するIPアドレス。 |
| 電話番号 |  ユーザーの電話番号。 |
| ターゲティングのオプトアウト |  このイベントをリターゲティングに使用しないことを示すフラグ。`true`に構成されている場合、イベントは帰属のみに使用します。 |
