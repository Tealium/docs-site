コネクタ

Tealiumで、イベントおよびカスタマーデータのリアルタイムアクティベーションを行うサーバー側APIを統合する方法について説明します。

データの収集やアクティベーションを行うAPIを提供している場合や、クライアント側のJavaScriptタグと同等のサーバー側を備えている場合は、Tealium Customer Data Hubでのアウトバウンド用API統合の機能をこのガイドでご確認ください。

コネクタの仕組み

_コネクタ_とは、データを送信するために使用する、Tealiumとベンダーの間の統合機能です。コネクタは、ベンダーのAPIに含まれるメソッドを表す_アクション_を提供します。Tealiumユーザーは、コネクタを使用すると、簡単なワークフローによりデータをリアルタイムでアクティベートし、ベンダーに直接送ることができます。

イベントおよびカスタマーデータ

Web解析などの_イベントデータ_を送信するコネクタは、Tealium EventStreamで使用されます。

メールサブスクリプションステータスなどの_カスタマーデータ_を送信するコネクタは、Tealium AudienceStreamで使用されます。

構成

コネクタは、以下のパーツで管理されます。

  • コネクタ コネクタは、ベンダーへの接続を確立します。これには通常、APIキーまたはアカウントIDが使用されます。認証資格情報が使用されることもあります。
  • アクション アクションは、特定のベンダーAPIメソッドを実装します。各アクションには、データのソース（イベントまたはカスタマー）、データマッピングのほか、必要なベンダー構成の指定が求められます。単一のコネクタに、複数のアクションが関連付けられます。
  • データマッピング データマッピングでは、イベントまたはカスタマーからのどの動的値をベンダーに送信するかを決定します。

アクションの例

以下の例は、さまざまなベンダーカテゴリの一般的なコネクタアクションを示します。

  • アナリティクス ページビュー、注文購入、ユーザー登録などのアナリティクスイベントを送信します。
  • ディスプレイ広告 広告セグメントに対して、追加または削除を行います。
  • メール リストに対する追加または削除、メールのトリガー、サブスクリプション顧客の関連データの追加を行います。
  • CRM リードまたは連絡先の作成、リードの変換、レコードのupsert（挿入または更新）を行います。
  • モバイル SMS通知またはプッシュ通知の送信、アプリインストールのトラッキング、アプリ内イベントの送信を行います。

コネクタの詳細については、こちらを参照してください。

例: Facebook Audiencesコネクタ

Facebook 広告ユーザー向けの Facebook Audiences コネクタ を使用すると、Tealium ユーザーは、1 つ以上のデータソースから収集されたユーザー行動に基づき、カスタムオーディエンスをリアルタイムで管理できます。

このコネクタは、以下のアクションを提供します。

  • ユーザーをカスタムオーディエンスに追加する
  • ユーザーをカスタムオーディエンスから削除する
  • ユーザーを全てのカスタムオーディエンスからオプトアウトする

各アクションについて、Facebook Custom Audiences用のフィールド（必須またはオプション）のリストがあり、データマッピングに使用できます。

認証

ベンダーAPIが認証を必要とする場合に備え、Tealiumでは2つのソリューションを用意しています。

  • Tealiumが提供する認証情報 ユーザーに代わって動作するサーバー側アプリをTealiumが作成および管理します。Tealiumが、ベンダーとの認証関係を維持します。
  • 顧客が提供する認証情報 Tealiumユーザーが、接続の認証に必要なAPIアクセス権をベンダーから獲得します。

いずれのソリューションについても、Tealiumは以下の認証タイプをサポートします。

  • 基本認証（ユーザー名とパスワード）
  • OAuth 2.0
  • JSON Web Token（JWT）

Tealiumは常に全世界のサーバー数十台に分散しているため、APIに同時トークン数の制限がある場合は問題が発生する可能性があります。

ベストプラクティス

アセットの作成

アセットの作成は、TealiumユーザーがTealium Customer Data Hub（CDH）から離れずに、コネクタと共に使用するリソースを作成できる機能です。たとえば、Facebook Audiencesコネクタでは、ユーザーがFacebookの新しいカスタムオーディエンスを作成し、そのオーディエンスを新しいコネクタアクションのターゲットとして選択できます。

このオプション機能は、コネクタアクションに関連したリソースを作成するためのAPI メソッドをサポートするすべてのベンダーに提供されます。作成メソッドの例を以下に示します。

  • ディスプレイ広告 広告主セグメントまたはオーディエンスを作成します。
  • メール サブスクリプション顧客リストまたは連絡先リストを作成します。

動的ベンダーオプション

ユーザーによるコネクタアクションの構成は、Tealiumによって支援されます。ベンダーのプラットフォームから直接取得したデータを使用して、コネクタオプションが動的に構成されます。

たとえば、Facebook Audiencesコネクタの構成画面では、TealiumがAPI呼び出しを行い、ユーザーのFacebook広告アカウントから、カスタムオーディエンスのリストを取得します。これによりユーザーは、見慣れた名前と便利なメタデータを使って、外部リソースを参照できるため、正しいリソースを確実に選択できます。

これらの動的構成オプションは、場合によっては順次呼び出しとして実行する必要があります。CRMコネクタでは、たとえばCRMオブジェクトのプライマリキーフィールドを取得するには、まずCRMオブジェクトのリストを取得する必要があります。

デフォルトのUpsert

ベンダープラットフォームでレコードを作成または更新するAPIメソッドについて、Tealiumでは、単一リクエスト内のupsertに対するネイティブサポートを使用するようお勧めしています。これにより、Tealiumから発信するAPI呼び出しの数を最小限に抑えることができます。たとえば、レコードの作成または更新前に、APIユーザーはまず、レコードが存在するかどうかを確認する必要があります。レコードが存在しない場合は、次のAPI呼び出しでレコードが作成されます。レコードが存在する場合は、次のAPI呼び出しで、既存のレコードが更新されます。

バッチとレートの制限

Tealiumの目標は、リアルタイムのパフォーマンスを提供することですが、ベンダーによっては、リソースコストの管理目的でデータを小さなバッチで受信する方が望ましいとされる場合もあります。大量のデータを処理する場合は特に、このことが当てはまります。

Tealiumでは、ベンダーのレートで個々のアカウントでのAPIの使用を制限するようにお勧めしています。これにより、ボリュームの高い小さなグループによってシステム全体が影響を受ける状況を回避できます。

Tealiumのコネクタは、使用されているAPIエンドポイントに求められるしきい値に適合します。以下はレート制限の例です。

  • リクエストあたり50イベント、1秒あたり最大10リクエスト（500リクエスト/秒）
  • 1分あたり10,000イベント、リクエストあたり最大10KB（最大167リクエスト/秒）

制限事項

現在、Tealiumのコネクタでは以下がサポートされていません。

  • ファイル転送
  • 5000ミリ秒以内に接続しないベンダーAPI
  • リクエストに対して5000ミリ秒（バッチリクエストの場合は10秒）以内に応答しないベンダーAPI
