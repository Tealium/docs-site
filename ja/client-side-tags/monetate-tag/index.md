## タグの構成

まず、タグマーケットプレイスにアクセスし、プロファイルにMonetateタグを追加します（[タグの追加方法](https://docs.tealium.com/manage-tags/)を参照）。

タグを追加した後、以下の構成を構成します：

* **ページ上でのライブラリの読み込み**: Tealiumを介さずにページ上でMonetateを読み込む場合はTrueに構成
* **Monetate HTTPS URL**: Monetateから提供されたトラッキングコードスニペット内のURL
* **すべてのページでaddCartRowsを使用**: カートデータを自動的にすべてのページタイプで送信することを許可

## ロードルール

[ロードルール](https://docs.tealium.com/about-load-rules/)は、サイト上でこのタグのインスタンスをいつ、どこでロードするかを決定します。

推奨されるロードルール：**すべてのページ**

## データマッピング

マッピングは、[データレイヤー変数](https://docs.tealium.com/data-layer-variables/)からベンダータグの対応する宛先変数にデータを送信するプロセスです。変数をタグの宛先にマッピングする方法については、[データマッピング](https://docs.tealium.com/ja/iq-tag-management/data-mappings/manage/)を参照してください。

Monetateタグの宛先変数は、その**データマッピング**タブに組み込まれています。利用可能なカテゴリは以下の通りです：

### 標準

| **宛先名**                | **説明**                                                                                                                                                                                             |
|:------------------------------------|:------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Monetate HTTPS URL (`base_url`)     | Monetateから提供されたトラッキングコードスニペット内のURL                                                                                                                                           |
| ページタイプ (`page_type`)             | ユーザーが閲覧しているページを識別するために使用。`setPageType`メソッドで使用される。デフォルトのページタイプを上書きするために使用                                                                      |
| パンくずリスト (`breadcrumbs`)         | ユーザーのナビゲーションのパンくずリストを表す順序付き配列。`addBreadcrumbs`メソッドで使用される。すべてのページタイプで送信される                                                |
| カスタム変数 (`variable.custom`) | `setCustomVariables`を使用してカスタム変数を構成するために使用。マッピング名の`custom`を変数の名前に置き換えて、その変数にマップします。すべてのページタイプで送信される |

### E-コマース

MonetateタグはE-コマース対応であり、デフォルトのE-コマース拡張マッピングを自動的に使用します。以下の場合を除き、このカテゴリでの手動マッピングは通常必要ありません：

* 拡張マッピングを上書きしたい場合
* 拡張機能で提供されていないE-コマース変数が必要な場合

| **宛先名**                    | **説明**                                                                                                                                                                                                                                                                                  |
|:----------------------------------------|:-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| カスタム注文データ (`orderData.custom`)  | 現在のページまたはトランザクションの注文データのカスタム項目。マッピング名の`custom`を変数の名前の形式`_CUSTOM`に置き換えて、その変数にマップします。すべてのイベントで送信され、特定のイベントのプロパティを構成するためにイベントパラメータを使用します。 |
| 注文割引 (`order_discount`)       | 現在のページまたはトランザクションの注文割引。                                                                                                                                                                                                                                          |
| 注文ID (`order_id`)                   | 現在のトランザクションの一意の注文ID。デフォルトのE-コマース値`_corder`を上書きするために使用します                                                                                                                                                                                     |
| 小計 (`order_subtotal`)            | 現在のページまたはトランザクションの注文小計。デフォルトのE-コマース値`_csubtotal`を上書きするために使用します                                                                                                                                                                           |
| 配送料 (`order_shipping`)      | 現在のページまたはトランザクションの配送料。デフォルトのE-コマース値`_cship`を上書きするために使用します                                                                                                                                                                               |
| 税額 (`order_tax`)                | 現在のページまたはトランザクションの税額。デフォルトのE-コマース値`_ctax`を上書きするために使用します                                                                                                                                                                                     |
| 通貨 (`order_currency`)             | 現在のページまたはトランザクションの通貨。デフォルトのE-コマース値`_ccurrency`を上書きするために使用します                                                                                                                                                                           |
| 商品IDリスト (`product_id`)      | 現在のページまたはトランザクションの商品IDリスト。デフォルトのE-コマース値`_cprod`を上書きするために使用します                                                                                                                                                                            |
| SKUリスト (`product_sku`)            | 現在のページまたはトランザクションの商品SKUリスト。デフォルトのE-コマース値`_csku`を上書きするために使用します                                                                                                                                                                            |
| カテゴリリスト (`product_category`) | 商品カテゴリのリスト。`addCategories`メソッドで使用される。デフォルトのE-コマース値`_ccat`を上書きするために使用します                                                                                                                                                   |
| 数量リスト (`product_quantity`) | 現在のページまたはトランザクションの商品数量リスト。デフォルトのE-コマース値`_cquan`を上書きするために使用します                                                                                                                                                                     |
| 価格リスト (`product_unit_price`)   | 現在のページまたはトランザクションの商品価格リスト。デフォルトのE-コマース値`_cprice`を上書きするために使用します                                                                                                                                                                        |
### 非金銭的な変換

|:--------------------------------------------------|:-----------------------------------------------------------------------------------------------------|
| 変換IDリスト（conversion_id）                     | 非金銭的な変換に関連付けられたitemIdプロパティを構成するために `addConversionRows` と共に使用されます。 |

### ページタイプ

これらの目的地にマッピングして、ページ上で特定のページタイプイベントをトリガーします。イベントをトリガーするには：

1. ドロップダウンリストからページタイプを選択します。カスタムページイベントもドロップダウンリストから送信できます。カスタムページイベントの場合、それを識別する名前を入力します。
1. **トリガー**フィールドにマッピングされる変数の値を入力します。
1. さらにイベントをマッピングするには、**+**ボタンをクリックして手順＃1と＃2を繰り返します。
1. **適用**をクリックします。

提供された値がデータレイヤーで見つかったときにページタイプイベントがトリガーされます。

| **目的地名**                                      | **説明**                                                                                                                                                           |
|:--------------------------------------------------|:------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| ホームページビュー (`home`)                       | 通常、サイト訪問がウェブサイトをナビゲートする際に最初に着陸するページです。                                                                                   |
| ログインページビュー (`login`)                    | サイト訪問が **ログイン**、**マイアカウント**、または **サインアップ** をクリックすると表示されるページです。チェックアウトプロセス中以外の場所でこれらがクリックされた場合です。 |
| カテゴリーページビュー (`category`)               | このページは通常、サイト訪問を特定のインデックスページに誘導するサムネイル画像またはリンクを提供します。                                                       |
| チェックアウトページビュー (`checkout`)           | 注文を行う直前にサイト訪問に提示されるページです。                                                                                                               |
| 製品インデックスページビュー (`product_index`)   | これらのページは通常、製品のサムネイルビュー、説明、および価格を表示します。                                                                                       |
| 検索 (`search`)                                   | このページは通常、サイト訪問の特定のリクエストに関連するサムネイル画像またはリンクを提供します。                                                               |
| 製品詳細ページビュー (`product_detail`)           | インデックスページから製品画像またはリンクをクリックしてナビゲートすると表示されるページです。製品情報を詳細に表示し、カートにアイテムを追加することができます。 |
| カート (`cart`)                                   | サイト訪問が購入を準備しているすべての製品を表示するページです。                                                                                                |
| 購入 (`purchase`)                                 | 注文を完了した後に顧客に表示されるページです。                                                                                                                     |
| カスタムページイベント                             | サイト上で何か重要なことが起こったことを示す方法を提供します。イベント名はイベントキーとしてMonetateに送信されます                                                |

### 旅行イベント

これらの目的地にマッピングして、ページ上で特定のイベントをトリガーします。イベントをトリガーするには：

1. ドロップダウンリストからイベントを選択します。事前定義されたリストから選択するか、'カスタム'イベントを作成することができます。'カスタム'イベントの場合、それを識別する名前を入力してください。カスタムパラメーターやイベントを構成する前に、BloomReachの担当者に相談してください。
1. **トリガー**フィールドにマッピングされる変数の値を入力します。
1. さらにイベントをマッピングするには、**+**ボタンをクリックして手順＃1と＃2を繰り返します。
1. **適用**をクリックします。

提供された値がデータレイヤーで見つかったときにイベントがトリガーされます。

| **目的地名**                                | **説明**                                                                                      |
|:--------------------------------------------|:---------------------------------------------------------------------------------------------|
| クルーズ検索 (`cruise_search`)              | ユーザーがクルーズを検索するときにサイトからMonetateに情報を渡します                        |
| クルーズレビュー (`cruise_review`)          | ユーザーがカートにクルーズを追加するときにサイトからMonetateに情報を渡します                  |
| クルーズ予約 (`cruise_booking`)             | ユーザーがクルーズの購入を完了するときにサイトからMonetateに情報を渡します                    |
| ホテル検索 (`hotel_search`)                 | ユーザーがホテルを検索するときにサイトからMonetateに情報を渡します                            |
| ホテルレビュー (`hotel_review`)             | ユーザーがカートにホテルを追加するときにサイトからMonetateに情報を渡します                    |
| ホテル予約 (`hotel_booking`)                | ユーザーがホテルの購入を完了するときにサイトからMonetateに情報を渡します                      |
| フライト検索 (`flight_search`)              | ユーザーがフライトを検索するときにサイトからMonetateに情報を渡します                          |
| フライトレビュー (`flight_review`)          | ユーザーがカートにフライトを追加するときにサイトからMonetateに情報を渡します                  |
| フライト予約 (`flight_booking`)             | ユーザーがフライトの購入を完了するときにサイトからMonetateに情報を渡します                    |
