## 動作方法

* 注文IDが入力されると、購入追跡が自動的に行われます。
* Pixel ID フィールドには、カンマ区切りの Pixel ID リストを入力できます。
* **イベントIDの生成**: `true`に構成すると、Facebook Pixel で `event_id` パラメータが生成され、これらのイベントIDがサーバーサイド属性として利用可能になります。
  * 最新バージョンの Tealium Collect タグが必要です。
  * `event_id`にカスタム変数をマッピングする予定の場合は、`true`に構成する必要があります。
  * 詳細については、[Web用コンバージョンAPI](#conversions-api-for-web)を参照してください。
* テンプレートは、製品数量の配列が存在する場合に自動的に `num_items` 値を計算しようとします。
* マッピングを使用して以下を行います：
  * `trackSingle`の切り替え（デフォルトでは`false`）— 有効にすると、`fbq('trackSingle')`を使用して明示的にマッピングされたピクセルIDにのみイベントを送信し、無効にすると、`fbq('track')`を使用して初期化されたすべてのピクセルIDにイベントをブロードキャストします。
  * `disablePushState`の切り替え（デフォルトでは`false`）
  * タグ構成を動的に構成
  * E-Commerce 拡張の値を上書き
* 次の E-Commerce 拡張パラメータをサポート:
  * 注文 ID
  * 小計
  * 通貨
  * 製品IDのリスト
  * 製品カテゴリのリスト
  * 製品価格のリスト
  * 製品数量のリスト
  * 製品名のリスト

  
<blockquote>
Facebook Pixel タグは自動的にユーザーデータをハッシュ化します。この機能は変更できません。
</blockquote>


### Web 用コンバージョン API


<blockquote>
この機能にはアクティブな [Tealium Collect タグ](https://docs.tealium.com/tealium-collect-tag/)が必要です。
</blockquote>


Web用FacebookコンバージョンAPIをサポートするには、**イベントIDの生成**を`true`に構成します。**イベントIDの生成**が有効になっている場合、このタグは追跡された各イベントに対して一意のイベントIDを生成し、それをTealium EventStreamのFacebookコンバージョンコネクタで使用する属性として送信し、`event_id`パラメータでFacebook Pixelに渡します。このイベントID属性は、サーバーサイド統合とWebベースのタグを同期するためにコネクタでマッピングできます。

タグは次の命名規則を使用してイベント ID を送信します：

```nohl
fb_event_id_{FACEBOOK_EVENT}_{TAG_UID}
```

たとえば、タグ＃32からの購入イベントは次の属性と値を送信します：

```json
{
  "fb_event_id_Purchase_32": "028b2ade7478..."
}
```

同じタグからのページビューイベントは次の属性と値を送信します：

```json
{
  "fb_event_id_PageView_32": "084b1cda7461..."
}
```

#### 重複排除

適切なイベントの重複排除を確保するために、Facebook Pixel タグからのイベントIDをTealium Collect タグによって送信されるペイロードに含める必要があります。これを行うには、次の手順を実行します：

* **タグタイミング**ドロップダウンから**優先**を選択します。
* **バンドルフラグ**トグルを`オン`に構成します。
* [ロードオーダーマネージャー画面](https://docs.tealium.com/load-order-manager/)を使用して、Tealium Collect タグの前に Facebook Pixel タグを発火させます。Tealium Collect タグは最後に発火させることをお勧めします。

これらのイベントID属性の使用に関する情報については、[Facebook Conversions コネクタ：Webイベントの重複排除](https://docs.tealium.com/facebook-conversions-connector/#deduplication-for-web-events)を参照してください。

#### レガシーイベント ID

Facebook Pixel タグの古いバージョンでは、`fb_event_id_{FACEBOOK_EVENT}`の形式でイベントID属性が生成されていました。たとえば、`fb_event_id_Purchase`です。これらの古い属性名はタグによって引き続き生成されますが、同じページ上で複数のFacebookタグをサポートしないため、レガシーと見なされます。

Facebook Pixel タグの最新バージョンは、生成された属性名にタグUIDを含めることで、ページ上で複数のFacebookタグが発火するのをサポートします。もしまだレガシー形式のイベントID属性を使用している場合は、タグの最新バージョンに更新し、新しいイベントID属性を参照することをお勧めします。

## タグ構成

タグマーケットプレースにアクセスして新しいタグを追加します。タグを追加する方法についての詳細は、[タグの管理](https://docs.tealium.com/manage-tags/)を参照してください。

タグを追加した後、次の構成を構成します：

* **Pixel ID**
  * あなたのFacebook Pixel ID。
  * Pixel IDはコードスニペット内で長い数字のシリーズとして表示されます。
  * 複数のPixel IDをスペースなしのカンマ区切りリストで入力できます。
  * 例：`fbq("init", "12345678901234");`
* **高度なマッチング**
  * Facebook Advanced Matchingサービスにデータを送信して有効にします。
  * 例：`fbq("init", "12345678901234", {em:"6dd8b7d7..."});`
  * **高度なマッチング**を有効にして個人を特定できる情報（PII）のパラメータをマッピングする場合は、**Meta Events Manager**で**自動高度なマッチング**を無効にし、パラメータを手動でのみマッピングすることを強くお勧めします。**自動高度なマッチング**は、必要および範囲外の追加のPIIパラメータをマッピングする可能性があり、セキュリティおよびプライバシーの問題を引き起こす可能性があります。詳細については、Facebookの[ウェブサイトまたはモバイルアプリの自動高度なマッチングの構成](https://www.facebook.com/business/help/1993001664341800?id=1205376682832142)を参照してください。
* **イベントIDの生成**
  * Tealium Collect タグがサーバーサイドでイベントIDを送信するために必要です。
  * すべてのFacebookトラッキングイベントに対して自動的にイベントIDを生成します。
  * イベントID属性は次の命名規則に従います：`fb_event_id_{FACEBOOK_EVENT}_{TAG_UID}`
  * 例：`"fb_event_id_Purchase_32" : "028b2ade7478..."`
* **自動ページビューイベント**: `True`に構成すると、ページビューイベントを自動的に追跡します。

## イベントマッピング

Facebook Pixel イベントは、イベントにマッピングする変数を選択し、トリガーを入力することでデータマッピングを通じて構成されます。次のユースケースで示されるように、変数が特定の値と等しい場合にイベントがトリガーされます。

### ユースケース：購入イベント

購入イベントを構成するには：

1. 変数ドロップダウンリストから `tealium_event` を選択し、**宛先を選択**をクリックします。
1. **カテゴリ**セクションで、**イベント**を選択します。
1. **マッピングされた変数が等しい場合**フィールドに `Purchase` を構成します。これは `tealium_event` が購入をトリガーするために等しくなければならない値です。
1. **トリガーイベント**ドロップダウンリストで、`Purchase` を選択します。これは `tealium_event` が購入と等しい場合にトリガーされるイベントです。![](https://docs.tealium.com/images/client-side-tags/a.png)購入イベントに送信する必要がある追加データが、組み込みの E-Commerce 拡張マッピングによってカバーされていない場合は、**カスタムパラメータ**を使用して追加の購入データをマッピングします。

次の例では、購入イベントに製品ブランドを渡します：

1. 変数ドロップダウンリストから `product_brand` を選択し、**宛先を選択**をクリックします。
1. **カテゴリ**セクションで、**イベント固有のパラメータ**を選択します。
1. **宛先**フィールドにカスタムパラメータの名前 `brand` を構成します。
1. **イベントのために**ドロップダウンリストで、カスタムパラメータが収集するイベント `Purchase` を選択します。  
![](https://docs.tealium.com/images/client-side-tags/c.png)

[Facebook Pixel Helper](https://developers.facebook.com/docs/facebook-pixel/support/pixel-helper/) またはブラウザが提供する開発者ツールを使用してマッピングをテストします。

1. **ネットワークパネル**を使用し、Facebookをキーワードに含むすべてのネットワークリクエストを表示するためにFacebookでフィルタリングします。
1. リクエストの名前を見て、購入イベントが成功したことを確認します。
1. リクエストを開き、**クエリ文字列パラメータ**までスクロールダウンして、構成されたデータマッピングの値を確認します：`purchase`、`product_id`、および`product_brand`。  
![](https://docs.tealium.com/images/client-side-tags/d.png)
## データマッピング

データマッピングは階層構造を持ち、特定のマッピング（例えばイベント固有のパラメーター）は一般的なマッピングよりも優先されます。例えば、**標準トラッキングパラメーター** タブで変数を **Currency** 宛先にマッピングし、**ViewContent** イベントのためにも **Currency** 宛先にマッピングした場合、**ViewContent** のマッピングが一般的なマッピングよりも優先されますが、これは **ViewContent** イベントにのみ適用されます。他のすべてのイベントでは、標準トラッキングマッピングが使用されます。

マッピングとは、[データレイヤー変数](https://docs.tealium.com/data-layer-variables/)からベンダータグの対応する宛先変数へデータを送信するプロセスです。変数をタグ宛先にマッピングする方法については、[データマッピング](https://docs.tealium.com/ja/iq-tag-management/data-mappings/manage/)を参照してください。

利用可能なカテゴリーは以下の通りです：

### タグ構成

#### コンテンツタイプ

**標準トラッキングパラメータ**タブまたは**イベント固有パラメータ**マッピングで直接`content_type`パラメータにマッピングして構成します。

**コンテンツタイプ**タブでは、以下のサポートされている値のいずれかに`content_type`パラメータを条件付きで構成できます：

* `destination`
* `flight`
* `home_listing`
* `hotel`
* `product`
* `product_group`
* `vehicle`

マッピングされた変数がフィールドに入力した値と等しい場合、ドロップダウンリストから選択したタイプに値が構成されます。ほとんどの場合、`product`または`product_group`が使用され、その値はFacebookにアップロードされた製品フィードで指定した`content_type`と一致する必要があります。他の値は主にダイナミック広告で使用されます。

|変数| タイプ | 説明|
|---| ---| ---|
|`destination`|  文字列 | 目的地。 |
|`flight`|  文字列 | フライト。 |
|`home_listing`|  文字列 | ホームリスティング。 |
|`hotel`| 文字列 |  ホテル。 |
|`product`|  文字列 | 製品。 |
|`product_group`|  文字列 | 製品グループ。 |
|`vehicle`| 文字列 |  車両。 |
