## タグのヒント

* 注文IDが入力されると、購入追跡が自動的に行われます。
* イベントトラッキングIDフィールドには、カンマ区切りのイベントトラッキングIDリストを入力できます。
* マッピングを使用して以下を行います：
    * タグ構成を動的に構成
    * E-コマース拡張値を上書き
* これらのE-コマース拡張パラメータをサポート：
    * 注文ID
    * 小計
    * 通貨
    * 商品IDリスト
    * 商品カテゴリリスト
    * 商品価格リスト
    * 商品数量リスト
    * 商品名リスト

## タグ構成

タグマーケットプレイスにアクセスして新しいタグを追加します。詳細については、[タグについて]()を参照してください。

タグを追加する際には、以下の構成を構成します：

* **マーチャントID**：マーチャントIDは、オンボーディングプロセス中にThe Trade Deskから提供されます。
* **イベントトラッキングID**：プラットフォームUIで構成されたイベントトラッキングタグID（またはIDのカンマ区切りリスト）(`pixel_id`)。詳細については、[The Trade Desk: リアルタイムコンバージョンイベント](https://partner.thetradedesk.com/v3/portal/data/doc/DataConversionEventsApi)を参照してください。

## ロードルール

すべてのページでタグをロードするか、タグがロードされる条件を構成します。詳細については、[ロードルールについて]()を参照してください。

## データマッピング

マッピングは、データレイヤー変数からベンダータグの対応する宛先変数へデータを送信するプロセスです。詳細については、[データマッピングについて]()を参照してください。

利用可能なカテゴリは以下の通りです：

### タグ構成

| 変数 | タイプ/値 | 説明 |
|:---------|:-----|:------------|
| `pixel_id` | 文字列 | イベントトラッキングID。 |
| `merchant_id` | 文字列 | マーチャントID。 |

### E-コマース

| 変数 | タイプ/値 | 説明 |
|:---------|:-----|:------------|
| `order_id` | 文字列 | 注文ID（`_corder`を上書き）。 |
| `value` | 文字列 | 小計（`_csubtotal`を上書き）。 |
| `currency` | 文字列 | 通貨（`_ccurrency`を上書き）。 |
| `items.cat` | 配列 | 商品カテゴリリスト（`_ccat`を上書き）。 |
| `items.item_code` | 配列 | 商品IDリスト（`_cprod`を上書き）。 |
| `items.name` | 配列 | 商品名リスト（`_cprodname`を上書き）。 |
| `items.qty` | 配列 | 商品数量リスト（`_cquan`を上書き）。 |
| `items.price` | 配列 | 商品価格リスト（`_cprice`を上書き）。 |

### ユーザーデータ

| 変数 | タイプ/値 | 説明 |
|:---------|:-----|:------------|
| `country` | 文字列 | 国。 |
| `region` | 文字列 | 地域。 |
| `metro` | 文字列 | メトロ。 |
| `city` | 文字列 | 市。 |
| `zip` | 文字列 | 郵便番号。 |

| イベント | 説明 |
|:------|:------------|
| `addtocart` | カートに追加。 |
| `purchase` | 購入。 |
| `viewitem` | 商品を見る。 |
| `searchitem` | 商品を検索。 |
| `searchcategory` | カテゴリを検索。 |
| `login` | ログイン。 |
| `messagebusiness` | ビジネスにメッセージを送る。 |
