Braze Web SDK タグの構成ガイド

この記事では、Tealium iQ Tag Management アカウントで Braze Web SDK タグを構成する方法について説明します。

サポートされているバージョン

  • 3.5
  • 4.0

Braze Web SDK タグは、タグテンプレートバージョン 3.3 およびそれ以前をサポートしています。最新のタグテンプレートに更新することをおすすめします。

タグのヒント

  • Purchase イベントは、Order ID が存在する場合に発生します。

  • 以下の E-Commerce 拡張パラメータをサポートしています：

    • Customer ID (_ccustid を上書き)
    • City (_ccity を上書き)
    • Country (_ccountry を上書き)
    • Order ID (_corder を上書き)
    • Sub Total (_csubtotal を上書き)
    • Currency (_ccurrency を上書き)
    • Product IDs (_cprod を上書き)
    • Product Quantities (_cquan を上書き)

  • 標準の構成値を動的に上書きするためにマッピングを使用します。

タグの構成

まず、Tealiumの Tag Marketplace に移動し、Braze Web SDK タグをプロファイルに追加します（タグの追加方法）。

タグを追加した後、以下の構成を構成します：

  • Code Version

    • 使用する Braze Web SDK のメジャーマイナーバージョンを入力します。
    • 例えば、バージョン 2.6.1 をデプロイする場合は、2.6 と入力します。
    • 最新の Braze Web SDK のバージョンは、changelog を参照してください。

  • API Key

    • Braze ダッシュボードの Manage App Group 構成で利用可能な一意の API キーです。

  • Base URL

    • 割り当てられた Braze クラスタを指す API エンドポイントです。
    • 4.0 の場合の例：https://js.appboycdn.com/web-sdk/4.0/braze.no-amd.min.js
    • 3.5 およびそれ以前のバージョンの場合の例：https://js.appboycdn.com/web-sdk/3.5/appboy.no-amd.min.js
    • オプションの一覧については、Braze SDK エンドポイント を参照してください。

  • Enable Logging

    • 有効にすると、ウェブコンソールにデバッグ情報が記録されます。

ロードルール

ロードルール は、サイト上でこのタグのインスタンスをいつ、どこにロードするかを決定します。

データマッピング

マッピングは、データレイヤ変数 からベンダータグの対応する宛先変数にデータを送信するプロセスです。(詳細を見る)

Braze Web SDK タグの宛先変数は、タグのデータマッピングタブに組み込まれています。以下のテーブルには、利用可能な宛先カテゴリと各宛先名の説明がリストされています。

標準

宛先名 説明
initOpt.allow_crawler
  • クローラーを許可
  • Web クローラーによるすべてのアクティビティを Braze に記録します。
  • オプションは true または false です。
initOpt.app_ver
  • アプリバージョン
  • データを関連付けるアプリのバージョンです。
initOpt.base_url
  • ベース URL
  • Braze が提供するカスタムエンドポイントの完全な URL です。
initOpt.no_cookies
  • クッキーを使用しない
  • Braze がユーザー情報をクッキーに保存しないようにします。
  • オプションは true または false です。
initOpt.no_fntawsm
  • FontAwesome を使用しない
  • Braze が FontAwesome を読み込まないようにします。
  • オプションは true または false です。
initOpt.enable_htmlmsgs
  • メッセージ内の HTML を有効にする
  • Braze ユーザーがインアプリメッセージを介して HTML を送信できるようにします。
  • オプションは true または false です。
initOpt.enable_logging
  • ログを有効にする
  • Braze のエラーメッセージをウェブコンソールに表示します。
  • オプションは true または false です。
initOpt.localization
  • ローカライゼーション
  • ブラウザのデフォルト言語を上書きするために使用される ISO 639-1 言語コードです。
initOpt.min_trggrinterval
  • アクション間の最小インターバル
  • 別のトリガーアクションが発生するまでの時間（秒単位）です。
  • デフォルト値は 30 です。
initOpt.msg_innewtab
  • インアプリメッセージを新しいタブで開く
  • インアプリメッセージのリンクを新しいタブまたはウィンドウで開くようにします。
  • オプションは true または false です。
initOpt.card_innewtab
  • ニュースフィードカードを新しいタブで開く
  • ニュースフィードカードのリンクを新しいタブまたはウィンドウで開くようにします。
  • オプションは true または false です。
initOpt.explicit_dissmisal
  • 明示的な閉じる操作が必要
  • ユーザーがメッセージを閉じるためにボタンをクリックする必要があります。
  • オプションは true または false です。
initOpt.safari_pushid
  • Safari ウェブサイトプッシュ ID
  • Safari プッシュ証明書からの ID です。
initOpt.srvcewrkr_location
  • サービスワーカーの場所
  • ルートディレクトリにない場合、service-worker.js ファイルへのパスです。
initOpt.session_timeout
  • セッションタイムアウト
  • セッションがタイムアウトするまでの時間（分単位）です。
  • デフォルト値は 30 です。
initOpt.enableSdkAuthentication
  • 現在のユーザーの最後の既知の JSON Web トークンを Braze サーバーに送信する
  • この構成は、バージョン 3.5 以降で利用可能です。
  • オプションは true または false です。

E-Commerce

Braze Web SDK タグは、E-Commerce に対応しており、デフォルトの E-Commerce 拡張マッピングが自動的に使用されます。一般的には、このカテゴリで手動でマッピングする必要はありませんが、拡張マッピングを上書きしたり、希望する E-Commerce 変数が拡張で提供されていない場合に使用することができます。

宛先名 説明
order_id
  • 注文 ID
  • _corder を上書きします。
order_subtotal
  • 小計
  • _csubtotal を上書きします。
order_currency
  • 通貨
  • _ccurrency を上書きします。
customer_id
  • 顧客 ID
  • _ccustid を上書きします。
customer_city
  • 顧客の都市
  • _ccity を上書きします。
customer_country
  • 顧客の国
  • _ccountry を上書きします。
product_id
  • 配列
  • 製品 ID のリスト
  • _cprod を上書きします。
product_quantity
  • 配列
  • 数量のリスト
  • _cquan を上書きします。

イベント

特定のイベントをページでトリガーするための宛先にマップします。

イベントをトリガーするための手順は以下の通りです：

  1. ドロップダウンリストからイベントを選択します。
  • プリセットリストから選択するか、カスタムイベントを作成します。
  • カスタムイベントの場合は、識別するための名前を入力します。
  1. トリガー フィールドにマッピングする変数の値を入力します。
  2. 追加のイベントをマッピングするには、+ ボタンをクリックして、手順 1 と 2 を繰り返します。
  3. 適用 をクリックします。

以下のテーブルは、データレイヤで指定された値が見つかった場合にイベントがトリガーされるイベントトリガーをリストアップしています。

パラメータ

事前に定義されたイベントにデータを渡すために、以下の宛先にマップします。パラメータは、プリセットイベントとのみ使用されます。カスタムイベントでパラメータを渡す方法については、以下の「カスタムイベントデータ」セクションを参照してください。

以下の手順で事前に定義されたイベントにパラメータを渡します：

  1. イベントフィールドで、ドロップダウンリストからイベントを選択します。
  2. パラメータフィールドで、ドロップダウンリストからパラメータを選択します。
  3. カスタムパラメータの場合は、送信するパラメータの名前を入力します。
  4. + 追加 をクリックします。

  • _cprod を上書きします。

  • これは、E-Commerce の値 _cquan を上書きします。

  • _corder を上書きします。

  • インアプリの購入のための小計です。
  • _csubtotal を上書きします。

  • _ccurrency を上書きします。

  • 現在のユーザーの識別子です。

  • エイリアスのラベル（エイリアスのソースなど）です。
key
  • カスタム属性キー
  • カスタム属性の識別子です。
  • 送信するパラメータの名前を入力します。
value
  • カスタム属性の値
  • カスタム属性の値です。
  • 送信するパラメータの名前を入力します。
inc_value
  • カスタム属性の増分値
  • カスタムユーザー属性の値を増分する値です。
  • 送信するパラメータの名前を入力します。
  • 減算する場合は負の数を使用します。
  • デフォルト値は 1 です。
avatar_url
  • アバター画像の URL
  • 現在のユーザーの選択したアバターの URL です。
  • バージョン 4.0 以降では使用できません。
longitude
  • ユーザーの経度
  • ユーザーの位置の経度（度単位）です。
  • 値は -180 から 180 までです。
latitude
  • ユーザーの緯度
  • ユーザーの位置の緯度（度単位）です。
  • 値は -90 から 90 までです。
accuracy
  • 位置の精度
  • ユーザーの経度と緯度の精度（メートル単位）です。
altitude
  • ユーザーの高度
  • ユーザーの位置の高度（メートル単位）です。
altitude_accuracy
  • 高度の精度
  • ユーザーの高度の精度（メートル単位）です。
year
  • ユーザーの誕生年
  • 現在のユーザーの誕生日に関連付けられた年です。
month
  • ユーザーの誕生月
  • 現在のユーザーの誕生日に関連付けられた月です。
day
  • ユーザーの誕生日
  • 現在のユーザーの誕生日に関連付けられた日です。
email
  • ユーザーのメールアドレス
  • 現在のユーザーの検証済みメールアドレスです。
notification_type
  • 通知の購読タイプ
  • 現在のユーザーの選択した通知タイプです。
  • オプション: opted_in、subscribed、unsubscribed
first_name
  • ユーザーの名
  • 現在のユーザーの名です。
subscriptionGroupId
  • 購読グループ ID
  • 購読グループの一意の識別子です。
last_name
  • ユーザーの姓
  • 現在のユーザーの姓です。
gender
  • ユーザーの性別
  • 現在のユーザーの性別コードです。
  • オプションは男性 (m)、女性 (f)、その他 (o) です。
customer_city
  • ユーザーのホームの都市
  • 現在のユーザーのホームの都市です。
  • _ccity を上書きします。
customer_country
  • ユーザーの国
  • 現在のユーザーの国です。
  • _ccountry を上書きします。
localization
  • ユーザーの言語
  • 現在のユーザーの表示言語の ISO 639-1 言語コードです。
phone_number
  • ユーザーの電話番号
  • 現在のユーザーの電話番号です。
  • 数字、スペース、+、.、-、(、) のみが許可されます。

カスタムイベントデータ

カスタムイベントでカスタムパラメータを渡すために、カスタムイベントデータの宛先にマップします。以前にイベントタブでマッピングしたカスタムイベントと同じ名前を入力します。

カスタムイベントデータ変数をマップする手順は以下の通りです：

  1. イベント名 に、カスタムイベントの名前を入力します（イベントタブで指定したものと同じ）。
  2. パラメータ に、送信するパラメータの名前を入力します。
  3. + 追加 をクリックします。
