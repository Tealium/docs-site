## タグのヒント

* 構成値を上書きするか、カスタムデータを構成するためにマッピングを使用します。
* **変換タイプ**は、**タグタイプ**が `conv` に構成されている場合のみ使用されます。

## タグの構成

タグマーケットプレイスに行って新しいタグを追加します。詳細については、を参照してください。

タグを追加する際には、以下の構成を構成します：

* **アカウントID**：あなたのFreakOutアカウントID。
* **変換タイプ**：FreakOutからの正確な大文字小文字を区別する変換アクション名を入力します（例：`purchase`、`signup`、`leadform`）。
* **タグタイプ**：
    * `segment`：標準的なリターゲティング。
    * `conv`：標準的な変換。

## ベストプラクティス

* 新しい変換アクションを追加する前に：
    1. FreakOutで変換を構成します。
    1. FreakOut内でマッピングテーブルとテストを更新します。
    1. QAにデプロイします。
    1. ペイロードを検証します。
    1. 短い監視ウィンドウで本番環境に公開します。
* シングルページアプリケーション（SPA）や複数ステップのファネルの場合、中間ステップではなく最終成功状態でのみタグを発火させます。

## ロードルール

すべてのページでタグをロードするか、タグがロードされる条件を構成します。詳細については、[ロードルールについて]()を参照してください。

## データマッピング

マッピングは、データレイヤー変数からベンダータグの対応する宛先変数へデータを送信するプロセスです。詳細については、[データマッピングについて]()を参照してください。

利用可能なカテゴリーは以下の通りです：

### 標準

| 変数                     | タイプ   | 説明                                             |
|:----------------------|:-------|:--------------------------------------------------|
| `account_id`          | String | アカウントID。                                     |
| `conversion_type`     | String | 変換タイプ。                                       |
| `segment_data.custom` | String | セグメンテーション目的の追加カスタムデータ。       |
