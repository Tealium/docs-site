Pepperjamタグ構成ガイド

この記事では、Tealium iQタグ管理アカウントでPepperjamタグを構成する方法について説明します。

タグのヒント

  • 以下のE-Commerce拡張値をサポートしています：
    • 注文ID
    • 小計
    • プロモーションコード
    • 商品IDのリスト
    • 価格のリスト
    • 数量のリスト
    • カテゴリのリスト
  • “Standard Integration Type"の値は、タグの統合値が"Standard"の場合にのみ適用されます。

タグの構成

新しいタグを追加するためにタグマーケットプレイスに移動します。タグを追加する一般的な手順については、Tag Overviewの記事を参照してください。

タグを追加する際には、以下の構成を行います：

  • Program ID
    Pepperjamから提供されたID。
  • Integration
  • Standard Integration Type
    この選択は、このタグのStandard統合にのみ影響します。

データマッピング

マッピングは、data layer variableからベンダータグの対応する宛先変数にデータを送信するプロセスです。変数をタグの宛先にマップする方法については、Data Mappingsを参照してください。

利用可能なカテゴリは以下の通りです：

Standard

変数 説明
base_url
  • ベースURL
pid
  • プログラムID
integration
  • 統合
  • 値は次の通りです：
    • STANDARD
    • DYNAMIC
    • ITEMIZED
type
  • Standard統合タイプ
  • 値は1または2です。
new_to_file
  • 新規ファイル
  • 値は1です。

E-Commerce

変数 説明
order_id
  • 注文ID
  • _corderを上書きします。
order_subtotal
  • 小計
  • _csubtotalを上書きします。
order_coupon_code
  • プロモーションコード
  • _cpromoを上書きします。
product_id
  • 配列
  • 商品IDのリスト
  • _cprodを上書きします。
product_category
  • 配列
  • カテゴリのリスト
  • _ccatを上書きします。
product_quantity
  • 配列
  • 数量のリスト
  • _cquanを上書きします。
product_unit_price
  • 配列
  • 価格のリスト
  • _cpriceを上書きします。
