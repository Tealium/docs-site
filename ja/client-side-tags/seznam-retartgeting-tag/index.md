Seznam リターゲティングは、あなたのサイトを訪れたユーザーにリーチするために設計されています。

## タグのヒント

* マッピングを使用して：
    * 標準の構成値を動的に上書きします。
    * E-Commerce 拡張の値を動的に上書きします。

## タグの構成

タグマーケットプレイスにアクセスして新しいタグを追加します。詳細については、[タグについて]()を参照してください。

タグを追加する際には、以下の構成を構成します：

* **Rtg Id**: Seznam から提供されたマーチャントID。
* **Auto Tracking**: このオプションを有効にすると、タグがロードされるたびに `retargetingHit()` 関数を呼び出します。

## ロードルール

すべてのページでタグをロードするか、タグがロードされる条件を構成します。詳細については、[ロードルールについて]()を参照してください。

## データマッピング

マッピングは、データレイヤー変数からベンダータグの対応する宛先変数へデータを送信するプロセスです。詳細については、[データマッピングについて]()を参照してください。

利用可能なカテゴリーは以下の通りです：

### 構成

| 変数 | タイプ | 説明 |
|:---------|:-----|:------------|
| `rtgId` | 文字列 | Seznam から提供されたマーチャントID。 |
| `auto_tracking` | ブール値 | 自動追跡。このオプションを有効にすると、タグがロードされるたびに `retargetingHit()` 関数を呼び出します。 |

### 標準

| 変数 | タイプ | 説明 |
|:---------|:-----|:------------|
| `eid` | 文字列 | ユーザーのメールアドレス。 |
| `a1` | 文字列 | ユーザーの国。 |
| `a2` | 文字列 | ユーザーの都市。 |
| `a3` | 文字列 | ユーザーの通り。 |
| `a4` | 文字列 | ユーザーのホーム番号。 |
| `a5` | 文字列 | ユーザーの郵便番号。 |
| `tid` | 文字列 | ユーザーの電話番号。 |

### リターゲティング

| 変数 | タイプ | 説明 |
|:---------|:-----|:------------|
| `consent` | 数値  | ユーザーの同意。 &lt;ul&gt;&lt;li&gt;`0`: 同意なし。&lt;/li&gt;&lt;li&gt;`1`: 同意あり。&lt;/li&gt;&lt;/ul&gt; |
| `itemId` | 文字列 | オファー識別子。 |
| `category` | 文字列 | カテゴリー。 |
| `pageType` | 文字列 | ページタイプ。 |
| `rtgUrl` | 文字列 | クエリ文字列。 |

### コンバージョン

| 変数 | タイプ| 説明 |
|:---------|:-----|:------------|
| `id` | 文字列 | コンバージョン識別子。 |
| `value` | 文字列 | 注文値（`_ctotal`を上書き）。 |
| `orderId` | 文字列 | 注文識別子（`_corder`を上書き）。 |
| `zboziType` | 文字列 | 測定タイプ（`_ctype`を上書き）。 |
| `zboziId` | 文字列 | 施設ID。 |
