## タグのヒント

* クライアントIDを動的に構成するためにマッピングを使用します。
* 以下のEコマース拡張パラメータをサポートします：
    * 注文小計（`_csubtotal`）
    * 注文通貨（`_ccurrency`）
    * 数量リスト（`_cquan`）
* 数量は特定のイベントのすべてのアイテムの合計数です。マッピングを通じて数量値が提供されない場合、Eコマース拡張機能から利用可能な場合、製品数量のリスト配列から数量が計算されます。

## タグの構成

タグマーケットプレイスにアクセスして新しいタグを追加します。詳細については、を参照してください。

タグを追加する際には、以下の構成を構成します：

* **クライアントID**：Sprinklrが提供するクライアントID。
* **イベントIDの生成**：すべてのSprinklrトラッキングイベントに対して自動的にイベントIDを生成します。

## ロードルール

すべてのページでタグをロードするか、タグがロードされる条件を構成します。詳細については、[ロードルールについて]()を参照してください。

## データマッピング

マッピングは、データレイヤー変数からベンダータグの対応する宛先変数にデータを送信するプロセスです。詳細については、[データマッピングについて]()を参照してください。

利用可能なカテゴリは以下の通りです：

### タグ構成とグローバルパラメータ

| 変数 | タイプ/値 | 説明 |
|:---------|:-----|:------------|
| `client_id` | 文字列 | クライアントID。 |
| `eventId` | 文字列 | イベントID。 |
| `leadId` | 文字列 | リードID。 |
| `custom.myvar` |  | カスタムパラメータ。 |
| `generate_event_id` | ブール | イベントIDを生成。 |

### Eコマース

| 変数 | 説明 |
|:---------|:------------|
| `value` | 値（`_csubtotal`を上書き）。 |
| `currency` | 通貨（`_ccurrency`を上書き）。 |
| `product_quantity` | 製品数量のリスト（`_cquan`を上書き）。 |
| `quantity` | イベントのすべてのアイテムの合計数。 |

### ユーザーデータ

| 変数 | タイプ/値 | 説明 |
|:---------|:-----|:------------|
| `userInfo.firstName` | 文字列 | 顧客の名前。 |
| `userInfo.lastName` | 文字列 | 顧客の姓。 |
| `userInfo.fullName` | 文字列 | 顧客のフルネーム。 |
| `userInfo.customerId` | 文字列 | 顧客の一意の識別子。 |
| `userInfo.anonymousId` | 文字列 | 登録されていないユーザーの顧客の一意の識別子。Tealium訪問IDを自動的に送信し、変数をマッピングして上書きします。 |
| `userInfo.phoneNo` | 文字列 | 顧客の電話番号。 |
| `userInfo.email` | 文字列 | 顧客のメールアドレス。 |
