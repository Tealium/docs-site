Ipsos irisは、全国代表的な設立調査、受動的に測定されたシングルソースパネル、およびサイト中心のセンサス測定を組み合わせた最先端のハイブリッド方式です。タグを通じて収集されたデータは、公開社のブランドを市場に完全に表現するために、すべてのデバイスでIpsos irisの測定を完全に保証します。Ipsos irisは、柔軟で直感的なレポーティングインターフェースを通じて、デジタルオーディエンスの行動を日々、週ごと、月ごとに豊かに理解し、最終的により情報に基づいた決定をサポートします。

## タグのヒント

* マッピングを使用して、タグ構成をオーバーライドまたは動的に構成します。
* 各ウェブサイトタグにはユニークなセクションIDが含まれています。ユーザーのウェブサイトが複数のサイトセクションで構成されており、それぞれのセクション（例えば、ニュース、スポーツ、天気など）のデータを別々に測定したい場合。
* このドキュメント全体でサポートされている英数字値には、以下が含まれます：
  * 最大20文字の長さ
  * 小文字
  * 文字 [a-z]
  * 数字 [0-9]

## タグの構成

タグマーケットプレイスに移動して新しいタグを追加します。タグを追加する一般的な手順については、[タグの概要]()の記事を読んでください。

タグを追加する際、以下の構成を構成します：

* **Numeric Site Section Id**: サイトセクション名文字列が提供されていない場合、デフォルトで使用されます。
* **Site Section Name String**: ウェブサイトセクションの英数字キーワード。
* **Region**: ベーススクリプトの3つのバリエーションがあります。Ipsos irisにはジオロケーション機能がないため、クライアントは地域に基づいて異なるタグをデプロイして正しい地域サーバーにヒットさせます。

## ロードルール

すべてのページでタグをロードするか、タグがロードされる条件を構成します。ロードルールについての詳細は、[ロードルール]()のドキュメントを参照してください。

## データマッピング

マッピングは、データレイヤー変数からベンダータグの対応する宛先変数へデータを送信するプロセスです。変数をタグ宛先にマッピングする手順については、[データマッピング](/ja/iq-tag-management/data-mappings/manage/)を参照してください。

利用可能なカテゴリは以下の通りです：

### 標準

|変数| 説明|
|---| ---|
|Numeric Site Section Id (numericSiteSectionId)| [文字列]|
|Site Section Name String (siteSectionNameString)| [文字列]|
|Region (region)| [文字列]|

### イベント

イベントをマッピングするには、[イベントマッピングの作成](/ja/iq-tag-management/data-mappings/manage/)を参照してください。

|変数| 説明|
|---| ---|
|Page View (pageview)| pageview|
