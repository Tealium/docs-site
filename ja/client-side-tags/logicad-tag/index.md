Logicadは、独自のアルゴリズムを使用して、大規模な配信ログとシステムインフラを持つ広告を効果的に伝送します。

## タグのヒント

* ダイナミッククリエイティブリターゲティングタグタイプには、商品グループIDと広告主商品IDのマッピングが必要です。
* ダイナミッククリエイティブコンバージョンタグタイプには、商品グループIDのマッピングが必要です。
* 以下のE-Commerce拡張パラメータをサポートしています：
    * 注文ID (`_corder`)
    * 小計 (`_csubtotal`)
    * 商品IDのリスト (`_cprod`)
    * 数量のリスト (`_cquan`)
    * 価格のリスト (`_cprice`)

## タグの構成

タグマーケットプレイスに移動して新しいタグを追加します。詳細については、[タグについて]()を参照してください。

タグを追加する際には、以下の構成を行います：

* **広告主ID**：So-net Media Networkから提供された広告主ID。
* **タグタイプ**：ビジネスケースに合わせたタグのタイプを選択します。
* **フィールド区切り文字**：商品フィールド情報の間の区切り文字。レコード区切り文字と同じにはできません。
* **レコード区切り文字**：商品レコード情報の間の区切り文字。フィールド区切り文字と同じにはできません。

## ロードルール

すべてのページでタグをロードするか、タグがロードされる条件を構成します。詳細については、[ロードルールについて]()を参照してください。

## データマッピング

マッピングは、データレイヤー変数からベンダータグの対応する宛先変数にデータを送信するプロセスです。詳細については、[データマッピングについて]()を参照してください。

利用可能なカテゴリは以下の通りです：

### スタンダード

| 変数 | 説明 |
|---|---|
| `advertiser_id` | 広告主ID |
| `type` | タグタイプ |
| `field_delim` | 商品フィールド区切り文字 |
| `record_delim` | 商品レコード区切り文字 |
| `product_group_id` | 商品グループID |
| `advertiser_product_id` | 広告主商品ID |
| `retargeting_param` | リターゲティングパラメータ |
| `log_param1` | ログパラメータ1 |
| `log_param2` | ログパラメータ2 |
| `log_param3` | ログパラメータ3 |
| `log_param4` | ログパラメータ4 |
| `log_param5` | ログパラメータ5 |
| `log_param6` | ログパラメータ6 |
| `log_param7` | ログパラメータ7 |
| `log_param8` | ログパラメータ8 |
| `log_param9` | ログパラメータ9 |
| `log_param10` | ログパラメータ10 |

### E-Commerce

| 変数 | 説明 |
|:---------|:------------|
| `order_id` | 注文ID  (`_corder`を上書き) |
| `order_subtotal` | 合計価格/小計  (`_csubtotal`を上書き) |
| `order_quantity` | 合計数量  (商品数量の合計を上書き) |
|`product_id`  商品IDのリスト (`_cprod`を上書き)  | 
| `product_quantity` | 数量のリスト (`_cquan`を上書き)  | 
| `product_unit_price` | 価格のリスト (`_cprice`を上書き)  |
