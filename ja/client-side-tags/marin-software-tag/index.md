## タグの仕様と要件

### 仕様

**名前**: Marin Software

**ベンダー**: Marin Software

**タイプ**: Search

### 必須

**サービス**: Marin Softwareアカウント

**構成**:

* クライアントID

## Tealium iQでのタグ構成

### 1. タグの追加

Tealium iQのタグマーケットプレイスは、さまざまなタグを提供しています。プロファイルにタグを追加する方法については、[こちら]()をクリックしてください。

### 2. タグの構成

1. **タイトル** (必須): タグインスタンスを識別するための記述的なタイトルを入力します。
1. **クライアントID** (必須): Marin Softwareから割り当てられたクライアントIDを入力します。

### 3. ロードルールの適用

[ロードルール]()は、このタグのインスタンスをいつ、どこでロードするかを決定します。**すべてのページでロード**ルールは、デフォルトのロードルールです。このタグを特定のページでロードするには、関連する条件を持つ新しいロードルールを作成します。

**ベストプラクティス**: 注文確認ページのみでこのタグをロードするロードルールを作成します。さらに、注文確認ページでこのタグのインスタンスを1つだけロードすることを確認してください。同じページで複数のインスタンスをロードすると、Marin Softwareタグが失敗します。

### 4. マッピングの構成

[マッピング](/ja/iq-tag-management/data-mappings/manage/)は、データレイヤーのデータソースからベンダータグの対応する宛先変数にデータを送信するシンプルなプロセスです。

このタグでは、[E-Commerce extension]()を追加し、構成する必要があります。以下のデータポイントは、E-Commerce extensionによって自動的にMarinに送信されます。マッピングツールボックス内のこれらの宛先へのマッピングは、該当する場合、E-Commmerceマッピングを上書きします。

* **convType**
E-Commerce extensionの`_ctype`変数がこの宛先にマッピングされます。このデータポイントは、カートまたは注文のタイプを識別します。

* **orderid**
E-Commerce extensionの`_corder`変数がこの宛先にマッピングされます。このデータポイントは、注文のIDを識別します。

* **category**
E-Commerce extensionの`_ccat`変数がこの宛先にマッピングされます。このデータポイントは、製品のカテゴリを識別します。

* **currency**
E-Commerce extensionの`_ccurrency`変数がこの宛先にマッピングされます。このデータポイントは、注文の通貨タイプを識別します。

* **product**
E-Commerce extensionの`_cprod`変数がこの宛先にマッピングされます。この変数は、マッピングツールボックスの宛先として表示されません。これにマッピングするには、マッピングフィールドに手動で`product`を入力する必要があります。これは、注文された製品のリストです。

* **quantity**
E-Commerce extensionの`_cquan`変数がこの宛先にマッピングされます。この変数は、マッピングツールボックスの宛先として表示されません。これにマッピングするには、マッピングフィールドに手動で`quantity`を入力する必要があります。これは、注文された製品の数量です。

* **price**
E-Commerce extensionの`_cprice`変数がこの宛先にマッピングされます。この変数は、マッピングツールボックスの宛先として表示されません。これにマッピングするには、マッピングフィールドに手動で`price`を入力する必要があります。これは、製品価格のリストです。

#### 廃止されたマッピング

* **Affiliation**
この宛先は、最新のMarin Softwareタグテンプレートで廃止されました。この宛先へのマッピングは何も行いません。
