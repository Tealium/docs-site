この記事では、Tealium iQタグ管理アカウントでNeustar PlatformOneコンバージョンタグを構成する方法について説明します。

Neustar® PlatformOneは、広告主がすべてのマーケティング活動にわたって精密マーケティングを提供できる統合マーケティングプラットフォームです。NeustarのAuthoritative Identity Engineによって駆動されるPlatformOneは、顧客と見込み客の単一で偏りのない正確なビューを提供し、コンバージョン率を改善し、支出を最適化し、最終的には売上を増加させます。

## タグのヒント

*   マッピングを使用して、標準の構成値を動的に上書きします。
*   次のE-Commerce変数を使用します
    *   `_corder`
    *  ` _ctotal`
    *   `_ccustid`
    *   `_cquan`
*   `product_quantity`
    *   これは配列としてマッピングできます（通常通り、E-Commerceの値を上書きし）、合計を出すために合計されます
    *   または、直接割り当てられる数値としてマッピングできます

## タグの構成

タグマーケットプレイスに移動して新しいタグを追加します。タグの追加方法については、[タグの概要]()の記事を参照してください。

タグを追加する際には、以下の構成を行います：

*   **タグID**：整数値。例：`123456`。
*   **タイプID**：整数値。例：`123456`。

## ロードルール

すべてのページでタグをロードするか、タグがロードされる条件を構成します。ロードルールについての詳細は、[ロードルール]()のドキュメンテーションを参照してください。

## データマッピング

マッピングは、データレイヤー変数からベンダータグの対応する宛先変数にデータを送信するプロセスです。タグの宛先に変数をマッピングする方法については、[データマッピング]()を参照してください。

利用可能なカテゴリーは以下の通りです：

### 標準

| 変数 | 説明 |
|:---------|:------------|
| タグID   | (`tag_id`)  |
| タイプID  | (`type`)    |

### E-Commerce

| 変数                                                     | 説明                            |
|:-------------------------------------------------------------|:---------------------------------------|
| 注文ID                                                     | (`order_id`) (`_corder`を上書き)     |
| 注文合計                                                  | (`order_total`) (`_ctotal`を上書き)  |
| 顧客ID                                                  | (`customer_id`) (`_ccustid`を上書き) |
| 数量のリスト (`product_quantity`) (`_cquan`を上書き) | \[配列/数値\]                       |
| カスタム属性1 (`cust1`)                                 | \[配列/文字列\]                       |
| カスタム属性2 (`cust2`)                                 | \[配列/文字列\]                       |
