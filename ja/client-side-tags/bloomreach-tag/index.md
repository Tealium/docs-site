## タグの構成

まず、タグマーケットプレイスにアクセスし、プロファイルにBloomReachタグを追加します（[タグの追加](https://docs.tealium.com/manage-tags/#add-a-tag)を参照）。

タグを追加した後、以下の構成を構成します：

* アカウントID：あなたのBloomReachアカウントID。
* ビューID：このBloomReachが提供するIDは、同じアカウントIDを使用して独自の製品カタログ特性を持つ複数のサイトがある場合にのみ必要です。
* ドメインキー：このBloomReachが提供するIDは、複数の言語バージョンのサイトにのみ必要です。
* Track TMS：Tealiumをあなたのタグ管理システム（TMS）として追跡するかどうかを切り替えます。BloomReachの実装を他のプラットフォームからTealiumに移行していて、両方のプラットフォームでBloomReachをロードしている場合にのみ`Yes`に切り替えます。デフォルトは`No`です。これにより、BloomReachは各プラットフォームからの追跡呼び出しを区別できます。
* ページビュー時に`br_data`をクリア：各ページビューで`br_data`オブジェクトをクリアするか、永続的な値を保持するかを切り替えます。これは通常、シングルページまたはAJAXアプリケーションで`Yes`に構成されます。デフォルトは`No`です。

## ロードルール

[ロードルール](https://docs.tealium.com/about-load-rules/)は、サイト上でこのタグのインスタンスをいつ、どこでロードするかを決定します。

推奨されるロードルール：**全ページ**

## データマッピング

マッピングは、[データレイヤー変数](https://docs.tealium.com/ja/iq-tag-management/data-mappings/manage/)からベンダータグの対応する宛先変数にデータを送信するプロセスです。変数をタグ宛先にマッピングする方法については、[データマッピング](https://docs.tealium.com/ja/iq-tag-management/data-mappings/manage/)を参照してください。

BloomReachタグの宛先変数は、その**データマッピング**タブに組み込まれています。利用可能なカテゴリは以下の通りです：

### 標準/グローバル

| **宛先名**           | **説明**                                                                                                           |
|:---------------------|:--------------------------------------------------------------------------------------------------------------------------|
| アカウントID           | 構成された**アカウントID**を上書きします。                                                                                 |
| ビューID              | 構成された**ビューID**を上書きします。                                                                                    |
| ドメインキー           | 構成された**ドメインキー**を上書きします。                                                                                 |
| タイトル                | 現在のページのタイトルでページタイトルを上書きします。                                                              |
| 通貨             | トランザクションの通貨コードを上書きします。                                                                             |
| ユーザーID              | E-Commerce拡張の`_ccustid`値を上書きします。                                                                    |
| Track TMSフラグ       | 構成されたTrack TMS構成を上書きします。値は`yes`の文字列である必要があります。                    |
| Clear br_dataフラグ   | 構成されたページビュー時のClear `br_data`構成を上書きします。値は`yes`の文字列である必要があります。 |
| イベントグループ          | デフォルトのイベントグループを上書きするためのマップです。BloomReach担当者と相談した後にのみこれを上書きしてください。         |
| イベントタイプ           | デフォルトのイベントタイプを上書きするためのマップです。BloomReach担当者と相談した後にのみこれを上書きしてください。          |

### ページタイプ (`ptype`)

| **宛先名** | **説明**                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
|:---------------------|:-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| ptype                | (全てのページビューに必要) BloomReachには7つのページタイプ分類があります：`homepage`, `product`, `category`, `search`, `content`, `thematic`, `other`。マップされた変数の値を入力し、ドロップダウンから適切な`ptype`を選択することで、`ptype`を動的に構成します。<br><br>ページタイプがBloomReachのサポートされているページタイプのいずれかに該当しない場合は、`other`を使用します。BloomReach担当者と相談した後にのみカスタムptypeを構成してください。 |

### カテゴリーページ

| **宛先名** | **説明**                                                                                                                                                                                                                                                     |
|:---------------------|:--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| カテゴリーID          | BloomReachデータベース/カタログで参照される一意のカテゴリーIDです。カテゴリーIDはフィードの`crumb_id`と一致する必要があります。これは通常、数値IDまたは一意のクラムパスです。<br><br>クラムパスを渡す場合は、個々のクラム値を区切ることはありません。 |
| カテゴリー名        | ページのブレッドクラムです。各クラムはパイプ(&#124;)で区切られる必要があります。                                                                                                                                                                                        |

### コンバージョンページ

| **宛先名** | **説明**                                                                                                 |
|:---------------------|:----------------------------------------------------------------------------------------------------------------|
| 注文ID             | コンバージョンの注文またはトランザクションIDです。                                                                 |
| 注文合計          | 注文の合計価格で、税金と送料を含みます。                                                     |
| 製品IDのリスト  | 製品IDの配列です。                                                                                        |
| 名前のリスト        | 製品名の配列です。                                                                                      |
| SKUのリスト         | 製品SKUの配列です。                                                                                       |
| カテゴリのリスト   | 製品カテゴリの配列です。                                                                                 |
| 価格のリスト       | 製品価格の配列です。                                                                                     |
| 数量のリスト       | 注文内の各製品の数の配列です。                                                            |
| コンバージョンフラグ   | コンバージョンまたはサンキューページでこれを`1`に構成します。注文IDが入力されると自動的に`1`に構成されます。 |


### 製品ページ

| **目的地名** | **説明** |
|:---------------------|:---------------------------|
| 製品IDリスト | 製品IDの配列。 |
| 名前リスト | 製品名の配列。 |
| SKUリスト | 製品SKUの配列。 |

### 検索ページ

| **目的地名** | **説明** |
|:---------------------|:---------------------------------------------------|
| 検索語 | ページを説明する検索クエリの値。 |

### バーチャルページビュー

| **目的地名** | **説明** |
|:-----------------------|:-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 元の参照URL | バーチャルページビューイベントを発火する際、最初にロードされたページのURLです。バーチャルページビューが発火されると自動的にキャプチャされますが、ここでマッピングすることによって上書きすることができます。 |

### イベント

ページ上で特定のイベントをトリガーするためにこれらの目的地にマップします。イベントをトリガーするには：

1. ドロップダウンリストからイベントを選択します。事前定義されたリストから選択するか、カスタムイベントを作成することができます。カスタムイベントの場合、それを識別する名前を入力してください。カスタムパラメーターやイベントを構成する前に、BloomReachの担当者に相談してください。
1. **トリガー**フィールドにマッピングされる変数の値を入力します。
1. より多くのイベントをマップするには、**+**ボタンをクリックして手順1と2を繰り返します。
1. **適用**をクリックします。

提供された値がデータレイヤーで見つかったときにイベントがトリガーされます。

| **目的地名** | **説明** |
|:---------------------|:-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| カートに追加 | 訪問がアイテムをカートに追加するときにトリガーします。デフォルトイベントグループ：`cart`。デフォルトイベントタイプ：`click-add`。 |
| コンテンツページビュー | 訪問が製品ページ以外の専用ページ（例：レシピ、ブログ、記事ページ）を開くときにトリガーします。 |
| コンテンツ検索 | 訪問が検索を実行するときにトリガーします。 |
| クイックビュー | 訪問が製品の詳細を表示するためにクイックビューを開くときにトリガーします。これは通常、リスティングページやカテゴリーページで発生します。デフォルトイベントグループ：`product`。デフォルトイベントタイプ：`quickview`。 |
| 検索提案 | 訪問が検索語を入力して送信をクリックするか、自動提案語が表示されて訪問がそれをクリックするときにトリガーします。デフォルトイベントグループ：`suggest`。デフォルトイベントタイプ：`submit`, `click`。 |
| バーチャルページビュー | 現在のページが新しいページビューとして追跡されるべき方法で更新されるときにトリガーします。これはAJAXサイトやシングルページアプリで一般的です。 |
| ウィジェットカートに追加 | 訪問が推薦からアイテムをカートに追加するときにトリガーします。 |
| ウィジェットクリック | 訪問が推薦からアイテムをクリックするときにトリガーします。 |
| ウィジェットビュー | 訪問が推薦からアイテムを表示するときにトリガーします。 |
| カスタム | カスタムイベントをトリガーします。これを構成する前に、BloomReachの担当者に相談してください。 |

### イベントパラメータ

これにより、特定のイベントのパラメータにマップすることができます。

| **目的地名** | **説明** |
|:---------------------------|:------------------------------------------------------------------------------------|
| 製品ID (`prod_id`) | カートに追加またはクイックビューイベントの製品ID。 |
| 製品名 (`prod_name`) | クイックビューイベントの製品名。 |
| 製品SKU (`sku`) | カートに追加またはクイックビューイベントの製品SKU。 |
| 検索クエリ (`q`) | 検索提案イベントのために訪問が入力した検索語。 |
| 表示クエリ (`aq`) | 検索提案イベントのために訪問に表示された自動提案検索語。 |


<blockquote>
検索提案イベントの表示クエリパラメータを渡す場合、イベントタイプは自動的に`click`に構成されます。それ以外の場合は、イベントタイプは`submit`になります。
</blockquote>


#### セグメントパラメータ

`br_data.SEGMENT_NAME`の目的地名でカスタムセグメントパラメータを追加できます。ここで、`SEGMENT_NAME`はセグメント名を表します。

例：`br_data.customer_tier = "vip"`
