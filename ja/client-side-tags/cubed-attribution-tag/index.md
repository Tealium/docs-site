## タグの構成

まず、タグマーケットプレイスにアクセスし、プロファイルにCUBEDタグを追加してください（[タグの追加](https://docs.tealium.com/manage-tags/#add-a-tag)を参照）。

タグを追加した後、以下の構成を行います：

* **アカウントトークン**: CUBEDから提供されるユニークなトークン
* **イベント詳細の送信**: イベントと共に製品の詳細を送信するかどうかを選択します。デフォルトは **false**

## ロードルール

[ロードルール](https://docs.tealium.com/about-load-rules/)は、サイト上でこのタグのインスタンスをいつ、どこでロードするかを決定します。

推奨されるロードルール：**全ページ**

## データマッピング

マッピングは、[データレイヤー変数](https://docs.tealium.com/ja/iq-tag-management/data-mappings/manage/)からベンダータグの対応する宛先変数にデータを送信するプロセスです。変数をタグの宛先にマッピングする方法については、[データマッピング](https://docs.tealium.com/about-data-mappings/)を参照してください。

CUBEDタグの宛先変数は、そのデータマッピングタブに組み込まれています。利用可能なカテゴリーは以下の通りです：

### 標準

| **宛先名**                         | **説明**                                                                                                                                                   |
|:---------------------------------------------|:------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| アカウントトークン (`account_token`)              | アカウントトークンを動的に上書きする場合にマッピングします                                                                                                |
| フォワードパラメータ (`forward_param`)             | マーケティングチャネルを分類するために使用されるランディングページURLパラメータの配列                                                                                   |
| 整数ラベル (`addIntLabel.custom`)        | CUBEDデータと共に送信する整数ラベル値。マッピングでカスタムをラベルのキー名に置き換えます                                       |
| 浮動小数点ラベル (`addFloatLabel.custom`)        | CUBEDデータと共に送信する浮動小数点ラベル値。マッピングでカスタムをラベルのキー名に置き換えます                                          |
| 文字列ラベル (`addStringLabel.custom`)      | CUBEDデータと共に送信する文字列ラベル値。マッピングでカスタムをラベルのキー名に置き換えます                                         |
| 日付/時間ラベル (`addDataTimeLabel.custom`) | CUBEDデータと共に送信する日付/時間ラベル値。マッピングでカスタムをラベルのキー名に置き換えます。日付はISO 8601形式である必要があります |
| ブールラベル (`addBoolLabel.custom`)       | CUBEDデータと共に送信するブールラベル値。マッピングでカスタムをラベルのキー名に置き換えます                                        |
| 同期ID (`sync_ids`)                        | デバイスやプラットフォーム間で顧客レコードを結合するための一意の識別子の配列                                                                          |

### E-コマース

CUBEDタグはE-コマース対応しているため、デフォルトのE-コマース拡張マッピングを自動的に使用します。以下の場合を除き、このカテゴリでの手動マッピングは通常必要ありません：

* 拡張マッピングを上書きしたい場合
* 拡張機能で提供されていないE-コマース変数が必要な場合

| **宛先名**                    | **説明**                                                                                                                                                                         |
|:----------------------------------------|:----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 注文ID/トランザクションID (`order_id`)    | イベントに関連付けられた一意のID。マッピングされていない場合、E-コマース値 `_corder` がデフォルトで使用されます                                                                         |
| 注文合計/収益 (`order_total`)     | イベントに関連する収益。マッピングされていない場合、E-コマース値 `_ctotal` がデフォルトで使用されます                                                                           |
| 商品名リスト (`product_name`)          | イベントに関連する商品名のリスト。イベントの詳細を送信する場合は必須です。マッピングされていない場合、E-コマース値 `_cprodname` がデフォルトで使用されます |
| SKUリスト (`product_sku`)            | イベントに関連する商品SKUのリスト。マッピングされていない場合、E-コマース値 `_csku` がデフォルトで使用されます                                                        |
| 数量リスト (`product_quantity`) | イベントに関連する商品の数量リスト。マッピングされていない場合、E-コマース値 `_cquan` がデフォルトで使用されます                                             |
| 価格リスト (`product_unit_price`)   | イベントに関連する商品の価格リスト。マッピングされていない場合、E-コマース値 `_cprice` がデフォルトで使用されます                                                    |

### イベントデータ：カスタムイベント

| **宛先名**                | **説明**                                                                                             |
|:------------------------------------|:------------------------------------------------------------------------------------------------------------|
| イベント名 (`event_name`)           | 送信されるイベントの名前。これはCUBEDインターフェースで事前に構成されたイベントのいずれかと一致する必要があります |
| イベント詳細の送信 (`send_details`) | イベントと共に製品固有のE-コマースの詳細が送信されるかどうかを決定するブール値                     |
