Namogooは、ウェブセッションに侵入する不正な広告をブロックすることで、オンラインの顧客体験を向上させ、集中力を保ち、邪魔が入らないブラウジング旅行を保証します。

## タグのヒント

* マッピングを使用して、標準の構成値を動的に上書きします。
* 以下のE-Commerce拡張パラメーターをサポートしています：
    * カート合計
    * 割引後の注文合計
    * クーポンコード
    * アイテム数
    * 通貨

## タグ構成

新しいタグを追加するためにタグマーケットプレイスに移動します。詳細については、[タグについて]()を参照してください。

タグを追加する際には、以下の構成を行います：

* **クライアントID**：Namogooクライアント（アカウント）ID

## ロードルール

すべてのページでタグをロードするか、タグがロードされる条件を構成します。詳細については、[ロードルールについて]()を参照してください。

## データマッピング

マッピングは、データレイヤー変数からベンダータグの対応する宛先変数にデータを送信するプロセスです。詳細については、[データマッピングについて]()を参照してください。

利用可能なカテゴリーは以下の通りです：

### 標準

| 変数 | タイプ | 説明 |
|:---------|:------------|:------------|
| `client_Id`  | 文字列 | クライアントID。 |
| `cartTotal`  | 数値 | カート合計（`_csubtotal`を上書き）。 |
| `orderTotalBeforeDiscount`  | 数値 | 割引前の注文合計。 |
| `orderTotalAfterDiscount`  | 数値 | 割引後の注文合計（`_ctotal`を上書き）。 |
| `couponCode`  | 文字列 | クーポンコード（`_cpromo`を上書き）。 |
| `numOfItems`  | 数値 | アイテム数（`_cquan`を上書き）。 |
| `currency`  | 文字列 | 通貨（`_ccurrency`を上書き）。 |

### イベント
イベントをマップするには、[イベントマッピングの作成]()を参照してください。

| 変数 | 説明 |
|:---------|:------------|
|  `addCartData` | カートデータを追加。 |
|  `addOrderData` | 注文データを追加。 |

### イベント固有のパラメーター
イベントをマップするには、[イベントマッピングの作成]()を参照してください。

| 変数 | 説明 |
|:---------|:------------|
| `addCartData` | カートデータを追加。 |
| `addOrderData` | 注文データを追加。 |
