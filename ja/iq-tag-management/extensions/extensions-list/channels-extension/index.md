たとえば、多くの広告チャンネルを運用しており、顧客のチャンネルライフサイクルがどのようなものかを知りたい場合があります。人々は1つの広告を見ているのか、それとも多くの広告を見ているのか？どの広告を最初に、または最後に見ているのか？購入する人々にとって、購入前の流れはどうなっているのか？

この拡張機能は、オリジネーター、インフルエンサー、クローザーのアフィリエイトに対してチャンネルコンバージョンクレジットを構成する機能も提供します。

## 前提条件

* 開始する前に、[拡張機能について]()を参照してください。

## 要件

* utag v4.38以降。`utag.js`テンプレートを更新する方法についての詳細は、ナレッジベースの記事[utag.jsの最新バージョンへの更新のベストプラクティス](https://support.tealiumiq.com/en/support/solutions/articles/36000363470)を参照してください。

## 拡張機能の使用

拡張機能を追加すると、以下の構成オプションが利用可能になります：

### チャンネル

* **チャンネル名**：チャンネルの名前
* **カテゴリ**：ドロップダウンメニューからチャンネルが属するカテゴリを選択
* **条件**：このチャンネルを構成するタイミングを決定する条件を適用。**+** ボタンをクリックして新しい条件を追加するか、**-** ボタンをクリックして条件を削除します。

**チャンネルを追加** ボタンをクリックして、必要に応じて追加のチャンネルを追加および構成します。

![](https://docs.tealium.com/images/iq-tag-management/no-title-176icaa0d1be9b890a0e.png)

### 帰属期間と人口

出力データソースを配置するページを選択します。これらのデータソースは、カートページや注文確認ページなどのコンバージョンタイプのページでのみ配置することをお勧めします。

**帰属期間/人口** タブに移動します。このタブから、特定のチャンネルにコンバージョンを帰属させたい期間と、それを行いたいページを構成します。

* **帰属期間**：整数を入力し、ドロップダウンメニューから単位を選択
    * **セッション**：ユーザーのウェブサイト訪問の期間にわたって続き、クッキーに「0」と表示されます。
    * **訪問**：最大値は365日に構成され、タイムスタンプが表示されます。
    * **日数**：データを保持する日数で、タイムスタンプが表示されます。

* **データソース**：（デフォルト：全ページ）チャンネル拡張機能を実行するタイミングと場所を決定するロードルールを選択します。デフォルト値の使用をお勧めします。
* **データソースの人口における永続値のクリア**：チェックボックスを選択すると、各ページの読み込みごとにすべての出力データソースからデータがクリアされます。
    * 現在のチャンネルのみを追跡したい場合、または単一のコンバージョンを超えてライフタイムチャンネルの動作に興味がない場合は、ボックスをチェックします。
    * 複数のコンバージョンがあるサイトの場合は、ボックスをチェックしないでください。

![](https://docs.tealium.com/images/iq-tag-management/no-title-175i03abcc3041938f48.png)

### コンバージョン

チャンネルがコンバージョンにどのように貢献したかに応じて、チャンネルに与えるコミッション額を決定します：

* **コンバージョン値ソース**：これは、クレジットが基づく数値のコンバージョン額を含むデータソースです。これは通常、コンバージョンの小計額です。
* **オリジネータークレジット**：これは、コンバージョンのオリジネーターに帰属するコンバージョン値ソースの割合です。
* **インフルエンサークレジット**：これは、コンバージョンのインフルエンサーまたはインフルエンサーに帰属するコンバージョン値ソースの組み合わせた割合です。
    * **繰り返しチャンネル応答を許可** をチェックすることで、チャンネルがインフルエンサーとして複数回表示されることを有効にします。この値のチェックを外すと、各ユニークなインフルエンサーは一度だけリストされます。

* **クローザークレジット**：これは、コンバージョンのクローザーに帰属するコンバージョン値ソースの割合です。

![](https://docs.tealium.com/images/iq-tag-management/no-title-174icc0c5df949e4995c.png)

## クッキーデータのフォーマット

この拡張機能からのデータは `channelflow` というクッキーに保存されます。クッキーデータは、始めから終わりまでの完全なチャンネルフローを提供します。各チャンネルには3つの値があります：チャンネル名、カテゴリ、帰属期間。各値はバー「`|`」で区切られた文字列として保存されます。各個別のチャンネルはコンマで区切られます。以下は例です：

```
Channel1 Name|Category|Attribution Period, Channel2 Name|Category|Attribution Period
```

帰属期間が `Same Session` に構成されている場合、その値はクッキーで `0` です。帰属期間が `Visitor` または `Days` に構成されている場合、帰属期間の値はゼロより大きい数です。

したがって、カテゴリがそれぞれ `affiliate`, `display ads`, `sponsorship` に構成され、すべての帰属期間が `Same Session` に構成されている3つのチャンネル（`channel1`, `channel2`, `channel3`）がある場合、データは次のようになります：

```
channel1|affiliate|0,channel2|displayads|0,channel3|sponsorship|0
```

同じチャンネルとカテゴリで、帰属期間が `Visitor` または `Days` に構成されている場合、データは次のようになります：

```
channel1|affiliate|1358975968083,channel2|displayads|1358975970419,channel3|sponsorship|1358975973256
```

### クッキーデータの読み取り

クッキーデータは、始めから終わりまでの完全なチャンネルフローを提供します。クッキーは左（始め）から右（終わり）へ読まれます。

* クッキーとページ変数は、チャンネル、カテゴリ、有効期限で構成されます。
* 上記のクッキー例は `channel|category|expiration` として読まれます
    * **チャンネル**：チャンネルの名前、例えば `PepperJam`。
    * **カテゴリ**：チャンネルのカテゴリ、`PepperJam` の場合は `Affiliate` を選択します。
    * **有効期限**：帰属期間に基づいて構成されたチャンネルフローのライフサイクルを追跡します。

* 販売/コンバージョンのクレジットが誰に行くかを決定する際、クレジットは最後に訪れたチャンネルに行きます。私たちの例では、それはチャンネル `SF` です。
## チャネル変数

Channels拡張機能を使用すると、拡張機能によって保存されたデータは、ロードルール、拡張機能、およびデータマッピングで使用できる新しいデータレイヤー変数に保存されます。これらの変数は、カテゴリからチャネルを分割することにより、より詳細な情報を提供し、最初に訪れたチャネル、最後に訪れたチャネル、訪れたすべてのチャネルとカテゴリを提供します。

以下の変数が作成されます：

| 変数                             | 説明                                                                                                                                                                                                                                                                         |
|:---------------------------------|:----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `channel_category_originator`    | 最初に訪れたカテゴリ。                                                                                                                                                                                                                                                       |
| `channel_originator`             | 最初に訪れたチャネル。                                                                                                                                                                                                                                                       |
| `channel_category_influencer`    | クローザーを含まない訪れたカテゴリのリスト。                                                                                                                                                                                                                                 |
| `channel_influencer`             | クローザーを含まない訪れたチャネルのリスト。                                                                                                                                                                                                                                 |
| `channel_category_closer`        | 最後に訪れたカテゴリ。                                                                                                                                                                                                                                                        |
| `channel_closer`                 | 最後に訪れたチャネル。                                                                                                                                                                                                                                                        |
| `channel_category_path`          | 訪れたすべてのカテゴリ。                                                                                                                                                                                                                                                      |
| `channel_path`                   | 訪れたすべてのチャネル。                                                                                                                                                                                                                                                      |
| `channel_originator_credit`      | コンバージョン値の割合に基づいてオリジネーターに与えられる金額を表示します。                                                                                                                                                                                                 |
| `channel_influencer_credit`      | コンバージョン値の割合に基づいて各インフルエンサーに与えられる金額。これは、インフルエンサークレジットの割合をコンバージョン変数の値で乗算し、`channel_influencer`変数にリストされているチャネルの数で割ることによって計算されます。 |
| `channel_closer_credit`          | コンバージョン値の割合に基づいてクローザーに与えられる金額。                                                                                                                                                                                                                  |

特定のページでのみチャネル情報をベンダータグに渡したい場合は、次の手順に従ってください：

1. 新しいデータレイヤー変数を作成します。
1. 特定のページで新しいデータソースの値をチャネルクローザーに構成するようにSet Data Values拡張機能を構成します。
1. ベンダータグの構成を開き、`channel_closer`の代わりに新しい変数を適切なタグ先にマッピングします。

### チャネル変数の読み取り

各ページ訪問で変数がどのように変化するかを見るために、顧客がPJ、EM、SFチャネルを見る上記のシナリオを想像してください。変数が次のように変化することに注意してください：

1. 最初のチャネルフロー：
    * `channel_category_originator: affiliate`
    * `channel_originator: PJ`
    * `channel_category_path: affiliate`
    * `channel_path: PJ`
    * `channel_category_closer: affiliate`
    * `channel_closer: PJ`

1. 二番目のチャネルフロー：
    * `channel_category_originator: affiliate`
    * `channel_originator: PJ`
    * `channel_category_path: affiliate,displayads`
    * `channel_path: PJ,EM`
    * `channel_category_closer: displayads`
    * `channel_closer: EM`

1. 三番目のチャネルフロー：
    * `channel_category_originator: affiliate`
    * `channel_originator: PJ`
    * `channel_category_path: affiliate,displayads,sponsorships`
    * `channel_path: PJ,EM,SF`
    * `channel_category_closer: sponsorships`
    * `channel_closer: SF`

## カスタム処理 - 高度

カスタム処理コードは拡張機能の最後に実行されるため、このコードが呼び出されるときにはすべての計算と割り当てが完了しています。

コンテナオブジェクトは`obj`という名前で、カスタム処理に使用する以下の属性を含んでいます：

* `channel_originator`
* `channel_influencer`
* `channel_closer`
* `channel_category_originator`
* `channel_category_influencer`
* `channel_category_closer`
* `channel_path`
* `channel_category_path`
* `channel_originator_credit`
* `channel_influencer_credit`
* `channel_closer_credit`

以下はカスタムハンドラの例です：

```
  // カスタムハンドラの例
  // もしcommisionjunctionがオリジネーターであれば
  // それはクローザーでもあります
  if(obj.channel_originator=='commisionjunction'){
    obj.channel_closer='commission junction';
  }
```

## テスト

Channels拡張機能が環境に公開された後、完全なテストを行います。

異なるチャネルが存在するページのURLをメモし、データがキャプチャされるように各URLにアクセスします。

[HTTPFox](https://addons.mozilla.org/en-us/firefox/addon/httpfox/)や[Firebug](http://getfirebug.com)などのFirefoxツールやChromeデバッグツールを使用すると、ページ変数やクッキーの状態を確認できます。これらのツールは、問題のデバッグとチャネルフロートラッキングが必要に応じて機能していることを検証するのに役立ちます。
