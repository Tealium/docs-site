## 必要条件

* utag v4.38以降。`utag.js`テンプレートの更新についての詳細は、ナレッジベースの記事 [utag.jsの最新バージョンへの更新のベストプラクティス](https://support.tealiumiq.com/en/support/solutions/articles/36000363470) を参照してください。

## 動作方法

ハッシュ変更イベントは、ページのURLのハッシュ記号 (`#`) 以降のアンカー部分が変更されたときにトラッキングします。例えば、URL `https://www.example.com/page#part2` において、URLのアンカー部分は `#part2` です。ページのハッシュのアンカー部分が変更されると、トラッキング呼び出しがトリガーされます。

ハッシュ変更イベントリスナーは主にシングルページアプリケーション（SPA）で使用するためのものです。

イベントリスナーの追加方法についての詳細は、[イベントの管理]()を参照してください。

## イベントトリガー

イベントトリガーは、イベントリスナーがトラッキングする特定のアクションです。

ハッシュ変更イベントは以下のイベントトリガーをトラッキングできます：

* **ハッシュ変更** – ページのURLのハッシュ記号 (`#`) 以降のアンカー部分が変更されたときにトリガーします。

以下の閾値をトリガーに構成できます：

* **遅延** - トラッキング呼び出しを遅延させる時間（ミリ秒）。

### トリガー頻度

トリガー頻度は、イベントトリガーがトラッキング呼び出しを引き起こす回数を決定します。SPAはDOMのロードが1回しかないため、ハッシュ変更イベントのトリガー頻度は**常に**構成することをお勧めします。これにより、イベントリスナーはすべてのハッシュ変更をトラッキングします。

詳細については、[イベントトリガー]()を参照してください。

## イベントトリガー変数

イベントトリガー変数は、イベントリスナーがトラッキング呼び出しと共に送信する値です。ハッシュ変更イベントには以下のデフォルトのイベントトリガー変数があります：

### ハッシュ変更

|識別子| 説明|
|---| ---|
|`tealium_event=&#34;hash_change&#34;`| ページのURLのアンカー部分が変更されました。 |
|`iq_event_id`| イベントを送信したイベントリスナーのUID。|

**例**

```json
{
   &#34;tealium_event&#34;  : &#34;hash_change&#34;,
   &#34;iq_event_id&#34;   : &#34;hash_change_events_3&#34;
}

```
