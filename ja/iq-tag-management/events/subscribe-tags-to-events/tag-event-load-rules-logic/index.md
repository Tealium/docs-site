イベントリスナーのロードルールとタグのロードルールとイベントリスナーが組み合わさると、ロードと発火の複雑なロジックと挙動が生じます。

## タグとイベントがロードする方法

購読されたタグとイベントリスナーは、以下のロジックを使用してロードするタイミングを決定します：

* イベントリスナーのロードルールは、それに購読しているタグのロードルールに関係なく、イベントがロードするタイミングを決定します。
* タグがAND論理関係でイベントリスナーに購読されている場合、イベントリスナーのロードルールがタグがロードするタイミングを決定します。
* タグがOR論理関係でイベントリスナーに購読されている場合、タグのロードルールがタグがロードするタイミングを決定します。

## タグとイベントリスナーが発火する方法

イベントリスナーがタグのロードルールのロジックにどのように表示されるかによって、イベントがイベントリスナーのペイロードにタグデータを含むかどうかが決まります：

* イベントリスナーは、訪問がそのアクションを実行すると、それに購読しているタグに関係なく、その変数を発火して送信します。
* タグがAND論理関係でイベントリスナーに購読されている場合、訪問がイベントリスナーのアクションを実行すると、イベントリスナーはタグを発火し、イベントリスナーの変数とタグデータをタグのエンドポイントに送信します。
* タグがOR論理関係でイベントリスナーに購読されている場合、タグはページがロードされたときに発火し、その変数をエンドポイントに送信します。訪問がイベントリスナーのアクションを実行すると、イベントリスナーはタグを発火し、イベントリスナーの変数とタグデータをタグのエンドポイントに送信します。
