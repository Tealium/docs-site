## ロードルールなしの例

以下の例では、ホームページと製品ページがあるウェブサイトを使用します。そのサイトでは、さまざまな構成と関係性を持つタグとイベントリスナーのセットを使用します。

### イベントにサブスクライブされていないタグ

ユーザーが以下の構成でイベントリスナーを作成します：

* Element Visibility（要素の可視性）イベントタイプ。
* 広告バナーのCSS要素ID。
* 5秒間完全に表示。
* 一度だけトリガー。
* DOM Readyスコープを使用（デフォルト）。
* All Pages（全ページ）とイベントリスナーのロードルールを使用。
* ページ上にTealium Collectタグが存在し、All PagesとEvents（イベント）のロードルールを使用します。

![](/images/iq-tag-management/tc-all-pages.png)

**全ページ：**

* 製品詳細ページは、ページがDOM Readyになった後にこのイベントリスナーをロードします。
* ページはTealium Collectタグをロードします。
* タグが発火し、そのペイロードをタグエンドポイントに送信します。
* 訪問がそのページで初めて広告バナー全体を5秒以上見た場合、イベントリスナーが発火し、その変数をトラッキングコールで送信します。

### イベントにサブスクライブされているタグ

ユーザーが以下の構成でイベントリスナーを作成します：

* Element Visibility（要素の可視性）イベントタイプ。
* 広告バナーのCSS要素ID。
* 5秒間完全に表示。
* 一度だけトリガー。
* DOM Readyスコープを使用（デフォルト）。
* All Pages（全ページ）とイベントリスナーのロードルールを使用。
* ページ上にTealium Collectタグが存在し、Element visible（要素が可視）イベントにサブスクライブされています。

![](/images/iq-tag-management/screen-shot-2022-09-19-at-15.39.47.png)

**全ページ：**

* 製品詳細ページは、ページがDOM Readyになった後にこのイベントリスナーをロードします。
* ページはTealium Collectタグをロードします。
* 訪問がそのページで初めて広告バナー全体を5秒以上見た場合、イベントリスナーがTealium Collectタグを発火させ、その変数をTealium Collectタグのペイロードとともにトラッキングコールで送信します。

## ロードルールありの例

以下の例では、ホームページと製品ページがあるウェブサイトを使用します。そのサイトでは、さまざまな構成と関係性を持つタグ、イベントリスナー、ロードルールのセットを使用します。

### イベントにサブスクライブされていないタグ

ユーザーが以下の構成でイベントリスナーを作成します：

* Element Visibility（要素の可視性）イベントタイプ。
* 広告バナーのCSS要素ID。
* 5秒間完全に表示。
* 一度だけトリガー。
* DOM Readyスコープを使用（デフォルト）。
* 製品詳細ページのみにロードされるようにロードルールを使用。
* ページ上にTealium Collectタグが存在し、All PagesとEvents（イベント）のロードルールを使用します。

![](/images/iq-tag-management/tc-all-pages.png)

**製品ページ**

* 製品詳細ページは、ページがDOM Readyになった後にこのイベントリスナーをロードします。
* ページはTealium Collectタグをロードします。
* タグが発火し、そのペイロードをタグエンドポイントに送信します。
* 訪問がそのページで初めて広告バナー全体を5秒以上見た場合、イベントリスナーが発火し、その変数をトラッキングコールで送信します。

**ホームページ**

* ホームページは、ロードルールが製品ページのみにイベントリスナーをロードするように指定しているため、このイベントはロードされません。
* ホームページはTealium Collectタグをロードします。
* タグが発火し、そのペイロードをタグエンドポイントに送信します。

### イベントにサブスクライブされているタグ（ANDロジック）

ユーザーが以下の構成でイベントリスナーを作成します：

* Element Visibility（要素の可視性）イベントタイプ。
* 広告バナーのCSS要素ID。
* 5秒間完全に表示。
* 一度だけトリガー。
* DOM Readyスコープを使用（デフォルト）。
* 製品詳細ページのみにロードされるようにロードルールを使用。
* Tealium CollectタグのルールとイベントリスナーにANDロジックで追加され、その他のタグルールとイベントリスナーは`true`と評価されます。![](/images/iq-tag-management/tag-and-logic.png)

**製品ページ：**

* 製品詳細ページは、ページがDOM Readyになった後にこのイベントリスナーをロードします。
* ページはまた、そのロードルールとイベントリスナーのロードルールがすべて`true`と評価されるため、Tealium Collectタグをロードしますが、すぐには発火しません。
* 訪問がそのページで初めて広告バナー全体を5秒以上見た場合、イベントリスナーがTealium Collectタグを発火させ、その変数をTealium Collectタグのペイロードとともにトラッキングコールで送信します。

**ホームページ：**

* ホームページは、ロードルールが製品ページのみにイベントリスナーをロードするように指定しているため、このイベントはロードされません。
* Tealium ConnectタグがANDロジックでイベントにサブスクライブされているため、タグのロードルールとイベントリスナーのルールが組み合わさってすべて`false`と評価されます。
* ホームページはTealium Connectタグをロードしません。

### 2つのイベントリスナーにサブスクライブされているタグ（ORロジック）

ユーザーが以下の構成で2つのイベントリスナーを作成します：

**イベント＃1 - 要素が可視**

* Element Visibility（要素の可視性）イベントタイプ。
* 広告バナーのCSS要素ID。
* 5秒間完全に表示。
* 一度だけトリガー。
* DOM Readyスコープを使用（デフォルト）。
* 製品詳細ページのみにロードされるようにロードルールを使用。
* Tealium CollectタグのルールとイベントリスナーにORロジックで追加され、その他のタグルールとイベントリスナーは`true`と評価されます。

**イベント＃2 - フォームの送信**

* Form Submission（フォーム送信）イベントタイプ。
* フォームのCSS要素ID。
* 一度だけトリガー。
* DOM Readyスコープを使用（デフォルト）。
* 製品詳細ページのみにロードされるようにロードルールを使用。
* Tealium CollectタグのルールとイベントリスナーにORロジックで追加され、その他のタグルールとイベントリスナーは`true`と評価されます。
![](/images/iq-tag-management/mat-multi-event.png)

**製品ページ：**

* 製品詳細ページは、ページがDOM Readyになった後にこれら2つのイベントリスナーをロードします。
* ページはTealium Collectタグをロードします。
* タグが発火し、そのペイロードをタグエンドポイントに送信します。
* 訪問がそのページで初めて広告バナー全体を5秒以上見た場合、イベントリスナーがTealium Collectタグを発火させ、その変数をTealium Collectタグのペイロードとともにトラッキングコールで送信します。
* 訪問が初めてフォームを送信した場合、イベントリスナーがTealium Collectタグを発火させ、その変数をTealium Collectタグのペイロードとともにトラッキングコールで送信します。

**ホームページ：**

* ホームページは、ロードルールが製品ページのみにイベントリスナーをロードするように指定しているため、どちらのイベントリスナーもロードされません。
* Tealium ConnectタグがORロジックでイベントリスナーにサブスクライブされているため、タグのロードルールとイベントリスナーのルールがすべて`true`と評価されます。
* ホームページはTealium Connectタグをロードし、発火します。
