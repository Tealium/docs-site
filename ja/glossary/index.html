A

AJAX (非同期JavaScriptとXML)

AJAXは、ウェブサーバーからメッセージを送受信するために使用される一連の技術です。

AudienceDB (以前の名前はManaged Redshift for Audiences)

Tealiumのデータベースソリューションの一つで、構造化データを使用して視聴者レベルのデータを整理します。

AudienceStore

Tealiumのデータベースソリューションの一つで、半構造化データを使用してオーディエンスレベルのデータをフォーマットします。

B

bオブジェクト

bオブジェクトは、utag.dataオブジェクトのエイリアスおよびローカルインスタンスであり、データレイヤーなどが該当します。

Back to top

C

CDN (コンテンツデリバリーネットワーク)

コンテンツデリバリーネットワークは、ウェブページのコンテンツを訪問にできるだけ迅速かつ効率的に配信することを目指して、大規模に分散したサーバーネットワークです。

CDP (カスタマーデータプラットフォーム)

マーケターが管理するシステムで、他のマーケティングシステムがアクセス可能な永続的で統一された顧客データベースを作成します。

CLI（コマンドラインインターフェース）

ウィンドウやボタンの代わりにテキストベースのコマンドでコンピュータシステムを操作するためのツール。これは、グラフィカルインターフェースと比較してより簡潔または強力な制御を可能にすることが多く、エンジニアや他の高度なコンピュータユーザーの間で人気があり、一部のアプリケーションはCLIを通じてのみアクセスするように設計されています。

CMS (コンテンツ管理システム)

ウェブサイトのコンテキストで通常使用されるデジタルコンテンツの作成と変更をサポートするコンピュータアプリケーション。多くの場合、複数のユーザーが協力して作業する環境をサポートするために使用されます。

CPM（千回あたりのコスト）

特定のウェブページ上での1,000回の広告表示の価格を示す用語。直接的な頭文字ではなく、「M」はローマ数字の1,000を意味します。

CRM (顧客関係管理)

企業が現在および潜在的な顧客との関係を管理するアプローチで、顧客が企業との歴史に関するデータ分析を使用してビジネスとの関係を改善し、特に顧客の保持に焦点を当て、最終的には売上成長を促進します。

CSS (カスケーディングスタイルシート)

CSSは、フォントのサイズやタイプ、色、レイアウトなど、ウェブページの外観とスタイルを記述するための言語です。

CTR（クリックスルーレート）

あなたの広告が受け取るクリック数を、広告が表示される回数で割ったもの：クリック数 ÷ インプレッション数 = CTR。例えば、クリックが5回、インプレッションが1000回だった場合、CTRは0.5%になります。あなたの広告やキーワードそれぞれには、アカウントで確認できる独自のCTRがあります。

D

DOM (ドキュメントオブジェクトモデル)

HTML、XML、またはXHTMLドキュメント、たとえばウェブページの要素を識別し操作するための規約。

DOM ID

これはHTMLドキュメント内の要素の一意の識別子値です。

DOM完全

DOMは、外部リソースを含む全ページがロードされ、完全にレンダリングされたときに「完全」とみなされます。

DOM準備完了

DOMは、すべてのオブジェクト、プロパティ、メソッドがロードされたときに「準備完了」とみなされます。これはページのロードが完全に終了する前かもしれません。したがって、DOM準備完了は、DOMが準備完了するまで拡張機能の実行を待つJavaScriptコード拡張のスコープオプションです。

DSP（Demand-Side Platform：需要側プラットフォーム）

デジタル広告の在庫を購入するバイヤーが、一つのインターフェースを通じて複数の広告交換とデータ交換のアカウントを管理できるシステム。

Back to top

E

ESP (電子メールサービスプロバイダ)

電子メールマーケティングまたは一括電子メールサービスを提供する会社。

EventDB (以前はManaged S3 for Audiencesと呼ばれていました)

Tealiumのデータベースソリューションの一つで、構造化データを使用してイベントレベルのデータを整理します。

EventStore (以前はManaged S3 for Eventsと呼ばれていました)

Tealiumのデータベースソリューションの一つで、半構造化データを使用してイベントレベルのデータを整理します。

Back to top

G

GDPR（一般データ保護規則）

欧州政府が欧州連合（EU）内のすべての個人のデータ保護をエンリッチメントし、統一することを目指すオプトインモデルの規則です。また、EU外への個人データの輸出にも対応しています。

GETリクエスト

GETリクエストは、サーバーからデータを取得するためのHTTPメソッド（HTTP参照）です。リクエストはURLのクエリストリングパラメータに含まれています。例えば、ブラウザはGETリクエストを使用して、ウェブサイトをホスティングしているサーバーから*.htmファイルを取得することができます。

Git

Gitはソフトウェア開発プロジェクトのためのソースコードバージョン管理システムです。

GitHub

GitHubは、Gitを使用したソフトウェア開発プロジェクトのコードリポジトリです。

GPC (グローバルプライバシーコントロール)

グローバルプライバシーコントロール (GPC)は、ユーザーが一般的に自分の個人データの販売や共有に同意しないことをドメイン間で示すことができる提案されたブラウザの標準です。

Back to top

H

HTML (ハイパーテキストマークアップ言語)

HTMLは、特にウェブブラウザでのウェブページを整理し表示するためのマークアップ言語です。

HTMLフッター

フッターエレメントは、通常ページの下部に位置し、ウェブページに関する追加情報（著者、著作権情報、連絡先情報など）を含みます。

HTMLヘッド

head要素はHTMLドキュメントの上部に位置しており、ページの説明情報、CSS情報、メタタグデータを含んでいます。

HTMLボディ

body要素は、テキスト、リンク、画像など、ウェブページの表示可能なコンテンツを含みます。一部のベンダータグなどの非表示コンテンツは通常、bodyに含まれます。Tealiumのコードはbody内に配置します。

HTTP (HyperText Transfer Protocol)

HTTPは、World Wide Web上でクライアントとサーバー間のデータ通信を行うためのさまざまな方法を定義しています。

Back to top

I

iframe

iframeは、その範囲内で別のウェブドキュメントを表示できるウェブページ要素です。これは、ページ内にページを表示するようなものです。ピクセルトラッキングのために、ピクセルリクエストの代わりにiframeリクエストを使用することができます。

iOS

Appleのモバイルオペレーティングシステム。

IoT (モノのインターネット)

物理的なデバイス、車両、建物、および電子機器、ソフトウェア、センサー、アクチュエータ、ネットワーク接続機能を組み込んだ他のアイテムの相互接続。これらのオブジェクトがデータを収集し、交換し、既存のネットワークインフラストラクチャを通じて遠隔で感知し、制御されることを可能にします。

iPaaS (統合プラットフォームサービス)

iPaaS、または統合プラットフォームサービスは、クラウド内、およびクラウドとエンタープライズ間の統合を構築し、デプロイするためのプラットフォームです。大企業は、エンタープライズレベルのシステムを運用している場合、公開クラウドとプライベートクラウドの両方でアプリケーションとローカルデータを統合するためにiPaaSを使用します。

J

JavaScript

インターネットで一般的に使用されるスクリプト言語。

JavaScriptページ変数

これは、UDOの外部に存在するウェブページ上の変数を識別するための変数タイプです。

JavaScriptオブジェクト表記法 (JSON)

人間が理解しやすいデータ伝送のためのオープンスタンダードフォーマット。JSONは、DataAccessがデータベーススキーマにデフォルトで使用する標準でもあります。

jQuery

jQueryは、クライアントサイドのHTMLスクリプティング用のJavaScriptライブラリです。

K

KPI (主要業績評価指標)

組織、従業員、または他の定義された項目がパフォーマンスの目標を達成するための成功を評価するために使用される定量的な指標。

M

MA (マーケティングオートメーション)

マーケティング部門や組織がオンラインの複数のチャネル（メール、ソーシャルメディア、ウェブサイトなど）でより効果的にマーケティングを行い、繰り返しのタスクを自動化するために設計されたソフトウェアプラットフォームと技術。高度なマーケティングオートメーションツールの助けを借りて、マーケターはキャンペーン管理、顧客データの区分と統合などのマーケティングプロセスを自動化することができ、これによりリードの質が向上し、売上が増加します。

MarTechスタック

顧客を獲得、維持、拡大するためのデジタルツールの集まり。

MFA (多要素認証)

コンピュータへのアクセス制御の方法で、ユーザーが認証メカニズムに対して複数の別々の証拠を正常に提示した後にのみアクセスが許可されます - 通常は以下のカテゴリーのうち少なくとも2つ：

  • 知識
    パスワードのような、彼らが知っているもの。
  • 所有
    特別なハードウェアキーフォブのような、彼らが持っているもの。
  • 固有性
    彼らのユニークな指紋認証のような、彼らがあるもの。
MVT (多変量テスト)

インターネットマーケティングでは、ウェブサイトの複数のコンポーネントをライブ環境でテストするプロセスのことを指します。これは、単純に言えば、1ページ上で同時に実行される複数のA/Bテストと考えることができます。A/Bテストは通常、2つのコンテンツバリエーションのうちどちらが優れているかを判断するために実行されます。多変量テストは、複数の変数を使用して理想的な組み合わせを見つけ出します。

P

PHI (保護された健康情報)

個々の健康や状態に関連するあらゆる健康情報、個々に対する健康ケアの提供、または個々への健康ケアの提供の支払いに関する情報。

PII (個人を特定できる情報)

特定の個人を識別できる可能性のある任意のデータ。一人を他の人と区別するために使用でき、匿名データの非匿名化に使用できる情報は、PIIと見なすことができます。

POSTリクエスト

POSTリクエストは、サーバーにデータを送信するためのHTTPメソッド（HTTP参照）です。データはPOSTリクエストの本文に含まれます。この方法は、HTMLフォームを介してユーザーパスワードなどの機密情報を送信するのに理想的です。

Back to top

Q

QA

QAはTealium iQの公開環境の一つであり、自身のQAまたはステージング環境にリンクすることを意図しています。

Back to top

S

SaaS（Software as a Service）

SaaS、またはSoftware as a Serviceは、ソフトウェアのローカルバージョンをインストールするのではなく、ウェブインターフェースを介してソフトウェアへのアクセスとしてサービスを購読する顧客に配信モデルです。

SDK (ソフトウェア開発キット)

SDKは、特定のプラットフォーム向けのソフトウェアを開発するための開発ツールのセットです。

SEM (検索エンジンマーケティング)

インターネットマーケティングの一形態で、主に有料広告を通じてウェブサイトの視認性を検索エンジンの結果ページ（SERPs）で高めることを含む。SEMは、ウェブサイトのコンテンツやサイト構造を調整または書き換えて検索エンジンの結果ページでのランキングを高め、ペイパークリック（PPC）リストをエンリッチメントするための検索エンジン最適化（SEO）を組み込むことがあります。

SKU（在庫管理単位）

販売のための明確な種類のアイテム、例えば製品やサービス、そしてそのアイテムの種類を他のアイテムの種類と区別するために関連するすべての属性。製品に対して、これらの属性には、製造元、説明、材料、サイズ、色、パッケージング、保証条件などが含まれるが、これに限定されるものではない。ビジネスが在庫を取るとき、それは各SKUの数量を数えます。

SSP（サプライサイドプラットフォーム）

デジタル広告インベントリの購入者が複数の広告交換およびデータ交換アカウントを管理できるシステム。世界の大手ウェブパブリッシャーの多くが、オンラインメディアスペースの販売を自動化し、最適化するためにサプライサイドプラットフォームを使用しています。

T

TLC (ティーリアム学習センター)

ティーリアムの顧客や従業員が交流し、解決策や経験を共有できるオンラインコミュニティです。学習センターには製品ガイド、開発者向けドキュメンテーション、フォーラム、教育とトレーニング、サポートデスクとプロダクションステータスページへのリンクが含まれています。

TMS (タグ管理システム)

ベンダータグを管理するためのSaaSシステム。

Back to top

U

UAT（ユーザー受け入れテスト）

ユーザーにとってテクノロジーソリューションが機能することを確認するプロセス。システムテストはソフトウェアがクラッシュせず、文書化された要件を満たすことを保証する一方で、UATはソリューションがユーザーにとって機能し、ユーザーがソリューションを受け入れることを保証します。ソフトウェアベンダーはこれを「BETAテスト」と呼ぶことがあります。

UDO (ユニバーサルデータオブジェクト)

ユニバーサルデータオブジェクト、またはUDOは、あなたのサイトで構成された utag_data という名前のデータオブジェクトで、あなたのTealium実装のリソースとなるよう構成されています。UDOの構成方法についての詳細は、ユニバーサルデータオブジェクトガイドをご覧ください。

URL (Uniform Resource Locator)

ウェブアドレスとも呼ばれるURLは、ウェブページなどのリソースを識別する一連の文字です。

utag.footer.js

utag.footer.jsファイルは、コンテンツ変更拡張を介して変更されたコンテンツを配信するためのutag.jsファイルの非推奨のバリアントです。

utag.js

utag.jsファイルには、サイト上でTealium実装をロードするために必要なすべての参照と構成が含まれています。

utag.sync.js

utag.sync.jsファイルは、サイトでJavaScriptを同期的にロードするためのオプションの解決策です。

UTUI (ユニバーサルタグユーザーインターフェース)

めったにTealium iQを参照するために使用されません。

W

Webhookコネクタ

Webhookコネクタは、Audienceデータをサーバーに配信するための一般的なHTTPリクエスト（GET、PUT、POSTリクエストなど）を行うサービスです。Tealiumは、WebhookとWebhookJの2つのWebhookコネクタを提供しています。

XML (拡張可能マークアップ言語)

XMLは、データを整理、転送、保存するためのマークアップ言語です。

XPath

XPathは、HTMLドキュメント内のルート要素またはノードに対する選択された要素の位置パス名です。

カ

カート回復

企業がカートを放棄したユーザーをターゲットにすると、そのユーザーはウェブサイトに戻ってカートに残された商品の購入を完了します。

カート放棄

誰かがウェブサイトを訪れて、オンラインカートに何かを入れ、購入せずにウェブサイトを退出すること。

Back to top

サ

サーバーサイドトラッキング

サーバーサイドの状況では、「セッション」とも呼ばれ、個々の情報はすべてサーバーに保管され、 いつでもアクセスできます。ユーザーのマシンに保存される唯一のデータは セッションIDで、これは広告主のサーバー上の追加データに接続できます。

Back to top

タ

ターゲット属性

ターゲット属性は、モデルを定義し、予測されるユーザーのアクションを表すために選択されたAudienceStream属性です。これらの属性は、アクションが実行されたことを示すために選択された訪問または訪問レベルのブール値です。例えば、“Has Purchased"というブール値の訪問属性は、訪問中に購入イベントが発生したことを示します。

Back to top

ア

アトリビューション

複数のウェブサイトを跨いで訪問のタッチポイントを追跡し、どのキャンペーンがコンバージョンにつながったかを判断します。

Back to top

バ

バージョン

バージョンは、Tealium実装のための構成の離散的なセットです。名前を付けて保存アクションを実行すると新しいバージョンが作成され、クライアント側で上書きアクションを実行するか、サーバー側で保存を行うと既存のバージョンが上書きされます。

ア

アプリケーションプログラミングインターフェース (API)

二つのソフトウェアが互いに通信することを可能にするプログラム。

マ

マーケティングテクノロジー (MarTech)

製品やサービスの宣伝や販売のためのデジタルツールの使用。

ア

アンドロイド

Googleのモバイルオペレーティングシステム。

Back to top

シ

シーケンス（ビジター属性）

シーケンスは、順序付けられたイベントのリストです。ファネルとは異なり、シーケンスには終了目標は必要ありません。このタイプの属性を使用して、サイトを通じた訪問の旅を追跡することができます。

ウ

ウェブコンパニオン

あなたのウェブサイトにTealiumをインストールするための無料ツールです。ウェブコンパニオンを使用すると、タグと拡張機能がウェブページにロードされていることを検査し、検証することができます。

Back to top

データ冗長性

我々が通常見る冗長性は、以下の2つの形態のいずれかで現れます：

  • 複数のソリューションを使用することに関連した冗長性
  • ページ上で既に利用可能なデータに関連した冗長性

Tealiumは、デフォルトで、Webページに複数のタグがある場合にデータ冗長性を有効にします。注文確認ページのすべてのデジタルマーケティングタグと、各タグが注文IDを構成するために異なる変数名を使用する方法を考えてみてください。ページに7つのタグがある場合、注文IDを参照する7つの異なる変数が存在します。しかし、Tealiumでは、utag_dataオブジェクトで一度注文ID変数を構成し、Tealiumがそれを7つの異なるベンダーに渡す処理を行います。これにより、ページの読み込み時間が短縮され、ページのパフォーマンスが向上します。

ビジネスで使用される用語で、顧客の期待、好み、嫌悪感を捉えるための詳細なプロセスを表します。具体的には、VoCは市場調査技術で、顧客の要望とニーズを詳細な階層構造にまとめ、相対的な重要性と現在の代替案に対する満足度の観点から優先順位をつけます。適切に行われると、VoCプログラムは顧客の経験についてリアルタイムの洞察を提供します。このフィードバックから、顧客体験を改善するためのトレンドと機会を特定することができます。

Tealium Predictでは、混同行列（エラー行列とも呼ばれます）は、実際の値と予測値を比較するモデルエクスプローラーで報告される訓練済みモデルのパフォーマンス測定です。業界用語では、混同行列は真の値が既知の一連のテストデータを使用し、実際の値と予測値をテーブル形式で表示して、特定のアルゴリズムのパフォーマンスを視覚化することを可能にします。

受信者動作特性 (ROC)

ROC/AUC（曲線下面積）は、訓練済みモデルのパフォーマンス測定を指します。業界用語では、ROCは真の陽性率として知られ、これは真の陽性数を真の陽性数と偽陰性数の合計で割ったものとして計算されます。ROCは、実際の結果が陽性である場合にモデルが陽性クラスをどれだけよく予測するかを説明します。真の陽性率は感度とも呼ばれます。

配

配列

配列とは、変数の値の集合です。例えば、product_idsという配列の変数は、複数の商品IDをリスト化し、各商品IDをその配列の要素に配置することができます。

product_ids=["ProdID001","ProdID002","ProdID003"]

配列を使用する一つの利点は、一つの配列の要素を別の配列と関連付けることができることです。例えば、product_idsとproduct_pricesのように、データを配列として格納する二つの変数がある場合、各配列の要素を関連付けて、最初の商品IDに最初の商品価格を関連付けることができます。

product_ids=["ProdID001","ProdID002","ProdID003"];
product_prices=["30.99","12.99","10.00"];

したがって、最初の商品の価格は$30.99、二つ目の商品の価格は$12.99、三つ目の商品の価格は$10.00となります。

予

予測期間

目標属性のアクションが発生する予測したい日数、週数、または月数の期間。例えば、ユーザーの「再訪の可能性」を次の「x」日、週、または月で予測します。

予測値

デプロイされたモデルによって生成されるスコアです。このスコアは、次回の訪問の訪問時（次回の訪問が予測時間窓内に発生すると仮定）にターゲット属性値の結果が真に構成される可能性を表しています。この値は、モデルの対応する出力属性に保存されます。

予測閾値

予測値を目標属性値と比較するために選択された数値閾値。例えば、予測閾値が0.5に構成され、予測値が0.51に構成された場合、目標属性値の結果は真とみなされます。
