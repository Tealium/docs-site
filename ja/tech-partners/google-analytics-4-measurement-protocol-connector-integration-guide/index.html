Tealium + Google Analytics 4 Measurement Protocol 統合ガイド

この記事では、Google Analytics 4 Measurement Protocol タグと Tealium を一緒に構成して使用する方法について説明します。

動作原理

Google Analytics 4 Measurement Protocol Connector を使用すると、開発者はイベントを直接 Google Analytics サーバーに送信する HTTP リクエストを行うことができます。これにより、開発者は任意の HTTP 対応環境からビジネスとのユーザーの相互作用を測定することができます。これにより、サーバー間で発生する相互作用を簡単に測定することができます。

Tealium EventStream は、Google Analytics 4 のトラッキングを有効にするためのコンジットとして機能します。これにより、以下のような機能が提供されます。

  • ウェブアプリケーションとアプリの両方をトラッキングします。
  • オンラインとオフラインの動作を関連付けます。
  • クライアントサイドとサーバーサイドの両方で相互作用を測定します。
  • オフラインの会話など、標準的なユーザーの相互作用の外で発生するイベントを送信します。

Tealium は、Google Analytics 4 Measurement Protocol Connector で以下のアクションをサポートしています。

  • ページビューイベントの送信
  • イベントの送信
  • Firebase イベントの送信

図に表示されているダイアグラムに表示されている Google Analytics 4 タグの追加の構成の詳細については、Google Analytics 4 (GA4) タグの構成ガイド を参照してください。

考慮事項

Google Analytics 4 コネクタの Google Analytics Measurement Protocol は、クライアントサイドのタグソリューションの完全な代替ではありません。

制限事項

  • セッションまたはユーザーデータが必要な場合は、クライアントサイドのタグが必要です。
  • user_id キーは、クライアントサイドと Measurement Protocol イベントの両方に user_id を含める必要があります。
  • 現在の GA4 Measurement Protocol API ではセッションデータはサポートされていません。自動的な Google Analytics セッションデータを取得するには、Google Analytics 4 (GA4) タグを使用する必要があります。
    • タグを有効にすると、session_start イベントと first_visit イベントも送信されます。
  • utm パラメータはサポートされていません。
  • 地理的データとユーザーエージェントは含まれていません。

認証

Measurement Protocol API は、URL のクエリパラメータとして含まれる測定 ID と API シークレットを使用します。

https://google-analytics.com/mp/collect?measurement_id=G-1234567890&api_secret=xxxxxxxxxx

構成

コネクタを構成するには、API シークレットとデータストリームの測定 ID または Firebase ID を入力します。

応答/デバッグ

Measurement Protocol は、HTTP リクエストが受信された場合は常に 2xx ステータスコードを返します。Measurement Protocol は、ペイロードデータが不正である場合や、ペイロード内のデータが正しくない場合や Google Analytics で処理されなかった場合にエラーコードを返しません。

Google はイベントを検証するための別のエンドポイントを提供しています。

https://google-analytics.com/debug/mp/collect?

Tealium Measurement Protocol コネクタは、イベントデータの検証プロセスを表示する Trace 中にこのエンドポイントを使用します。

推奨事項と構成の詳細

上記の制限事項を回避し、すべてのデータ要件を完全にカバーするためには、Tealium iQ タグ管理で Google Analytics 4 タグ を実装します。タグをすべてのページで読み込むように構成し、Send Page View を false に構成します。さらに、Google Analytics Cookie Matching Service タグを追加して、Google clientId を utag_main__ga クッキー値として自動的に追加します。

その後、Google Analytics コネクタで Google の自動収集イベント以外の必要なイベントを構成できます。

Google Analytics 4 タグは、order_id が存在する場合に自動的に購入イベントをトリガーします。E-commerce 拡張マッピングを介して購入を追跡する場合は、Set Data Values 拡張を Google Analytics 4 にスコープを構成して _corder 属性をクリアするように構成します。

Tealium は、クライアントサイドの実装をサポートしてウェブクライアントの統合をより良く対応するだけでなく、Tealium の EventStream を介してモバイルやオフラインイベントをサポートするため、両方のソリューションと組み合わせたアプローチを作成することができます。

クライアントサイドタグの構成の詳細については、Google Analytics 4 (GA4) タグの構成ガイド を参照してください。

ページビューイベントの送信

概要

このアクションは、ウェブサイトのページビューイベントを Google Analytics に送信するために使用されます。

ボディパラメータ

PARAMETER 説明
クライアント ID 必須。クライアントの一意の識別子。

この識別子は、Google Analytics 4 タグによって構成されたクライアント ID を使用する必要があります。詳細については、Google Analytics 4 (GA4) タグの構成ガイド を参照してください。
測定 ID のオーバーライド 測定 ID。たとえば、データストリームの識別子。
Firebase App ID のオーバーライド 必須。Firebase アプリの識別子。この構成は、構成セクションで使用される Firebase App ID を上書きします。
ユーザー ID ユーザーの一意の識別子。Google Analytics 4 Measurement Protocol イベントに user_id を含める場合は、クライアントサイドのイベントにも user_id を含める必要があります。
タイムスタンプマイクロ秒 イベントに関連付ける時間の Unix タイムスタンプ（マイクロ秒単位）。これを構成して、過去に発生したイベントを記録します。この値は user_property またはイベントのタイムスタンプを介して上書きすることができます。イベントは、プロパティのタイムゾーンに基づいて、カレンダー上の過去 3 日間まで遡ることができます。
パーソナライズされていない広告 これらのイベントがパーソナライズされた広告に使用されないことを示すために、これを true に構成します。

イベントパラメータ

PARAMETER 説明
ページタイトル ページのタイトル。
ページの場所 必須。ページの完全な URL。

追加のパラメータを渡すこともできますが、ページビューイベントのトラッキングを許可するためには必須ではありません。

Google ペイロード

次の例のペイロードに示すように、events.name の値は page_view であり、必須パラメータは page_location と client_id です。

{
  "events": [
    {
      "name": "page_view",
      "params": {
        "page_location": "http://demosite.com/ecomm/category.html",
        ...other mapped parameters...
      }
    }
  ],
  "client_id": "017f6ae7e7a40014b19d17739b3d05079002707100fb8"
}

イベントの送信

概要

このアクションは、Google Analytics に非ページビューイベントを送信するために使用されます。Google が定義したイベントのリストからイベントタイプを選択することができます（詳細については、Measurement Protocol (Google Analytics 4) Events を参照してください）。さらに、カスタムイベントを選択し、カスタムイベントの値を Custom Event Type にマッピングすることもできます。

ボディパラメータ

PARAMETER 説明
クライアント ID 必須。クライアントの一意の識別子。Google Analytics 4 タグによって構成されたクライアント ID を使用する必要があります。Cookie Matching タグが推奨される実装方法です。
測定 ID のオーバーライド 測定 ID。たとえば、データストリームの識別子。
Firebase App ID のオーバーライド 必須。Firebase アプリの識別子。この構成は、構成セクションで使用される Firebase App ID を上書きします。
ユーザー ID ユーザーの一意の識別子。Google Analytics 4 Measurement Protocol イベントに user_id を含める場合は、クライアントサイドのイベントにも user_id を含める必要があります。
タイムスタンプマイクロ秒 イベントに関連付ける時間の Unix タイムスタンプ（マイクロ秒単位）。過去に発生したイベントを記録するためにこれを構成します。この値は user_property またはイベントのタイムスタンプを介して上書きすることができます。イベントは、プロパティのタイムゾーンに基づいて、カレンダー上の過去 3 日間まで遡ることができます。
パーソナライズされていない広告 これらのイベントがパーソナライズされた広告に使用されないことを示すために、これを true に構成します。

アチーブメントの解除 ユーザーがアチーブメントを解除した場合。ユーザーがゲームをどのように体験しているかをよりよく理解するために、このイベントを使用します。 イベントパラメータ:
  • アチーブメント ID（必須）
カートの表示 ユーザーがカートを表示した場合。 イベントパラメータ:
  • 通貨（必須）
  • 値（必須）
アイテムパラメータ:
  • アイテム ID（必須）
  • アイテム名（必須）
アイテムの表示 ユーザーが特定のコンテンツを表示した場合。最も人気のある表示されたアイテムを特定するために、このイベントを使用します。 イベントパラメータ:
  • 通貨（必須）
  • 値（必須）
アイテムパラメータ:
  • アイテム ID（必須）
  • アイテム名（必須）
アイテムリストの表示 特定のカテゴリのアイテムのリストをユーザーが表示した場合。 イベントパラメータ:
  • アイテムリスト ID（必須）
  • アイテム名（必須）
アイテムパラメータ:
  • アイテム ID（必須）
  • アイテム名（必須）
プロモーションの表示 ユーザーがリストからプロモーションを表示した場合。 イベントパラメータ:
  • クリエイティブ名（オプション）
  • クリエイティブスロット（オプション）
  • ロケーション ID（オプション）
  • プロモーション ID（オプション）
  • プロモーション名（オプション）
アイテムパラメータ:
  • アイテム ID（必須）
  • アイテム名（必須）
検索結果の表示 ユーザーが検索結果を表示した場合。 イベントパラメータ:
  • 検索語句（オプション）
カスタムイベント Google の事前構成されたイベントで定義されていないカスタムイベントをサポートするために、このイベントを使用します。 このイベントには必須パラメータはありません。

サポートされているイベントの完全なリストについては、Measurement Protocol - Google Analytics 4 events を参照してください。

Firebase イベントの送信

概要

このアクションは、Firebase イベントを Google Analytics に送信するために使用されます。構成は Send Event と同じですが、App Instance ID も提供する必要があります。

このコネクタアクションのイベントタイプは、主に Firebase にパスするためのモバイルアプリケーションデータのトラッキングに使用されます。

PARAMETERS 説明
App Instance ID 必須。Firebase アプリの特定のインストールを一意に識別します。この値は Firebase SDK を介して取得できます。
Firebase App ID のオーバーライド 必須。Firebase アプリの識別子。この構成は、構成セクションで使用される Firebase App ID を上書きします。
ユーザー ID (オプション) ユーザーの一意の識別子。
タイムスタンプマイクロ秒 (オプション) イベントに関連付ける時間の Unix タイムスタンプ（マイクロ秒単位）。
パーソナライズされていない広告 (オプション) これを true に構成すると、これらのイベントはパーソナライズされた広告に使用されないことを示します。

イベントパラメータ

このアクションは、Send Event アクションと同じ一般的なイベントパラメータをサポートしており、同じイベント固有のパラメータもサポートしています。

ユーザープロパティ

ユーザープロパティは、言語構成や地理的位置など、ユーザーベースのセグメントを説明します。Analytics は一部のユーザープロパティを自動的にログします。追加のプロパティを収集する場合は、プロジェクトごとに最大 25 個の追加のユーザープロパティを構成できます。GA4 カスタムディメンションとメトリックおよびFirebase カスタムユーザープロパティの構成と登録方法については、GA4 カスタムディメンションおよびメトリックとFirebase カスタムユーザープロパティを参照してください。
