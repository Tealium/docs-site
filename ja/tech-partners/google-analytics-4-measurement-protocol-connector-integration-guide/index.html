Tealium + Google Analytics 4 測定プロトコル統合ガイド

この記事では、Google Analytics 4 測定プロトコルタグとコネクタをTealiumと一緒に構成し使用する方法について説明します。

仕組み

Google Analytics 4 測定プロトコルコネクタは、開発者がHTTPリクエストを作成し、イベントを直接Google Analyticsサーバーに送信することを可能にします。これにより、開発者はHTTP対応環境からビジネスとのユーザーのやり取りを測定することが容易になり、サーバー間のやり取りを簡単に測定できます。

Tealium EventStreamは、Google Analytics 4のトラッキングを可能にするためのパイプ役を果たします。モバイル、IoT、オフラインイベントおよびページビュートラッキングのサポートを統合することで：

  • ウェブアプリケーションとアプリの両方をトラッキングします。
  • オンラインとオフラインの行動を結びつけます。
  • クライアントサイドとサーバーサイドの両方でのやり取りを測定します。
  • 標準的なユーザーのやり取りの外で発生するイベントを送信します。例えば、オフラインでの会話など。

Tealiumは、Google Analytics 4 測定プロトコルコネクタと共に以下のアクションをサポートします：

  • ページビューイベントの送信
  • イベントの送信
  • Firebaseイベントの送信

図に表示されるGoogle Analytics 4タグの追加構成詳細については、Google Analytics 4 (GA4) タグ構成ガイドを参照してください。

考慮事項

Google Analytics 4コネクタのGoogle Analytics測定プロトコルは、クライアントサイドのタグソリューションを完全に置き換えるものではありません。

制限事項

  • セッションデータやユーザーデータが必要な場合、クライアントサイドのタグが必要です。
  • user_idキーは、クライアントサイドと測定プロトコルのイベントの両方でuser_idを含める必要があります。
  • セッションデータは、現在のGA4測定プロトコルAPIではサポートされていません。自動的なGoogle Analyticsセッションデータを取得するには、Google Analytics 4 (GA4) タグを使用する必要があります。
    • タグを有効にすると、session_startとfirst_visitのイベントも送信されます。
  • パラメータ値は100文字に制限されています。100文字を超える値は、最終的なGA4レポートから切り捨てられるか、削除されます。
  • utmパラメータをサポートしていません。
  • キャンペーンやトラフィックソースデータの送信をサポートしていません。
  • 地理的なデータやユーザーエージェントを含んでいません。

認証

測定プロトコルAPIは、URLのクエリパラメータとして含まれる測定IDとAPIシークレットを使用します。

https://google-analytics.com/mp/collect?measurement_id=G-1234567890&api_secret=xxxxxxxxxx

構成

コネクタを構成するには、APIシークレットとデータストリームの測定IDまたはFirebase IDを入力します。

レスポンス/デバッグ

測定プロトコルは、HTTPリクエストが受信された場合は常に2xxのステータスコードを返します。ペイロードデータが不正な形式であった場合、またはペイロードのデータが不正であったりGoogle Analyticsによって処理されなかった場合でも、測定プロトコルはエラーコードを返しません。

Googleは、イベントの検証用に別のエンドポイントを提供しています：

https://google-analytics.com/debug/mp/collect?

Tealiumの測定プロトコルコネクタは、このエンドポイントをTrace中に使用し、イベントデータの検証プロセスを表示します。

推奨事項と構成詳細

制限事項を緩和し、すべてのデータ要件を完全にカバーするために、Tealium iQタグ管理でGoogle Analytics 4タグを実装します。すべてのページでタグをロードし、ページビューの送信をfalseに構成します。さらに、GoogleのclientIdが自動的にutag_main__gaクッキー値として追加されるように、Google Analytics Cookie Matching Serviceタグを追加します。

Googleの自動収集イベント以外のすべての必要なイベントは、Tealium EventStreamのGoogle Analyticsコネクタで構成できます。

Google Analytics 4タグは、E-commerce拡張マッピングを通じてorder_idが存在するときに自動的に購入イベントをトリガーします。E-commerce拡張を有効にしていて、EventStreamを通じて購入をトラッキングしたい場合は、Google Analytics 4にスコープを構成したデータ値構成拡張を構成して、_corder属性をクリアします。

Tealiumは、ウェブクライアントの統合をより適切に対応するクライアントサイドの実装をサポートすることができるため、またTealiumのEventStreamを通じてモバイルやオフラインのイベントをサポートすることができるため、Tealiumは両方のソリューションと統合し、組み合わせたアプローチを作成することができます。

クライアントサイドタグの構成についての詳細情報は、Google Analytics 4 (GA4) タグ構成ガイドを参照してください。

PageViewイベントの送信

概要

このアクションは、ウェブサイトのページビューイベントをGoogle Analyticsに渡すために使用されます。

ボディパラメータ

パラメータ 説明
クライアントID 必須。クライアントの一意の識別子。

この識別子は、Google Analytics 4タグによって構成されたクライアントIDを使用する必要があります。詳細については、Google Analytics 4 (GA4) タグ構成ガイドを参照してください。
測定IDの上書き 測定ID。例えば、データストリームの識別子。
FirebaseアプリIDの上書き 必須。Firebaseアプリの識別子。この構成は、構成セクションで使用されるFirebaseアプリIDを上書きします。
ユーザーID ユーザーの一意の識別子。Google Analytics 4測定プロトコルイベントに user_id を含める場合、クライアントサイドイベントにもそれを含める必要があります。
タイムスタンプマイクロ イベントに関連付ける時間のUnixタイムスタンプ（マイクロ秒）。過去に発生したイベントを記録するためにこれを構成します。この値は user_property またはイベントタイムスタンプを介して上書きすることができます。イベントは、プロパティのタイムゾーンに基づいて最大3日間遡って日付を付けることができます。
非パーソナライズ広告 これらのイベントがパーソナライズ広告に使用されないことを示すために、これをtrueに構成します。

イベントパラメータ

パラメータ 説明
ページタイトル ページのタイトル。
ページロケーション 必須。ページへの完全なURL。

追加のパラメータを渡すことができますが、ページビューイベントの追跡を許可するために必要ではありません。

Google Payload

以下の例のペイロードに示すように、events.name の値は page_view で、必須パラメータは page_location と client_id です：

{
  "events": [
    {
      "name": "page_view",
      "params": {
        "page_location": "http://demosite.com/ecomm/category.html",
        ...other mapped parameters...
      }
    }
  ],
  "client_id": "017f6ae7e7a40014b19d17739b3d05079002707100fb8"
}

イベントの送信

概要

このアクションは、ページビュー以外のイベントをGoogle Analyticsに送信するために使用されます。Googleが定義したイベントのリストからイベントタイプを選択できます（詳細については、Measurement Protocol (Google Analytics 4) Eventsを参照してください）。また、カスタムイベントを選択し、カスタムイベントの値をカスタムイベントタイプにマッピングすることもできます。

ボディパラメータ

パラメータ 説明
クライアントID 必須。クライアントの一意の識別子。Google Analytics 4タグによって構成されたクライアントID。Cookie Matching Tagが推奨される実装方法です。
測定IDの上書き 測定ID。例えば、データストリームの識別子。
FirebaseアプリIDの上書き 必須。Firebaseアプリの識別子。この構成は、構成セクションで使用されるFirebaseアプリIDを上書きします。
ユーザーID ユーザーの一意の識別子。Google Analytics 4測定プロトコルイベントに user_id を含める場合、クライアントサイドイベントにもそれを含める必要があります。
タイムスタンプマイクロ イベントに関連付ける時間のUnixタイムスタンプ（マイクロ秒）。過去に発生したイベントを記録するためにこれを構成します。この値は user_property またはイベントタイムスタンプを介して上書きすることができます。イベントは、プロパティのタイムゾーンに基づいて最大3日間遡って日付を付けることができます。
非パーソナライズ広告 これらのイベントがパーソナライズ広告に使用されないことを示すために、これをtrueに構成します。

チュートリアル開始 ユーザーがオンボーディングプロセスを開始した場合。このイベントをtutorial_completeと一緒にファネルで使用して、チュートリアルを完了するユーザーの数を測定します。 このイベントに必要なパラメータはありません。
チュートリアル完了 ユーザーがオンボーディングプロセスを完了した場合。このイベントをtutorial_beginと一緒にファネルで使用して、チュートリアルを完了するユーザーの数を測定します。 このイベントに必要なパラメータはありません。
実績解除 ユーザーが実績を解除した場合。このイベントを使用して、ユーザーがゲームをどのように体験しているかをよりよく理解します。 イベントパラメータ:
  • Achievement ID (必須)
カート表示 ユーザーがカートを表示した場合。 イベントパラメータ:
  • Currency (必須)
  • Value (必須)
アイテムパラメータ:
  • Item ID (必須)
  • Item Name (必須)
アイテム表示 ユーザーが特定のコンテンツを表示した場合。このイベントを使用して、最もよく表示されるアイテムを特定します。 イベントパラメータ:
  • Currency (必須)
  • Value (必須)
アイテムパラメータ:
  • Item ID (必須)
  • Item Name (必須)
アイテムリスト表示 ユーザーが特定のカテゴリーのアイテムのリストを表示した場合。 イベントパラメータ:
  • Item List ID (必須)
  • Item Name (必須)
アイテムパラメータ:
  • Item ID (必須)
  • Item Name (必須)
プロモーション表示 ユーザーがリストからプロモーションを表示した場合。 イベントパラメータ:
  • creative name (任意)
  • Creative Slot (任意)
  • Location ID (任意)
  • Promotion ID (任意)
  • Promotion Name (任意)
アイテムパラメータ:
  • Item ID (必須)
  • Item Name (必須)
検索結果表示 ユーザーに検索結果が表示された場合。 イベントパラメータ:
  • Search Term (任意)
カスタムイベント Googleのプリセットイベントで定義されていないカスタムイベントをサポートするためのこのイベントを使用します。 このイベントに必要なパラメータはありません。

サポートされているイベントの完全なリストを表示するには、Measurement Protocol - Google Analytics 4 eventsを参照してください。

Firebaseイベントの送信

概要

このアクションは、FirebaseイベントをGoogle Analyticsに送信するために使用されます。構成はSend Eventと同じですが、App Instance IDも提供する必要があります。

このコネクタアクションイベントタイプは、主にモバイルアプリケーションデータをFirebaseに渡して分析するために使用されます。

パラメータ 説明
App Instance ID 必須。Firebaseアプリの特定のインストールを一意に識別します。この値はFirebase SDKを通じて取得できます。
Firebase App ID Override 必須。Firebaseアプリの識別子。この構成はConfigurationセクションで使用されるFirebase App IDを上書きします。
User ID (任意) ユーザーの一意の識別子。
Timestamp Micros (任意) イベントに関連付ける時間のUnixタイムスタンプ（マイクロ秒）。
Non Personalized Ads (任意) これらのイベントがパーソナライズされた広告に使用されないことを示すために、これをtrueに構成します。

イベントパラメータ

このアクションは、Send Eventアクションと同じ一般的なイベントパラメータをサポートし、同じイベント固有のパラメータもサポートします。

ユーザープロパティ

ユーザープロパティは、言語の構成や地理的な位置など、ユーザーベースのセグメントを記述します。Analyticsは自動的に一部のユーザープロパティをログに記録します。追加のプロパティを収集したい場合は、プロジェクトごとに最大25の追加ユーザープロパティを構成できます。ユーザープロパティの構成と登録方法については、GA4 Custom dimensionsとメトリクスとFirebase Custom user propertiesを参照してください。
