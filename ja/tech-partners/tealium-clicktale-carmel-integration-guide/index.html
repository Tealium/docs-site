Tealium + Clicktale Carmel 統合ガイド

この記事では、iQタグ管理（TiQ）アカウントでClicktaleタグを構成する方法について説明します。

TealiumタグマーケットプレイスからのClicktaleタグとiQタグ管理の正常な統合により、顧客はクリックストリームデータに関連するセッションの再生をシームレスに実行し、タグ付けされたWebページに関連するエンド顧客の摩擦を検出し、最小限に抑えることができます。

前提条件

  • Tealium
    • iQタグ管理
    • AudienceStream
    • EventStream（オプション）
  • Clicktale
    • Clicktaleのリアルタイム信号は、データエクスポートへのアクセスを持つすべての顧客に利用可能です。

仕組み

Clicktaleは、ブランドがユニークな行動洞察に基づいて行動し、ユーザーエクスペリエンスを計測可能な利点に変えることを可能にします。Contentsquare社のClicktaleについての詳細は、contentsquare.comをご覧ください。

不正行為

  • 過度なペースト
    同一の訪問セッション内で異常で潜在的に不正なペースト行動を特定する閾値が満たされたときにトリガーされます。不正の可能性は3つのレベルがあります：低、中、高。
  • 過度なリロード
    同一の訪問セッション内で異常で潜在的に不正なリロード行動を特定する閾値が満たされたときにトリガーされます。不正の可能性は3つのレベルがあります：低、中、高。

広告ブロック

  • Ad Blocker Signal
    特定のファイルがブロックされているかどうかを特定し、広告ブロッカーが存在することを特定します。

EventStreamイベント仕様とAudienceStream訪問属性からの信号をオプションで使用することができます。

タグのヒント

  • 注文IDが構成されるとコンバージョンが発生します。
  • Clicktale Replay LinkをUDHに送信するには、以下の手順を実行する必要があります：
    • 再生リンクの生成を有効にするために、Clicktaleの担当者に連絡します。
    • Send Replay Link構成をTrueに構成します。
  • Contentsquareタグは、ページ上でロードされるTealiumアカウントとプロファイルを自動的に検出し、再生リンクをそこに送信します。再生リンクを別のアカウントまたはプロファイルに送信するには、tealium_accountとtealium_profileの目的地にそれぞれ希望するアカウントとプロファイルをマッピングしてデフォルトの値を上書きします。
  • Tealium AudiencesとBadgesをClicktaleに渡すには：
    • このデータの受信を有効にするために、Clicktaleの担当者に連絡します。
    • ClicktaleにBadgeを送信するには、マッピングツールボックスのBadgesタブに移動し、ClicktaleシステムでBadgeが識別される名前を入力します。
    • すべてのAudiencesをClicktaleに送信するには、Send UDH AudiencesのトグルをTrueに切り替えます。

タグ構成

まず、Tealiumのタグマーケットプレイスに移動し、Clicktale Carmelタグを追加します（タグの追加方法について詳しくはこちらをご覧ください）。

タグを追加した後、Data Mappingsセクションで以下の構成を行います。

ロードルール

ロードルールは、このタグのインスタンスをいつ、どこでロードするかを決定します。‘すべてのページでロード’ルールはデフォルトのロードルールです。このタグを特定のページでロードするには、関連する条件を持つ新しいロードルールを作成します。

  • 訪問のマウスの動きを追跡したいページでこのタグをロードします。

データマッピング

マッピングは、データレイヤー変数からベンダータグの対応する宛先変数にデータを送信するプロセスです。変数をタグの宛先にマッピングする方法については、データマッピングを参照してください。

利用可能なカテゴリーは次のとおりです：

標準

変数 説明
title
  • 必須。
  • タグインスタンスを識別します。
  • Clicktale Carmelがデフォルトの名前です。
  • 同じベンダーの複数のタグを使用する場合は、一意の名前を割り当てます。
partition
  • パーティション。
  • データが送信されるパーティション。
  • サーバーパーティション値を動的に構成するために、この変数にマッピングします。
project_guid
  • プロジェクトGUID。
  • プロジェクトガイドフィールドを構成するために、この変数にマッピングします。
send_replay_link
  • Clicktale Replay Linkを送信
  • 値はtrueまたはfalseです。
send_udh_audiences
  • UDH Audiencesを送信。
  • 値はtrueまたはfalseです。
tealium_account
  • Tealiumアカウント。
tealium_profile
  • Tealiumプロファイル。

E-Commerce

変数 説明
order_id
  • 注文ID。
  • _corderを上書きします。
  • トランザクションに必要です。
order_total
  • 注文合計。
  • _ctotalを上書きします。
  • トランザクションに必要です。
order_shipping
  • 送料。
  • _cshipを上書きします。
order_tax
  • 税金。
  • _ctaxを上書きします。
product_id
  • 配列
  • 製品IDのリスト。
  • _cprodを上書きします。
product_name
  • 配列
  • 名前のリスト。
  • _cprodnameを上書きします。
  • トランザクションに必要です。
product_sku
  • 配列
  • SKUのリスト。
  • _cskuを上書きします。
  • トランザクションに必要です。
product_category
  • 配列
  • カテゴリのリスト。
  • _ccatを上書きします。
product_quantity
  • 配列
  • 数量のリスト。
  • _cquanを上書きします。
  • トランザクションに必要です。
product_unit_price
  • 配列
  • 価格のリスト。
  • _cpriceを上書きします。
  • トランザクションに必要です。

Badges

変数 説明
title
  • Badge識別子。
  • マッピングされたバッジをClicktaleに送信するには、バッジがClicktaleで識別される名前を入力します。
  • Tealiumは自動的にバッジのタイトルを新しいオブジェクトsend_udh_dataの子プロパティとして含めます
  • バッジの名前はsend_udh_dataにマッピングされた状態で表示されます。
