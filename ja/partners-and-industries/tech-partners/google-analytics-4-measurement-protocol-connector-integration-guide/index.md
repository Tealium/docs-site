## 動作原理

[Google Analytics 4 測定プロトコルコネクタ]()は、開発者がHTTPリクエストを行い、イベントを直接Google Analyticsサーバーに送信することを可能にします。これにより、開発者はHTTPが有効な環境からビジネスとのユーザーのやり取りを測定でき、サーバー間でのやり取りが発生する場合でも簡単に測定できます。

Tealium EventStreamは、Google Analytics 4のトラッキングを可能にするための伝送路として機能します。モバイル、IoT、オフラインイベントおよびページビューのトラッキングをサポートすることにより、以下を実現します：

* Webアプリケーションとアプリの両方をトラッキング。
* オンラインとオフラインの行動を結びつける。
* クライアントサイドとサーバーサイドの両方でのやり取りを測定。
* 標準的なユーザーインタラクション以外で発生するイベントを送信。

![](/images/server-side-connectors/googleanalytics4measurementprotocoldiagram.png)

Tealiumは、Google Analytics 4 測定プロトコルコネクタで以下のアクションをサポートしています：

* ページビューイベントを送信
* イベントを送信
* Firebaseイベントを送信

図に表示されているGoogle Analytics 4タグの追加の構成詳細については、[Google Analytics 4 (GA4) タグ構成ガイド]()を参照してください。

### 考慮事項

Google Analytics 4の測定プロトコルコネクタは、クライアントサイドタグソリューションの完全な代替品ではありません。

### 制限事項

* セッションまたはユーザーデータが必要な場合、クライアントサイドタグが必要です。
* `user_id`キーは、クライアントサイドと測定プロトコルのイベントの両方に`user_id`を含める必要があります。
* 現在のGA4測定プロトコルAPIではセッションデータはサポートされていません。自動的なGoogle Analyticsセッションデータを取得するには、Google Analytics 4 (GA4) タグを使用する必要があります。
  * タグを有効にすると、`session_start`および`first_visit`イベントも送信されます。
* パラメータ値は100文字に制限されています。100文字を超える値は、最終的なGA4レポートから切り捨てられるか、削除されます。
* `utm`パラメータはサポートされていません。
* キャンペーンおよびトラフィックソースデータの送信はサポートされていません。
* 地理データおよびユーザーエージェントを含まない。

## 認証

測定プロトコルAPIは、URLのクエリパラメータとして含まれる測定IDとAPIシークレットを使用します。

```
https://google-analytics.com/mp/collect?measurement_id=G-1234567890&amp;api_secret=xxxxxxxxxx
```

## 構成

コネクタを構成するには、APIシークレットとデータストリームの測定IDまたはFirebase IDを入力します。

## レスポンス/デバッグ

測定プロトコルは、HTTPリクエストが受信された場合に常に`2xx`ステータスコードを返します。ペイロードデータが不正であったり、ペイロードのデータが誤っていたり、Google Analyticsによって処理されなかった場合、測定プロトコルはエラーコードを返しません。

Googleはイベントの検証用に別のエンドポイントを提供しています：

```
https://google-analytics.com/debug/mp/collect?
```

Tealium測定プロトコルコネクタは、このエンドポイントをTrace中に使用し、イベントデータの検証プロセスを表示します。

### 推奨事項と構成の詳細

指摘された制限を緩和し、すべてのデータ要件を完全にカバーするために、Tealium iQタグ管理で[Google Analytics 4タグ]()を実装し、すべてのページでタグをロードするように構成します。**Send Page View**は`false`に構成します。さらに、Google `clientId`が自動的に`utag_main__ga`クッキー値として追加されるように、Google Analytics Cookie Matching Serviceタグを追加します。

Googleの自動収集イベント以外の必要なイベントは、Tealium EventStreamのGoogle Analyticsコネクタで構成できます。

Google Analytics 4タグは、E-commerce拡張マッピングを通じて`order_id`が存在する場合に自動的に購入イベントをトリガーします。E-commerce拡張機能を有効にしており、EventStreamを通じて購入を追跡したい場合は、Google Analytics 4にスコープされた[Set Data Values]()拡張機能を構成して、`_corder`属性をクリアします。

![](/images/server-side-connectors/googleanalytics4measurementprotocolsetdatavalues.png)

Tealiumは、Webクライアント統合に対応するクライアントサイド実装をサポートするだけでなく、TealiumのEventStreamを通じてモバイルおよびオフラインイベントもサポートするため、両方のソリューションと統合し、組み合わせたアプローチを作成できます。

クライアントサイドタグの構成に関する詳細情報については、[Google Analytics 4 (GA4) タグ構成ガイド]()を参照してください。

## PageViewイベントの送信

### 概要

このアクションは、ウェブサイトのページビューイベントをGoogle Analyticsに渡すために使用されます。

### ボディパラメータ

| パラメータ | 説明 |
| --- | --- |
| クライアントID | （必須）クライアントの一意の識別子です。&lt;br&gt;&lt;br&gt;この識別子は、Google Analytics 4タグによって構成されたクライアントIDを使用する必要があります。詳細については、[Google Analytics 4 (GA4) タグ構成ガイド]()を参照してください。|
| 測定IDオーバーライド | 測定IDです。例えば、データストリームの識別子です。|
| FirebaseアプリIDオーバーライド | （必須）Firebaseアプリの識別子です。この構成は**構成**セクションで使用されるFirebaseアプリIDを上書きします。|
| ユーザーID | ユーザーの一意の識別子です。Google Analytics 4測定プロトコルイベントに`user_id`を含める場合、クライアントサイドのイベントにもそれを含める必要があります。|
| タイムスタンプマイクロ秒 | イベントに関連付ける時間のUnixタイムスタンプ（マイクロ秒単位）です。過去に発生したイベントを記録するためにこの値を構成します。この値は`user_property`またはイベントのタイムスタンプを介して上書きすることができます。イベントは、プロパティのタイムゾーンに基づいて最大3カレンダー日まで遡って日付を構成することができます。|
| 非パーソナライズ広告 | これらのイベントがパーソナライズ広告に使用されないことを示すために、これをtrueに構成します。|

### イベントパラメータ

| パラメータ | 説明 |
| --- | --- |
| ページタイトル | ページのタイトルです。 |
| ページ位置 | （必須）ページの完全なURLです。 |

追加のパラメータを渡すことができますが、ページビューイベントの追跡には必要ありません。

#### Googleペイロード

次の例のペイロードに示されているように、`events.name`の値は`page_view`であり、必要なパラメータは`page_location`と`client_id`です：

```json
{
  &#34;events&#34;: [
    {
      &#34;name&#34;: &#34;page_view&#34;,
      &#34;params&#34;: {
        &#34;page_location&#34;: &#34;http://demosite.com/ecomm/category.html&#34;,
        ...他のマップされたパラメータ...
      }
    }
  ],
  &#34;client_id&#34;: &#34;017f6ae7e7a40014b19d17739b3d05079002707100fb8&#34;
}
```

## イベントの送信

### 概要

このアクションは、Google Analyticsに非ページビューイベントを送信するために使用されます。Googleが定義したイベントタイプのリストからイベントタイプを選択することができます（詳細については、[Measurement Protocol (Google Analytics 4) Events](https://developers.google.com/analytics/devguides/collection/protocol/ga4/reference/events)を参照してください）。さらに、カスタムイベントを選択し、カスタムイベント値を**カスタムイベントタイプ**にマッピングすることができます。

| パラメータ | 説明 |
| --- | --- |
| クライアントID | （必須）クライアントの一意の識別子です。Google Analytics 4タグによって構成されたクライアントIDです。Cookie Matching Tagが推奨される実装方法です。|
| 測定IDオーバーライド | 測定IDです。例えば、データストリームの識別子です。|
| FirebaseアプリIDオーバーライド | （必須）Firebaseアプリの識別子です。この構成は**構成**セクションで使用されるFirebaseアプリIDを上書きします。|
| ユーザーID | ユーザーの一意の識別子です。Google Analytics 4測定プロトコルイベントに`user_id`を含める場合、クライアントサイドのイベントにもそれを含める必要があります。|
| タイムスタンプマイクロ秒 | イベントに関連付ける時間のUnixタイムスタンプ（マイクロ秒単位）です。過去に発生したイベントを記録するためにこの値を構成します。この値は`user_property`またはイベントのタイムスタンプを介して上書きすることができます。イベントは、プロパティのタイムゾーンに基づいて最大3カレンダー日まで遡って日付を構成することができます。|
| 非パーソナライズ広告 | これらのイベントがパーソナライズ広告に使用されないことを示すために、これをtrueに構成します。|

## Firebaseイベントの送信


### 概要

このアクションは、FirebaseイベントをGoogleアナリティクスに送信するために使用されます。構成は「イベント送信」と同じですが、アプリインスタンスIDも提供する必要があります。

このコネクタアクションイベントタイプは、主にモバイルアプリケーションデータをFirebaseに渡して分析するために使用されます。

| パラメータ | 説明 |
| --- | --- |
| アプリインスタンスID | (必須) Firebaseアプリの特定のインストールを一意に識別します。この値はFirebase SDKを通じて取得できます。 |
| FirebaseアプリIDオーバーライド | (必須) Firebaseアプリの識別子です。この構成は**構成**セクションで使用されるFirebaseアプリIDを上書きします。 |
| ユーザーID (オプション) | ユーザーの一意の識別子です。 |
| タイムスタンプマイクロ秒 (オプション) | イベントに関連付ける時間のUnixタイムスタンプ（マイクロ秒単位）。 |
| 非パーソナライズ広告 (オプション) | これらのイベントがパーソナライズされた広告に使用されないことを示すために、これをtrueに構成します。 |

### イベントパラメータ

このアクションは、「イベント送信」アクションと同じ一般的なイベントパラメータをサポートし、同じイベント固有のパラメータもサポートします。

### ユーザープロパティ

ユーザープロパティは、言語の好みや地理的な位置など、ユーザーベースのセグメントを記述します。アナリティクスは[自動的にいくつかのユーザープロパティを記録します](https://support.google.com/analytics/answer/9268042)。追加のプロパティを収集したい場合は、プロジェクトごとに最大25の追加ユーザープロパティを構成できます。[GA4カスタムディメンション](https://support.google.com/analytics/answer/9269570)とメトリクス、および[Firebaseカスタムユーザープロパティ](https://developers.google.com/analytics/devguides/collection/protocol/ga4/user-properties?client_type=firebase#example_usage)について読んで、ユーザープロパティの構成と登録方法を学びましょう。
