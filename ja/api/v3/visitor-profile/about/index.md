## 要件

この機能には以下が必要です：

* サーバーサイドのリーダー、エディター、またはパブリッシャーのレガシーパーミッション、または
* 最低限の**ビュー**ビジタールックアッププラットフォームパーミッション

## 仕組み

このAPIを使用すると、特定の訪問について既知のデータを取得し、ウェブサイトへの初回訪問のパーソナライゼーションを向上させることができます。ビジターの検索は、アカウントからの[Visitor ID]()属性に基づいています。

## レート制限

ビジタープロファイルAPIのデフォルトのレート制限は50リクエスト/秒です。より高いレート制限が必要な場合は、予想される使用量を添えてTealiumのアカウントマネージャーに連絡してください。

## 認証

ベアラートークンは、すべてのAPI呼び出しを認証するために使用され、APIキーではありません。APIキーは認証呼び出しでのみ使用されます。ベアラートークンに加えて、認証応答には、後続のサーバーサイドAPI呼び出しで使用する必要がある地域固有のホスト名が含まれています。

APIキーからベアラートークンを生成する方法については、[認証]()を参照してください。
