Tealiumは、リアルタイムデータワークフローにAIモデルを統合するための2つのアプローチをサポートしています：

* **AIコネクタ**：サポートされているAIプロバイダーと統合し、プロンプトベースのLLMインタラクションまたは直接モデルエンドポイントの呼び出しを行います。
* **Tealium機能**：独自のモデルを呼び出すか、カスタムエンドポイントを呼び出します。

このガイドを使用して、各アプローチの動作と使用タイミングを理解してください。

## AIコネクタ

### 動作方法

AIコネクタは、サポートされているプロバイダーのホストされたAIサービスと統合します。2つのインタラクションパターンをサポートしています：

* **プロンプトベース（LLM）**：プロンプトとTealiumデータをLLMエンドポイントに送信します。モデルは構造化されたJSONレスポンスを返し、それがTealium Collectに転送されてプロファイルが豊かになります。
* **モデルエンドポイントの呼び出し**：Tealiumのイベント属性または訪問属性を直接デプロイされたモデルエンドポイント（例：Snowflake Model Registryエンドポイント）に送信します。モデルは予測またはスコアを返し、それがTealium Collectに転送されてプロファイルが豊かになります。

AIコネクタを使用するには、プロバイダーの認証資格情報と、モデルに送信するTealiumデータを定義する少なくとも1つの構成済みアクションが必要です。

AIコネクタは、構造化された予測可能な出力を返すユースケース用に設計されています。長文のコンテンツやメディア生成には向いていません。自由形式の出力が必要なユースケースの場合は、本番環境に移行する前に、組織のAIガバナンスプロセスに沿ってください。

### データフロー

![](/images/guides/ai-connectors-flow.png)

1. **トリガーが発火**：イベントまたはオーディエンスの変更がコネクタアクションをトリガーします。
1. **リクエスト送信**：コネクタはプロンプトまたは入力データをAIモデルエンドポイントに送信します。
1. **レスポンス受信**：モデルは要求された値を生成し、JSONオブジェクトを返します。
1. **イベント取り込み**：コネクタはレスポンスをTealium Collectに転送します。
1. **プロファイルが豊かに**：エンリッチメントルールがレスポンス値を訪問プロファイル属性に書き込みます。
1. **オーディエンスがアクティブに**：オーディエンスルールが更新された属性を評価し、下流のコネクタが発火します。

### プロンプト

プロンプトは、モデルに評価するコンテキストデータを指示し、特定の質問をし、期待されるレスポンスを定義します。ベンダーコネクタによっては、レスポンス出力に関する具体的な指示を含める必要があるかもしれませんし、必要ないかもしれません。詳細は特定のコネクタの指示を参照してください。

一部のコネクタ、例えばOpenAIの**Send Prompt to OpenAI**アクションは、スキーマを通じて自動的にJSON構造を強制します。これらのコネクタの場合、プロンプトにレスポンス形式の指示を含めないでください。以下の例は、手動でレスポンス形式を構成する必要があるコネクタに適用されます。

#### コンテキストデータ

プロンプトには、特定の属性またはイベント全体または訪問データオブジェクトをコンテキストとして含めることができます。

特定の属性を含めるには、コネクタアクションでそれらをマップし、プロンプトで参照します。たとえば、`product_review`をマップすると、プロンプトで`{{product_review}}`を参照できます。

データオブジェクト全体をプロンプトに含めるには、コネクタアクション構成で**Add Event Payload**または**Add Visitor Profile**を有効にし、プロンプトで`{{event_payload}}`または`{{visitor_profile}}`を参照します。

オーディエンスアクションの場合、訪問プロファイルオブジェクト内に現在の訪問データを含めるために**Add Current Visit**も有効にすることができます。

**重要**  
コネクタがコンテキストデータに関するプロンプト指示を要求する場合、プロンプトの最初の行にそれを含めます。例えば：

```wrap
顧客イベントを説明するJSONオブジェクトを受け取ります：
{{event_payload}}
```

コネクタがレスポンス形式に関するプロンプト指示を要求する場合、プロンプトの最後にそれを含めます。例えば：

```wrap
次の構造で単一のJSONオブジェクトを1行で返してください：
{
  &#34;tealium_account&#34;: &#34;{{tealium_account}}&#34;,
  &#34;tealium_profile&#34;: &#34;{{tealium_profile}}&#34;,
  &#34;tealium_visitor_id&#34;: &#34;{{tealium_visitor_id}}&#34;,
  &#34;tealium_event&#34;:  &#34;ai_response&#34;,
  &#34;ai_review_sentiment&#34;: &#34;&lt;customer review sentiment&gt;&#34;
}
```

#### 質問

プロンプトは、モデルにデータの評価方法を伝え、1つ以上の特定の値を生成するように依頼します。

たとえば、このプロンプトは、顧客のレビューの感情を表すテキスト値を生成するように依頼します：

```wrap
提供された製品レビューイベントに基づいて、顧客の感情を「不満足」、「中立」、または「満足」として分類してください。
```

### レスポンスイベント

コネクタはAIレスポンスを解析し、JSONオブジェクトを抽出し、イベントとしてTealium Collectに送り返します。

レスポンスイベントの例：

```json
{
  &#34;tealium_account&#34;: &#34;acme&#34;,
  &#34;tealium_profile&#34;: &#34;main&#34;,
  &#34;tealium_visitor_id&#34;: &#34;383...05d&#34;,
  &#34;tealium_event&#34;: &#34;ai_response&#34;,
  &#34;ai_review_sentiment&#34;: &#34;satisfied&#34;
}
```

### アクティベーション

モデルが有効なJSONオブジェクトを返すと、コネクタはそれをTealium Collectにイベントとして転送します。その後、エンリッチメントルールがレスポンス値を訪問プロファイル属性に書き込みます。そこから、オーディエンスと下流のコネクタがエンリッチされたプロファイルに基づいてアクティブになります。

`tealium_event`にコネクタ固有のユニークな値を構成し、レスポンス属性をユースケースに特有の名前にすることで、これらのイベントに一致するイベントフィードやルールを簡単に作成できます。

AIレスポンスイベントのアクティベーションのための例のルール：


[
  [
    {
      &#34;input&#34;: &#34;tealium_event&#34;,
      &#34;operator&#34;: &#34;equals&#34;,
      &#34;filter&#34;: &#34;ai_response&#34;
    },
    {
      &#34;input&#34;: &#34;ai_review_sentiment&#34;,
      &#34;operator&#34;: &#34;is populated&#34;
    }
  ]  
]

### テスト

本番環境でAIコネクタを有効にする前に、コネクタのマッピングで**デバッグモード**を有効にし、トレースを使用して構成をテストして、構成とプロンプトの動作を検証してください。デバッグモードを使用すると、本番コストが発生する前やライブデータに影響を与える前に、エラーをキャッチし、プロンプトを最適化するのに役立ちます。

デバッグモードでは、コネクタはAIリクエストを実行しますが、レスポンスイベントをアカウントに送信せずに結果をログに記録します。トレースツールを使用して、完全なリクエストとレスポンスを検査し、生成された出力を確認し、トリガー条件と属性マッピングが期待通りに機能していることを確認してください。

## AIのための機能

### 仕組み

Tealiumの機能は、独自のモデル（IYOM）を呼び出すためのコードベースのアプローチを提供します。事前に構成されたコネクタを使用する代わりに、イベントまたは訪問機能から直接任意のモデルエンドポイントを呼び出すためにJavaScriptを記述します。使用例には、数値スコアを返す従来の機械学習モデル、専用のAIコネクタを持たないプラットフォームでホストされているLLM、クラウドデータプラットフォーム上のエンドポイントが含まれます。

Tealiumの機能は、イベント機能（イベント処理後にトリガーされる）と訪問機能（訪問プロファイルの更新後にトリガーされる）の両方をサポートし、即時のイベントコンテキストまたは蓄積された訪問履歴のいずれかをスコアリングすることができます。

機能が発火すると、次のステップに従います：

1. イベント属性または訪問プロファイルデータを収集します。
1. モデルエンドポイントのためのリクエストペイロードを構築します。
1. `fetch()`を使用してエンドポイントを呼び出します。
1. レスポンスを解析します。
1. `track()`を使用して予測をTealium Collectに送信します。
1. エンリッチメントが訪問プロファイルに予測値を書き込みます。

機能は直接訪問プロファイルを変更することはできないため、予測はTealium Collectを通じてプロファイルに書き込まれる必要があります。

機能は同期および非同期のモデル呼び出しをサポートしています。10秒以内に応答するモデルの場合は、標準の`await`パターンを使用します。遅いモデルの場合は、ファイア・アンド・フォーゲットパターンを使用します。機能はリクエストを送信し、すぐに終了します。その後、モデルが推論を完了すると、コールバックURLに結果をPOSTします。

実装の詳細については、[AIアクティベーションのための機能を作成する]()を参照してください。

## コネクタと機能の選択

### AIコネクタを使用する場合

* プロンプトベースの推論のために大規模な言語モデルを呼び出す場合。
* サポートされているプラットフォーム（例：Snowflake Model Registry）上のデプロイされたモデルエンドポイントをリアルタイムで推論する場合。
* サポートされているAIプロバイダーに対応するユースケースがある場合。
* モデルがTealiumイベントとして直接転送可能な構造化された出力を返す場合。
* Tealium UIを通じてコードを書かずに統合を構成したい場合。
* ページビューなどの高ボリュームで低価値のイベントではなく、ターゲット指定された高価値のイベントやオーディエンスで構成されるユースケースがある場合。

AIコネクタは、分類ラベル、信頼値、スコア、または事前定義されたリストからの識別子など、構造化された予測出力を返すユースケース用に設計されています。長文コンテンツやメディア生成が必要なユースケースの場合は、本番に移行する前に組織のAIガバナンスプロセスに沿ってください。

### Tealium機能を使用する場合

* モデルが専用のAIコネクタを持たないプラットフォームでホストされている場合。
* ペイロードがカスタムロジック、データ変換、またはPIIフィルタリングを必要とする場合。
* 非同期モデル応答またはカスタムネットワーク構成を処理する必要がある場合。

モデルにデータを送信する前にPIIを削除またはトークン化する必要がある場合は、Tealium機能を使用してカスタムフィルタリングを適用し、承認されたフィールドのみをエンドポイントに送信します。

## 比較概要

| 考慮事項 | AIコネクタ | Tealium機能 |
|---------------|---------------|-------------------|
| 構成 | UIを通じて構成 | JavaScriptコードを書く |
| サポートされているモデルタイプ | LLMおよびサポートされているプラットフォーム上のデプロイされたモデルエンドポイント | 任意のMLまたはAIモデル、従来のMLを含む |
| サポートされているプロバイダー | [サポートされているベンダー](#supported-vendors)を参照 | 任意のHTTPSエンドポイント |
| プロンプト制御 | 変数置換を伴うテンプレートベース | 完全にプログラム可能 |
| 認証 | コネクタ構成で管理 | Tealium機能内の認証トークンとして保存 |
| ペイロード構造 | イベントまたは訪問データに基づいて固定 | 完全にカスタマイズ可能 |
| モデル呼び出し前のPIIフィルタリング | プラットフォームレベルのみ（同意カテゴリと制限された属性）。コネクタ内にカスタム前処理コードはない | JavaScriptでの完全にカスタマイズ可能なフィルタリングと変換 |
| レスポンス処理 | 自動JSON解析とイベント転送 | コード内のカスタム解析 |
| 非同期サポート | 限定的。Bedrock AI WorkflowアクションはLambdaコールバックを通じて非同期応答をサポート | ファイア・アンド・フォーゲットパターンとTealium Collectへのモデルコールバックを通じてサポート |

## 一般的なユースケース

以下の表は、一般的なAIユースケースとそれぞれの推奨統合アプローチをリストしています。

| ユースケース | 説明 | 推奨アプローチ |
|----------|-------------|----------------------|
| 感情分析 | 顧客のフィードバックやレビューイベントを肯定的、中立的、または否定的に分類します。 | AIコネクタ |
| 意図分類 | 閲覧履歴や購入履歴に基づいて顧客の意図を特定します。 | AIコネクタ |
| 次善のアクション（プレイブック選択） | 訪問のコンテキストに基づいて承認されたアクションまたはオファーの有限セットから選択します。 | AIコネクタ |
| 傾向スコアリング | 訓練された機械学習モデルを使用して、コンバージョン、離脱、または高価値アクションの可能性を予測します。 | AIコネクタ（モデルがサポートされているプラットフォーム上にデプロイされている場合）またはTealium機能 |
| 次善のアクション（スコア最適化） | 数値モデルを使用して候補アクションのセットをスコアリングし、最も高いスコアのものを選択します。 | AIコネクタ（モデルがサポートされているプラットフォーム上にデプロイされている場合）またはTealium機能 |
| フィーチャーストアルックアップ | データプラットフォームから事前に計算されたスコアまたはフィーチャーを取得してアクティベーションします。 | Tealium機能 |
| カスタムエンドポイント推論 | 専用のAIコネクタを持たないプラットフォームでホストされているモデルを呼び出します。 | Tealium機能 |
