イベントと訪問機能は、イベントまたは訪問が処理された後のデータパイプラインの最後に実行されます。他のシステムからデータを取得したり、Tealiumのデータを増強したり、他のエンドポイントにデータを送信するために、イベントと訪問機能を使用します。

![](/images/server-side/functions-event-data-pipeline.png)

関数が外部システムにアクセスするための認証が必要な場合は、[関数に認証を追加する]()を参照してください。

このデータパイプラインの時点で、イベントまたは訪問データの変更はシステムの他の部分には影響しません。イベントを変更するには、イベント属性のエンリッチメントまたはイベント変換機能を使用します。訪問プロファイルを変更するには、訪問属性のエンリッチメントを使用します。

## Action V2 と Action V3 ランタイム

Action V2 ランタイムは廃止され、サポートされていません。V2 ランタイムを使用する関数は実行されていますが、コードの変更を保存することはできません。コードの変更を保存するには、ランタイムバージョンを更新する必要があり、関数コードの変更が必要になる場合があります。必要なコードの変更については、[V2関数をV3ランタイムに移行する]()を参照してください。

イベントと訪問機能の入力データは、機能のタイプ（イベントまたは訪問）とランタイムバージョン（Action V2またはAction V3）によって異なります。Action V2 ランタイムを使用する関数は、入力データに名前付きエクスポートを持っています。Action V3 ランタイムを使用する関数は、入力パラメータとしてデータを受け取ります。

さらに、V2およびV3のイベントと訪問機能は、以下の点で異なります：
* サービスプロバイダーの認証を取得する。
* Tealium Collectにイベントを送信する。
* グローバル変数を取得する。
* 属性IDによって属性名または値を取得する。
* fetch() APIを使用してHTTPリクエストを行う。
