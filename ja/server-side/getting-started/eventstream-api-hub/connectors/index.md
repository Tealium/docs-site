ここではその動作について説明します：

* **コネクタ**  
コネクタには、ベンダーに接続し、彼らがサポートするAPIアクションをトリガーするために必要な資格情報が含まれています。たとえば、アカウントID、ユーザー名/パスワード、またはAPIキーなどです。
* **アクション**  
アクションは、ベンダーの操作またはAPIメソッドであり、メールのトリガー、カスタムオーディエンスの構築、または購入イベントの追跡などがあります。アクションの設定は、ベンダーが受け取るイベントと属性を決定します。
* **イベントフィード**  
各アクションはイベントフィードによってトリガーされます。イベントフィードがイベントを受け取るたびに、それが使用されるコネクタアクションをトリガーします。
* **データマッピング**  
データマッピングは、イベント属性をベンダーが期待するデータパラメータに変換するためにアクションで設定されます。
