## よくある質問

#### クライアントサイドのプロファイルからユーザーを削除すると、サーバーサイドのプロファイルからも削除されますか？
いいえ。ユーザーがクライアントサイドとサーバーサイドの両方に存在する場合、一方からユーザーを削除しても他方には影響しません。

#### クライアントサイドのプロファイルへのアクセスを許可せずに、ユーザーにサーバーサイドのプロファイルへのアクセスを許可できますか？
はい。アカウントに既に存在するユーザーがサーバーサイドのプロファイルへのアクセス権を付与されている場合、**Tag Management > Manage Users** に移動して、そのユーザーをクライアントサイドのインターフェースから削除します。詳細については、上記のサーバーサイド専用ユーザーを追加するを参照してください。

#### 「すべての現在および将来のプロファイル」オプションはサーバーサイドの権限にどのような影響を与えますか？
サーバーサイドのユーザーには影響しません。すべてのサーバーサイドユーザーは、新しいサーバーサイドプロファイルに対してデフォルトの権限ロールである**No Access**を持っています。

#### ユーザーがサーバーサイドで1つのプロファイルのみにアクセス権を持ち、クライアントサイドでプロファイルアクセスがない場合、どうなりますか？
ユーザーがログイン画面でサーバーサイドを選択すると、自動的にアカウント/プロファイルにロードされます。クライアントサイドにログインしようとすると、アクセス拒否メッセージが表示されるモーダルが表示されます。

#### サーバーサイドの管理者は自身のサーバーサイドプロファイルの権限を変更できますか？
はい。**Manage Account** 権限によって付与されたサーバーサイドの管理ユーザーは、自身を含むすべてのサーバーサイドユーザーの権限を変更することができます。

#### サーバーサイドの管理者はすべてのサーバーサイドプロファイルの権限を管理できますか？
はい。サーバーサイドの管理ユーザーのアクセスは、アカウントレベルの権限である**Manage Account** 権限によって付与されます。この権限レベルを持つユーザーは、サーバーサイドインターフェースの**Manage Users** 画面にアクセスし、すべてのプロファイルのユーザー権限を管理できます。

#### 新しいサーバーサイドの権限はOmnichannel File Status APIにどのような影響を与えますか？
[Omichannel File Status API](https://docs.tealium.com/about-omnichannel-file-status-api/)へのアクセス制御は、サーバーサイドの権限機能の最終リリース時に変更されます。標準的な要件である[APIキーを使用したAPI認証](https://docs.tealium.com/api-keys/)に加えて、ユーザーは関連するCustomer Data Hubアカウントへの読み取りアクセスを持っている必要があります。[v1 API](https://docs.tealium.com/omnichannel-file-status-objects-v1/)を使用するユーザーは、Customer Data Hubアカウントへの読み取りアクセスのみが必要です。

#### サーバーサイドの管理者は新しいユーザーをアカウントに追加できますか？
いいえ。**Manage Account** 権限は新しいユーザーを追加したり、アカウントからユーザーを削除するアクセスを許可しません。プロファイルへのアクセスを削除することはできますが、新しいユーザーをアカウントに招待したり、アカウントからユーザーを完全に削除することは、iQ Tag Managementワークフローで**Manage Users** 権限が割り当てられているユーザーのみが可能です。

#### この機能はTiQ内でのユーザー管理アクセスの付与や制御の方法を変更しますか？
いいえ。iQ Tag Managementの**Manage Users** 権限は、ビジネスアカウントからユーザーを追加または削除するアクセスをユーザーに付与する権限のままです。**Manage Users** 権限のみを持つユーザーは、サーバーサイドの権限を管理するアクセスを持っていません。
