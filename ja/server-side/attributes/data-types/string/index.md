## 仕組み

文字列は、顧客情報（名前や郵便番号）、注文情報（注文IDや配送方法）、またはサイト情報（サイトセクションやサイト言語）などに使用されます。

文字列属性は、イベント、訪問、訪問の範囲で利用可能です。

![](https://docs.tealium.com/images/server-side/attribute-scopes-table-all.png)

### サイズ制限

EventStreamの文字列属性は、最大1,500文字です。この制限を超えると、文字列は1,500文字に切り捨てられます。

AudienceStreamの文字列属性値は、最大1,000文字です。1,000文字を超える文字列がエンリッチメントの結果として生成された場合、その値は保存されません。

文字列属性は、暗号化および圧縮後のプロファイルの最大サイズ（400 KB）によっても制限されます。

## エンリッチメント

文字列属性に利用可能なエンリッチメントは以下の通りです。

### 文字列を構成

このエンリッチメントは、提供された定数値または別の文字列属性の値から文字列属性の値を構成します。ソース値は文字列、iQ変数、またはオムニチャネル属性のみ可能です。

**属性名**: `customer_type`

* **初期値**: `""`
* **エンリッチされた値**: `"unknown"`
* **結果の値**: `"unknown"`

### 文字列を分割

このエンリッチメントでは、分布割合に基づいて複数の値を構成します。構成する各値には割合構成もあります。複数の値/割合エントリーが可能ですが、分布は合計で100%になる必要があります。分布は乱数ジェネレータに基づいているため、小規模なサンプルでは分布と一致しない場合がありますが、より多くの値が割り当てられるにつれて分布比率は正確になります。ソース値は文字列、iQ変数、またはオムニチャネル属性のみ可能です。

この例では、`test_group`という属性を使用して、ユーザーを「GroupA」と「GroupB」という2つの等しいグループ（50/50）にセグメント化します。この属性は、各グループに関連する活動を識別するために使用できます。

**属性名**: `test_group`

* **初期値**: `""`
* **結果の値**: 文字列の50%に対して`"GroupA"`を構成<br> 文字列の50%に対して`"GroupB"`を構成

参照: [エンリッチメント例: 文字列を分割](https://support.tealiumiq.com/en/support/solutions/articles/36000363487-enrichment-example-split-string)

### 文字列を削除

このエンリッチメントは、属性から値全体を削除します。

* **初期値**: `"Jane Smith"`
* **結果の値**: (削除されました)

### 文字列を小文字に

このエンリッチメントは、文字列属性の現在の値を小文字にします。

**属性名**: `email_address`

* **初期値**: `"First.Last@Example.com"`
* **結果の値**: `"first.last@example.com"`

### 属性を結合

このエンリッチメントは、区切り文字を使用して複数の値を単一のテキスト値に結合します。区切り文字は1文字以上です。たとえば、`site_section`、`page_category`、その他のページレベル属性を組み合わせてページ階層値を作成することができます。

**属性名**: `page_path`

* **初期値**: `""`
* **エンリッチされた値**:  
属性1: `site_region="en-us"`  
属性2: `site_section="Electronics"`  
属性3: `category_name="Tablets"`  
区切り文字: `":"`
* **結果の値**: `"en-us:Electronics:Tablets"`

### 文字列を日付に構成

このエンリッチメントは、日付値を文字列に変換し、カスタムフォーマットを許可します。ソース値は日付、文字列、iQ変数、またはファイルインポート属性です。日付フォーマッターについての詳細は、[日付属性](https://docs.tealium.com/date-attribute/#date-formatter)を参照してください。

**属性名**: `last_purchase_date`

* **初期値**: `""`
* **エンリッチされた値**: 最終ログイン日付、フォーマット`"yyy-MM-dd"`
* **結果の値**: `"2019-12-31"`
