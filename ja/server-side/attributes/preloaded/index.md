## 動作原理

便宜上、AudienceStream および EventStream には、事前に読み込まれたイベント、訪問、および訪問属性のセットが含まれています。各属性は、各訪問または訪問について有用な情報を決定するのに役立つ統計情報を収集するように構成されています。これらの属性には、[Tealium Collect タグ]()からの標準イベントデータを超える追加データは必要ありません。

 事前読み込みされた属性は変更できませんが、複製してそのコピーを変更することができます。複製には元の属性のすべてのエンリッチメントとルールが含まれています。

### IPv6のサポート

このセクションでは、どのコンポーネントがIPv6をサポートしているか、およびIPv6アドレスが利用できない場合のフォールバックの構成方法を示します。

**タグホスト名**

| ホスト名 | IPv4サポート | IPv6サポート |
|----------|--------------|--------------|
| `tags.tiqcdn.com` | ✓ | ✓ |
| `tags.tiqcdn.cn` | ✓ | ✓ |
| `tags-eu.tiqcdn.com` | ✓ | ✗ |

**ファーストパーティドメイン**

ファーストパーティドメインのIPv6サポートは、構成が作成された時期に依存します：

* ファーストパーティドメイン機能のリリース前に作成された構成はIPv6をサポートします。
* 機能リリース後に製品UIを通じて作成された構成は、デフォルトではIPv6をサポートしません。IPv6を有効にするには、ドメインのAレコードと同じディストリビューションを指すAAAA DNSレコードを追加します。

**その他のコンポーネント**

以下のコンポーネントはIPv6をサポートしていません：

* Tealium CollectタグおよびCollect HTTP API（`collect.tealiumiq.com`および`collect-&lt;region&gt;.tealiumiq.com`）
* ファーストパーティCollectドメイン
* Tealium v1、v2、およびv3 API（`platform.tealiumapis.com`）
* コネクターおよび外部ファイルのインポート

IPv6アドレスが利用できない場合にIPv4アドレスをフォールバックとして使用する属性を作成する方法については、[IPv6を使用する際のフォールバックの構成](https://support.tealiumiq.com/en/support/solutions/articles/36000521207)のナレッジベース記事を参照してください。

## 例

以下の例は、プリロードされた属性を使用する方法を示しています：

* **頻繁な訪問**バッジを使用してオーディエンスを作成し、そのオーディエンスにコネクタを使用して「定期的な訪問であることに感謝します」というメールを送信します。
* **使用したオペレーティングシステムのライフタイム（お気に入り）**属性を使用して、特定のダウンロードの配置を変更し、より関連性の高いアプリが最初に表示されるようにします。
* **合計ダイレクト訪問数**と**合計リファード訪問数**を使用して、訪問ごとのダイレクト訪問とリファード訪問の比率を計算し、各訪問または訪問を理解します。
