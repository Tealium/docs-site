ホストされたデータレイヤーAPIは、ホストされたデータレイヤー機能の一部です。このコンポーネントを使用すると、Tealiumのサーバー上でデータレイヤーオブジェクトをアップロードし、アクセスすることができます。データレイヤーオブジェクトは静的なJSONオブジェクトで、ページ上のデータレイヤーを補完するためにウェブサイトから取得することができます。

## 開始する前に

開始する前に、[ホストされたデータレイヤー機能の主要な概念]()について読むことをお勧めします。

## 前提条件

この機能を実装するためには、以下の項目が必要です：

* **ホストされたデータレイヤー拡張**  
これはホストされたデータレイヤー機能の最初のコンポーネントです。このコンポーネントの目的は、ホストされたデータレイヤーオブジェクトをターゲットのデータレイヤーにエンリッチすることです。データレイヤーオブジェクトをアップロードする前に、ホストされたデータレイヤー拡張を[追加および構成]()する必要があります。
* **保存または名前を付けて保存の権限**  
現在[保存または名前を付けて保存の権限]()がない場合は、アカウント管理者に連絡して、いずれかの権限を付与してもらってください。

## 命名ガイドライン

データレイヤーオブジェクトをアップロードする前に、オブジェクトに何を名前とするかを知っておく必要があります。アップロードされたデータオブジェクトの名前は、ページ上のルックアップ変数（データレイヤー）の値と一致しなければなりません。これらの製品に関する静的情報をアップロードするために、ホストされたデータレイヤーAPIを使用します。

まずルックアップ変数の値を知り、次にホストされたデータレイヤーオブジェクトの名前を期待される値と一致するように命名する必要があります。ルックアップ変数の名前は、ページ上のデータレイヤーとホストされたデータレイヤーオブジェクトとの間の共有リンクとなります。

### 例

例えば、ルックアップ変数`product_id`のデータレイヤーオブジェクトの名前は、ページ上で入力された値と一致しなければなりません。例えば：&#34;prod123456&#34;。ページ上の変数が`product_id`であれば、その値は表示されている製品のID、例えば&#34;prod123456&#34;になります。

* ページ上のデータレイヤー：`utag_data.product_id = [&#34;prod123456&#34;];`
* ホストされたデータレイヤーオブジェクトの名前：&#34;prod123456&#34;
* データレイヤーオブジェクトの名前は、`dl_id`パラメータの値となります。

名前には空白や以下のUniform Resource Identifier（URI）予約文字を含めることはできません：/ : ? # [ ] @ ! $ &amp;amp; &#39; ( ) \* &#43; , ; = % . &#34;
