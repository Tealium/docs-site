Data Connect API (Early Access)

This article describes the methods provided by the Data Connect API and provides examples of use for each method.

Data Connect is currently in Early Access (EA) and only available to select customers. If you are interested in trying this feature, contact your Tealium account team.

How It works

The Data Connect API is an interface you can use to create Data Connect tables and to create, read, update, and delete (CRUD operations) items in Data Connect tables.

API path: /storage/v1/accounts/{account}/profiles/{profile}/tables

The Data Connect API supports the following request types:

  • GET
  • POST
  • DELETE
  • PUT

Authentication

A bearer token is used to authenticate all API calls, not the API key. The API key is only used to call the Tealium API v3 to obtain the bearer token. In addition to the bearer token, the authentication response includes a region-specific hostname that must be used in subsequent server-side API calls. See the Tealium API V3 Getting Started guide to learn about generating a bearer token from the API key.

  • Method: POST
  • Path: https://platform.tealiumapis.com/v3/auth/accounts/{account}/profiles/{profile>}
  • Header: Content-Type: application/x-www-form-urlencoded
  • Parameters: Body Parameters:
    • username
    • key

Permissions

Interaction with the API is based on the same permissions configured for the Tealium Data Connect product.

  • If a table contains masked PII data and the user does not have the permissions required to view PII data, a series of asterisks (****) is displayed in place of the data. API responses in this scenario also display asterisks in place of the masked data.

  • If a table contains masked data and the user does have the permissions required to delete a table, the table is not deleted and the following message is displayed: "code":1,"message":"Current user is not authorized to perform this operation"

For an overview of table permissions used with Data Connect, including information relevant to PII data obfuscation, see About Data Connect.

Request information and examples

The following sections describe the request types that can be used with the API and provide the information needed to create the request and request result examples to view for comparison.

Get All Tables (GET)

This method retrieves all existing tables from Data Connect.

  • Method: GET
  • Path: /storage/v1/accounts/{account}/profiles/{profile}/tables
  • Header: Authorization: Bearer {token_from_authentication_response}

    • account
    • profile

Create a Table (POST)

This method creates a table in Data Connect.

  • Method: POST
  • Path: /storage/v1/accounts/{account}/profiles/{profile}/tables
  • Header: Authorization: Bearer {token_from_authentication_response}
  • Parameters
    • URL parameters:
      • account
      • profile
    • Body parameters
      • name
      • description
      • expirationPeriodInSeconds
      • fieldsToObfuscate
      • sensitiveData

Request Example

 https://{api_host_from_authentication_response}/v3/dataconnect/storage/v1/accounts/*****/profiles/*****/tables 
 {    
  "name": "products_catalog",    
  "description": "table contains list of items available in products catalog",    
  "expirationPeriodInSeconds": "800",    
  "fieldsToObfuscate": [        
    "product_id",
    "product_price"    
  ],    
  "sensitiveData": true 
 }

Request Result

201 Created

Update Table Attributes (PUT)

This method updates attributes of Data Connect tables.

  • Method: PUT
  • Path: /storage/v1/accounts/{account}/profiles/{profile}/tables/{table}
  • Header: Authorization: Bearer {token_from_authentication_response}
  • Parameters:
    • URL parameters
      • account
      • profile
      • table
    • Body parameters
      • name
      • description
      • expirationPeriodInSeconds
      • fieldsToObfuscate
      • sensitiveData

Request Example

 https://{api_host_from_authentication_response}/v3/dataconnect/storage/v1/accounts/******/profiles/*****/tables/products_catalog 
 {    
 "name": "products_catalog",
 "description": "table contains list of items available in products catalog",
 "expirationPeriodInSeconds": "300",
 "fieldsToObfuscate": [        
   "product_id"    
 ],
 "sensitiveData": true 
}

Request Result

204 No Content

Remove Items in a Table (POST)

This method removes a items, specified by a list of lookup keys, from a Data Connect table.

  • Method: POST
  • Path: /storage/v1/accounts/{account}/profiles/{profile}/tables/{table}/batch/remove
  • Header: Authorization: Bearer {token_from_authentication_response}
  • Parameters:
    • URL parameters
      • account
      • profile
      • table
    • Body parameters
      • lookupKeys

Request Example

https://{api_host_from_authentication_response}/v3/dataconnect/storage/v1/accounts/******/profiles/****/tables/products_catalog/batch/remove  
{    
  "lookupKeys": [        
  "prod-100",        
  "prod-101"    ] 
}

Request Result

200 OK { "unprocessedItems": [] }

Create/Update an Item in a Table (PUT)

This method creates or updates an item in a Data Connect table. If the specified lookup key already exists in the table, its value is updated. If the lookup is not found in the table, a new item is added.

  • Method: PUT
  • Path: /storage/v1/accounts/{account}/profiles/{profile}/tables/{table}/data/{lookup_key}
  • Header: Authorization: Bearer {token_from_authentication_response}
  • Parameters:
    • URL parameters
      • account
      • profile
      • table
      • lookup_key
    • Body parameters
      • “string”

Request Example

https://{api_host_from_authentication_response}/v3/dataconnect/storage/v1/accounts/*****/profiles/****/tables/products_catalog/data/prod-102   
{    
  "spreadsheet_id": "1D1aGNuj5fEZmMDWkOm5buh7s3QMy7r9OIL_VxHONiOA",    
  "name": "Product Test Sheet - Data Connect",    
  "sheet_name": "Sheet1",    
  "row": 4,    
  "id": "4",    
  "columns": {        
    "col_1": "prod-102",        
    "col_2": "name 102",        
    "col_3": ["shirt"],        
    "col_4": "9.99",        
    "col_5": "no",        
    "col_6": "yes"    
  },    
"updated": false 
}

Request Result

200 OK - for update 201 Created - for adding new record
