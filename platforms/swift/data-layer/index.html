Data Layer

A full list of the data layer variables provided by each module.

AppData Module Variables

The following variables are available in the AppData module.

Variable Name Description
app_build Minor build version of the app

Example: 2213
app_name Name of the app (usually same as App Store name)

Example: Digital Velocity
app_rdns Fully-qualified name of the app bundle

Example: com.tealium.digitalvelocity
app_uuid Random uuid. Persists for the duration of the app install, so long as one of the persistent storage modules is also enabled. Resets if app is uninstalled.

Example: 123e4567-e89b-12d3-a456-426655440000
app_version Version of the app bundle

Example: 1.0
app_version_full App Version + Build Number

Example: 1.0 (2213)
tealium_vid Persistent Tealium visitor ID

Example: 123e4567e89b12d3a456426655440000
tealium_visitor_id Same as tealium_vid. Included for legacy reasons

Example: 123e4567e89b12d3a456426655440000

Attribution Module Variables

The following variables are available in the Attribution module.

Variable Name Description
ad_campaign_id The ID of the campaign the corresponding ad was part of

Example: 1234567890
ad_campaign_name The name of the campaign the corresponding ad was part of

Example: CampaignName
ad_group_id The ID of the ad group the corresponding ad was part of

Example: 1234567890
ad_group_name The name of the ad group the corresponding ad was part of.

Example: AdGroupName
ad_keyword The keyword that drove the ad impression which led to the corresponding ad click.

Example: Keyword
ad_org_name The organization that owns the campaign the corresponding ad was part of.

Example: OrgName
ad_user_clicked_last_30_days True if user clicked on a Search Ads impression within 30 days prior to app download.

Example: true
ad_user_date_clicked Date and time the user clicked on a corresponding ad

Example: 2016-12-05T17:31:40Z
ad_user_date_converted Date and time the user downloaded your app

Example: 2016-12-05T17:31:40Z
device_advertising_id User-resettable advertising identifier (IDFA)

Example: 6D92078A-8246-4BA4-AE5B-76104861E7DC
device_advertising_enabled Boolean to indicate whether the user has allowed ad tracking. True if ad tracking enabled, otherwise false. Note: if false, the advertising ID will appear as a string of zeroes

Example: true
device_advertising_vendor_id Unique ID guaranteed to be the same across all apps on the same device from a single vendor (apps with the same 1st 2 parts of the RDNS bundle identifier e.g. com.tealium or com.acme)

Example: 6D92078A-8246-4BA4-AE5B-76104861E7DC

Autotracking Module Variables

The following variables are available in the Autotracking module.

Variable Name Description
autotracked Set to true and added to each autotracked tracking call

Example: true

ConsentManager Module Variables

The following variables are available in the ConsentManager module.

Variable Name Description
consent_categories An array of the user’s selected consent categories

Example: [“analytics”, “cdp”]
consent_status The user’s current consent status

Example: consented/notConsented
policy Consent Policy

Example: “gdpr”

Connectivity Module Variables

The following variables are available in the Connectivity module.

Variable Name Description
tealium_datasource Specifies the data source name. Added to every dispatch.

Example: abc123
was_queued Indicates whether a tracking call was queued due to no connectivity. Only present for queued events; absent for all other events

Example: true

Core Module Variables

The following variables are available in the VolatileData module.

Variable Name Description
screen_title Screen title passed to trackView call

Example: “homescreen”
tealium_event The name of the event being tracked

Example: “checkout”

Crash Reporter Module Variables

Variable Name Description/Example
crash_cause Cause of the crash

Example: Crash Reason
crash_name Friendly name of the crash, if available

Example: Crash Name
crash_libraries List of loaded libraries at the time of the crash

Example:
`[{“baseAddress”: “0xa3e0000”, “codeType”: { “arch”: 64, “typeEncoding”: “Mach”}, “imageName”: “/Applications/Xcode9.3.app/Contents/Developer/Platforms/iPhoneOS.platform/Developer/Library/CoreSimulator/Profiles/Runtimes/iOS.simruntime/Contents/Resources/RuntimeRoot/usr/lib/dyld_sim”, “imageSize”: 212992, “imageUuid”: “4015e9b70bde”}]``
crash_process_id PID of the app at time of crash

Example: 84351
crash_process_path Path the app was running under

Example: /DemoApp.app/DemoApp
crash_parent_process Parent process that launched the app

Example: launchd_sim
crash_parent_process_id

Example: Parent process PID		 83183
crash_signal_address Signal address of the crash

Example: 4572945982
crash_signal_code Signal code causing the crash

Example: #0
crash_signal_name Signal name causing the crash

Example: SIGABRT
crash_threads Returns all active threads at the time of the crash

Example:
`{“crashed”: 1,“registers”: {“cs”: “0x07”},“stack”: {“instructionPointer”: 4572945982,“symbolInfo”: {“symbolName”: “”,“symbolStartAddr”: 0},“threadId”: “”}}`
crash_uuid Unique identifier for this specific crash

Example: CC2DA0E9-E544-429A-AC5E-A268FC62F02A
device_memory_usage Current memory used by the app on the device at the time of the crash

Example: 143.57MB
device_memory_available Free memory on device at the time of the crash

Example: 1068.88MB
device_os_build Current OS build number

Example: 15E217

DeviceData Module Variables

The following variables are available in the DeviceData module.

Variable Name Description
app_memory_usage Memory being used by the current process/app in Megabytes

Example: 35.75MB
battery_percent Current battery percentage at time of track call

Example: 58
cpu_architecture CPU architecture

Example: 64
cpu_type CPU type

Example: ARM64v8
device_is_charging Boolean to indicate if device is currently charging at time of track call

Example: false
device_orientation Simple orientation

Example: Portrait, Landscape
device_orientation_extended Full orientation

Example: Face Up
device_resolution Screen resolution

Example: 1080x1920
memory_active* Total active memory on the device

Example: 997.78MB
memory_compressed* Total compressed memory on the device

Example: 153.39MB
memory_free* Total free memory on the device

Example: 120.81MB
memory_inactive* Total inactive memory on the device

Example: 441.83MB
memory_physical* Total physical memory on the device (RAM) in Megabytes

Example: 2013.50MB
memory_wired* Total wired memory on the device

Example: 207.12MB
model_name Consumer device name

Example: iPhone 7 Plus
model_variant Model variant (generally indicates which radio technology is available on the device)

Example: CDMA, GSM, WiFi, Cellular
network_connection_type** Current connection type

Example: wifi, cellular
network_iso_country_code Mobile network ISO country code

Example: gb
network_mcc Mobile network Country Code (see https://en.wikipedia.org/wiki/Mobilecountrycode)

Example: 234
network_mnc Mobile network code (see https://en.wikipedia.org/wiki/Mobilecountrycode)

Example: 34
network_name Mobile network carrier name

Example: EE
os_build Operating System Build Version

Example: 15J580
os_name Operating System Name

Example: iOS, tvOS, watchOS, macOS
os_version Operating system version

Example: 11.1
user_locale Current device language

Example: en

Lifecycle Module Variables

The following variables are available in the Lifecycle module.

Variable Name Description
lifecycle_dayofweek_local Local day of week that call was made - 1=Sunday, 2=Monday, etc.)

Example: 4
lifecycle_dayssincelastwake Days since last detected wake in integer increments

Example: 1
lifecycle_dayssincelaunch Days since first launch in integer increments

Example: 23
lifecycle_dayssinceupdate Days since the last detected app version update in integer increments

Example: 46
lifecycle_diddetect_crash Was a crash detected during this launch/wake event. Populated only if true.

Example: true
lifecycle_firstlaunchdate GMT timestamp of very first detected launch/wake in ISO8601 format from Zulu time

Example: 2013-07-11T17:55:04Z
lifecycle_firstlaunchdate_MMDDYYYY GMT Timestamp formatted as MM/DD/YYYY

Example: 01/18/2012
lifecycle_hourofday_local Local hour of day that call was made (24 hour format)

Example: true
lifecycle_isfirstlaunch Only present if call is very first launch/wake call

Example: true
lifecycle_isfirstlaunchupdate Only present if call is first launch/wake after a detected updated

Example: true
lifecycle_isfirstwakemonth Only present if call is first launch/wake of the month

Example: true
lifecycle_isfirstwaketoday Only present if call is first launch/wake of the day

Example: true
lifecycle_launchcount Total number of launches this version of your app (since the last update)

Example: 3
lifecycle_priorsecondsawake Whole seconds app was awake since last launch only. Aggregates total from all wakes prior. Sent only with lifecycle_type: launch calls.

Example: 126
lifecycle_secondsawake Whole seconds app was awake since most recent wake/launch.

Example: 30
lifecycle_sleepcount Total number of times your app has gone to sleep (resets if updated)

Example: 5
lifecycle_totalcrashcount Total number of crashes counted since install (only reset if app deleted)

Example: 21
lifecycle_totallaunchcount Total number of launches since install (only reset if app deleted)

Example: 3
lifecycle_totalsecondsawake Total number of seconds your app has been in a woken/active state since app install (only reset if app deleted)

Example: 36
lifecycle_totalsleepcount Total number of times your app has gone into the background since app install (only reset if app deleted)

Example: “400”
lifecycle_totalwakecount Total number of launches + wakes since install (only reset if app deleted)

Example: “563”
lifecycle_type Type of lifecycle call

Example: “launch”, “wake”, “sleep”
lifecycle_updatelaunchdate GMT timestamp of first wake/launch after a version update has been detected

Example: 2014-09-08T18:10:01Z
lifecycle_wakecount Total number of launches + wakes in this version of your app (resets if updated)

Example: “29”

The following variables are available in the TagManagement module.

Variable Name Description
dispatch_service Static string to indicate which module the tracking call came from

event_timestamp_offset_hours Current offset from UTC in hours

Example: -8
event_timestamp_unix Current Unix timestamp (1970 epoch) in seconds

Example: 1511189280
event_timestamp_unix_millis Current Unix timestamp (1970 epoch) in milliseconds

Example: 1511189280337
tealium_account Current Tealium account (from TealiumConfig object)

Example: tealium
tealium_environment Current Tealium environment (from TealiumConfig object)

Example: dev
tealium_library_name Tealium library name

Example: swift
tealium_library_version Tealium library version

Example: 1.4.0
tealium_profile Current Tealium profile (from TealiumConfig object)

Example: mobile
tealium_random Random number

Example: 8449989974958830
tealium_session_id Tealium Session ID

Example: 1511262829677

Example: 1511262829
