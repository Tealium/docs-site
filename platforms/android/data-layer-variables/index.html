ANDROID

Data Layer

Full list of data layer variables available from the Android library.

The following variables are populated by the Tealium Android Library as part of the mobile data layer. These variables are included automatically in each tracking call from the library and can be used in your Universal Data Hub configuration.

These variables are added to your iQ Tag Management account when you activate the profile for mobile use.

Addition variables are available in the Lifecycle Tracking Module for 5.0.x.

Variable Name Android Version Description
app_build 5.4+ App build version e.g. “1”
app_name 1.1+ Application name (from the  manifest attribute “android:label”)
app_memory_usage 5.4+ Memory in use by app process (MB) e.g. “57”
app_rdns 3.0+ Reverse DNS application ID (from the root manifest attribute “package”)
app_version 1.1+ Application version [any string]
call_type 2.0+ Call type [view/link]
carrier 1.1+ Carrier device is currently on [ATT, Verizon, etc.]
carrier_iso 1.1+ Carrier iso country code - ISO 3166-1
carrier_mcc 1.1+ Mobile country code - ITU-T Recommendation E.212
carrier_mnc 1.1+ Mobile network code
connection_type 1.1+ [wifi, cellular, etc.]
device 1.1+ [iphone, ipad, etc.]
device_android_runtime 5.4+ Android Runtime version e.g. “2.1.0”
device_available_external_storage 5.4+ Total available external storage on device
device_available_system_storage 5.4+ Total available storage on device e.g. 273772544
device_architecture 3.0+ Bit architecture of the device [32, 64, etc]
device_battery_percent 4.0+ Whole number percent representation of device’s remaining power [20, 50,…]
device_cputype 3.0+ Cpu type [arm6, arm7, arm7s, etc]
device_ischarging 4.0+ Is the device actively charging [true, false]
device_language 3.0+ Two-letter ISO-639-1 language setting identifier [en, ja, etc.]
device_orientation 5.0+ Orientation of device at time of call [Portrait, Landscape Left, Landscape Right, Portriat UpsideDown, Face Up, Face Down, Unknown]
device_os_build 5.4+ Operating System build version e.g. “4499259”
device_os_version 5.0+ Operating System version on device [7.0]
event_name 5.0+ “mobile_link” required by vdata
library_version 1.2+ The version of the Tealium library your app has installed
lifecycle_dayofweeklocal* 1.2+ Local day of week that call was made (ie 1=Sunday, 2=Monday, etc.)
lifecycle_dayssincelastwake* 1.2+ Days since last detected wake in integer increments
lifecycle_dayssincelaunch* 1.2+ Days since first launch in integer increments
lifecycle_dayssinceupdate* 1.2+ Days since the last detected app version update in integer increments
lifecycle_diddetectcrash* 5.0.3+ Populated ‘true’ on launch events if crash was determined.
lifecycle_firstlaunchdate* 1.2+ GMT timestamp of very first detected launch/wake in ISO8601 format from Zulu time
(ie 2013-07-11T17:55:04Z)
lifecycle_firstlaunchdate_MMDDYYYY* 1.2+ GMT Timestamp formatted as MM/DD/YYYY
(ie 01/18/2012)
lifecycle_hourofday_local* 1.2+ Local hour of day that call was made (24 hour format)
lifecycle_isfirstlaunch* 1.2+ Only present if call is very first launch/wake call (value = true)
lifecycle_isfirstlaunchupdate* 1.2+ Only present if call is first launch/wake after a detected updated
lifecycle_isfirstwakemonth* 1.2+ Only present if call is first launch/wake of the month (value = true)
lifecycle_isfirstwaketoday* 1.2+ Only present if call is first launch/wake of the day (value = true)
lifecycle_lastlaunchdate* 1.2+ GMT timestamp of last recorded launch in ISO8601 format from Zulu time
(ie 2013-07-11T17:55:04Z)
lifecycle_lastsleepdate* 1.2+ GMT timestamp of last recorded sleep in ISO8601 format from Zulu time
(ie 2013-07-11T17:55:04Z)
lifecycle_lastwakedate* 1.2+ GMT timestamp of last recorded wake in ISO8601 format from Zulu time
(ie 2013-07-11T17:55:04Z)
lifecycle_launchcount* 1.2+ Total number of launches this version of your app (resets if app version updates)
lifecycle_priorsecondsawake* 1.2+ Whole seconds app was awake since last launch only. Aggregates total from all wakes prior. Sent only with lifecycle_type: launch calls
lifecycle_secondsawake* 1.2+ Whole seconds app was awake since last wake/launch. Sent only with lifecycle_type:sleep
lifecycle_sleepcount* 1.2+ Total number of times your app has gone to sleep (resets if app version updates)
lifecycle_totalcrashcount* 5.0.4+ Total number of times crashes were detected. Does not reset unless app resintalled.
lifecycle_totallaunchcount* 1.2+ Total number of launches since install.
Does not reset unless app resintalled.
lifecycle_totalsecondsawake* 1.2+ Total number of seconds your app has been in a woken/active state.
Does not reset unless app resintalled.
lifecycle_totalsleepcount* 1.2+ Total number of times your app has gone into the background since install.
Does not reset unless app resintalled.
lifecycle_totalwakecount* 1.2+ Total number of launches + wakes since install.
Does not reset unless app resintalled.
lifecycle_type* 1.2+ Type of lifecycle call (ie launch, wake, sleep)
lifecycle_updatelaunchdate* 1.2+ GMT timestamp of first wake/launch after a version update has been detected.ISO8601 format from Zulu time(ie 2013-07-11T17:55:04Z)
lifecycle_wakecount* 1.2+ Total number of launches + wakes in this version of your app (resets if app version updates)
link_id 1.0+ (EVENT CALLS ONLY) General event or link title - this could be a button title or an object’s Tealium ID [any string]
~orientation~ 2.0+ DEPRECATED Orientation of device at time of call [Portrait, Landscape Left, Landscape Right, Portriat UpsideDown, Face Up, Face Down, Unknown]
origin 3.0+ Constant that can be used in mapping to distinguish mobile from web implementations [mobile]
os_version 1.1+ Operating System version on device [7.0]
page_type 5.0+ “mobile_view” constant that can be used in mapping to distinguish mobile view / page changes from mobile or web events - required by vdata
platform 1.0+ [iOS, Android, BB10, etc.]
screen_title 1.0+ (VIEW CALLS ONLY) General view title [any string]
timestamp 1.1+ Timestamp to seconds of event occurrence (ie 2013-07-11T19:57:47Z) [ISO8601 at Zulu]
timestamp_local 2.0+ Local Timestamp to seconds of event occurrence (ie 2013-07-11T19:57:47) [ISO8601 without offset]
timestamp_offset 2.0+ Local timezone offset of device’s location in hours (ie -3)
timestamp_unix 1.1+ GMT/UTC Unix timestamp (ie 1373498679) of event occurrence.
timestamp_unix_milliseconds 5.5.1 GMT/UTC Unix timestamp in milliseconds (ie 1373498679123) of event occurrence.
uuid 1.2+ (Universally Unique Identifier) Unique for each installation of an app on a device. [any string]
visitor_id 5.0+ Tealium-generated visitor_id
