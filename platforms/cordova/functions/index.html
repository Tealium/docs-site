CORDOVA

Function Definitions

The function definitions of the Tealium for Cordova plugin.

Config

init()

Initializes the Tealium Cordova plugin.

Parameter Data Type Description/Usage
config JavaScript Object* See separate table
successCallback Function Overrides the default Success Callback to allow custom success handling
errorCallback Function Overrides the default Success Callback to allow custom success handling

*The config object:

Parameter Data Type Description/Usage
account String Tealium Account Name
Example: “tealium”
profile String Tealium Profile Name
Example: “main”
environment String Tealium Environment Name
Example: “dev”
instance String Arbitrary Tealium instance name
Example: “main”
dataSourceId String Optional alphanumeric data source key
Example: “3obb8c”
isLifecycleEnabled String Enables or disables automatic lifecycle tracking
Example: “false”
logLevel Integer (Enum) Sets the log level of the Cordova plugin (does not affect core SDK log level). Defaults to tealium.logLevels.DEV if not specified.
collectDispatchURL String Overrides the default Tealium Collect dispatch URL
Example: https://collect.tealiumiq.com/vdata/i.gif?tealium_account=tealium&tealium_profile=mobile
collectDispatchProfile String Overrides the default Tealium Collect dispatch profile (default is always “main”)
Example: “mobile”
isCrashReporterEnabled String If set, enables the crash reporter module (default disabled)
Example: “true”

Tracking

trackView()

Sends a screen view tracking request to the Tealium Cordova plugin.

Parameter Data Type Description/Usage
Data Object JavaScript object Populates the data layer for this tracking call with the key/value pairs specified

Example:
{"tealium_event" : "page_view"}
instance String Arbitrary instance name constant, used to refer to a specific tracking instance.

Example:
“main”

trackEvent()

Sends an event tracking request to the Tealium Cordova plugin.

Parameter Data Type Description/Usage
Data Object JavaScript object Populates the data layer for this tracking call with the key/value pairs specified

Example:
{"tealium_event": "Click to call"}
instance String Arbitrary instance name constant, used to refer to a specific tracking instance.

Example:
“main”

track()

Sends a tracking request to the Tealium Cordova plugin.

Parameter Data Type Description/Usage
Event Type String Tealium event type

Example:
“link” (event) or “view” (screen view)
Data Object JavaScript/JSON object Populates the data layer for this tracking call with the key/value pairs specified

Example:
{"tealium_event" : "page_view"}
instance String Arbitrary instance name constant, used to refer to a specific tracking instance.

Example:
“main”

Data Management

addPersistent()

Adds data to the Tealium persistent data store.

Parameter Data Type Description/Usage
Key Name String Key name of value to be persisted

Example:
“user_hashed_email”
Value String or Array of strings The value to be persisted for this key

Example:
“293c2529495b45cbbc7b451df5720e8c”
instance String Arbitrary instance name constant, used to refer to a specific tracking instance.

Example:
“main”

addVolatile()

Adds data to the Tealium volatile data store.

Parameter Data Type Description/Usage
Key Name String Key name of value to be stored in volatile memory
Example: “user_hashed_email”
Value String or Array of strings, Object/Dictionary The value to be stored in volatile memory for this key
Example: “293c2529495b45cbbc7b451df5720e8c”
instance String Arbitrary instance name constant, used to refer to a specific tracking instance.
Example: “main”

removePersistent()

Removes data from the Tealium persistent data store.

Parameter Data Type Description/Usage
Key Name String Key name of value to be removed from persistent storage

Example:
“user_hashed_email”
instance String Arbitrary instance name constant, used to refer to a specific tracking instance.

Example:
“main”

removeVolatile()

Removes data from the Tealium volatile data store.

Parameter Data Type Description/Usage
Key Name String Key name of value to be removed from volatile memory

Example:
“user_hashed_email”
instance String Arbitrary instance name constant, used to refer to a specific tracking instance.

Example:
“main”

getPersistent()

Retrieves a value from the Tealium Persistent data store Returns: Persisted data with the same data type that was originally stored (String or Array of strings).

Parameter Data Type Description/Usage
Key Name String Key name of value to be retrieved from persistent storage

Example:
“user_hashed_email”
instance String Arbitrary instance name constant, used to refer to a specific tracking instance.

Example:
“main”

getVolatile()

Retrieves a value from the Tealium Volatile data store Returns: Stored data with the same data type that was originally stored (String, Array of strings, or Object/Dictionary).

Parameter Data Type Description/Usage
Key Name String Key name of value to be retrieved from volatile storage

Example:
“user_hashed_email”
instance String Arbitrary instance name constant, used to refer to a specific tracking instance.

Example:
“main”

addRemoteCommand()

Adds a remote command for later execution, initiated by Tealium iQ.

Parameter Data Type Description/Usage
Remote Command ID String ID of the remote command. Matching id must be used in Tealium iQ to trigger the code block

Example:
“changeBackgroundColor”
instance String Arbitrary instance name constant, used to refer to a specific tracking instance.

Example:
“main”
Code block Function A function literal or reference, which will be available for later remote execution from Tealium iQ

Example:
function (message) { //do something}

getVistorId()

Returns the current Tealium visitor ID for the current user. It takes no parameters. Returns: String representation of the current visitor ID.
